# 本地备份功能实现计划

## 2. 功能需求

### 2.1 导出功能

| 功能 | 说明 |
|------|------|
| JSON 导出 | 完整数据备份，可恢复 |
| CSV 导出 | 仅待办列表，Excel 兼容 |
| 密码保护 | 可选，AES-256-GCM 加密 |
| SAF 选择路径 | 用户选择保存位置 |

### 2.2 恢复功能

| 功能 | 说明 |
|------|------|
| JSON 导入 | 从备份恢复所有数据 |
| 密码解密 | 如果备份有密码 |
| 确认覆盖 | 恢复前警告用户 |
| 版本验证 | 检查备份版本兼容性 |

### 2.3 自动备份

| 功能 | 说明 |
|------|------|
| 每周触发 | WorkManager 定期任务 |
| 保留最近 5 个 | 自动清理旧备份 |
| 可配置 | 用户可开启/关闭 |

## 3. 技术方案

### 3.1 序列化方案

- **库**: Kotlinx Serialization
- **JSON**: `kotlinx-serialization-json`
- **优势**: 类型安全、代码生成、零反射开销

### 3.2 加密方案

| 项目 | 选择 |
|------|------|
| 算法 | AES-256-GCM |
| 密钥派生 | PBKDF2 (100,000 迭代) |
| 盐值 | 随机 16 字节/文件 |
| IV | 随机 12 字节/文件 |

### 3.3 文件格式

#### JSON 备份结构

```json
{
  "version": 1,
  "exportTime": "2024-01-15T10:30:00Z",
  "appVersion": "1.0.0",
  "encrypted": false,
  "data": {
    "todos": [...],
    "categories": [...],
    "corgiData": {...},
    "moodHistory": [...],
    "preferences": {...}
  }
}
```

#### CSV 格式

| 列名 | 数据来源 |
|------|---------|
| 标题 | TodoItem.title |
| 内容 | TodoItem.content |
| 分类 | Category.name |
| 优先级 | 高/中/低 |
| 状态 | 待办/已完成 |
| 截止日期 | yyyy-MM-dd HH:mm |
| 提醒时间 | yyyy-MM-dd HH:mm |
| 创建时间 | yyyy-MM-dd HH:mm |

## 4. 实现步骤

### 阶段 1：依赖与数据模型

1. **添加 Kotlinx Serialization 依赖**
   - 修改 `build.gradle.kts` (Project)
   - 修改 `build.gradle.kts` (App)

2. **创建备份数据模型**
   - `BackupData.kt`: 顶层数据结构
   - `BackupMeta.kt`: 元数据
   - `SerializableTodoItem.kt`: 可序列化的待办（或给现有类加注解）

### 阶段 2：核心功能

3. **实现序列化模块**
   - `JsonSerializer.kt`: JSON 序列化/反序列化
   - `CsvSerializer.kt`: CSV 生成

4. **实现加密模块**
   - `EncryptionManager.kt`: AES-256-GCM 加密/解密

5. **实现备份管理器**
   - `BackupManager.kt`: 导出/恢复核心逻辑

### 阶段 3：UI 集成

6. **创建对话框组件**
   - `ExportDialog.kt`: 导出格式选择 + 密码
   - `ImportDialog.kt`: 恢复确认
   - `PasswordDialog.kt`: 密码输入

7. **修改设置页面**
   - `SettingsScreen.kt`: 添加"数据管理"区域
   - `SettingsViewModel.kt`: 添加备份相关状态

8. **实现自动备份**
   - `AutoBackupWorker.kt`: WorkManager 任务
   - DataStore 添加自动备份设置

### 阶段 4：测试与验证

9. **单元测试**
   - 序列化测试
   - 加密解密测试

10. **集成测试**
    - 导出/恢复流程测试
    - 边界情况测试

## 6. 测试场景

### 手动测试

1. **JSON 导出**
   - 创建待办 → 导出 JSON → 验证文件生成

2. **CSV 导出**
   - 导出 CSV → 用 Excel 打开验证格式

3. **数据恢复**
   - 删除数据 → 恢复备份 → 验证数据完整性

4. **密码保护**
   - 带密码导出 → 错误密码恢复 → 正确密码恢复

5. **自动备份**
   - 开启自动备份 → 验证定期执行

### 边界情况

- 空数据库导出
- 大量数据导出（性能）
- 版本不兼容的备份文件
- 损坏的备份文件

---

## 7. 风险与注意事项

| 风险 | 应对措施 |
|------|---------|
| 数据损坏 | 恢复前验证、保留原数据 |
| 密码丢失 | 无法恢复（用户需自行保管） |
| 版本升级 | 版本号检查、迁移逻辑 |
| 大文件处理 | 协程 + IO 线程、进度提示 |
