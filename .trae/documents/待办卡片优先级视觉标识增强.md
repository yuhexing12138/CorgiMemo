# 待办卡片优先级视觉标识增强 - 实施计划

## Context（背景）

CorgiMemo 项目的待办卡片目前只在左侧 4dp 竖条上区分优先级（高/中/低），且无优先级（priority=0）时竖条完全透明。这导致以下视觉体验问题：

1. **无优先级卡片缺乏存在感**：透明竖条使卡片"无标识"，用户难以快速判断任务重要性
2. **优先级信息冗余度低**：仅靠一根 4dp 竖条，扫视时识别成本高
3. **视觉装饰单一**：与项目"治愈、温暖、舒适"的设计理念有进一步可挖掘空间

**本次改造目标**：在保留现有 4dp 优先级竖条的基础上，新增**边框 + 阴影**两层视觉标识（与竖条同色系），并为无优先级新增浅绿色，统一所有待办卡片的视觉语言。

**预期效果**：
- 用户扫视列表时可"一眼"识别每个任务的重要性等级
- 无优先级任务有明确的"低压力感"视觉（浅绿），区别于高/中/低
- 整体视觉更精致、有层次感

```kotlin
fun dimColorOf(priority: Int): Color = when (priority) {
    3 -> HighDim
    2 -> MediumDim
    1 -> LowDim
    else -> NoneDim  // ← 从 None 改为 NoneDim
}

// 文件末尾新增
data class PriorityVisual(
    val bar: Color,     // 4dp 竖条（不透明）
    val border: Color,  // 边框（调用方 .copy(alpha=0.6f)）
    val shadow: Color   // 阴影（已带 alpha=0.3f）
)

fun priorityVisualOf(priority: Int, isCompleted: Boolean = false): PriorityVisual {
    val base = if (isCompleted) dimColorOf(priority) else colorOf(priority)
    return PriorityVisual(
        bar = base,
        border = base,
        shadow = base.copy(alpha = 0.3f)
    )
}
```

**关键注释**：
- `colorOf(0)` 原本返回 `None`（透明），现自动返回 `#C8E6C9`，**正好契合用户需求 1**

- `TodoGroupContainer`（`CheckboxEditText.kt`）调用 `colorOf(priority)`，无优先级分支会自动变浅绿色

### 步骤 5：回收站卡片 `DeletedTodoCard.kt` 全面改造

**文件**：[DeletedTodoCard.kt](file:///c:/Users/EDY/Desktop/CorgiMemo/app/src/main/java/com/corgimemo/app/ui/screens/recyclebin/DeletedTodoCard.kt)

**变更点**：整个 Card 组件 + 内部布局

**修改后结构**：

```kotlin
@Composable
fun DeletedTodoCard(
    item: DeletedTodoListItem,
    onRestore: () -> Unit,
    onPermanentDelete: () -> Unit
) {
    val visual = PriorityColors.priorityVisualOf(
        priority = item.priority,
        isCompleted = false  // 回收站待办视为未完成态（已删除非主页完成态）
    )
    val barWidth = 4.dp

    Card(
        modifier = Modifier
            .fillMaxWidth()
            .padding(horizontal = 16.dp, vertical = 4.dp)
            // 优先级边框
            .border(
                width = 1.5.dp,
                color = visual.border.copy(alpha = 0.6f),
                shape = RoundedCornerShape(12.dp)
            )
            // 优先级阴影
            .shadow(
                elevation = 2.dp,
                shape = RoundedCornerShape(12.dp),
                ambientColor = visual.shadow,
                spotColor = visual.shadow
            ),
        shape = RoundedCornerShape(12.dp),
        colors = CardDefaults.cardColors(
            containerColor = MaterialTheme.colorScheme.surface
        ),
        elevation = CardDefaults.cardElevation(defaultElevation = 0.dp)
    ) {
        // 左侧 4dp 竖条 + 右侧内容（与主页 PriorityBar 同样的 4dp 宽度）
        Row(modifier = Modifier.fillMaxWidth()) {
            Box(
                modifier = Modifier
                    .width(barWidth)
                    .fillMaxHeight()
                    .background(visual.bar)
            )
            Column(modifier = Modifier
                .weight(1f)
                .padding(16.dp)
            ) {
                // 原内容：标题 / 副标题 / 行内按钮
                Text(text = item.title, ...)
                Spacer(Modifier.height(4.dp))
                Text(text = buildSubtitle(item), ...)
                Spacer(Modifier.height(12.dp))
                Row(...) {
                    TextButton(onClick = onRestore) { Text("↩ 恢复") }
                    TextButton(onClick = onPermanentDelete) { Text("🗑 永久删除", color = error) }
                }
            }
        }
    }
}
```

**新增 import**：
- `androidx.compose.foundation.background`
- `androidx.compose.foundation.border`
- `androidx.compose.foundation.layout.Box`
- `androidx.compose.foundation.layout.Row`
- `androidx.compose.foundation.layout.fillMaxHeight`
- `androidx.compose.foundation.layout.width`
- `androidx.compose.ui.draw.shadow`

**关键设计点**：
- 回收站卡片**不**调用 `pressFeedback`（只读卡片，无需按压反馈）
- 因此 `Modifier.shadow` 直接放在 Card modifier 上即可，无需担心与缩放动画的交互
- `Row` + `weight(1f)` 实现"4dp 竖条 + 右侧内容撑满"，比主页 `Layout` measurePolicy 简单
- `isCompleted = false` 因为回收站待办是"已删除"而非"已完成"，保持原始优先级色

## 风险点

| # | 风险 | 等级 | 缓解方案 |
|---|---|---|---|
| 1 | `Modifier.shadow` + `Modifier.border` 绘制顺序 | 中 | Compose modifier 顺序：左→右 = 外→内 = 后绘制。`border().shadow()` 顺序正确：border 在 surface 之上、shadow 之外 |
| 2 | `Modifier.shadow` 与 `pressFeedback` graphicsLayer 缩放叠加 | 低 | scaleDown=0.94（仅 6% 形变），肉眼几乎不可察觉；且与"卡片整体下沉"物理感一致 |
| 3 | `Modifier.shadow` 性能开销 | 低 | LazyColumn 视图复用机制会复用 modifier，硬件加速渲染，开销可控 |
| 4 | Card `elevation = 0.dp` 后失去 Material 3 默认阴影 | 低 | 已被自定义 `Modifier.shadow` 完全替代，视觉等价 |
| 5 | `colorOf(0)` 从 Transparent 改为绿色 → 影响 `PriorityBar` priority=0 行为 | 低 | 正是用户需求 1 所要求的"无优先级显示浅绿色竖条" |
| 6 | 暗色模式下优先级色（红/橙/蓝/绿）过亮 | 中 | 当前 `PriorityColors` 不区分暗色模式，与现有行为一致；如需适配，作为后续优化 |
| 7 | `DeletedTodoCard` 内 Row 高度对齐 | 中 | 竖条 Box 用 `Modifier.fillMaxHeight()`，Column 内部 padding 控制内容；如高度对不齐，参考主页 `Layout` measurePolicy 方案 |

## 后续优化建议

1. **暗色模式适配**：当前 `PriorityColors` 在暗色模式下颜色可能过亮，后续可分亮/暗色
2. **子任务卡片视觉统一**：`SubTaskInTodoListItem` 未改造，可让子卡片继承父卡片的优先级色
3. **拖拽中的视觉强化**：拖拽时把 alpha 从 0.3 提升到 0.5，强化状态感
4. **批量模式选中态边框**：选中态可让边框变粗到 2dp 或加 dashed stroke
5. **阴影可访问性优化**：在白色 surface 上 alpha 0.3 的彩色阴影可能偏淡，可考虑调整为 0.4-0.5
6. **优先级排序的视觉提示**：让首页 sort order 段位标题也用对应优先级色，形成"区域 = 重要性"的直觉
