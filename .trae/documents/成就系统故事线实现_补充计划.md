## 实施步骤（1 步 + 验证）

### 步骤 1：修改 `AchievementScreen.kt`

**文件路径**: `app/src/main/java/com/corgimemo/app/ui/screens/achievement/AchievementScreen.kt`

**修改内容**：

1. **新增 import**：

   - `animateColorAsState`, `tween`（动画库）
   - `Box`, `background`（布局）
   - `StageProgressSection`, `NextAchievementPreview`（新组件）
   - `stageUnlockedCount`, `stageTotalCount`, `nextUnlockableAchievement`, `nextUnlockableProgress`
2. **新增状态收集**（与现有状态收集并列）：

   - `val stageUnlockedCount by viewModel.stageUnlockedCount.collectAsState()`
   - `val stageTotalCount by viewModel.stageTotalCount.collectAsState()`
   - `val nextAchievement by viewModel.nextUnlockableAchievement.collectAsState()`
   - `val nextProgress by viewModel.nextUnlockableProgress.collectAsState()`
3. **新增动态背景色**：

   - 使用 `animateColorAsState` 根据 `currentStage` 变化
   - 初见 → `Color(0xFFF1F5F9)` 石板灰
   - 成长 → `Color(0xFFECFDF5)` 翠绿
   - 飞跃 → `Color(0xFFEFF6FF)` 蓝色
   - 巅峰 → `Color(0xFFFFF7ED)` 暖橙
   - 动画 800ms 平滑过渡
4. **页面包裹 Box + 背景**：

   - 将现有 Scaffold 内容包裹在 `Box(Modifier.fillMaxSize().background(bgColor))` 中
   - (注意：Scaffold 本身不能直接包在 Box 里，需将 Scaffold 的 content 区域包裹 Box 或设置 Scaffold containerColor)
5. **在整体进度卡片下方添加 StageProgressSection**：

   - 位置：`OverallProgressCard` 和 `CorgiStageCard` 之间
   - 传入：`currentStage`, `currentStage.displayName`, `stageUnlockedCount`, `stageTotalCount`
6. **在柯基展示卡片下方添加 NextAchievementPreview**：

   - 位置：`CorgiStageCard` 和成就 Grid 之间
   - 传入：`nextAchievement`, `nextProgress`
7. **移除 BottomSheet 相关逻辑**（可选优化）：

   - 删除 `AchievementDetailSheet` 相关代码
   - 删除 `selectedAchievement`/`selectedIsUnlocked`/`selectedProgress` 相关状态处理
   - (因 3D 翻转已替代底部 Sheet 的详情展示功能)

**⚠️ 注意**：Scaffold 的 backgroundColor 不能直接设置动态颜色。最佳实践是通过 `Scaffold` 的 `containerColor` 参数设置，或在 Scaffold 的 content lambda 中用 Box 包裹背景。
