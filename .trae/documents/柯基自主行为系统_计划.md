### 问题分析

1. **无空闲检测机制**：没有追踪用户最后操作时间的逻辑
2. **无行为管理类**：5 种自主行为逻辑需要分散在 ViewModel 中实现
3. **无行为状态**：UI 层无法感知当前行为类型（如打哈欠显示 "Zzz"）
4. **数据持久化不足**：需要追踪最后活跃日期用于"被忽略想念"检测

## 二、需求分析

### 5 种柯基自主行为

| 行为类型 | 触发条件 | 表现效果 | 特殊需求 |
|---------|---------|---------|---------|
| **空闲打哈欠** | 用户 10 秒无操作 | 打哈欠动画 + "Zzz" | 冷却 2 分钟 |
| **深夜入睡** | 23:00 后首次打开 APP | 睡觉姿态 | 仅首次打开触发 |
| **待办堆积担心** | 待办数 > 10 且 > 50% 超期 | 皱眉表情 | 持续显示直到改善 |
| **被忽略想念** | 3 天未打开 APP | 委屈表情 + "柯基想你了" | 首次打开显示 |
| **连续完成开心** | 30 秒内连续完成 3 个任务 | 尾巴摇得更欢 | 30 秒窗口 |

### 打哈欠资源说明

用户明确说明：**打哈欠暂无帧图片资源，暂时用占位符，后续再替换。**

## 三、实现方案

### 3.1 新增 CorgiBehaviorManager 类

**文件：** `CorgiMemo/app/src/main/java/com/corgimemo/app/animation/CorgiBehaviorManager.kt`

**职责：**
- 统一管理 5 种自主行为的检测逻辑
- 提供行为状态判断的工具函数
- 管理行为冷却时间

```kotlin
/**
 * 柯基行为类型枚举
 */
enum class BehaviorType {
    NONE,           // 无特殊行为
    YAWNING,        // 打哈欠（空闲）
    SLEEPING_NIGHT, // 深夜入睡
    WORRIED,        // 待办堆积担心
    MISSED_YOU,     // 被忽略想念
    HAPPY_STREAK    // 连续完成开心
}

/**
 * 柯基行为状态
 */
data class BehaviorState(
    val type: BehaviorType = BehaviorType.NONE,
    val message: String = "",
    val startTime: Long = 0L
)

object CorgiBehaviorManager {
    // 时间常量
    const val IDLE_YAWN_THRESHOLD_MS = 10_000L      // 10秒无操作触发打哈欠
    const val YAWN_COOLDOWN_MS = 120_000L           // 打哈欠冷却 2 分钟
    const val NIGHT_SLEEP_HOUR = 23                 // 23点后入睡
    const val MISSED_DAYS_THRESHOLD = 3             // 3天未打开
    const val HAPPY_STREAK_WINDOW_MS = 30_000L      // 30秒窗口
    const val HAPPY_STREAK_COUNT = 3                // 连续完成 3 个
    
    // 检测函数...
}
```

## 四、详细实现步骤

### 阶段一：数据层准备（CorgiPreferences）

**目标：** 增加行为追踪所需的持久化存储

**步骤：**
1. 新增 `longPreferencesKey` 用于存储最后活跃时间戳
2. 新增 `stringPreferencesKey` 用于标记今日深夜入睡检测
3. 实现对应的读写方法

**文件修改：**
- `datastore/CorgiPreferences.kt`

---

### 阶段二：行为管理器（CorgiBehaviorManager）

**目标：** 创建集中管理所有行为检测逻辑的类

**步骤：**
1. 定义 `BehaviorType` 枚举（5 种行为）
2. 定义 `BehaviorState` 数据类
3. 实现行为检测工具函数：
   - `isNightTime(hour: Int): Boolean` - 判断是否 23 点后
   - `shouldWorry(todoCount: Int, overdueCount: Int): Boolean` - 判断待办堆积
   - `wasMissed(daysDiff: Int): Boolean` - 判断是否 3 天未打开
   - `hasHappyStreak(times: List<Long>, windowMs: Long): Boolean` - 判断连续完成

**文件修改：**
- 新增 `animation/CorgiBehaviorManager.kt`

---

### 阶段三：ViewModel 集成（HomeViewModel）

**目标：** 集成行为检测逻辑到 ViewModel

**步骤：**

**3.1 状态初始化**
1. 新增 `currentBehavior` 状态流
2. 新增 `lastUserInteractionTime` 变量（空闲检测）
3. 新增 `idleCheckJob` 协程任务
4. 新增 `recentCompletedTimes` 列表（连续完成检测）

**3.2 启动时行为检测**
1. 在 `init { }` 中调用 `checkStartupBehaviors()`
2. 检查"被忽略想念"（3 天未打开）
3. 检查"深夜入睡"（23 点后首次打开）

**3.3 空闲打哈欠机制**
1. 实现 `startIdleDetection()` - 每 1 秒检查一次
2. 实现 `reportUserInteraction()` - 用户操作时重置计时器
3. 检测逻辑：
   - 如果 10 秒无操作 → 触发打哈欠
   - 冷却 2 分钟内不再触发
   - 打哈欠持续约 2 秒后恢复正常

**3.4 连续完成检测**
1. 在 `handleTaskCompleted()` 中记录完成时间
2. 检查最近 30 秒内是否完成 3 个任务
3. 若是 → 设置 `HAPPY_STREAK` 行为

**3.5 待办堆积担心**
1. 监听 `todos` 状态变化
2. 每次变化时检查：待办数 > 10 且 > 50% 超期
3. 若是 → 设置 `WORRIED` 行为

**3.6 暴露用户操作入口**
- 新增 `onUserInteraction()` 方法供 UI 层调用

**文件修改：**
- `viewmodel/HomeViewModel.kt`

## 七、风险评估

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 打哈欠动画资源缺失 | 中 | 使用现有姿态占位，添加 TODO 注释 |
| 空闲检测协程泄漏 | 高 | ViewModel 生命周期管理，`onCleared()` 中取消 |
| 行为冲突（多个同时触发） | 中 | 定义行为优先级：MISSED_YOU > SLEEPING > WORRIED > HAPPY_STREAK > YAWNING |
| 连续完成时间窗口不准确 | 低 | 使用 `System.currentTimeMillis()` 精确计算 |

---

## 八、优先级说明

当多个行为条件同时满足时，按以下优先级显示：

```
1. MISSED_YOU（被忽略想念）- 启动时最高优先级
2. SLEEPING_NIGHT（深夜入睡）- 时间相关
3. WORRIED（待办堆积）- 状态相关
4. HAPPY_STREAK（连续开心）- 近期行为
5. YAWNING（打哈欠）- 最低优先级，可被覆盖
```
