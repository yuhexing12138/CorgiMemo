# 侧滑导航栏（Drawer）实现计划

## 概述
基于 `docs/superpowers/specs/侧滑导航栏深度优化指导文档.md`，在现有 CorgiMemo 应用中实现侧滑导航栏功能。使用 Jetpack Compose Material 3 的 `ModalNavigationDrawer` 组件。

## 技术方案
- **Drawer 类型**：Material 3 `ModalNavigationDrawer`
- **状态管理**：`DrawerState` 控制开/关，`HomeViewModel` 管理数据状态
- **数据流**：`HomeViewModel` 提供分类列表 + 待办计数，抽屉组件只负责展示

## 需要修改/创建的文件（共 9 个）

### 1. 🆕 新建：`DeletedTodoEntity` 实体
- **路径**: `app/src/main/java/com/corgimemo/app/data/model/DeletedTodo.kt`
- 新建一个 Room 实体，包含 TodoItem 的所有字段 + `deletedAt` 时间戳

### 4. 🔧 修改：`CorgiMemoDatabase.kt`
- **路径**: `app/src/main/java/com/corgimemo/app/data/local/db/CorgiMemoDatabase.kt`
- 新增 `DeletedTodoEntity` 到 entities 列表
- 新增 `deletedTodoDao()` 抽象方法
- 新增数据库迁移 `MIGRATION_15_16`（创建 deleted_todos 表）

### 5. 🆕 新建：`AppDrawer.kt`（核心 UI 组件）
- **路径**: `app/src/main/java/com/corgimemo/app/ui/components/AppDrawer.kt`

  - `CategoryGroupSection`：分组管理区域（标题+暖橙色下划线+分组列表）

### 6. 🔧 修改：`HomeViewModel.kt`
- **路径**: `app/src/main/java/com/corgimemo/app/viewmodel/HomeViewModel.kt`
- **新增状态**：
  - `selectedCategoryId: StateFlow<Long?>` — 当前选中的分类 ID（null=全部）
  - `todoCountByCategory: Map<Long, Int>` — 每个分类的待办计数（computed from _todos）

- **新增方法**：

  - `deleteCategory(id: Long)` — 删除自定义分类
  - `pinCategory(id: Long)` — 置顶分类（改变排序）
  - `restoreDeletedTodo(todoId: Long)` — 恢复删除的待办
  - `permanentlyDeleteOldTodos()` — 清空过期删除记录
- **修改现有方法**：
  - `deleteTodo()` — 删除时先存入 DeletedTodo 表
  - `batchDelete()` — 批量删除时先存入 DeletedTodo 表
  - `filteredTodos` — 组合 _selectedCategoryId 过滤

## 数据流设计

```
用户操作 → HomeScreen (ModalNavigationDrawer)
              ↓
         AppDrawerContent (展示)
              ↓
         HomeViewModel (状态管理)
              ↓
    ┌───────┴───────┐
    ↓               ↓
CategoryRepo  DeletedTodoRepo
    ↓               ↓
  Room DB        Room DB
```

## 抽屉组件树

```
ModalNavigationDrawer (Material 3)
  ├── drawerContent:
  │   └── AppDrawerContent
  │       ├── UserProfileSection
  │       │   ├── 圆形头像 (CircleShape, 48dp)
  │       │   ├── 昵称 (18sp, FontWeight.Bold)
  │       │   └── Lv.5 标签 + 阶段描述
  │       ├── CategoryGroupSection
  │       │   ├── 标题 "分组管理" (16sp Bold + 暖橙色下划线)
  │       │   ├── CategoryItem "全部待办 (N)" → selectedCategoryId=null
  │       │   ├── CategoryItem "未分类 (N)" → selectedCategoryId=0
  │       │   ├── CategoryItem (自定义分组 × N) → selectedCategoryId=id
  │       │   └── CategoryItem "最近删除 (N)" (灰色, 🗑️)
  │       ├── AddCategoryButton (暖橙色圆角按钮)
  │       └── 底部入口区
  │           ├── ⚙️ 设置
  │           └── ❓ 帮助与反馈
  └── content:
      └── Scaffold (现有的 HomeScreen 内容)
```

## 交互逻辑

| 操作 | 行为 |
|------|------|
| 左边缘右滑 | 打开 Drawer（ModalNavigationDrawer 内置） |
| 点击遮罩层 | 关闭 Drawer（ModalNavigationDrawer 内置） |
| 点击 "全部(N)" | 关闭抽屉，selectedCategoryId=null，显示所有待办 |
| 点击 "未分类(N)" | 关闭抽屉，筛选 categoryId 不在任何分类中的待办 |
| 点击 自定义分组 | 关闭抽屉，筛选该分类的待办，标题显示分组名 |
| 点击 ⋮ 菜单 | 弹出 CategoryOperationSheet BottomSheet |
| 点击 "添加分组" | 弹出 AddCategoryDialog 输入弹窗 |
| 删除分组 | 弹出二次确认弹窗 |
| 点击 "最近删除" | 展开最近删除列表 / 导航到最近删除页 |

## 视觉规范
- 抽屉宽度: 280dp

- 分组项高度: 56dp
- 暖橙色: `UiColors.Primary` = `Color(0xFFFF9A5C)`
- 添加按钮: 暖橙色圆角矩形，高度 48dp
- "最近删除": 灰色文字
