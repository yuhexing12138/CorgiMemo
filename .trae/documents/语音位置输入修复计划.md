# 语音输入和位置输入修复计划

## 问题概述

新建 `TodoCreateBottomSheet` 组件时，遗漏了原有 `TodoEditScreen` 中的两个重要功能：
- 语音输入（语音转文字）
- 位置输入（地理围栏）

## 调研结论

### 原有功能实现位置

| 功能 | 实现文件 | 关键代码位置 |
|------|----------|-------------|
| 语音输入 | `TodoEditScreen.kt` | 84-128 行（状态管理）、155-202 行（UI组件） |
| 位置输入 | `TodoEditScreen.kt` | 76-82 行（状态）、257-278 行（UI组件） |

### 需要添加的组件

| 组件 | 功能说明 |
|------|----------|
| `SpeechViewModel` | 语音识别核心逻辑 |
| `RecordingIndicator` | 录音状态指示器 |
| `RecordingWaveAnimation` | 录音波形动画 |
| `LocationPicker` | 位置选择器（地理围栏） |

## 实施步骤

### 第一阶段：添加语音输入功能

**修改文件**：`TodoCreateBottomSheet.kt`

**修改内容**：
1. 添加必要的导入（权限、ActivityResultLauncher 等）
2. 在组件中创建 `SpeechViewModel` 实例
3. 添加录音权限请求逻辑
4. 在标题 TextField 右侧添加麦克风图标按钮
5. 添加录音状态显示（"正在聆听..."、"正在识别..."）
6. 添加录音波形动画
7. 将语音识别结果自动填入标题

### 第二阶段：添加位置输入功能

**修改文件**：`TodoCreateBottomSheet.kt`

**修改内容**：
1. 添加 `LocationPicker` 组件导入
2. 从 `TodoEditViewModel` 收集地理围栏相关状态
3. 在日期选择按钮后添加 `LocationPicker` 组件
4. 实现位置变更回调（更新 ViewModel 状态）

### 第三阶段：编译验证

**操作**：执行 `gradle :app:compileDebugKotlin`

**验证标准**：
- 编译无错误
- 无新增警告
- 底部弹窗包含语音输入按钮和位置选择器

## 潜在风险

| 风险 | 影响 | 规避措施 |
|------|------|----------|
| 权限请求在 BottomSheet 中可能有问题 | 语音功能无法使用 | 确保权限 launcher 在组件顶层创建 |
| LocationPicker 组件过大，BottomSheet 可能不美观 | UI 体验差 | 保持 LocationPicker 的折叠/展开设计 |
| 录音动画和底部弹窗动画冲突 | 视觉体验差 | 使用 AnimatedVisibility 控制动画显示 |

## 文件变更清单

| 文件 | 变更类型 | 说明 |
|------|----------|------|
| `app/src/main/java/com/corgimemo/app/ui/components/TodoCreateBottomSheet.kt` | 修改 | 添加语音输入和位置输入功能 |

## 预期效果

修改后，用户点击右下角 "+" 按钮时：
1. 底部弹窗滑出（柯基在背景可见，坐立姿态）
2. 标题输入框右侧有麦克风图标按钮
3. 点击麦克风可以语音输入待办标题
4. 底部有位置选择器，可以设置地理围栏
5. 关闭弹窗后柯基恢复趴卧姿态
