# 通知横幅操作按钮实现计划

## 一、需求分析

### 1.1 功能需求

| 需求 | 说明 |
|------|------|
| 通知展开时显示三个操作按钮 | "完成"、"稍后10分钟"、"稍后1小时" |
| 点击"完成" | 标记待办为已完成，关闭通知 |
| 点击"稍后10分钟/1小时" | 修改提醒时间，重新设定闹钟 |
| 使用 NotificationCompat.Builder.addAction() | 添加操作按钮 |
| 使用 BroadcastReceiver | 接收 Action 并处理 |
| Android 12+ 适配 | 使用 NotificationCompat.CATEGORY_REMINDER |

### 1.2 技术要求

- Android 通知操作按钮 + BroadcastReceiver + AlarmManager

## 三、实现方案

### 3.1 整体架构

```
时间到期 / 位置触发
    ↓
BroadcastReceiver (ReminderActionReceiver)
    ↓
onReceive()
    ↓
根据 Action 类型判断
    ├─ ACTION_COMPLETE → 标记待办完成 + 关闭通知
    ├─ ACTION_SNOOZE_10M → 更新 reminderTime +10分钟 + 重新设闹钟
    └─ ACTION_SNOOZE_1H → 更新 reminderTime +1小时 + 重新设闹钟
```

### 3.2 架构设计

#### 3.2.1 通知 Action 常量

| Action 常量 | 说明 |
|-----------|------|
| `ACTION_COMPLETE` | 标记完成 |
| `ACTION_SNOOZE_10M` | 稍后10分钟 |
| `ACTION_SNOOZE_1H | 稍后1小时 |

#### 3.2.2 Intent Extra 数据

| Extra Key | 类型 | 说明 |
|------------|------|------|
| `EXTRA_TODO_ID` | Long | 待办 ID |
| `EXTRA_NOTIFICATION_ID` | Int | 通知 ID |

### 3.3 通知构建流程

```
NotificationCompat.Builder(context, CHANNEL_ID)
    ├─ setSmallIcon()
    ├─ setContentTitle()
    ├─ setContentText()
    ├─ setStyle(NotificationCompat.BigTextStyle())  ← 展开显示完整内容
    ├─ setContentIntent()                            ← 点击通知打开 App
    ├─ setCategory(NotificationCompat.CATEGORY_REMINDER) ← Android 12+ 适配
    ├─ addAction(完成按钮)                          ← 新功能
    ├─ addAction(稍后10分钟按钮)                    ← 新功能
    ├─ addAction(稍后1小时按钮)                     ← 新功能
    └─ notify()
```

## 四、详细实现步骤

### 步骤 1：创建 BroadcastReceiver

#### 1.1 创建 ReminderActionReceiver.kt

**文件路径：`receiver/ReminderActionReceiver.kt

**功能说明：**
- 接收通知操作按钮的点击事件
- 处理三种 Action：完成、稍后10分钟、稍后1小时
- 使用 WorkManager 或直接操作数据库

**代码结构：**

```kotlin
package com.corgimemo.app.receiver

import android.content.BroadcastReceiver
import android.content.Context
import android.content.Intent
import androidx.work.OneTimeWorkRequestBuilder
import androidx.work.WorkManager
import androidx.work.workDataOf

/**
 * 通知操作按钮广播接收器
 * 处理通知中的"完成"、"稍后"等操作
 */
class ReminderActionReceiver : BroadcastReceiver() {

    companion object {
        // Action 常量
        const val ACTION_COMPLETE = "com.corgimemo.app.ACTION_COMPLETE"
        const val ACTION_SNOOZE_10M = "com.corgimemo.app.ACTION_SNOOZE_1H" = "com.corgimemo.app.ACTION_SNOOZE_1H"

        // Extra 常量
        const val EXTRA_TODO_ID = "extra_todo_id"
        const val EXTRA_NOTIFICATION_ID = "extra_notification_id"

        // Snooze 时间（毫秒）
        const val SNOOZE_10_MINUTES = 10 * 60 * 1000L
        const val SNOOZE_1_HOUR = 60 * 60 * 1000L
    }

    override fun onReceive(context: Context, intent: Intent) {
        val todoId = intent.getLongExtra(EXTRA_TODO_ID, -1)
        val notificationId = intent.getIntExtra(EXTRA_NOTIFICATION_ID, 0)

        if (todoId == -1L) return

        when (intent.action) {
            ACTION_COMPLETE -> {
                // 标记待办完成
                scheduleCompleteTodo(context, todoId, notificationId)
            }
            ACTION_SNOOZE_10M -> {
                // 稍后10分钟
                scheduleSnoozeTodo(context, todoId, notificationId, SNOOZE_10_MINUTES)
            }
            ACTION_SNOOZE_1H -> {
                // 稍后1小时
                scheduleSnoozeTodo(context, todoId, notificationId, SNOOZE_1_HOUR)
            }
        }
    }

    /**
     * 调度完成待办的 Worker
     */
    private fun scheduleCompleteTodo(
        context: Context,
        todoId: Long,
        notificationId: Int
    ) {
        val workRequest = OneTimeWorkRequestBuilder<CompleteTodoWorker>()
            .setInputData(
                workDataOf(
                    CompleteTodoWorker.KEY_TODO_ID to todoId,
                    CompleteTodoWorker.KEY_NOTIFICATION_ID to notificationId
                )
            )
            .build()
        WorkManager.getInstance(context).enqueue(workRequest)
    }

    /**
     * 调度稍后提醒的 Worker
     */
    private fun scheduleSnoozeTodo(
        context: Context,
        todoId: Long,
        notificationId: Int,
        delayMillis: Long
    ) {
        val workRequest = OneTimeWorkRequestBuilder<SnoozeTodoWorker>()
            .setInputData(
                workDataOf(
                    SnoozeTodoWorker.KEY_TODO_ID to todoId,
                    SnoozeTodoWorker.KEY_NOTIFICATION_ID to notificationId,
                    SnoozeTodoWorker.KEY_DELAY_MILLIS to delayMillis
                )
            )
            .build()
        WorkManager.getInstance(context).enqueue(workRequest)
    }
}
```

### 步骤 2：创建 Worker 类

#### 2.1 CompleteTodoWorker.kt

**文件路径：** `worker/CompleteTodoWorker.kt

**功能：** 标记待办完成 + 关闭通知

```kotlin
package com.corgimemo.app.worker

import android.app.NotificationManager
import android.content.Context
import androidx.work.CoroutineWorker
import androidx.work.WorkerParameters
import com.corgimemo.app.CorgiMemoApplication
import com.corgimemo.app.data.repository.TodoRepository
import javax.inject.Inject

/**
 * 完成待办的 Worker
 * 在后台线程执行数据库操作
 */
class CompleteTodoWorker(
    context: Context,
    params: WorkerParameters
) : CoroutineWorker(context, params) {

    companion object {
        const val KEY_TODO_ID = "todo_id"
        const val KEY_NOTIFICATION_ID = "notification_id"
    }

    @Inject
    lateinit var todoRepository: TodoRepository

    override suspend fun doWork(): Result {
        // 获取 App 组件注入
        (applicationContext as CorgiMemoApplication).appComponent.inject(this)

        val todoId = inputData.getLong(KEY_TODO_ID, -1)
        val notificationId = inputData.getInt(KEY_NOTIFICATION_ID, 0)

        if (todoId == -1L) return Result.failure()

        return try {
            // 1. 从数据库获取待办
            val todo = todoRepository.getTodoById(todoId)
            todo?.let {
                // 2. 更新为已完成
                val updatedTodo = it.copy(
                    status = 1,
                    completedAt = System.currentTimeMillis(),
                    updatedAt = System.currentTimeMillis()
                )
                todoRepository.updateTodo(updatedTodo)
            }

            // 3. 关闭通知
            val notificationManager = applicationContext.getSystemService(
                Context.NOTIFICATION_SERVICE) as NotificationManager
            notificationManager.cancel(notificationId)

            Result.success()
        } catch (e: Exception) {
            Result.retry()
        }
    }
}
```

#### 2.2 SnoozeTodoWorker.kt

import android.content.Context
import androidx.work.CoroutineWorker
import androidx.work.WorkerParameters
import com.corgimemo.app.CorgiMemoApplication
import com.corgimemo.app.data.repository.TodoRepository
import javax.inject.Inject

/**
 * 稍后提醒的 Worker
 * 更新 reminderTime 并重新设置闹钟
 */
class SnoozeTodoWorker(
    context: Context,
    params: WorkerParameters
) : CoroutineWorker(
    override suspend fun doWork(): Result {
        // 获取 App 组件注入
        (applicationContext as CorgiMemoApplication).appComponent.inject(this)

        val todoId = inputData.getLong(KEY_TODO_ID, -1)
        val notificationId = inputData.getInt(KEY_NOTIFICATION_ID, 0)
        val delayMillis = inputData.getLong(KEY_DELAY_MILLIS, 0)

        if (todoId == -1L) return Result.failure()

        return try {
            // 1. 从数据库获取待办
            val todo = todoRepository.getTodoById(todoId)
            todo?.let {
                // 2. 计算新的 reminderTime
                val newReminderTime = System.currentTimeMillis() + delayMillis
                val updatedTodo = it.copy(
                    reminderTime = newReminderTime,
                    updatedAt = System.currentTimeMillis()
                )
                todoRepository.updateTodo(updatedTodo)

                // 3. 重新设置闹钟（如果有 AlarmScheduler 实现）

            }

            // 4. 关闭当前通知
            val notificationManager = applicationContext.getSystemService(
                Context.NOTIFICATION_SERVICE) as NotificationManager
            notificationManager.cancel(notificationId)

            Result.success()
        } catch (e: Exception) {
            Result.retry()
        }
    }
}

## 六、注意事项

### 6.1 PendingIntent 的 requestCode 唯一性

**问题：** 多个通知的 PendingIntent 需要不同的 requestCode 会互相覆盖。

**解决方案：**
- 使用 `todo.id.toInt()` 作为基础 requestCode
- 完成按钮：`todo.id.toInt()`
- 稍后10分钟：`todo.id.toInt() + 1`
- 稍后1小时：`todo.id.toInt() + 2`

### 6.2 Android 12+ PendingIntent 标志

**必须使用 `FLAG_IMMUTABLE` 或 `FLAG_MUTABLE`

当前代码使用 `FLAG_IMMUTABLE`（推荐）

### 6.3 NotificationCompat.CATEGORY_REMINDER 的作用

| 版本 | 行为 |
|------|------|
| Android 12+ | 系统可以智能处理提醒类型通知 |
| Android 12- | 忽略该属性 |

### 6.4 WorkManager vs 直接在 BroadcastReceiver 中直接操作数据库

**为什么使用 WorkManager：

| 方案 | 优点 | 缺点 |
|------|------|------|
| **WorkManager（推荐） | 后台线程、可重试、生命周期管理 | 需要额外配置 |
| **直接操作 | 简单 | 主线程阻塞、ANR 风险 |

选择 **WorkManager** 方案更稳健。

### 6.5 通知 ID 唯一性

**问题：** 多个待办同时提醒时，通知 ID 需要唯一。

**解决方案：**
- 当前使用 `NOTIFICATION_ID = 1001`（固定）
- 建议：使用 `todo.id.toInt()` 作为通知 ID
