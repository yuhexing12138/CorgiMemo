### 任务 A: 相机/相册实现

#### 修改文件: `TodoEditScreen.kt`

**步骤 1**: 在 Composable 函数顶部添加 Launcher 声明

```kotlin
// 在现有状态声明区域（约第 130 行附近）添加：

/** 相机拍照 Launcher */
val cameraLauncher = rememberLauncherForActivityResult(
    contract = ActivityResultContracts.TakePicture()
) { uri: Uri? ->
    uri?.let {
        /** 将拍摄的照片复制到应用内部存储 */
        val savedPath = ImageUtils.copyUriToInternalStorage(context, it)
        viewModel.addImagePath(savedPath)
    }
}

/** 相册选择 Launcher（支持多选） */
val galleryLauncher = rememberLauncherForActivityResult(
    contract = ActivityResultContracts.GetMultipleContents()
) { uris: List<Uri> ->
    uris.forEach { uri ->
        val savedPath = ImageUtils.copyUriToInternalStorage(context, uri)
        viewModel.addImagePath(savedPath)
    }
}

/** 相机权限请求 Launcher */
val cameraPermissionLauncher = rememberLauncherForActivityResult(
    contract = ActivityResultContracts.RequestPermission()
) { isGranted: Boolean ->
    if (isGranted) {
        /** 权限已授予，启动相机 */
        val photoUri = ImageUtils.createImageUri(context)
        cameraLauncher.launch(photoUri)
    } else {
        /** 权限被拒绝，显示提示 */
        // TODO: 显示 Snackbar 提示用户去设置中开启权限
    }
}
```

**步骤 2**: 替换 ImagePickerDialog 的空回调（第 916-927 行）

```kotlin
if (showImagePicker) {
    ImagePickerDialog(
        onCameraSelected = {
            /** 检查并请求相机权限，然后启动相机 */
            checkAndRequestCameraPermission(
                context = context,
                permissionLauncher = cameraPermissionLauncher,
                onPermissionGranted = {
                    val photoUri = ImageUtils.createImageUri(context)
                    cameraLauncher.launch(photoUri)
                },
                onPermissionDenied = {
                    // 显示权限拒绝提示
                }
            )
        },
        onGallerySelected = {
            /** 打开系统相册选择器 */
            galleryLauncher.launch("image/*")
        },
        onDismiss = { showImagePicker = false }
    )
}
```

**新增依赖**: 需要确认 `ImageUtils` 工具类是否存在以下方法：
- `createImageUri(context)`: 创建用于相机拍照的临时 URI（FileProvider 兼容）

### 任务 B: 背景色持久化

#### B.1 修改 `TodoItem.kt` - 添加字段

在 [TodoItem.kt](file:///c:\Users\EDY\Desktop\CorgiMemo\app\src\main\java\com\corgimemo\app\data\model\TodoItem.kt) 的 data class 末尾（`imagePaths` 字段之后）添加：

```kotlin
/**
 * 背景颜色值（ARGB 整数）
 * 用于持久化用户选择的卡片背景色
 * 默认值为 0xFFFFFFFF（白色/透明）
 *
 * 存储方式：Color.toArgb() → Int → 数据库
 * 读取方式：数据库 Int → Color(ColorInt)
 */
@ColumnInfo(defaultValue = "16777215") // 0xFFFFFFFF = 白色
val backgroundColor: Int = 0xFFFFFFFF
```

> **设计决策**: 使用 `Int` 类型（ARGB 值）而非 `String`，原因：
> 1. 更节省存储空间（4 字节 vs 变长字符串）
> 2. 与 Android Color 类原生兼容（`Color.toInt()` / `Color()`）
> 3. 避免 JSON 解析开销

### 任务 C: 富文本编辑器集成

#### 修改文件: `TodoEditScreen.kt`

**核心思路**: 用 `RichTextEditor` + `TextFormatToolbar` 替换现有的 `OutlinedTextField`（内容编辑区），同时保持与 ViewModel 的 `content` StateFlow 双向同步。

**步骤 1**: 添加 RichTextEditorState 状态

```kotlin
// 在状态声明区域添加：
/** 富文本编辑器状态（管理 AnnotatedString 和格式状态） */
val editorState = remember { RichTextEditorState() }

/** 初始化编辑器内容（从 ViewModel 加载时同步） */
LaunchedEffect(content) {
    if (editorState.textFieldValue.text != content) {
        editorState.textFieldValue = TextFieldValue(AnnotatedString(content))
    }
}
```

**步骤 2**: 替换 OutlinedTextField（第 519-560 行）

将以下代码：
```kotlin
OutlinedTextField(
    value = content,
    onValueChange = { newValue ->
        viewModel.setContent(newValue)
        // ... @ 检测逻辑
    },
    // ...
)
```

替换为：
```kotlin
/** 富文本编辑器（替代原有的 OutlinedTextField） */
RichTextEditor(
    state = editorState,
    onValueChange = { newValue ->
        /** 同步纯文本内容到 ViewModel（用于保存和搜索） */
        viewModel.setContent(newValue.text)
        editorState.textFieldValue = newValue

        /** 检测 @ 字符输入，触发关联选择弹窗 */
        if ("@" in newValue.text && !showMentionPopup) {
            showMentionPopup = true
            mentionQuery = ""
        }
    },
    modifier = Modifier
        .fillMaxWidth()
        .height(200.dp), /** 固定高度，避免高度跳动 */
    placeholder = "输入详细描述（支持 **粗体**、*斜体*、~~删除线~~ 等格式）",
    enabled = true
)
```

**步骤 3**: 连接 EditToolbar 的格式按钮回调

修改 EditToolbar 的回调（约第 380-400 行），连接到 RichTextEditor 的格式函数：

## 技术风险与缓解措施

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| FileProvider 配置缺失导致相机崩溃 | 高 | 检查 AndroidManifest.xml 是否已配置 provider；如未配置需添加 |
| Room 迁移失败导致数据丢失 | 高 | 使用 ALTER TABLE ADD COLUMN（安全操作）；defaultValue 设为白色 |
| RichTextEditor 与 @提及检测冲突 | 中 | 在 `onValueChange` 中保留原有 @ 检测逻辑不变 |
| Color.toArgb() 版本兼容性 | 低 | 使用 `androidx.compose.ui.graphics.Color.toArgb()` （Compose 1.0+ 均支持） |
