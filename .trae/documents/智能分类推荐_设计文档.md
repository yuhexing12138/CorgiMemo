**日期**: 2026-05-21

## 1. 项目背景与目标

### 1.1 现状

- 成就系统已实现
- 数据库中 TodoItem 含 categoryId 字段（分类管理已实现）
- 身份切换时默认分类列表已实现（上班族/学生不同分类）
- 分类选择器 UI 尚未实现
- 智能分类推荐功能尚未实现

### 1.2 目标

实现根据待办内容自动推荐分类的功能，包括：

1. **关键词-分类映射数据模型**

   - 预设关键词映射表（硬编码）
   - 支持用户自定义关键词（Room 数据库存储）
2. **智能推荐引擎**

   - 输入：待办标题 + 详情
   - 输出：推荐的分类（得分相同时返回 null）
   - 性能：匹配耗时 < 10ms
3. **用户自定义关键词管理**

   - 设置入口：我的 → 设置 → 智能分类设置
   - 支持添加、删除、修改关联分类
4. **UI 集成**

   - 分类选择器（卡片列表形式）
   - 智能推荐标签（暖橙色，点击自动选中）
   - 语音输入同样触发推荐

## 2. 核心决策

| 决策项           | 选择                                       | 原因                               |
| ---------------- | ------------------------------------------ | ---------------------------------- |
| 架构模式         | 分层架构（数据层→仓库层→匹配引擎→UI层） | 职责清晰，易于维护和测试           |
| 关键词存储       | 预设（硬编码）+ 用户自定义（Room）         | 预设不可修改，用户自定义可灵活管理 |
| 匹配方式         | 精确匹配优先 + 模糊匹配兜底                | 精确匹配权重更高，用户体验更好     |
| 分类选择器       | 卡片列表形式（类似优先级选择器）           | 与现有 UI 风格一致，操作直观       |
| 推荐触发         | 防抖 300ms                                 | 避免频繁匹配，减少性能开销         |
| 用户关键词优先级 | 高于预设关键词                             | 尊重用户自定义配置                 |

#### 3.2.1 MatchType 枚举

enum class MatchType(val weight: Int) {
    EXACT(10),      // 精确匹配权重
    FUZZY(3)        // 模糊匹配权重
}

#### 3.2.2 CategoryKeyword 数据类

```kotlin
data class CategoryKeyword(
    val id: Long = 0,
    val keyword: String,           // 关键词文本
    val categoryType: Int,         // 关联分类类型（CategoryType）
    val matchType: MatchType,      // 匹配类型
    val isUserDefined: Boolean     // 是否用户自定义
) {
    val weight: Int
        get() = if (isUserDefined) matchType.weight * 2 else matchType.weight
}
```

**权重规则**：

- 用户自定义关键词权重是预设关键词的 2 倍
- 精确匹配权重是模糊匹配的约 3 倍

## 5. 智能推荐引擎设计

### 5.1 CategoryMatcher 类

```kotlin
@Singleton
class CategoryMatcher @Inject constructor(
    private val categoryKeywordRepository: CategoryKeywordRepository
) {
  
    data class MatchResult(
        val categoryType: Int,
        val score: Int,
        val matchedKeywords: List<String>
    )
  
    data class Recommendation(
        val categoryType: Int,
        val score: Int,
        val matchedKeywords: List<String>
    )
  
    /**
     * 根据文本推荐分类
     * @param title 待办标题
     * @param content 待办详情（可选）
     * @return 推荐结果，若无明确推荐则返回 null
     */
    fun recommendCategory(title: String, content: String? = null): Recommendation? {
        // 1. 合并文本
        val text = if (content.isNullOrBlank()) {
            title.lowercase(Locale.getDefault())
        } else {
            "${title.lowercase(Locale.getDefault())} ${content.lowercase(Locale.getDefault())}"
        }
      
        if (text.isBlank()) return null
      
        // 2. 获取所有关键词（预设 + 用户自定义）
        val allKeywords = categoryKeywordRepository.getAllKeywordsSync()
      
        if (allKeywords.isEmpty()) return null
      
        // 3. 计算每个分类的得分
        val categoryScores = mutableMapOf<Int, MatchResult>()
        val textNormalized = text.replace("\\s+".toRegex(), "")
      
        allKeywords.forEach { keyword ->
            val keywordText = keyword.keyword.lowercase(Locale.getDefault())
          
            val matched = when (keyword.matchType) {
                MatchType.EXACT -> textNormalized.contains(keywordText)
                MatchType.FUZZY -> text.contains(keywordText)
            }
          
            if (matched) {
                val current = categoryScores.getOrDefault(
                    keyword.categoryType,
                    MatchResult(keyword.categoryType, 0, emptyList())
                )
                categoryScores[keyword.categoryType] = current.copy(
                    score = current.score + keyword.weight,
                    matchedKeywords = current.matchedKeywords + keyword.keyword
                )
            }
        }
      
        // 4. 找出得分最高的分类
        if (categoryScores.isEmpty()) return null
      
        val sortedResults = categoryScores.values
            .sortedByDescending { it.score }
      
        val topScore = sortedResults[0].score
        val topResults = sortedResults.filter { it.score == topScore }
      
        // 5. 得分相同时返回 null（无明确推荐）
        if (topResults.size > 1) return null
      
        val best = topResults[0]
        return Recommendation(
            categoryType = best.categoryType,
            score = best.score,
            matchedKeywords = best.matchedKeywords
        )
    }
}
```

### 5.2 性能优化策略

| 优化项       | 说明                                                   |
| ------------ | ------------------------------------------------------ |
| 内存缓存     | 关键词列表在 Repository 层内存缓存，避免频繁查询数据库 |
| 纯字符串操作 | 匹配逻辑不涉及 I/O，纯 CPU 计算                        |
| 批量匹配     | 单次遍历所有关键词，O(n) 时间复杂度                    |
| 文本预处理   | 合并标题和详情，统一转小写，去除空白                   |

### 5.3 匹配示例

| 输入文本               | 匹配关键词                               | 推荐分类       |
| ---------------------- | ---------------------------------------- | -------------- |
| "明天下午开会讨论项目" | "开会"（工作，精确匹配）                 | 工作           |
| "复习数学考试"         | "复习"、"考试"（学习）                   | 学习           |
| "去超市买牛奶和面包"   | "超市"、"买"（生活）                     | 生活           |
| "跑步半小时"           | "跑步"（运动）                           | 运动           |
| "会议和复习"           | "会议"（工作）+ "复习"（学习），得分相同 | null（无推荐） |

class CategoryKeywordRepository @Inject constructor(

    @IoDispatcher private val ioDispatcher: CoroutineDispatcher

    private val cacheInitialized = AtomicBoolean(false)

        CategoryKeyword("论文", CategoryType.STUDY, MatchType.FUZZY, false),
        CategoryKeyword("复习", CategoryType.STUDY, MatchType.FUZZY, false),
        CategoryKeyword("背单词", CategoryType.STUDY, MatchType.FUZZY, false),
        CategoryKeyword("阅读", CategoryType.STUDY, MatchType.FUZZY, false),
        CategoryKeyword("课程", CategoryType.STUDY, MatchType.FUZZY, false),

        CategoryKeyword("快递", CategoryType.LIFE, MatchType.FUZZY, false),
        CategoryKeyword("取件", CategoryType.LIFE, MatchType.FUZZY, false),
        CategoryKeyword("缴费", CategoryType.LIFE, MatchType.FUZZY, false),
        CategoryKeyword("挂号", CategoryType.LIFE, MatchType.FUZZY, false),
        CategoryKeyword("体检", CategoryType.LIFE, MatchType.FUZZY, false),
        CategoryKeyword("打扫", CategoryType.LIFE, MatchType.FUZZY, false),
        CategoryKeyword("洗衣", CategoryType.LIFE, MatchType.FUZZY, false),
        CategoryKeyword("做饭", CategoryType.LIFE, MatchType.FUZZY, false),
      
        // 运动类
        CategoryKeyword("跑步", CategoryType.SPORT, MatchType.FUZZY, false),
        CategoryKeyword("健身", CategoryType.SPORT, MatchType.FUZZY, false),
        CategoryKeyword("游泳", CategoryType.SPORT, MatchType.FUZZY, false),
        CategoryKeyword("瑜伽", CategoryType.SPORT, MatchType.FUZZY, false),
        CategoryKeyword("打球", CategoryType.SPORT, MatchType.FUZZY, false),
        CategoryKeyword("锻炼", CategoryType.SPORT, MatchType.FUZZY, false)
    )
  
    /**
     * 确保缓存已初始化
     */
    private suspend fun ensureCacheInitialized() {
        if (!cacheInitialized.get()) {
            val userKeywords = categoryKeywordDao.getUserDefined()
                .map { it.toDomainModel() }
            keywordsCache.set(presets + userKeywords)
            cacheInitialized.set(true)
        }
    }
  
    /**
     * 获取所有关键词（同步版本，供匹配引擎使用）
     */
    fun getAllKeywordsSync(): List<CategoryKeyword> {
        return keywordsCache.get()
    }
  
    /**
     * 获取所有关键词
     */
    suspend fun getAllKeywords(): List<CategoryKeyword> {
        ensureCacheInitialized()
        return keywordsCache.get()
    }
  
    /**
     * 获取用户自定义关键词（按分类分组）
     */
    suspend fun getUserDefinedGrouped(): Map<Int, List<CategoryKeyword>> {
        ensureCacheInitialized()
        return keywordsCache.get()
            .filter { it.isUserDefined }
            .groupBy { it.categoryType }
    }
  
    /**
     * 添加用户自定义关键词
     */
    suspend fun addUserKeyword(keyword: String, categoryType: Int, matchType: MatchType = MatchType.FUZZY) {
        ensureCacheInitialized()
      
        val entity = CategoryKeywordEntity(
            keyword = keyword,
            categoryType = categoryType,
            matchType = if (matchType == MatchType.EXACT) 0 else 1,
            isUserDefined = true
        )
        val id = categoryKeywordDao.insert(entity)
      
        val newKeyword = CategoryKeyword(
            id = id,
            keyword = keyword,
            categoryType = categoryType,
            matchType = matchType,
            isUserDefined = true
        )
        keywordsCache.set(keywordsCache.get() + newKeyword)
    }
  
    /**
     * 删除用户自定义关键词
     */
    suspend fun deleteUserKeyword(id: Long) {
        ensureCacheInitialized()
        categoryKeywordDao.deleteUserDefined(id)
        keywordsCache.set(keywordsCache.get().filter { it.id != id })
    }
  
    /**
     * 更新用户自定义关键词
     */
    suspend fun updateUserKeyword(id: Long, categoryType: Int) {
        ensureCacheInitialized()
      
        val current = keywordsCache.get().find { it.id == id } ?: return
      
        categoryKeywordDao.update(
            CategoryKeywordEntity(
                id = id,
                keyword = current.keyword,
                categoryType = categoryType,
                matchType = if (current.matchType == MatchType.EXACT) 0 else 1,
                isUserDefined = true
            )
        )
      
        val updated = current.copy(categoryType = categoryType)
        keywordsCache.set(
            keywordsCache.get().map { if (it.id == id) updated else it }
        )
    }
}

/**
 * 加载分类列表并设置默认分类
 */
fun loadCategories() {
    viewModelScope.launch {
        // 加载所有分类
        val allCategories = categoryRepository.getAllCategoriesList()
        _categories.value = allCategories
      
        // 如果是新建待办，设置默认分类
        if (existingTodo == null && _categoryId.value == 0L) {
            val defaultCategory = when (userPreferences.getUserType()) {
                UserType.WORKER -> allCategories.find { it.type == CategoryType.WORK }
                UserType.STUDENT -> allCategories.find { it.type == CategoryType.STUDY }
                else -> allCategories.firstOrNull()
            }
            defaultCategory?.let { _categoryId.value = it.id }
        }
    }
}

/**
 * 触发分类推荐
 */
fun triggerRecommendation() {
    viewModelScope.launch {
        val recommendation = categoryMatcher.recommendCategory(
            title = _title.value,
            content = _content.value.takeIf { it.isNotBlank() }
        )
      
        if (recommendation != null) {
            val category = _categories.value.find { it.type == recommendation.categoryType }
            _recommendedCategory.value = category
        } else {
            _recommendedCategory.value = null
        }
    }
}

/**
 * 接受推荐的分类
 */
fun acceptRecommendation() {
    _recommendedCategory.value?.let { category ->
        _categoryId.value = category.id
        _recommendedCategory.value = null
    }
}

/**
 * 设置标题并触发推荐（带防抖）
 */
fun setTitleWithRecommendation(title: String) {
    _title.value = title
    // 使用 debounce 避免频繁调用
    recommendationJob?.cancel()
    recommendationJob = viewModelScope.launch {
        delay(300)
        triggerRecommendation()
    }
}

## 13. 风险与注意事项

| 风险                 | 应对方案                                                         |
| -------------------- | ---------------------------------------------------------------- |
| 分类类型不匹配       | CategoryType 需与 Category 实体的 type 字段一致                  |
| 默认分类 ID 变化     | 按 type 查找分类，不依赖固定 ID                                  |
| 匹配性能问题         | 关键词内存缓存，纯字符串操作                                     |
| 用户关键词与预设重复 | 用户关键词权重更高，自动覆盖                                     |
| 数据库迁移失败       | 迁移脚本独立测试，保留回滚方案                                   |
| 防抖逻辑实现复杂     | 使用 Kotlin Coroutines 的 `debounce` 或简单 `delay + cancel` |
