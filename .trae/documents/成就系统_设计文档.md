# 成就系统设计文档

**日期**: 2026-05-20  
**版本**: v1.0  
**状态**: 设计阶段

---

## 1. 项目背景与目标

### 1.1 现状

- 子任务功能已实现（待办可添加多个子步骤，子任务可单独完成）
- 柯基成长系统已实现（经验值、等级、形态变化）
- 成就系统部分实现：`AchievementManager` 定义了 5 个成就，但数据存储在 JSON 字符串中，检测逻辑简单

### 1.2 目标

实现完整的成就系统，包括：
- 13 个成就定义（新 8 个 + 旧 5 个）
- Room 数据库持久化
- 丰富的成就检测逻辑
- 成就解锁弹窗（含动画）
- 成就墙页面
- 与柯基系统联动

---

## 2. 核心决策

| 决策项 | 选择 | 原因 |
|-------|------|------|
| 存储方案 | Room 实体替代 JSON | 查询效率高、数据完整、易于扩展 |
| 成就数量 | 新 8 个 + 旧 5 个 + 复合 1 个 = 14 个 | 保留用户历史解锁数据 |
| 等级检测 | 柯基升级时触发 | 与现有成长系统联动 |
| 早起判定 | 任务完成时间 < 8:00 | 简化实现，用户易理解 |
| 阶段定义 | 独立于柯基等级 | 用户明确要求 |
| 实现方式 | 渐进式重构 | 风险最低，便于回滚 |

### 4.1 AchievementEntity

### 4.2 AchievementDao

```kotlin
@Dao
interface AchievementDao {
    @Query("SELECT * FROM achievements")
    suspend fun getAll(): List<AchievementEntity>

    @Query("SELECT * FROM achievements WHERE unlockedAt IS NOT NULL ORDER BY unlockedAt DESC")
    suspend fun getUnlocked(): List<AchievementEntity>

    @Query("SELECT * FROM achievements WHERE id = :id")
    suspend fun getById(id: String): AchievementEntity?

    @Insert(onConflict = OnConflictStrategy.IGNORE)
    suspend fun insert(achievement: AchievementEntity)

    @Insert(onConflict = OnConflictStrategy.IGNORE)
    suspend fun insertAll(achievements: List<AchievementEntity>)

    @Query("UPDATE achievements SET unlockedAt = :timestamp WHERE id = :id")
    suspend fun unlock(id: String, timestamp: Long)

    @Query("SELECT COUNT(*) > 0 FROM achievements WHERE id = :id AND unlockedAt IS NOT NULL")
    suspend fun isUnlocked(id: String): Boolean

    @Query("SELECT COUNT(*) FROM achievements WHERE unlockedAt IS NOT NULL")
    suspend fun getUnlockedCount(): Int

    @Query("SELECT id FROM achievements WHERE unlockedAt IS NOT NULL")
    suspend fun getUnlockedIds(): List<String>
}
```

### 4.3 数据库迁移（7 → 8）

```kotlin
private val MIGRATION_7_8 = object : Migration(7, 8) {
    override fun migrate(database: SupportSQLiteDatabase) {
        // 1. 创建 achievements 表
        database.execSQL("""
            CREATE TABLE IF NOT EXISTS achievements (
                id TEXT PRIMARY KEY NOT NULL,
                unlockedAt INTEGER
            )
        """.trimIndent())

        // 2. 为 corgi_data 添加新字段
        database.execSQL("ALTER TABLE corgi_data ADD COLUMN consecutiveEarlyDays INTEGER NOT NULL DEFAULT 0")
        database.execSQL("ALTER TABLE corgi_data ADD COLUMN lastEarlyDate TEXT NOT NULL DEFAULT ''")

        // 3. 初始化所有成就（unlockedAt = null）
        // 在应用启动时通过 AchievementRepository 完成
    }
}
```

### 4.4 数据迁移逻辑

旧成就 ID 到新 ID 映射：

| 旧 ID | 新 ID |
|-------|-------|
| scholar_hat | study_100 |
| tie | work_100 |
| crown | consecutive_30_days |
| angel_wings | total_500 |
| cape | all_achievements |

迁移在 `AchievementRepository.initialize()` 中执行：

```kotlin
suspend fun initialize() {
    // 1. 插入所有新成就（忽略已存在的）
    val entities = allAchievements.map { AchievementEntity(id = it.id, unlockedAt = null) }
    achievementDao.insertAll(entities)

    // 2. 从旧 JSON 迁移数据
    val corgiData = corgiRepository.getCorgiData() ?: return
    val oldIds = AchievementManager.parseAchievementIds(corgiData.unlockedAchievements)

    oldIds.forEach { oldId ->
        val newId = mapOldToNewId(oldId)
        if (newId != null && !achievementDao.isUnlocked(newId)) {
            achievementDao.unlock(newId, System.currentTimeMillis())
        }
    }
}
```

### 5.1 触发时机

| 成就条件类型 | 触发时机 | 检测函数 |
|-------------|---------|---------|
| FIRST_COMPLETE | 任务完成时 | `checkOnTaskComplete()` |

| CUMULATIVE_STUDY | 任务完成时 | `checkOnTaskComplete()` |
| CUMULATIVE_WORK | 任务完成时 | `checkOnTaskComplete()` |
| CUMULATIVE_LIFE | 任务完成时 | `checkOnTaskComplete()` |

### 5.3 早起达人检测逻辑

```
每日首次打开时：
    if 当前时间 < 当天 8:00:
        等待第一个任务完成后检查
    else:
        consecutiveEarlyDays = 0

任务完成时（且是当天第一个完成的任务）：
    if 完成时间 < 当天 8:00:
        if lastEarlyDate == 昨天:
            consecutiveEarlyDays++
        else:
            consecutiveEarlyDays = 1
        lastEarlyDate = 今天日期

        if consecutiveEarlyDays >= 7:
            解锁 early_7_days 成就
    else:
        consecutiveEarlyDays = 0
```

| `checkDateChange()` 完成后 | `achievementChecker.checkOnDailyOpen()` |
| `addExperience()` 检测到升级后 | `achievementChecker.checkOnLevelUp(newLevel)` |
