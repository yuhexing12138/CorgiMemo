# 成就故事与动画增强设计文档

**日期**: 2025-06-24
**版本**: v1.0
**状态**: 设计完成

---

## 1. 背景与目标

### 1.1 现状

成就系统已完成基础实现：
- 26 个成就定义（通用 9 + 学生 10 + 上班族 7），4 个阶段（初见/成长/飞跃/巅峰）
- Room 数据库持久化 + DAO + Repository
- 多条件成就检测器（累计/连续/分类/早起/等级）
- 成就解锁弹窗（含 Confetti 动画）
- 成就墙页面（按阶段分组 + 底部详情 Sheet）
- 装扮联动（成就解锁自动解锁对应装扮）

### 1.2 目标

在现有成就系统基础上，增强以下 4 个模块：

1. **成就故事数据** — 为每个成就增加故事标题和柯基对话
2. **里程碑庆祝动画** — 区分重大/普通成就，增加柯基跳跃 + 音效
3. **成就徽章墙 3D 展示** — graphicsLayer 实现 3D 翻转效果
4. **成就进度追踪** — 阶段进度条 + 下一成就预览

---

## 2. 核心决策

| 决策项 | 选择 | 原因 |
|-------|------|------|
| 阶段阈值 | 保持当前 0/5/10/16 | 用户确认不与现有系统冲突 |
| 重大成就判定 | 含装扮 + 连续天数 + 里程碑 | 用户确认三类均为重大 |
| 3D 翻转 | graphicsLayer rotationY | 用户确认选择翻转方案 |
| 详情展示 | 翻转至背面直接查看 | 取代现有 BottomSheet |
| 动画实现 | Canvas 绘制柯基跳跃 | 无需引入 Lottie，减少依赖 |
| 音效实现 | MediaPlayer + raw 资源 | 轻量，尊重现有 soundEnabled |
| 故事线阶段 | 保持现有 4 阶段结构 | 与现有阶段定义一致 |

## 4. 模块 2：里程碑庆祝动画

### 4.1 重大成就判定

```kotlin
fun Achievement.isMajorAchievement(): Boolean {
    // 1. 含装扮成就
    if (outfitId != null) return true
    // 2. 连续天数成就
    val consecutiveIds = setOf(
        NewAchievementId.CONSECUTIVE_7_DAYS,
        NewAchievementId.CONSECUTIVE_30_DAYS,
        NewAchievementId.WORK_PROMOTION,
        NewAchievementId.STUDY_EXAM_WEEK,
        NewAchievementId.EARLY_7_DAYS
    )
    if (id in consecutiveIds) return true
    // 3. 里程碑成就
    val milestoneIds = setOf(
        NewAchievementId.TOTAL_100,
        NewAchievementId.TOTAL_500,
        NewAchievementId.CORGI_LEVEL_5,
        NewAchievementId.CORGI_LEVEL_10,
        NewAchievementId.ALL_ACHIEVEMENTS
    )
    if (id in milestoneIds) return true
    return false
}
```

**重大成就列表（共 12 个）：**
- first_complete（首次完成，含装扮的基础）
- consecutive_7_days（连续天数）
- daily_10_tasks（单天10个，里程碑）
- total_100（里程碑）
- corgi_level_5（里程碑）
- early_7_days（连续天数）
- study_100（含装扮）
- work_100（含装扮）
- consecutive_30_days（含装扮 + 连续天数）
- total_500（含装扮 + 里程碑）
- corgi_level_10（里程碑）
- all_achievements（含装扮 + 里程碑）

## 5. 模块 3：3D 翻转徽章墙

### 5.1 翻转实现方案

使用 Compose 的 `graphicsLayer` + `animateFloatAsState`：

```kotlin
var isFlipped by remember { mutableStateOf(false) }
val rotation by animateFloatAsState(
    targetValue = if (isFlipped) 180f else 0f,
    animationSpec = tween(400, easing = FastOutSlowInEasing)
)

Box(
    modifier = Modifier
        .graphicsLayer {
            rotationY = rotation
            cameraDistance = 12f * density
        }
) {
    // 正面（rotationY 0° ~ 90° 可见）
    if (rotation <= 90f) {
        FrontSide(achievement, isUnlocked, currentProgress)
    }
    // 背面（rotationY 90° ~ 180° 可见）
    else {
        BackSide(achievement, isUnlocked, currentProgress)
    }
}
```

### 6.2 下一成就预览

根据当前已解锁数量，确定每个阶段内下一个未解锁的成就：

```kotlin
fun getNextUnlockableAchievement(
    achievements: List<Pair<Achievement, Boolean>>,
    currentStage: AchievementStage
): Achievement? {
    val stageAchievements = achievements
        .filter { it.first.stage == currentStage && !it.second }
        .sortedBy { it.first.threshold }
    return stageAchievements.firstOrNull()?.first
}
```

## 9. 风险与注意事项

| 风险 | 应对方案 |
|------|---------|
| 3D 翻转在低端设备卡顿 | 使用 `cameraDistance` 优化透视，考虑 `useSmoothRefresh` 降级 |
| 音效文件体积 | 使用短促音效（<1秒），压缩为 mp3 格式 |
| 柯基跳跃动画分辨率 | Canvas 绘制自适应屏幕密度 |
| 现有 BottomSheet 与翻转冲突 | 翻转后移除 BottomSheet 逻辑，干净替换 |
| 同时解锁多个成就 | 队列显示弹窗，按解锁顺序逐个弹出 |
