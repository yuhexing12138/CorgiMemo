## 1. 概述

### 1.1 功能目标
为柯基备忘录应用添加桌面小部件支持，允许用户：
- 在桌面快速查看今日待办
- 一键创建新待办
- 在小部件上直接完成待办
- 无需打开应用即可快速操作

### 1.2 技术选型
- **框架：** Jetpack Glance（Compose 语法的 AppWidget 框架）
- **数据更新：** WorkManager（每30分钟） + 广播机制（立即刷新）
- **深色模式：** 自动适配系统主题

### 1.3 小部件尺寸

| 尺寸 | 类名 | 功能 | 最小单元格 | 调整范围 |
|------|------|------|-----------|---------|
| 1x1 | QuickAddWidget | 快速添加按钮 | 1x1 | 不可调整 |
| 2x2 | TodayPreviewWidget | 今日待办预览（3-5条） | 2x2 | 2x2 ~ 4x4 |
| 4x2 | TodoListWidget | 完整今日待办列表（可滚动） | 4x2 | 4x2 ~ 5x4 |

## 4. 数据管理

```kotlin
object WidgetDataManager {
    
    /**
     * 获取今日待办列表（用于小部件显示）
```

## 5. 数据更新机制

### 5.1 两种刷新方式

| 方式 | 触发条件 | 延迟 | 适用场景 |
|------|---------|------|---------|
| **WorkManager** | 定时调度 | 每 30 分钟 | 后台静默更新 |
| **广播机制** | 用户操作后立即触发 | 实时 | 添加/修改/删除待办 |

### 5.2 WorkManager 定期刷新

```kotlin
class WidgetUpdateWorker(context: Context, params: WorkerParameters) : CoroutineWorker(context, params) {
    
    override suspend fun doWork(): Result {
        // 刷新所有小部件
        QuickAddWidget().updateAll(context)
        TodayPreviewWidget().updateAll(context)
        TodoListWidget().updateAll(context)
        return Result.success()
    }
}

// 调度（在 Application.onCreate 中）
fun scheduleWidgetUpdates(context: Context) {
    val request = PeriodicWorkRequestBuilder<WidgetUpdateWorker>(
        repeatInterval = 30,
        repeatIntervalTimeUnit = TimeUnit.MINUTES
    ).build()
    
    WorkManager.getInstance(context).enqueueUniquePeriodicWork(
        "WidgetUpdate",
        ExistingPeriodicWorkPolicy.UPDATE,
        request
    )
}
```

### 5.3 广播立即刷新

```kotlin
class WidgetUpdateReceiver : BroadcastReceiver() {
    
    companion object {
        const val ACTION_REFRESH = "com.corgimemo.app.ACTION_REFRESH_WIDGET"
        
        fun sendRefreshBroadcast(context: Context) {
            val intent = Intent(ACTION_REFRESH)
            intent.setPackage(context.packageName)
            context.sendBroadcast(intent)
        }
    }
    
    override fun onReceive(context: Context, intent: Intent) {
        if (intent.action == ACTION_REFRESH) {
            // 立即刷新所有小部件
            CoroutineScope(Dispatchers.IO).launch {
                QuickAddWidget().updateAll(context)
                TodayPreviewWidget().updateAll(context)
                TodoListWidget().updateAll(context)
            }
        }
    }
}
```

### 5.4 调用刷新的时机

在以下操作后调用 `WidgetUpdateReceiver.sendRefreshBroadcast(context)`：
- 创建新待办
- 修改待办
- 删除待办
- 标记待办完成
- 恢复待办

## 6. 点击操作处理

### 6.1 操作类型

| 操作 | Intent Action | 处理类 |
|------|---------------|--------|
| 点击标题栏 | 跳转首页 | MainActivity (navigate_to=home) |
| 点击待办项 | 跳转编辑 | MainActivity (navigate_to=edit, todoId=xxx) |
| 点击 1x1 添加按钮 | 跳转创建 | MainActivity (navigate_to=create) |
| 点击完成按钮 | ACTION_COMPLETE_TODO | WidgetActionReceiver |

### 6.2 完成操作处理

```kotlin
class WidgetActionReceiver : BroadcastReceiver() {
    
    companion object {
        const val ACTION_COMPLETE_TODO = "com.corgimemo.app.ACTION_COMPLETE_TODO"
        const val EXTRA_TODO_ID = "extra_todo_id"
        
        fun getCompletePendingIntent(context: Context, todoId: Long): PendingIntent {
            val intent = Intent(context, WidgetActionReceiver::class.java).apply {
                action = ACTION_COMPLETE_TODO
                putExtra(EXTRA_TODO_ID, todoId)
            }
            return PendingIntent.getBroadcast(
                context,
                todoId.toInt(),
                intent,
                PendingIntent.FLAG_UPDATE_CURRENT or PendingIntent.FLAG_IMMUTABLE
            )
        }
    }
    
    override fun onReceive(context: Context, intent: Intent) {
        val todoId = intent.getLongExtra(EXTRA_TODO_ID, -1)
        if (todoId == -1L) return
        
        when (intent.action) {
            ACTION_COMPLETE_TODO -> handleCompleteTodo(context, todoId)
        }
    }
    
    private fun handleCompleteTodo(context: Context, todoId: Long) {
        CoroutineScope(Dispatchers.IO).launch {
            // 获取 Repository
            val database = CorgiMemoDatabase.getDatabase(context)
            val repository = TodoRepository(database.todoDao(), Dispatchers.IO)
            
            // 标记完成
            repository.getTodoById(todoId)?.let { todo ->
                val updated = todo.copy(
                    status = 1,
                    completedAt = System.currentTimeMillis(),
                    updatedAt = System.currentTimeMillis()
                )
                repository.updateTodo(updated)
            }
            
            // 刷新小部件
            WidgetUpdateReceiver.sendRefreshBroadcast(context)
        }
    }
}
```

## 7. 深色模式适配

Glance 的 `GlanceTheme` 会自动适配系统主题。需要确保：

1. 使用 `GlanceTheme.colors` 而不是硬编码颜色
2. 亮色/暗色模式的颜色在 `glance-provider` 中定义

**示例：**
```kotlin
// 在小部件 Content 中
GlanceTheme(colors = GlanceTheme.colors) {
    Surface(
        color = GlanceTheme.colors.widgetBackground,
        ...
    ) {
        // 内容
    }
}
```

## 13. 风险与应对

| 风险 | 应对措施 |
|------|---------|
| Glance API 限制（不支持所有 Compose 组件） | 使用 Glance 提供的组件，避免使用不支持的组件 |
| 滚动列表在 Glance 中的实现复杂 | 4x2 小部件使用 LazyColumn（Glance 支持） |
| 深色模式颜色适配问题 | 充分测试亮色/暗色模式 |
| 小部件刷新性能问题 | 合理控制刷新频率，避免频繁刷新 |
