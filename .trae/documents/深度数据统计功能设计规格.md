## 1. 功能概述

为 CorgiMemo 应用的"数据统计"页面添加 5 个深度分析模块，帮助用户更好地理解自己的任务完成模式和效率趋势。

### 核心功能模块

1. **分类完成率图表** - 横向条形图展示各分类完成率
2. **效率时段分析** - 6 个时段的完成数量统计 + 最佳时段标记
3. **任务类型分布饼图** - Canvas 绘制的甜甜圈图 + 图例交互
4. **趋势预测** - 基于历史数据的月度完成预测 + 鼓励提示
5. **统计页面整合** - TabRow 布局切换 4 个维度

### 5.2 关键算法

#### 时段划分算法

```kotlin
fun getTimePeriod(hour: Int): Int {
    return when (hour) {
        in 6..8 -> 0   // 6-9点
        in 9..11 -> 1  // 9-12点
        in 12..13 -> 2 // 12-14点
        in 14..17 -> 3 // 14-18点
        in 18..21 -> 4 // 18-22点
        in 22..23 -> 5 // 22-24点
        else -> -1     // 不计入统计
    }
}
```

#### 趋势预测算法

```kotlin
fun calculatePrediction(completedTodos: List<TodoItem>): Triple<Int, Int, Float> {
    val calendar = Calendar.getInstance()
    val today = calendar.get(Calendar.DAY_OF_MONTH)
    val daysInMonth = calendar.getActualMaximum(Calendar.DAY_OF_MONTH)
    val remainingDays = daysInMonth - today

    // 计算过去 4 周（28天）的日均完成数
    val fourWeeksAgo = System.currentTimeMillis() - 28L * 24 * 60 * 60 * 1000
    val recentCompleted = completedTodos.filter {
        it.completedAt >= fourWeeksAgo
    }
    val dailyAverage = recentCompleted.size / 28f

    // 预测值
    val predictedCompletion = (dailyAverage * remainingDays).toInt() + monthCount
    val predictedTotal = (dailyAverage * daysInMonth).toInt()
    val rate = predictedCompletion.toFloat() / predictedTotal.coerceAtLeast(1)

    return Triple(predictedCompletion, predictedTotal, rate)
}
```

## 7. 交互流程

### 7.1 用户操作流程

```
用户进入统计页面
    ↓
加载所有统计数据（loading 状态）
    ↓
显示概览 Tab（默认）
    ├─ 查看 3 个统计卡片
    ├─ 查看环形进度 + 预测
    └─ 查看 7 天趋势图
    ↓
用户点击其他 Tab
    ├─ 分类 Tab → 查看条形图 → 点击条形 → BottomSheet 详情
    ├─ 时段 Tab → 查看网格 → 识别最佳时段 → 阅读建议
    └─ 分布 Tab → 查看饼图 → 点击扇区/图例 → 高亮显示
```

### 7.2 数据刷新策略

* **进入页面时**: 自动加载最新数据

* **下拉刷新**: 支持 SwipeRefresh（可选，v2 实现）

* **数据缓存**: ViewModel 内存缓存，避免重复查询

* **更新频率**: 实时（每次进入页面重新计算）

## 8. 性能优化

### 8.1 列表性能

* 使用 `lazyColumn` / `LazyVerticalGrid` 实现虚拟化滚动

* 避免在 `remember` 外部进行复杂计算

* 对大数据集使用 `key()` 稳定 item identity

### 8.2 Canvas 性能

* 饼图/条形图使用 `DrawScope` 批量绘制

* 动画使用 `animateFloatAsState` 避免重绘

* 复杂图形使用 `withTransform` 减少状态保存

### 8.3 内存管理

* ViewModel 在页面销毁时自动清理

* 大型数据集合使用 StateFlow 冷流

* 图片/图标资源使用 Compose Resource 管理
