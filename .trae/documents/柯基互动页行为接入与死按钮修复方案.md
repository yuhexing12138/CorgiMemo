# 柯基互动页行为接入与死按钮修复方案

## Context（为什么做这个改动）

### 问题背景

用户反馈："柯基互动页完整吗？" 经调研发现 **CorgiDetailScreen 约 60% 完整**：

| 缺陷 | 严重度 | 影响 |
|------|--------|------|
| 5 种自主行为（YAWNING / SLEEPING_NIGHT / WORRIED / MISSED_YOU / HAPPY_STREAK）**完全不感知** | 🔴 P0 | 模型层在检测，UI 层完全没接，行为状态在详情页"失声" |
| "切换装扮"按钮是**死按钮** | 🔴 P0 | `toggleOutfitSheet()` 翻 StateFlow 但无 UI 消费 |
| 没有 `YawnOverlay` 组件 | 🔴 P0 | YAWNING 仅改 pose=SLEEP，与 SLEEPING_NIGHT 视觉混淆，HomeScreen 自身也有此盲点 |
| 情绪历史图表数据为空 | 🟡 P1 | 暂不修（需 Room 建表，单独任务） |
| `_corgiData` 参数加下划线 | 🟢 P2 | 编译警告，顺手清理 |

### 关键发现（重要）

**项目里 `OutfitQuickSwitchSheet` 已存在**（HomeScreen.kt:2844 文件级 public @Composable），是 HomeScreen 长按柯基触发的快速换装 BottomSheet。**直接复用零新组件**。

**项目里 `MissedYouDialog` 也在 HomeScreen.kt:2775**（同上可跨文件引用），但仅在 HomeScreen 挂载。

### 预期结果

修复后 CorgiDetailScreen 将：
- 弹出 5 种行为的视觉反馈（其中 3 种 SLEEPING_NIGHT / WORRIED / HAPPY_STREAK 已被 greeting 隐式覆盖；YAWNING 需新增 YawnOverlay；MISSED_YOU 需复用 MissedYouDialog）
- "切换装扮"按钮可用，弹出已解锁装扮的快速换装抽屉（与首页长按柯基同 UX）
- 整个修复不动 ViewModel 层（`HomeViewModel` 已提供所有必需 StateFlow / 方法）

## 风险与权衡

### 决策 1：HomeScreen 是否同时接 YawnOverlay？

**用户选"公共（推荐）"** 但本任务聚焦 CorgiDetailScreen。

- **取舍**：若仅在 CorgiDetailScreen 接 YawnOverlay，HomeScreen 仍存在 YAWNING/SLEEPING_NIGHT 视觉混淆 bug（pose 都是 SLEEP，greeting 都不变）
- **建议**：本任务**仅修 CorgiDetailScreen**（保持改动最小），HomeScreen 盲点列入"后续优化"
- **如果用户想一次性修完**：在 HomeScreen L196 之后加 `val currentBehavior by viewModel.currentBehavior.collectAsState()`，在 InteractiveCorgi 父 Box 内加 `if (currentBehavior == BehaviorType.YAWNING) YawnOverlay(Modifier.align(TopEnd))`，约 5 行

### 决策 2：OutfitQuickSwitchSheet 接收的 `currentOutfitId` 是哪个？

- 选 **`currentOutfit`**（ViewModel StateFlow，用户实际选择）
- **不**用 `effectiveOutfit`（节日优先）
- 原因：节日期间 `effectiveOutfit` 是"圣诞帽🎅"，但用户实际选择的是其他装扮，sheet 里高亮错位会让用户困惑。HomeScreen L1132 也用 `currentOutfit`

### 决策 3：是否修 `triggerYawn()` 内未写 greeting 的 bug？

`HomeViewModel.kt:2472-2484` 的 `triggerYawn()` 没写 `_greeting.value`，是已存在的 bug。

- **不修**：因为 YawnOverlay 浮层会显示 "Zzz 💤"，与 greeting 文字内容相近；同时改 greeting 会让气泡（问候语）和 YawnOverlay 同时显示，UX 重复
- **建议**：保持现状，YawnOverlay 作为唯一视觉差异化

### 决策 4：项目里 `OutfitQuickSwitchSheet` 和 `QuickOutfitCard` 当前可见性

- 两者都是文件级 `fun`（无 `private`），Kotlin 默认 public → 可跨文件访问
- **仍建议**抽到 `ui/components/`：让架构更清晰，参考 `AchievementUnlockDialog.kt` / `MoodHistoryChart.kt` 模式
