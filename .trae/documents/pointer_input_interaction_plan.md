# 触摸互动反馈（PointerInput 实现）计划

## 一、代码库调研结论

### 1.1 当前已实现的内容

**已存在的文件：**

| 文件路径 | 状态 | 使用方式 |
|---------|------|---------|
| `animation/InteractiveCorgi.kt` | ✅ 已存在 | 使用 `combinedClickable` 实现触摸检测 |
| `animation/HeartParticleEffect.kt` | ✅ 已存在 | 爱心粒子特效组件 |
| `animation/AnimationType.kt` | ✅ 已存在 | 包含 WINK、ROLL、SHY、WAG 等动画类型 |
| `animation/AnimationResourceManager.kt` | ✅ 已存在 | 所有动画帧资源映射 |
| `animation/FrameAnimation.kt` | ✅ 已存在 | 帧动画播放组件 |

### 1.2 当前实现与用户需求的差异

| 需求点 | 当前状态 | 差距 |
|--------|---------|------|
| 单击：眨眼动画 + 摇尾巴 | ✅ 眨眼已实现 | 缺少摇尾巴动画叠加 |
| 双击：打滚动画（4帧单次） | ✅ 已实现 | 无差距 |
| 长按：爱心粒子特效 | ✅ 已实现 | 无差距 |
| 快速连点：害羞躲到边缘 | ✅ 已实现 | 无差距 |
| 触摸检测使用 `pointerInput` | ❌ 使用 `combinedClickable` | **需要重写触摸检测逻辑** |
| 动画帧配置（眨眼2帧、打滚4帧） | ✅ 已配置 | 无差距 |

### 1.3 技术方案对比

**当前方案 (`combinedClickable`)：**
- ✅ 简单易用，内置单击/双击/长按检测
- ❌ 灵活性较差，难以自定义手势
- ❌ 无法控制点击时间阈值
- ❌ 难以实现复杂的手势组合

**目标方案 (`pointerInput`)：**
- ✅ 完全控制手势检测逻辑
- ✅ 可自定义时间阈值、距离阈值
- ✅ 可以实现更复杂的手势（如拖拽、滑动）
- ✅ 学习成本较低，但代码量稍多

## 四、详细实现步骤

### 步骤 1: 设计 PointerInput 手势检测逻辑

**位置**: `InteractiveCorgi.kt` 中的触摸检测部分

**手势检测策略**:

| 手势类型 | 触发条件 | 优先级 |
|---------|---------|--------|
| 长按 | 按住 500ms 不松开 | 1 (最高) |
| 双击 | 300ms 内连续点击 2 次 | 2 |
| 单击 | 点击后 300ms 无后续点击 | 3 |
| 快速连点 | 3 秒内连续点击 5 次 | 4 (最低，与单击合并检测) |

**核心逻辑**:
```kotlin
pointerInput(Unit) {
    awaitPointerEventScope {
        // 点击时间记录
        var lastClickTime = 0L
        var clickCount = 0
        
        while (true) {
            val event = awaitPointerEvent()
            val now = System.currentTimeMillis()
            
            // 检测按下事件
            if (event.type == Press) {
                // 检查是否在 300ms 内（双击检测）
                // 启动长按计时器
                // ...
            }
            
            // 检测抬起事件
            if (event.type == Release) {
                // 取消长按计时器
                // 判断是单击还是双击
                // ...
            }
        }
    }
}
```

### 步骤 2: 添加摇尾巴动画叠加

**问题**: 当前单击只播放眨眼动画，用户要求同时播放摇尾巴

**方案**: 同时显示两个动画层

## 五、潜在风险与应对

### 5.1 风险清单

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| 手势冲突（长按和单击） | 长按松手后触发单击 | 长按触发时标记 `longPressTriggered`，松手时检查 |
| 双击和快速连点冲突 | 双击被误判为快速连点的前两次 | 快速连点需要 5 次，双击优先处理 |
| `pointerInput` 的学习曲线 | 代码复杂度增加 | 使用 `detectTapGestures` 作为替代方案 |
| 动画叠加效果 | 两个动画同时播放可能卡顿 | 使用 `AnimatedVisibility` 控制显示时机 |

### 5.2 备选方案

如果 `pointerInput` 自定义实现过于复杂，可以使用 `detectTapGestures`:

```kotlin
Modifier.pointerInput(Unit) {
    detectTapGestures(
        onTap = { /* 单击 */ },
        onDoubleTap = { /* 双击 */ },
        onLongPress = { /* 长按 */ },
        onPress = { /* 按下状态 */ }
    )
}
```

**优点**: 
- 简洁易用
- 内置优先级处理

**缺点**:
- 无法自定义时间阈值
- 快速连点仍需额外处理
