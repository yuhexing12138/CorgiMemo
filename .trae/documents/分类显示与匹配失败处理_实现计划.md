# 分类显示与匹配失败处理 - 实现计划

## 一、需求分析

### 1. 核心需求

1. **待办列表页显示分类**

   - 每个待办卡片显示所属分类标签（如 "📚 学习"、"💼 工作"等）
2. **匹配失败场景 A：编辑页面无推荐提示**

   - 编辑页面输入时智能推荐没有命中（返回 null）
   - 显示"需要分类"的提示
3. **匹配失败场景 B：保存时无分类**

   - 保存待办时用户没有手动选择分类
   - 弹出分词对话框
4. **分词提取**

   - 对标题进行分词，列出可能的关键词供用户选择
   - 简单分词策略：n-gram + 停用词过滤
5. **自动加入关键词库**

   - 弹出对话框让用户选择分类
   - 用户选择后自动将关键词添加到该分类的关键词库

---

## 二、现状分析

### 1. 现有实现

1. **TodoListItem 数据模型**

   - 已有 `categoryId` 字段
   - 但 UI 层未显示分类
2. **HomeScreen 待办列表**

   - 使用 `TodoListItem` 组件
   - 已有 `categories` 列表（用于分享功能
3. **TodoEditScreen 编辑页面**

   - 已有分类选择器和智能推荐标签
   - `loadCategories()` 会根据用户身份设置默认分类
4. **智能推荐**

   - 已有 `CategoryMatcher` 实现匹配

### 2. 缺失功能

1. **待办列表分类显示
2. **简单分词器**
3. **匹配失败时的提示和处理流程**
4. **分词对话框**
