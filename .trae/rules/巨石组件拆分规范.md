# 巨石组件拆分规范

> **核心原则**：当单文件 ≥ 800 行 / ≥ 5 个 Composable / 多个职责共存时，必须按职责拆分到子包。
> 主入口保留为薄壳（Facade）层，对外 0 改动；真实实现迁移至 `xxxXxx/<model|sections|dialogs>/` 子目录。

## 1. 何时拆分（触发条件，三选一即触发）

| 维度 | 阈值 | 例子 |
| --- | --- | --- |
| 文件行数 | **≥ 800 行** | AppDrawer.kt 1130+ 行 |
| Composable 数量 | **≥ 5 个** | 5 个分区 + 5 个弹窗 |
| 职责数量 | **≥ 3 个独立职责** | 用户头 / 分类管理 / 弹窗 / 操作菜单 |

## 2. 拆分结构（按职责切分）

```
原文件: ui/components/XXDrawer.kt（1130+ 行）
    ↓
ui/components/XXDrawer.kt（< 150 行 薄壳层，保留所有原 API）
ui/components/xxdrawer/
    ├── model/        # 纯数据/操作类型（sealed class、enum、data class）
    │   ├── XxxAction.kt
    │   └── YyyAction.kt
    ├── sections/     # 主分区 UI 组件（每个分区一个文件）
    │   ├── XXContentImpl.kt    # 主入口实现（XxxContentImpl 后缀）
    │   ├── UserHeaderSection.kt
    │   ├── CategorySection.kt
    │   ├── FilterSection.kt
    │   └── SharedRow.kt        # 共享子组件，internal 可见性
    └── dialogs/      # 弹窗和菜单（独立子包）
        ├── AddDialog.kt
        ├── EditDialog.kt
        ├── ConfirmDialog.kt
        └── OperationSheets.kt
```

## 3. 薄壳层（Facade）规则

**必做**：
- 保留原文件所有顶层 Composable 函数，转发到新包实现
- sealed class 用 `typealias` 兼容（编译期展开为原类型，pattern match 完全等价）
- 转发函数与子包函数同名时，**薄壳内部用 FQN 调用**（避免同名导入冲突）

**禁止**：
- ❌ `rememberModalBottomSheetState()` 等 Composable 函数**不能**作为转发参数默认值（必须由调用方在 Composable 作用域创建并传入）
- ❌ 薄壳层不写任何业务逻辑，只透传参数

**薄壳样板**：
```kotlin
// 薄壳: ui/components/XXDrawer.kt
import com.corgimemo.app.ui.components.xxdrawer.model.XxxAction as XxxActionImpl
import com.corgimemo.app.ui.components.xxdrawer.sections.XXContentImpl

typealias XxxAction = XxxActionImpl  // sealed class 兼容

@Composable
fun XXContent(/* 原签名不变 */) {
    XXContentImpl(/* 透传所有 24+ 参数 */)
}
```

## 4. 可见性调整

| 原可见性 | 拆分后 | 原因 |
| --- | --- | --- |
| `private`（同文件） | `internal` | 跨文件调用必须 internal |
| `private const val` | `internal const val` | 常量被多个 section 复用 |
| `@Composable private fun` | `@Composable internal fun` | 跨 section 复用子组件 |

**禁止扩大为 `public`** —— 防止外部模块直接依赖内部实现细节，保留薄壳为唯一对外入口。

## 5. 命名约定

| 类别 | 命名规则 | 例子 |
| --- | --- | --- |
| 主入口实现 | `XxxImpl` 后缀 | `AppDrawerContentImpl` |
| 分区组件 | `<Xxx>Section` 后缀 | `CategoryGroupSection` |
| 弹窗 | `<Xxx>Dialog` 后缀 | `AddCategoryDialog` |
| 操作菜单 | `<Xxx>OperationSheet` 后缀 | `CategoryOperationSheet` |
| 共享子组件 | 描述性名字 | `CategoryItemRow` |
| sealed class | `<Xxx>Action` 后缀 | `CategoryAction` |
| 图标常量 | `DRAWER_ICON_<NAME>` 全大写 | `DRAWER_ICON_ALL = "📋"` |

## 6. 迁移路径（三阶段）

| 阶段 | 任务 | 调用方改动 |
| --- | --- | --- |
| **Phase 1** | 拆分 + 薄壳（保 API 兼容） | **0 改动** |
| **Phase 2** | 逐步迁移调用方到新路径 | 每次只动一个调用方 |
| **Phase 3** | 删除薄壳（可选） | 所有调用方迁完才删 |

> **不要跳级**：直接删原文件会让所有 import 报错，影响面巨大。先薄壳兼容、再渐进迁移、最后清理。

## 7. 常见陷阱

| 陷阱 | 症状 | 解决 |
| --- | --- | --- |
| **同名导入冲突** | 薄壳与子包同名函数，import 报错 | 薄壳内部用 FQN（如 `com.corgimemo.app.ui.components.xxdrawer.dialogs.AddDialog`） |
| **sheetState 默认值** | `@Composable fun Foo(sheetState: SheetState = rememberModalBottomSheetState())` 编译失败 | 不给默认值，调用方在 Composable 作用域创建 |
| **private 跨文件** | 子组件 A 在 sections/A.kt，调用方在 sections/B.kt，private 不可见 | 改为 internal |
| **薄壳中写业务逻辑** | 薄壳膨胀到 500+ 行 | 薄壳只做参数透传，所有逻辑在 `XxxImpl` |
| **删除原文件前未迁移** | 大量 import 报错 | 保留薄壳，所有调用方迁完再删 |

## 8. 收益度量

- **可读性**：单文件 1130+ 行 → 12 个文件平均 ~100 行
- **PR 冲突率**：多开发者并行修改不同分区，互不影响
- **可测试性**：每个 section / dialog 可独立预览和测试
- **可复用性**：共享子组件（CategoryItemRow 等）跨分区复用
- **职责清晰**：model / sections / dialogs 三层各司其职
