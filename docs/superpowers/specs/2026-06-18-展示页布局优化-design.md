# 刻记⁺ 展示页布局优化设计文档

> 针对 `【刻记+】APP/corgimemo-showcase.html` 的前端布局优化，打破全居中布局导致的 AI 同质化问题

## 1. 设计目标

### 1.1 核心问题

当前展示页存在严重的 AI 同质化问题，具体表现为：

| 问题类别 | 具体表现 | 评委视角风险 |
|---------|---------|------------|
| **对称疲劳** | Hero、页签、内容区全部水平居中 | 第一眼即识别为 AI 生成 |
| **节奏单一** | 7 个 Tab 全部采用"标题+3列网格"重复模式 | 滚动 2 屏后产生审美疲劳 |
| **卡片同质** | 所有 glass-card 样式完全一致 | 无法记住任何独特视觉元素 |
| **缺乏焦点** | 没有视觉引导线和层次变化 | 缺乏"设计感"和"匠心" |

### 1.2 优化目标

1. **打破居中布局限制**：创建不对称视觉结构，建立差异化设计语言
2. **建立视觉层次**：通过大数字章节号、错落网格、差异化卡片创造清晰节奏
3. **消除 AI 痕迹**：融合杂志编辑风、野兽派、故障艺术三种风格，形成独特视觉记忆点
4. **保留主题契合**：维持柯基治愈系温暖基调，确保设计为内容服务

## 2. 设计方案：混合风格策略

### 2.1 风格配比

采用"主基调+点缀"的混合策略，兼顾主题契合度与视觉冲击力：

| 风格 | 占比 | 角色 | 应用范围 |
|------|------|------|---------|
| **A 杂志编辑风** | 70% | 主基调 | Hero、Tab1-6 主体、页脚 |
| **B 新野兽派 Brutalism** | 15% | 强调点缀 | 核心价值、差异化优势、CTA 按钮 |
| **C 故障艺术 Glitch Art** | 15% | 科技点缀 | 原型展示 Tab7、成长系统、数据展示 |

### 2.2 风格选择理由

**为何选择混合方案而非单一风格：**

- **方案 A 单独使用**：主题契合度高但视觉冲击力不足，仍可能被识别为"精致但常规"
- **方案 B 单独使用**：反 AI 痕迹最强，但粗犷风格与柯基治愈主题存在反差，可能削弱产品温暖感
- **方案 C 单独使用**：视觉冲击最强，但赛博朋克风格与柯基温暖主题严重冲突，不适合面向大众评委
- **混合方案**：以杂志风为骨架确保主题契合，用野兽派和故障艺术作为视觉锚点创造记忆点，三者互补

## 3. 设计语言规范

### 3.1 配色系统

#### 3.1.1 主色调（杂志风基调）

| 用途 | 颜色值 | 说明 |
|------|--------|------|
| **主背景** | `#f5f0e8` | 米白纸张感，替代深色渐变背景 |
| **主色** | `#FF6B35` | 橙红（柯基色），比原 #FF9A5C 更饱和有力 |
| **主文字** | `#1a1a1a` | 深黑，高对比度 |
| **辅助色** | `#7EC8A0` | 薄荷绿，保留原成功色 |
| **次要文字** | `#666666` | 中灰 |
| **分割线** | `#1a1a1a` | 深黑细线，杂志感 |

#### 3.1.2 点缀色（故障艺术）

| 用途 | 颜色值 | 应用场景 |
|------|--------|---------|
| **霓虹绿** | `#00ff9f` | Tab7 原型展示边框、终端文字 |
| **霓虹粉** | `#ff006e` | 故障效果、RGB 分离阴影 |
| **电光蓝** | `#0095ff` | 故障效果、数据图表 |

#### 3.1.3 配色策略

- **整体浅色基调**：从深色渐变背景改为米白纸张感，降低 AI 生成的"科技感"痕迹
- **高对比度**：深黑文字 + 米白背景 + 橙红强调，符合杂志印刷美学
- **点缀克制**：霓虹色仅用于特定 section，避免视觉污染

### 3.2 字体系统

采用三字体混排策略，打破单一字体的 AI 痕迹：

| 用途 | 字体 | 字号 | 字重 | 说明 |
|------|------|------|------|------|
| **大标题** | Georgia (衬线) | 48-56px | 900 | 杂志封面感，hero 标题 |
| **章节号** | 系统无衬线 | 64-80px | 900 | 超大数字"01/02/03"作为视觉锚点 |
| **章节标题** | 系统无衬线 | 20-24px | 900 | 配合章节号使用 |
| **正文** | Nunito Sans | 13-15px | 400 | 保留原字体，阅读舒适 |
| **代码/数据** | monospace | 11-13px | 400 | 故障艺术 section 专用 |

### 3.3 布局系统

#### 3.3.1 网格规范

打破原有的等分网格，采用不对称布局：

| 布局类型 | 网格比例 | 应用场景 |
|---------|---------|---------|
| **Hero 分栏** | `3fr 2fr` | 左标题右柯基框 |
| **章节+内容** | `100px 1fr` 或 `1fr 100px` | 大数字章节号左右交替 |
| **错落网格** | `2fr 1fr 1fr` | 用户群体等非等高内容 |
| **野兽派卡片** | `1fr 1fr 1fr` + 硬阴影 | 核心价值强调 |
| **故障终端** | `80px 1fr` | Tab7 原型展示 |

#### 3.3.2 视觉节奏

通过以下维度变化创造错落有致的视觉节奏：

- **高度变化**：卡片高度不等高，有 40px / 60px / 80px 三档
- **宽度变化**：内容块宽度从 50% 到 90% 变化
- **位置变化**：章节号左右交替，内容块 margin 错落
- **样式变化**：普通卡片 / 野兽派卡片 / 故障边框交替出现

## 4. 各区域详细设计

### 4.1 Hero 区 — 杂志封面式不对称布局

**布局结构：**

```
┌─────────────────────────────────────────────┐
│ ISSUE 01·2026·06    刻记⁺创意展示    № 001  │ ← 顶部刊头
├─────────────────────────────────────────────┤
│                          ┌──────────────┐   │
│  — 治愈系记录应用        │   🐕 柯基    │   │
│                          │  摇尾中...   │   │
│  刻记⁺                   └──────────────┘   │
│  CORGIMEMO⁺              （粗边框+硬阴影）   │
│                                              │
│  「一个会陪伴你的治愈系记录应用」            │
├─────────────────────────────────────────────┤
│ ▌概览  柯基陪伴  身份自适应  三位一体  ...  │ ← 底部导航
└─────────────────────────────────────────────┘
```

**设计要点：**
- 左侧 3fr：大标题（Georgia 衬线 56px）+ 副标题 + 标语
- 右侧 2fr：柯基框（野兽派粗边框 + 硬阴影）
- 顶部刊头：ISSUE 编号 + 期号，营造杂志感
- 底部导航：横向文字导航，当前页用"▌"标记

### 4.2 Tab1 概览 — 大数字章节号 + 野兽派强调

**布局策略：A + B 混合**

- **章节 01 三大核心价值**：左侧大数字"01"（64px 橙红）+ 右侧野兽派卡片（粗边框+硬阴影）
- **章节 02 目标用户群体**：右侧大数字"02"（64px 薄荷绿）+ 左侧错落网格（2fr 1fr 1fr 非等高）
- **产品愿景引言**：上下双横线居中，Georgia 衬线斜体，杂志感强

### 4.3 Tab2 柯基陪伴 — 错落网格 + 横向滚动

**布局策略：A 杂志风**

- **8 种姿态**：打破 4×2 等高网格，采用错落布局（2 大 6 小，高度交替）
- **8 种表情**：横向滚动条展示，每个表情配大字号说明
- **情绪系统**：左侧公式说明 + 右侧动态进度条（故障艺术边框）
- **装扮系统**：卡片堆叠效果，hover 时展开

### 4.4 Tab3 身份自适应 — 纵向时间轴

**布局策略：A 杂志风**

- **三种身份对比**：纵向时间轴布局，非对称排列
- **问候语差异**：大引言式展示，每种身份用不同字体颜色
- **专属功能**：三栏非等高卡片，上班族/学生/老年高度递减

### 4.5 Tab4 三位一体 — 粗边框流程图

**布局策略：A + B 混合**

- **三大功能关系图**：野兽派粗边框流程图，硬阴影连接线
- **跨功能关联**：故障艺术风格的连线动画

### 4.6 Tab5 节日关怀 — 日历式网格

**布局策略：A 杂志风**

- **节日列表**：日历式网格布局，非均匀排列
- **柯基换装展示**：卡片堆叠效果，点击展开
- **冬至场景演示**：大场景插画 + 文字叙事

### 4.7 Tab6 用户故事 — 时间轴叙事

**布局策略：A 杂志风**

- **用户画像**：大引言式展示，配大字号用户语录
- **场景故事**：纵向时间轴，早中晚三个时段错落排列

### 4.8 Tab7 原型展示 — 故障艺术终端

**布局策略：A + C 混合**

- **章节号**：故障艺术风格"07"（RGB 分离效果）
- **手机模拟器**：霓虹绿边框 + 硬阴影（粉色），终端式操作提示
- **交互演示**：等宽字体显示操作日志，扫描线效果

## 5. 关键组件设计

### 5.1 章节号组件（Chapter Number）

```css
/* 大数字章节号 — 杂志风视觉锚点 */
.chapter-number {
    font-size: 64px;
    font-weight: 900;
    line-height: 0.85;
    color: var(--color-primary);
}

/* 左侧章节号 — 右边框分隔 */
.chapter-number.left {
    border-right: 3px solid #1a1a1a;
    padding-right: 12px;
}

/* 右侧章节号 — 左边框分隔（交替布局） */
.chapter-number.right {
    border-left: 3px solid #1a1a1a;
    padding-left: 12px;
    text-align: right;
}

/* 故障艺术章节号 — Tab7 专用 */
.chapter-number.glitch {
    color: #00ff9f;
    text-shadow: 2px 0 #ff006e, -2px 0 #0095ff;
}
```

### 5.2 野兽派卡片组件（Brutalism Card）

```css
/* 野兽派卡片 — 粗边框 + 硬阴影 */
.brutal-card {
    border: 3px solid #1a1a1a;
    padding: 12px;
    background: #fff;
    box-shadow: 5px 5px 0 var(--color-primary);
    transition: transform 0.2s, box-shadow 0.2s;
}

/* hover 效果 — 阴影偏移 */
.brutal-card:hover {
    transform: translate(-2px, -2px);
    box-shadow: 7px 7px 0 var(--color-primary);

/* 强调卡片 — 主色背景 */
.brutal-card.accent {
    background: var(--color-primary);
    color: #fff;
    box-shadow: 5px 5px 0 #1a1a1a;
}
```

### 5.3 故障艺术边框组件（Glitch Border）

```css
/* 故障艺术边框 — Tab7 专用 */
.glitch-border {

    box-shadow: 6px 6px 0 #ff006e;

    50% { text-shadow: -2px 0 #ff006e, 2px 0 #0095ff; }
}
```

## 6. 交互设计

### 6.1 滚动动画

- **章节号入场**：滚动到视区时，大数字从 0.5x 缩放到 1x，配合 opacity 0→1
- **卡片错落入场**：同 section 内卡片按 0.1s 间隔依次入场
- **故障效果触发**：Tab7 滚动到视区时，触发 RGB 分离动画

### 6.2 Hover 交互

- **野兽派卡片**：hover 时阴影偏移增大，卡片轻微上移
- **章节号**：hover 时颜色渐变切换
- **故障边框**：hover 时触发扫描线动画

### 6.3 页签切换

- **切换动画**：从右向左滑入，配合淡入效果
- **当前页标记**：底部导航用"▌"符号标记当前位置

## 7. 响应式设计

| 断点 | 布局调整 |
|------|---------|
| **桌面 ≥1200px** | 完整不对称布局，3 列网格 |
| **平板 768-1199px** | 简化为 2 列，章节号缩小至 48px |
| **手机 ≤767px** | 单列布局，章节号缩小至 36px，野兽派阴影减弱 |

## 8. 技术实现要点

### 8.1 文件结构

保持单文件 HTML 结构，所有 CSS/JS 内联，确保离线可运行。

### 8.2 CSS 变量重构

```css
:root {
    /* 杂志风主色 */
    --color-bg-paper: #f5f0e8;
    --color-primary-bold: #FF6B35;
    --color-text-ink: #1a1a1a;
    --color-accent-mint: #7EC8A0;

    /* 故障艺术点缀 */
    --color-neon-green: #00ff9f;
    --color-neon-pink: #ff006e;
    --color-electric-blue: #0095ff;

    /* 字体系统 */
    --font-serif: Georgia, 'Times New Roman', serif;
    --font-sans: 'Nunito Sans', sans-serif;
    --font-mono: 'Courier New', monospace;
}
```

### 8.3 性能优化

- **CSS 动画优先**：所有动画使用 CSS transform/opacity，避免 JS 重排
- **图片懒加载**：柯基帧动画图片使用 loading="lazy"
- **字体优化**：Google Fonts 使用 preconnect + display=swap

## 10. 验收标准

### 10.1 设计验收

- [ ] Hero 区采用不对称布局，无居中元素
- [ ] 每个 Tab 至少有 1 个大数字章节号
- [ ] 核心价值卡片采用野兽派粗边框样式
- [ ] Tab7 原型展示采用故障艺术风格
- [ ] 视觉节奏有明确的高低错落变化

### 10.2 技术验收

- [ ] 单文件 HTML，离线可运行
- [ ] 响应式设计覆盖桌面/平板/手机三端
- [ ] 所有动画支持 `prefers-reduced-motion`
- [ ] 柯基帧动画图片路径正确（相对路径）
- [ ] 页面加载性能良好（首屏 < 2s）

### 10.3 反 AI 痕迹验收

- [ ] 第一眼无法识别为 AI 生成
- [ ] 滚动 5 屏后无审美疲劳
- [ ] 至少 3 个独特视觉记忆点（大数字、粗边框、故障效果）
- [ ] 整体设计有"匠心感"和"设计感"
