# 搜索框滚动显隐交互设计

## 概述

待办页面的搜索框当前始终显示，占据屏幕顶部空间。本次修改为：搜索框默认隐藏，根据用户滚动方向动态滑入/滑出，释放更多列表显示空间。

## 交互规则

| 条件 | 搜索框状态 |
|------|-----------|
| 默认（页面加载） | 隐藏 |
| 手指向下滑（列表内容往上走） | 滑入显示 |
| 手指向上滑（列表内容往下走） | 滑出隐藏 |
| 到达列表顶部 | 隐藏 |
| 正在输入搜索词 | 始终显示（不受滚动影响） |
| 搜索结果为空 | 始终显示（不受滚动影响） |

### 状态公式

```kotlin
val searchBarVisible = (isScrollingDown && !isAtTop) || searchQuery.isNotBlank()
```

- `isScrollingDown`：手指向下滑动（列表内容向上走，即 `isScrollingUp = false`，用 LazyListState 的 firstVisibleItemIndex/offset 变化判断）
- `isAtTop`：列表滚到顶部（`firstVisibleItemIndex == 0 && firstVisibleItemScrollOffset == 0`）
- `searchQuery.isNotBlank()`：有搜索关键词时不隐藏

## 技术方案

### 选定方案：AnimatedVisibility + slideInVertically/slideOutVertically

搜索框作为一个整体从顶部上方滑入/滑出，列表自动填充空间。

### 动画参数

| 参数 | 值 | 说明 |
|------|-----|------|
| slideInVertically | `initialOffsetY = { -it }` | 从顶部上方滑入 |
| slideOutVertically | `targetOffsetY = { -it }` | 向顶部上方滑出 |
| 动画时长 | 200ms | 与项目其他过渡动画一致 |
| 缓动曲线 | FastOutSlowInEasing | 标准Material缓动 |

### 修改文件

| 文件 | 修改内容 |
|------|---------|
| ReorderableLazyColumn.kt | 暴露内部 `LazyListState`（添加 `listState` 参数，默认 `rememberLazyListState()`） |
| HomeScreen.kt | 使用 `derivedStateOf` 计算滚动方向 + AnimatedVisibility 包裹搜索框 |

### 不修改的文件

- **SearchBar.kt**：搜索框组件本身不变
- **HomeViewModel.kt**：搜索状态管理不变，searchQuery 已足够
- **排序按钮(📊)**：本次仍在 SearchBar 的 trailingIcon 中，随搜索框一起隐藏/显示，后续任务调整位置

### 详细实现

#### 1. ReorderableLazyColumn 暴露 LazyListState

当前 `ReorderableLazyColumn` 内部自行创建 `LazyListState`，外部无法读取滚动状态。

**修改**：添加可选参数 `listState: LazyListState = rememberLazyListState()`，内部使用传入的 listState 替代自行创建的。

```kotlin
fun <T> ReorderableLazyColumn(
    items: List<T>,
    isDragEnabled: Boolean,
    isDraggable: (T) -> Boolean = { true },
    isPinned: (T) -> Boolean,
    key: (T) -> Any,
    onReorder: (fromIndex: Int, toIndex: Int, crossedPinnedZone: Boolean) -> Unit,
    listState: LazyListState = rememberLazyListState(),  // 新增
    ...
)
```

内部将 `val listState: LazyListState = rememberLazyListState()` 删除，改用参数。

#### 2. HomeScreen 滚动方向检测

从 `ReorderableLazyColumn` 获取 `listState`，使用 `LaunchedEffect + snapshotFlow` 跟踪前后滚动位置变化，再用 `derivedStateOf` 计算当前可见性：

```kotlin
val lazyListState = rememberLazyListState()

// 记录上一次滚动位置，用于判断方向
var prevIndex by remember { mutableIntStateOf(0) }
var prevOffset by remember { mutableIntStateOf(0) }
var isScrollingDown by remember { mutableStateOf(false) }

// 通过 snapshotFlow 跟踪滚动位置变化，计算方向
LaunchedEffect(lazyListState) {
    snapshotFlow {
        lazyListState.firstVisibleItemIndex to lazyListState.firstVisibleItemScrollOffset
    }.collect { (currentIndex, currentOffset) ->
        isScrollingDown = currentIndex < prevIndex ||
            (currentIndex == prevIndex && currentOffset < prevOffset)
        prevIndex = currentIndex
        prevOffset = currentOffset
    }
}

val isAtTop by remember {
    derivedStateOf {
        lazyListState.firstVisibleItemIndex == 0 &&
            lazyListState.firstVisibleItemScrollOffset == 0
    }
}

val searchBarVisible by remember {
    derivedStateOf {
        (isScrollingDown && !isAtTop) || searchQuery.isNotBlank()
    }
}
```

#### 3. AnimatedVisibility 包裹搜索框

```kotlin
AnimatedVisibility(
    visible = searchBarVisible,
    enter = slideInVertically(
        initialOffsetY = { -it },
        animationSpec = tween(200, easing = FastOutSlowInEasing)
    ),
    exit = slideOutVertically(
        targetOffsetY = { -it },
        animationSpec = tween(200, easing = FastOutSlowInEasing)
    )
) {
    SearchBar(
        query = searchQuery,
        onQueryChange = { viewModel.updateSearchQuery(it) },
        onClear = { viewModel.clearSearch() },
        modifier = Modifier
            .fillMaxWidth()
            .padding(horizontal = 20.dp, vertical = 8.dp),
        trailingIcon = { /* 排序按钮 */ }
    )
}
```

#### 4. 将 listState 传入 ReorderableLazyColumn

```kotlin
ReorderableLazyColumn(
    items = displayItems,
    listState = lazyListState,  // 新增：传入外部创建的 listState
    isDragEnabled = ...,
    ...
)
```

## 边界情况处理

| 场景 | 处理方式 |
|------|---------|
| 搜索框隐藏时输入法仍打开 | 清空搜索词时收起键盘，隐藏后不再响应 |
| 搜索框动画进行中快速反向滚动 | AnimatedVisibility 自动处理反向动画 |
| 拖拽排序中滚动 | 不受影响，拖拽时滚动方向正常检测 |
| 下拉刷新 | SwipeRefresh 与搜索框独立，不互相影响 |
| 列表项不足一屏 | isAtTop 始终为 true，搜索框保持隐藏 |
