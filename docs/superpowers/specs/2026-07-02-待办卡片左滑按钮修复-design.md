# 待办卡片左滑按钮修复设计

> **创建日期**：2026-07-02
> **主题**：修复待办卡片左滑操作面板中分享/置顶/删除三个按钮无响应的问题，并实现置顶按钮根据卡片置顶状态动态切换文字与图标
> **约束**：不改动现有的待办卡片拖拽逻辑（`detectHorizontalDragGestures` 回调与 `ReorderableLazyColumn` 排序逻辑全部保留）

---

## 一、问题诊断

### 问题 1：左滑按钮无响应

**根本原因有两层 pointerInput 拦截**：

#### 拦截层 1：内容层 Box 的 modifier 链顺序问题

位置：[`SwipeableTodoBox.kt#L254-L405`](../../app/src/main/java/com/corgimemo/app/ui/components/SwipeableTodoBox.kt)

```kotlin
// 当前 modifier 链顺序
Modifier
    .pointerInput(...) { detectHorizontalDragGestures(...) }   // 外层
    .offset { IntOffset(cardOffsetX.value.roundToInt(), 0) }  // 中层
    .zIndex(10f)                                               // 内层
```

- Compose modifier 链从外到内，`.pointerInput` 在 `.offset` 之前应用
- `.offset` 只偏移其内部内容，不偏移外部的 `.pointerInput` 的命中测试区域
- 导致 `.pointerInput` 的命中区域始终为 `[0, cardWidth]`，不随卡片视觉偏移而移动
- 展开后按钮位于 `[cardWidth-216, cardWidth]`，与内容层 pointerInput 命中区域重叠
- 内容层 `zIndex(10f)` 高于按钮层 `zIndex(3f/2f/1f)`，事件优先派发给内容层
- `detectHorizontalDragGestures` 在 `onDragStart` 中消费 down 事件 → 按钮 `clickable` 收不到事件

#### 拦截层 2：外层 Layout 的 pointerInput 无条件消费 down

位置：[`SwipeableTodoBox.kt#L243-L250`](../../app/src/main/java/com/corgimemo/app/ui/components/SwipeableTodoBox.kt)

```kotlin
.pointerInput(isEnabled, isExpanded) {
    if (!isEnabled || !isExpanded) return@pointerInput
    awaitEachGesture {
        val down = awaitFirstDown(requireUnconsumed = false)
        down.consume()   // 无条件消费，本意阻止 ModalNavigationDrawer 抢手势
    }
}
```

- 本意：阻止父级 `ModalNavigationDrawer` 看到 down 后启动 Drawer 打开手势
- 副作用：过度消费事件，按钮区域 down 也被消费，按钮 `clickable` 无法触发

### 问题 2：置顶状态显示不切换

位置：[`SwipeableTodoBox.kt#L180-L195`](../../app/src/main/java/com/corgimemo/app/ui/components/SwipeableTodoBox.kt)

```kotlin
val buttons = remember {   // remember 无 key，不会响应 isPinned 变化
    listOf(
        SwipeButtonConfig("分享", R.color.ui_swipe_share, Icons.Outlined.Share, 3f, ...),
        SwipeButtonConfig("置顶", R.color.ui_primary, Icons.Outlined.PushPin, 2f, ...),  // 硬编码
        SwipeButtonConfig("删除", R.color.ui_swipe_delete, Icons.Outlined.Delete, 1f, ...)
    )
}
```

- `buttons` 列表用 `remember { ... }` 硬编码了 `label="置顶"` 和 `Icons.Outlined.PushPin`
- 未根据 `todo.isPinned` 状态动态切换
- 导致置顶后按钮仍显示"置顶"文字和空心图钉图标

## 二、修复目标

1. **按钮点击响应**：三个左滑按钮（分享/置顶/删除）均能正常触发已接入的业务逻辑
2. **置顶状态显示**：置顶按钮根据 `todo.isPinned` 状态动态切换
   - 未置顶：文字"置顶" + `Icons.Outlined.PushPin`（空心图钉）
   - 已置顶：文字"取消置顶" + `Icons.Filled.PushPin`（实心图钉）
   - 背景色保持橙色 `R.color.ui_primary` 不变
3. **右上角置顶图标**：保留 [`TodoListItem.kt#L621-L626`](../../app/src/main/java/com/corgimemo/app/ui/components/TodoListItem.kt) 已实现的右上角置顶图标显示

### 修复约束

- **不改动 `detectHorizontalDragGestures` 的任何回调**（onDragStart / onDragEnd / onDragCancel / onDrag）
- **不改动 `ReorderableLazyColumn` 的拖拽排序逻辑**
- **不改动 `HomeScreen.kt` 中三个按钮的 onClick 业务接入**（已正确实现）

## 四、数据流

```
HomeViewModel.todoList (StateFlow<List<TodoItem>>)
    ↓ 每个 todo 携带 isPinned 字段
HomeScreen.kt (items 列表渲染)
    ↓ todo.isPinned 作为参数传入
SwipeableTodoBox(isPinned = todo.isPinned, ...)
    ↓ 内部根据 isPinned 动态生成 buttons 列表
buttons = remember(isPinned) {
    listOf(
        SwipeButtonConfig("分享", ...),                    // 固定
        SwipeButtonConfig(                                    // 动态切换
            label = if (isPinned) "取消置顶" else "置顶",
            icon = if (isPinned) Icons.Filled.PushPin
                   else Icons.Outlined.PushPin,
            backgroundColorRes = R.color.ui_primary          // 背景色保持橙色
        ),
        SwipeButtonConfig("删除", ...)                      // 固定
    )
}
```

**关键设计决策**：
1. `remember(isPinned)` 而非 `remember`：将 `isPinned` 作为 remember 的 key，当置顶状态变化时自动重建 buttons 列表，确保 UI 即时刷新
2. `isPinned` 参数透传：通过 SwipeableTodoBox 的函数签名新增参数，由 HomeScreen 调用时传入，符合单向数据流
3. 按钮背景色不变：取消置顶状态仍使用橙色 `R.color.ui_primary`，仅通过文字和图标填充态区分

## 五、修复细节与代码变更点

### 改动点 A：内容层 Box modifier 链顺序调整

**位置**：`SwipeableTodoBox.kt` 第 254-405 行

**变更前**：

**变更后**：
```kotlin
Box(
    modifier = Modifier
        .offset { IntOffset(cardOffsetX.value.roundToInt(), 0) }   // ← 移到最前
        .pointerInput(isEnabled, isExpanded) {                      // ← 命中区域跟随 offset
            if (!isEnabled) return@pointerInput
            detectHorizontalDragGestures(...)                       // 拖拽回调零修改

        .zIndex(10f)

```

**效果**：展开后内容层 pointerInput 命中区域从 `[0, cardWidth]` 变为 `[-216, cardWidth-216]`，与按钮区域 `[cardWidth-216, cardWidth]` 自然分离。

### 改动点 B：外层 Layout pointerInput 智能分流

**位置**：`SwipeableTodoBox.kt` 第 243-250 行

**变更前**：
```kotlin
.pointerInput(isEnabled, isExpanded) {
    if (!isEnabled || !isExpanded) return@pointerInput
    awaitEachGesture {
        val down = awaitFirstDown(requireUnconsumed = false)
        down.consume()   // 无条件消费
    }
}
```

**变更后**：
```kotlin
.pointerInput(isEnabled, isExpanded) {
    if (!isEnabled || !isExpanded) return@pointerInput
    awaitEachGesture {
        val down = awaitFirstDown(requireUnconsumed = false)
        // 新增：down 落在按钮区域则放行，让事件透传给按钮 clickable
        val buttonAreaStartX = size.width - actionsWidthPx
        val isButtonArea = down.position.x >= buttonAreaStartX
        if (!isButtonArea) {
            down.consume()   // 仅消费非按钮区域，阻止 ModalNavigationDrawer 抢手势
        }
    }
}
```

**效果**：保留对 ModalNavigationDrawer 的手势隔离（非按钮区域仍消费 down），同时放行按钮区域的点击事件。

### 改动点 C：SwipeableTodoBox 新增 isPinned 参数 + buttons 动态生成

**位置**：`SwipeableTodoBox.kt` 函数签名（第 117 行附近）与 buttons 列表（第 180-195 行）

**变更前**：
```kotlin

            SwipeButtonConfig("删除", R.color.ui_swipe_delete, Icons.Outlined.Delete, 1f, ...)
        )
    }

```

**变更后**：
```kotlin
@Composable
fun SwipeableTodoBox(
    isEnabled: Boolean = true,
    isExpanded: Boolean = false,
    isPinned: Boolean = false,          // ← 新增参数
    onShareClick: () -> Unit = {},
    onPinClick: () -> Unit = {},
    onDeleteClick: () -> Unit = {},
    content: @Composable (isClickBlocked: Boolean) -> Unit

    val buttons = remember(isPinned) {  // ← remember 加入 isPinned 作为 key
        listOf(
            SwipeButtonConfig("分享", R.color.ui_swipe_share, Icons.Outlined.Share, 3f,
                RoundedCornerShape(topStart = cornerRadiusDp, bottomStart = cornerRadiusDp)),
            SwipeButtonConfig(
                label = if (isPinned) "取消置顶" else "置顶",                       // ← 动态切换文字
                icon = if (isPinned) Icons.Filled.PushPin else Icons.Outlined.PushPin, // ← 动态切换图标
                backgroundColorRes = R.color.ui_primary,                              // ← 背景色保持橙色
                zIndex = 2f,
                shape = RoundedCornerShape(0.dp)
            ),
            SwipeButtonConfig("删除", R.color.ui_swipe_delete, Icons.Outlined.Delete, 1f,
                RoundedCornerShape(topEnd = cornerRadiusDp, bottomEnd = cornerRadiusDp))
        )
    }
}
```

## 七、风险评估

| 风险点 | 可能性 | 影响 | 缓解措施 |
|-------|-------|------|---------|
| modifier 链顺序调整后拖拽命中区域变化 | 低 | 拖拽时手指需在卡片视觉区域内 | modifier 链顺序调整后，pointerInput 命中区域跟随视觉，符合直觉 |
| 外层 pointerInput 智能分流误判按钮区域 | 低 | 边缘点击可能失效 | 使用 `size.width - actionsWidthPx` 精确计算边界 |
| `Icons.Filled.PushPin` 在当前 Material Icons 版本中不可用 | 低 | 编译错误 | 项目已使用 `Icons.Outlined.PushPin`，同包下 Filled 版本必然存在 |
| remember(isPinned) 重建列表导致性能问题 | 极低 | 列表重建开销 | 仅 3 个 SwipeButtonConfig 对象，开销可忽略 |

## 八、后续可优化方向

1. **左滑按钮点击反馈**：当前按钮点击后仅收起面板，可考虑增加点击水波纹反馈
2. **置顶状态过渡动画**：置顶/取消置顶切换时，图标可增加 morphing 动画
3. **按钮长按操作**：分享按钮可支持长按直接复制图片到剪贴板
4. **批量删除确认**：当用户连续删除多个待办时，可考虑合并 Snackbar 提示
