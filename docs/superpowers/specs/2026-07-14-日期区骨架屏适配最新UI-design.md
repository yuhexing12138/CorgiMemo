# 日期区骨架屏适配最新 UI 设计

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:writing-plans to create implementation plan after spec approved.

**目标:** 让 `SpecialDateSkeleton` 与日期页最新 UI（3 段分组 + 置顶卡 + 3 列卡片）完全对齐，加载时无视觉跳变。

**架构:** 复用 `Skeleton.kt` 中的 `SkeletonSectionHeader`、`SkeletonCircle`、`SkeletonText`、`SkeletonCard` 等通用骨架组件；新增 `DateItemSkeleton` 与 `PinnedDateItemSkeleton` 两个私有 Composable，分别对应 `SpecialDateCard` 与 `PinnedDateCard` 的 3 列布局；硬编码与 `DateSectionHeader` 一致的分组颜色（不引入新的颜色常量文件）。

**技术栈:** Jetpack Compose（BOM 2026.04.01 + Compose 1.9.2）、Material3、LazyColumn

---

## 1. 背景与问题

日期页（`SpecialDateScreen`）在 2026-07-12 ~ 2026-07-14 期间经历了 3 次较大重构：

| 重构日期 | 关键变化 | 涉及文件 |
|---------|---------|---------|
| 2026-07-12 | 日期页 UI 全面重构（搜索框 / 3 段分组 / 卡片新视觉） | `SpecialDateScreen.kt`、`SpecialDateCard.kt` |
| 2026-07-13 | 卡片尺寸重设：圆形图 56→48dp、emoji 28→24sp、padding 16→12dp；移除"还有/已经/已归档 + 日期"行 | `SpecialDateCard.kt` |
| 2026-07-14 | 新增置顶日期卡 `PinnedDateCard`；分组语义"已过期"→"已归档" | `PinnedDateCard.kt`、`DateSectionHeader.kt` |

当前 `SpecialDateSkeleton.kt`（2026-07-13 版）已严重滞后于真实 UI：

- **左侧 12dp 优先级圆点** → 真实 UI 已改为 48dp 圆形图片区
- **两行文本骨架**（标题 + 副标题） → 真实 UI 已简化为单行标题 + 单行时间信息
- **右侧"▭▭天后"窄条骨架** → 真实 UI 已改为 22sp 大数字 + 11sp 单位两行结构
- **缺失置顶日期卡骨架** → 真实 UI 在最顶部多一个 `PinnedDateCard`
- **分区头用 `DateGroupTitleSkeleton`**（Box + SkeletonText 0.35f）→ 真实 UI 用 `DateSectionHeader`（图标 + 标签 + 计数）
- **未按分组颜色渲染分区头** → 真实 UI 倒计时/正计时/已归档分别用 #FF9A5C/#7EC8A0/#999999

这些差异导致首次加载（`!isDataInitialized`）时，骨架屏与真实内容布局错位，**从骨架屏切换到真实内容时出现明显的视觉跳变**，影响体验。

## 2. 设计目标

| 编号 | 目标 | 验收标准 |
|------|------|---------|
| G1 | 骨架屏与真实 UI 布局 1:1 对齐 | 切换到真实内容时无明显位置/高度跳变 |
| G2 | 复用 `Skeleton.kt` 现有组件 | 不新增颜色/形状常量；不破坏 `SkeletonSectionHeader` 调用约定 |
| G3 | 涵盖 3 段分组 + 置顶卡 + 普通卡 | 加载时完整还原页面信息层级 |
| G4 | 保持与日期页一致的分组颜色 | 倒计时 #FF9A5C / 正计时 #7EC8A0 / 已归档 #999999 |
| G5 | 不影响 `isDataInitialized` 流程 | 仅修改 `SpecialDateSkeleton.kt` 单一文件；签名不变 |

## 3. 方案对比与选择

### 3.1 方案 A：扩展现有 `SpecialDateSkeleton.kt`（推荐 ✅）

**做法:** 在 `SpecialDateSkeleton.kt` 内**重写**为新版结构，新增两个私有 Composable（`DateItemSkeleton`、`PinnedDateItemSkeleton`），删除旧的 `DateGroupTitleSkeleton` 与 `DateItemSkeleton`，复用 `SkeletonSectionHeader` 替代分区头。

**优点:**
- 改动范围最小（仅 1 个文件）
- 复用度高，沿用 `Skeleton.kt` 通用组件
- 分组颜色硬编码在文件内，与 `DateSectionHeader.kt` 的 `when` 分支保持同步（不引入新颜色对象，避免跨文件依赖）

**缺点:**
- 文件略长（约 130 行），但每个 Composable 职责单一，可读性可控

### 3.2 方案 B：拆分为 3 个新文件

**做法:** 新建 `DateItemSkeleton.kt`、`PinnedDateItemSkeleton.kt`、`DateSectionHeaderSkeleton.kt`，由 `SpecialDateSkeleton.kt` 组合。

**优点:** 文件拆分更细

**缺点:** 引入 3 个仅 1 处调用的私有组件，反而增加维护成本；不符合"避免过度抽象"原则 → **否决**

### 3.3 方案 C：抽到 `Skeleton.kt` 作为通用组件

**做法:** 把 `DateItemSkeleton` 等提到 `Skeleton.kt`，让其他页面（待办/灵感）复用。

**优点:** 长期复用潜力

**缺点:** 当前其他页面骨架屏差异较大（待办用 `TodoItemSkeleton`），且日期卡 3 列布局（48dp 圆 + weight=1f + 22sp 大数字）专属业务结构，泛化会导致参数爆炸 → **否决**

### 3.4 最终选择

**采用方案 A**：在 `SpecialDateSkeleton.kt` 内集中实现新版骨架屏。

## 4. 详细设计

### 4.1 组件结构

```
SpecialDateSkeleton (public, 签名不变)
├── SkeletonSearchBar      // 复用
├── PinnedDateItemSkeleton // 新增 (private)
│   ├── 左区: SkeletonText (0.4f, 16dp) // 标题
│   ├── 左区: SkeletonText (0.5f, 32dp) // 大数字
│   └── 右区: SkeletonCircle (56dp)     // 头像
├── SkeletonSectionHeader (倒计时, #FF9A5C)   // 复用
├── DateItemSkeleton × 2   // 新增 (private)
│   ├── 左: SkeletonCircle (48dp)        // 圆形图
│   ├── 中: SkeletonText (0.5f, 16dp)    // 标题
│   ├── 中: SkeletonText (0.4f, 14dp)    // 时间
│   └── 右: SkeletonText (0.18f, 22sp 容器) // 大数字
│       + SkeletonText (0.12f, 11sp 容器) // 单位
├── SkeletonSectionHeader (正计时, #7EC8A0)   // 复用
├── DateItemSkeleton × 1   // 复用
├── SkeletonSectionHeader (已归档, #999999)   // 复用
└── DateItemSkeleton × 2   // 复用
```

### 4.2 `DateItemSkeleton` 详细规格

模拟 `SpecialDateCard`（重构版）的 3 列布局：

| 区域 | 元素 | 尺寸 | 备注 |
|------|------|------|------|
| 左侧 | `SkeletonCircle` | 48dp | 对应 `Box.size(48.dp)` 圆形图 |
| 中间 | `SkeletonText` | width=0.5f, height=16dp | 标题（16sp Medium） |
| 中间 | `SkeletonText` | width=0.4f, height=14dp | 时间信息（14sp Medium） |
| 右侧 | `Column` | horizontalAlignment=End | 大数字 + 单位两行 |
| - 右侧上行 | `SkeletonText` | width=0.18f, height=22dp | 对应 22sp Bold 大数字 |
| - 右侧下行 | `SkeletonText` | width=0.12f, height=11dp | 对应 11sp 单位 |

**容器:** 用 `Card` 包裹（非 `SkeletonCard`，原因见 4.4）
- shape = `RoundedCornerShape(20.dp)`（与真实卡一致）
- containerColor = `MaterialTheme.colorScheme.surface`
- 内部 Row padding = 12dp（与真实卡一致）

### 4.3 `PinnedDateItemSkeleton` 详细规格

模拟 `PinnedDateCard` 的左 + 中 + 右三区布局：

| 区域 | 元素 | 尺寸 | 备注 |
|------|------|------|------|
| 左区 (widthIn min=70dp) | `SkeletonText` | width=0.4f, height=16dp | 标题 |
| 左区 | `SkeletonText` | width=0.5f, height=32dp | 大数字（40sp Bold 的视觉占比） |
| 中区 (weight=1f) | `SkeletonText` | width=0.6f, height=14dp | 单行倒数 |
| 右区 | `SkeletonCircle` | 56dp | 头像（与真实卡一致） |

**容器:** 与 `DateItemSkeleton` 一致（Card + Row + 16dp padding）

### 4.4 关键设计决策

#### 4.4.1 不复用 `SkeletonCard` 的原因

`SkeletonCard`（`Skeleton.kt:155-168`）用 `clip + background + padding(16.dp)` 实现，与真实 `SpecialDateCard` 的 `Card` + `padding(12.dp)` 视觉差异：
- `SkeletonCard` 是单色灰底 + 16dp padding
- 真实卡是 surface 色 + 2dp elevation + 12dp padding

→ **`DateItemSkeleton` / `PinnedDateItemSkeleton` 改用 `Card` + `surface` 色，与真实卡保持一致**，避免从骨架到真实内容的颜色跳变。

#### 4.4.2 分组颜色硬编码

| 分组 | 颜色 | 来源 |
|------|------|------|
| 倒计时 | `Color(0xFFFF9A5C)` | `DateSectionHeader.kt:39` |
| 正计时 | `Color(0xFF7EC8A0)` | `DateSectionHeader.kt:40` |
| 已归档 | `Color(0xFF999999)` | `DateSectionHeader.kt:42` |

**为何不抽到 `SectionHeaderColors`：** 日期分组颜色与待办页 `SectionHeaderColors`（Pinned/Pending/Completed）语义不同，混入会污染现有常量。骨架屏**直接硬编码**并加注释与 `DateSectionHeader.kt:39-43` 同步，避免单点改动遗漏。

#### 4.4.3 卡片数量分布

保持与当前 `SpecialDateSkeleton` 一致：
- 倒计时：2 张
- 正计时：1 张
- 已归档：2 张（折叠态下不渲染卡片，但骨架屏始终显示，模拟折叠按钮可点击）

#### 4.4.4 置顶卡位置

置于搜索栏之后、第一个分组头（倒计时）之前，与 `SpecialDateScreen.kt` 的渲染顺序一致。

### 4.5 与现有 `SkeletonSectionHeader` 的关系

`SkeletonSectionHeader`（`Skeleton.kt:247-281`）布局：
```
[▼] [标签] [▭计数]
```

`DateSectionHeader`（`DateSectionHeader.kt:31-56`）底层也是 `CollapsibleSectionHeader`（同结构）。

→ **直接复用 `SkeletonSectionHeader(label, color, modifier)`**，3 处调用：
```kotlin
SkeletonSectionHeader(label = "倒计时", color = Color(0xFFFF9A5C))
SkeletonSectionHeader(label = "正计时", color = Color(0xFF7EC8A0))
SkeletonSectionHeader(label = "已归档", color = Color(0xFF999999))
```

## 8. 风险与缓解

| 风险 | 等级 | 缓解措施 |
|------|------|---------|
| 分组颜色后续被 `DateSectionHeader.kt` 改动而不同步 | 低 | 在骨架屏代码处加注释指向 `DateSectionHeader.kt:39-43` |
| `Card` 比 `SkeletonCard` 略影响性能 | 极低 | 仅 6 个 Card（1 置顶 + 5 普通），无感知 |
| 未来 `PinnedDateCard` 视觉再变 | 中 | `PinnedDateItemSkeleton` 拆为独立 Composable，便于后续单点调整 |
