## 一、需求概述

将"我的待办"页面右上角的柯基爪子图标替换为竖向三点图标（MoreVert），点击后弹出功能菜单弹窗。菜单包含 7 个功能项，按垂直顺序排列。同时将"灵感"和"日期"页面的柯基爪子也替换为三点图标（点击提示开发中），"我的"页面保留柯基爪子。

### 功能项清单（按顺序）

| 序号 | 文字（切换态） | Material Icon | 行为 | 状态 |
|------|---------------|---------------|------|------|
| 1 | 隐藏详情 ↔ 显示详情 | `Icons.Outlined.Visibility` | 切换卡片简化/完整显示，持久化 | 启用 |
| 2 | 隐藏已完成 ↔ 显示已完成 | `Icons.Outlined.CheckBox` | 隐藏/显示所有已完成项，持久化 | 启用 |
| 3 | 待办排序 | `Icons.Outlined.BarChart` | 打开排序弹窗（迁移自搜索框） | 启用 |
| 4 | 管理分组 | `Icons.Outlined.CreateNewFolder` | Toast 提示"功能开发中..." | 占位 |
| 5 | 批量选择 | `Icons.Outlined.SelectAll` | 调用 `enterBatchMode()` 进入批量模式 | 启用 |
| 6 | 分享 | `Icons.Outlined.Share` | Toast 提示"功能开发中..." | 占位 |
| 7 | 创建待办副本 | `Icons.Outlined.ContentCopy` | Toast 提示"功能开发中..." | 占位 |

### 各页面右侧图标配置

| 页面 | 右侧图标 | 点击行为 |
|------|---------|---------|
| 我的待办 | `Icons.Default.MoreVert`（三点） | 弹出功能菜单（7项） |
| 灵感 | `Icons.Default.MoreVert`（三点） | Toast 提示"功能开发中..." |
| 日期 | `Icons.Default.MoreVert`（三点） | Toast 提示"功能开发中..." |
| 我的 | `Icons.Default.Pets`（柯基爪子） | 进入柯基详情页（保持现状） |

### 菜单样式

采用 **样式 A：经典下拉菜单**（Material 3 DropdownMenu 风格）：
- 白底卡片 + 圆角 12dp + 阴影
- 每项一行：图标（20dp）+ 文字（14sp），水平间距 12dp，垂直内边距 12dp
- 宽度 220dp
- 占位项 opacity = 0.4（仍可点击）
- 点击任意项后菜单立即关闭

### 菜单交互行为

- 点击三点图标弹出菜单，带缩放+淡入动画（`DropdownMenu` 默认动画）
- 点击菜单项后立即关闭菜单
- 点击遮罩区域关闭菜单
- 占位项点击后 Toast 提示"功能开发中..."

## 三、状态管理与数据流

### 3.1 新增状态

| 状态名 | 类型 | 存放位置 | 持久化键 | 语义 |
|--------|------|---------|---------|------|
| `hideDetails` | Boolean | HomeViewModel + DataStore | `hide_details` | 待办卡片是否简化显示 |
| `hideCompletedItems` | Boolean | HomeViewModel + DataStore | `hide_completed_items` | 是否隐藏所有已完成项 |

### 3.2 与现有状态的语义区分

| 状态名 | 类型 | 语义 | 影响范围 |
|--------|------|------|---------|
| `showCompleted`（现有） | Boolean | 已完成折叠区域是否展开 | 控制"已完成(N)"按钮点击后是否展开列表 |
| `hideCompletedItems`（新增） | Boolean | 是否隐藏所有已完成项 | true 时：隐藏折叠区域 + 列表中所有已勾选待办 |
| `hideDetails`（新增） | Boolean | 待办卡片是否简化显示 | true 时：卡片仅显示标题/提醒/优先级/置顶/勾选框 |

### 3.3 状态组合逻辑

- `hideCompletedItems=true` 时，`showCompleted` 强制视为 false（折叠区域也隐藏）
- `hideCompletedItems=true` 时，不显示"已完成(N)"分隔符按钮
- `hideCompletedItems` 切回 false 时，`showCompleted` 保持原值
- `hideDetails` 与 `hideCompletedItems` 相互独立，可任意组合

### 3.4 filteredTodos 数据流影响

```
现有：filteredTodos = pendingTodos + (showCompleted ? completedTodos : [])

改造后：
filteredTodos = pendingTodos + (hideCompletedItems ? [] : (showCompleted ? completedTodos : []))
```

即 `hideCompletedItems=true` 时直接过滤掉所有已完成项，不进入 `displayItems` 列表。同时 `completedCount` 在 `hideCompletedItems=true` 时视为 0。

### 3.5 HomeViewModel 新增方法

```kotlin
fun toggleHideDetails()           // 切换 hideDetails 并持久化
fun toggleHideCompletedItems()    // 切换 hideCompletedItems 并持久化
```

### 4.2 简化模式（isSimpleMode=true）显示规则

**显示的元素：**
- 父待办标题（必显示）
- 提醒时间（如有设置）
- 优先级图标（圆点或图标）
- 置顶图标（如已置顶，位于右上角）
- 完成勾选框（始终显示）

**隐藏的元素：**
- 描述内容（`todo.content`）
- 子任务进度（"子任务 2/5"）
- 子任务列表（展开的子项，强制收起，忽略 expandState）
- 分类标签（📁 工作）
- 关联提示（如有）
- 图片/附件预览（如有）

### 4.3 置顶图标位置调整

- **改造前**：置顶图标在 meta 行，带文字"置顶"
- **改造后**：置顶图标位于卡片右上角，使用 `Icons.Outlined.PushPin`，只图标无文字
- 使用 `Box` + `Alignment.TopEnd` 定位
- 两种模式（完整/简化）下置顶图标位置一致，都在右上角
- 内容区域需 `paddingEnd` 预留置顶图标空间

## 六、TodoMenuDropdown 组件设计

### 6.1 组件签名

```kotlin
@Composable
fun TodoMenuDropdown(
    expanded: Boolean,                              // 菜单是否展开
    onDismiss: () -> Unit,                          // 关闭菜单回调
    hideDetails: Boolean,                           // 当前 hideDetails 状态
    onToggleHideDetails: () -> Unit,                // 切换 hideDetails
    hideCompletedItems: Boolean,                    // 当前 hideCompletedItems 状态
    onToggleHideCompletedItems: () -> Unit,         // 切换 hideCompletedItems
    onSortClick: () -> Unit,                        // 打开排序弹窗
    onBatchSelectClick: () -> Unit,                 // 进入批量模式
    onPlaceholderClick: () -> Unit,                 // 占位项点击回调（Toast 提示）
)
```

### 6.2 实现要点

- 使用 Material 3 `DropdownMenu`（`expanded` + `onDismissRequest`）
- 每项使用 `DropdownMenuItem`，内含 `Icon` + `Text`
- 占位项（管理分组/分享/创建待办副本）保持 `enabled = true`（可点击），通过 `Modifier.alpha(0.4f)` 降低透明度，点击后触发 Toast 而非执行实际功能
- 切换类项文字根据状态动态变化（"隐藏详情" ↔ "显示详情"）
- 点击任意项后调用 `onDismiss()` 关闭菜单

### 6.3 占位项 Toast 实现

使用 `Toast.makeText(context, "功能开发中...", Toast.LENGTH_SHORT).show()`，在 HomeScreen 中通过 `LocalContext.current` 获取 context。

## 八、错误处理与边界情况

### 8.1 需处理的边界情况

1. **空状态**：`hideCompletedItems=true` 且所有待办都已完成时，显示空状态提示
2. **批量模式 + 菜单**：批量模式下点击三点菜单，先退出批量模式再打开菜单
3. **排序 + 隐藏**：`hideCompletedItems=true` 时排序仍作用于未完成项
4. **状态恢复**：从后台恢复时，`hideDetails` / `hideCompletedItems` 从 DataStore 正确加载
5. **子任务展开 + 简化**：`isSimpleMode=true` 时强制收起子任务（忽略 expandState）

### 8.2 手势冲突处理

遵循 project_memory 中的 pressFeedback 规则：
- 简化模式下子任务勾选、卡片点击不冲突
- `PressFeedback` Modifier 检查 consumed 事件并消费 down 事件
- 子任务元素消费自己的手势，防止父组件干扰

### 8.3 不做的事（YAGNI）

- 不实现管理分组/分享/创建副本的实际功能（仅 Toast）
- 不添加菜单项的长按操作
- 不实现菜单项的拖拽排序
- 不添加 `hideDetails` 的动画过渡（直接切换）
- 不为灵感/日期页的三点菜单实现实际功能

## 十一、设计决策记录

1. **菜单样式选择 A（经典下拉）**：贴近原生 Android 体验，Material 3 DropdownMenu 自带锚点定位和动画，代码简洁。
2. **独立 TodoMenuDropdown 组件**：符合项目现有的 `topBarActionButtons` 注入模式，组件解耦清晰，便于扩展。
3. **hideCompletedItems 与 showCompleted 分离**：两者语义不同（隐藏所有 vs 展开折叠区），分离避免状态混淆，支持灵活组合。
4. **置顶图标使用 PushPin**：语义明确，Material Icons 中明确存在，比 emoji 更专业。
5. **占位项可点击但 Toast 提示**：让用户知道功能存在，而非完全禁用造成困惑。
6. **灵感/日期页三点图标提示开发中**：统一交互模式，为后续功能扩展预留入口。
