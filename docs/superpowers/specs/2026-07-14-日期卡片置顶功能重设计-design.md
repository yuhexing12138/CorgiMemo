## 一、项目背景与目标

### 1.1 背景

当前 `SpecialDateScreen` 存在以下与"置顶"相关的不足：

1. **置顶卡位置不明显**：当前 `SQL ORDER BY isPinned DESC, targetDate ASC` 仅将置顶卡排在该卡片**所属分组**内的最前（"倒计时"或"正计时"），并未整体置顶突出。
2. **置顶卡视觉与普通卡无差异**：当前置顶卡仅在标题左侧多了一个小图钉图标（14dp），其它视觉（圆形头像 48dp / 标题 16sp / 时间 14sp / 右侧大数字 22sp）完全等同普通卡，置顶感不强。
3. **唯一性限制已在数据层实现**：`SpecialDateDao.togglePin` 改为 `clearPinExcept + setPinned` 流程；`SpecialDateViewModel._pinnedDateId` 跟踪当前置顶 id；新代码 `pinDate/unpinDate` 行为正确，但 UI 端 `pinnedDateId` 收集后**未被使用**。

### 1.2 目标

- 置顶卡显示在 LazyColumn **最顶部**（所有 DateSectionHeader 之上），不与倒计时/正计时/已归档分组混合。
- 置顶卡采用特殊布局：左"标题 + 大数字（天/年单位在右下）"、中"emoji 大背景 + 单行详细倒数信息"、右"圆形 emoji 头像"。
- 置顶卡与原"倒计时"分组内的同名卡**不重复显示**。
- 已有置顶互斥、归档联动等行为保持不变。

### 1.3 不在本设计内

- "已归档"入口的 UI 位置（用户已说明后续再确定）。
- `SpecialDateEditScreen` 编辑表单的字段调整。
- 标签 / 图片 / 关联等编辑能力。
- 现有 `SwipeableTodoBox` 的左滑三按钮（置顶/归档/删除）的内部行为改造。

## 二、关键决策摘要

| # | 决策 | 选择 |
|---|------|------|
| 1 | 中间装饰区 | 类别 emoji 背景（`date.category.emoji` 80sp 半透） |
| 2 | 高度增加 | 轻微（+25%，minHeight 100dp） |

| 4 | 数字下角单位 | 跟实际数单位：≤1年显示"X 天"；>1年显示"X 年 Y 天"；昨天/今天/明天直接显示文字 |

| 6 | 切换动画 | 不需额外动画，LazyColumn 原生 `animateItem()` 足够 |

| 8 | 右侧控件 | 圆形 emoji 头像（复用现状 48dp surfaceVariant + emoji 居中） |
| 9 | 置顶位置 | 全局顶部（所有 DateSectionHeader 之上） |
| 10 | 分组重复 | 从分组中隐藏（与顶部置顶卡去重） |
| 11 | 方案选型 | 方案 A：数据层分离（`pinnedDate` 独立 StateFlow） |

## 四、数据层 / ViewModel 设计

### 4.1 新增 `pinnedDate` StateFlow

**采用最小侵入方案**：复用 `groupByDisplayDates` 的转换逻辑，把"取已转换的 DisplayDate 列表"与"找第一个置顶的"两个步骤组合即可，避免重构 `groupByDisplayDates` 内部实现。

```kotlin
// SpecialDateViewModel.kt
val pinnedDate: StateFlow<DisplayDate?> = specialDates
    .map { dates ->
        // 1. 复用 groupedDates 内部的 groupByDisplayDates 做转换
        //    （过滤 isPinned=true && !isArchived）
        val candidates = groupByDisplayDates(dates.filter { it.isPinned && !it.isArchived })
        // 2. 取唯一置顶卡
        candidates.values.flatten().firstOrNull()
    }
    .stateIn(viewModelScope, SharingStarted.WhileSubscribed(5000), null)
```

**取值规则**：

| 条件 | `pinnedDate` |
|------|--------------|
| 1 卡 `isPinned=true && !isArchived` | 返回该卡的 `DisplayDate` |
| 0 卡 `isPinned=true` | null |
| 1 卡 `isPinned=true && isArchived=true` | null（已归档不展示置顶卡） |
| 2+ 卡 `isPinned=true` | 取第一个（理论上不会发生，因 `pinDate` 已 `clearPinExcept`） |

### 4.2 修改 `groupedDates`：过滤置顶卡

```kotlin
val groupedDates: StateFlow<Map<DateGroup, List<DisplayDate>>> =
    combine(specialDates, _searchQuery) { dates, query ->
        val filtered = if (query.isBlank()) dates
                       else dates.filter { it.title.contains(query, ignoreCase = true) }
        // ★ 新增：过滤掉已置顶卡，避免与 PinnedDateCard 重复显示
        val withoutPinned = filtered.filter { !it.isPinned }
        groupByDisplayDates(withoutPinned)
    }.stateIn(viewModelScope, SharingStarted.WhileSubscribed(5000), emptyMap())
```

### 4.3 其它不变

| 项 | 状态 |
|----|------|
| `pinDate(id)` / `unpinDate(id)` | 保持现有逻辑（已 `clearPinExcept` 互斥） |
| `_pinnedDateId` StateFlow | 保留（被 `archiveDate` 等内部逻辑使用） |
| `SpecialDateScreen` 中 `pinnedDateId` 的 collect | 保留（当前实际未使用，本次修改不清理，避免不必要变更） |
| `expandedDateId` / `pendingArchive` | 不变 |
| `archiveDate` 归档时清空 `_pinnedDateId` | 保持现有行为（VM 191-204 行） |
| `unarchiveDate` 恢复 `_pinnedDateId` | 保持现有行为（VM 245-249 行） |
| DB / Entity / Migration | **不动** |

### 5.3 单位显示规则（"天"/"年"）

```kotlin
// 数字部分 + 单位部分
val (labelText, unitText) = when (daysDiff) {
    0L   -> "今天" to null       // 不显示单位
    1L   -> "明天" to null
    -1L  -> "昨天" to null
    else -> {
        val abs = daysAbs
        when {
            // 超过1年：年+天组合
            abs >= 365L -> {
                val years = abs / 365L
                val days = abs % 365L
                "$years" to if (days > 0L) "年 $days 天" else "年"
            }
            // 1年以内（365天以下）：仅天
            else -> abs.toString() to "天"
        }
    }
}
```

**注**：与现有 `SpecialDateCard` 的 `daysLabel` / `daysUnit` 逻辑保持完全一致的行为；只是数字字号从 22sp 放大到 40sp。

### 5.4 关键代码骨架

```kotlin
/**
 * 置顶特殊日期卡片（特殊视觉版）
 *
 * 视觉结构（左→中→右）：
 * - 左区 (widthIn min=70dp)：标题 + 大数字(40sp) + 单位(16sp, 紧跟数字右下)
 * - 中间 (weight=1f)：emoji 大背景(80sp, alpha 0.4) + 单行详细倒数(14sp)
 * - 右侧：圆形头像 (48dp, surfaceVariant + emoji 居中)
 *
 * @param date 展示数据（已确保 isPinned=true）
 * @param nowMs 当前时间戳（毫秒），由 SpecialDateScreen 顶层 ticker 驱动
 * @param isClickBlocked 左滑操作面板是否展开
 * @param onClick 卡片整体点击回调（进入编辑）
 * @param modifier 外部 Modifier
 */
@Composable
fun PinnedDateCard(
    date: DisplayDate,
    nowMs: Long,
    isClickBlocked: Boolean = false,
    onClick: () -> Unit = {},
    modifier: Modifier = Modifier
) {
    // 颜色与时间差计算（与 SpecialDateCard 同样的规则）
    val isArchived = date.isArchived
    val diff = date.targetDate - nowMs
    val isFuture = diff >= 0
    val daysDiff = diff / 86_400_000L
    val daysAbs = abs(daysDiff)
    val timeColor = when {
        isArchived -> Color(0xFF999999)
        isFuture -> MaterialTheme.colorScheme.primary
        else -> Color(0xFF7EC8A0)
    }
    val (labelText, unitText) = /* 见 5.3 */

    Card(
        shape = RoundedCornerShape(20.dp),
        elevation = CardDefaults.cardElevation(defaultElevation = 2.dp),
        colors = CardDefaults.cardColors(containerColor = MaterialTheme.colorScheme.surface),
        modifier = modifier
            .fillMaxWidth()
            .then(
                if (isClickBlocked) Modifier
                else Modifier.combinedClickable(onClick = onClick)
            )
    ) {
        Row(
            modifier = Modifier
                .fillMaxWidth()
                .padding(16.dp)
                .then(if (isArchived) Modifier.alpha(0.6f) else Modifier),
            verticalAlignment = Alignment.CenterVertically
        ) {
            // 左区
            Column(modifier = Modifier.widthIn(min = 70.dp)) {
                Text(
                    text = date.title,
                    fontSize = 16.sp,
                    fontWeight = FontWeight.Medium,
                    color = MaterialTheme.colorScheme.onSurface,
                    maxLines = 1,
                    overflow = TextOverflow.Ellipsis
                )
                Row(verticalAlignment = Alignment.Bottom) {
                    Text(
                        text = labelText,
                        fontSize = 40.sp,
                        fontWeight = FontWeight.Bold,
                        color = timeColor
                    )
                    if (unitText != null) {
                        Text(
                            text = unitText,
                            fontSize = 16.sp,
                            fontWeight = FontWeight.Medium,
                            color = timeColor,
                            modifier = Modifier.padding(start = 2.dp, bottom = 4.dp)
                        )
                    }
                }
            }

            // 中间
            Box(
                modifier = Modifier
                    .weight(1f)
                    .fillMaxHeight(),
                contentAlignment = Alignment.Center
            ) {
                // 装饰：emoji 大背景（半透）
                Text(
                    text = date.category.emoji,
                    fontSize = 80.sp,
                    color = MaterialTheme.colorScheme.surfaceVariant,
                    modifier = Modifier.alpha(0.4f)
                )
                // 倒数文本（后声明，绘制顺序自然在上层）
                Text(
                    text = formatDuration(abs(diff)),
                    fontSize = 14.sp,
                    fontWeight = FontWeight.Medium,
                    color = timeColor,
                    maxLines = 1,
                    overflow = TextOverflow.Ellipsis
                )
            }

            // 右区：圆形头像
            Box(
                modifier = Modifier
                    .size(48.dp)
                    .background(
                        color = MaterialTheme.colorScheme.surfaceVariant,
                        shape = CircleShape
                    ),
                contentAlignment = Alignment.Center
            ) {
                Text(
                    text = date.category.emoji,
                    fontSize = 24.sp
                )
            }
        }
    }
}
```

### 6.4 左滑操作说明

- 置顶卡的左滑三按钮：固定为"取消置顶 / 归档 / 删除"（因为此卡已是置顶态）。
- `archiveDate` 归档时清空 `_pinnedDateId`（VM 现有逻辑），UI `pinnedDate` 重组为 null → 自动从顶部消失。
- 撤回归档时 `unarchiveDate` 恢复 `_pinnedDateId`（VM 现有逻辑 245-249 行），UI 重新显示置顶卡。

## 七、边界条件 & 错误处理

| 场景 | 处理 |
|------|------|
| 无置顶卡 | `pinnedDate = null` → LazyColumn 不插入置顶卡 item |
| 置顶卡已归档 | `pinnedDate` 过滤 `!isArchived` 返回 null → 顶部不显示（与已归档不进入分组一致） |
| 切换置顶（A → B） | `pinDate(B)` 时 SQL `clearPinExcept` 清掉 A，`pinnedDate` 自然重组为 B；LazyColumn 原生 `animateItem` 提供过渡 |
| 置顶卡左滑归档 | `archiveDate(id)` 清空 `_pinnedDateId`；`pinnedDate` 重组为 null → 顶部置顶卡消失 |
| 撤回归档（Snackbar 撤回） | `unarchive(id)` 恢复；`unarchiveDate` 内部若 `current.isPinned==true && pinnedDateId==null` 恢复 → UI 重新显示置顶卡 |
| 数字溢出（>999 天） | `Text(40sp)` 自动按宽度缩放；中栏 `weight(1f)` 自动让出空间 |
| 中间 emoji 显示不下 | emoji 半透背景（alpha 0.4），文本自然覆盖在上层 |
| `formatDuration` 异常 | 入参 `coerceAtLeast(0L)` 防负数；零值兜底为 "0秒" |
| 字体可访问性（系统放大） | 全部用 `sp` 单位，尊重用户字体设置 |
| 搜索过滤 | `groupedDates` 已过滤搜索；`pinnedDate` 也跟随（置顶卡命中搜索词时正常显示） |
