# 演示种子数据注入设计规范

> **创建日期**：2026-07-14
> **主题**：为 CorgiMemo 项目创建初始演示数据，覆盖待办事项、灵感记录、日期管理三大模块
> **数据来源**：依据 `初赛demo贴初稿.md` 的项目背景、功能需求和业务逻辑

---

## 一、设计目标

为 CorgiMemo 系统创建一套完整的初始演示数据，满足以下要求：

1. **待办事项**：不少于 5 条，覆盖高/中/低优先级、3+ 分组、待办/已完成/置顶状态、过去/当前/未来提醒、0-5 子任务、1-3 图片、不同时长语音
2. **灵感记录**：不少于 5 条，覆盖 0-5 不重复标签、过去/当前/未来创建时间、1-5 图片
3. **日期管理**：不少于 5 条，覆盖倒计时/正计时/归档三种状态、过去/当前/未来时间、至少 1 项置顶
4. **内容关联**：三大模块之间通过 CardRelation 建立明确关联，形成有机整体
5. **资源引用**：图片从 `drawable` 目录随机选用，语音生成静音音频文件

---

## 二、架构与文件结构

### 2.1 分层 Seeder 架构

采用分层 Seeder 架构，每个 Seeder 负责一个数据模块的注入，由统一的 `DemoDataSeeder` 编排：

| 类名 | 职责 |
|------|------|
| `DemoDataSeeder` | 总编排器，检查标志位、按顺序调用各 Seeder、管理事务 |
| `DemoResourceManager` | 资源处理：drawable 复制到内部存储、静音音频生成 |
| `TodoSeedData` | 待办事项种子数据定义与注入 |
| `InspirationSeedData` | 灵感记录种子数据定义与注入 |
| `DateSeedData` | 日期管理种子数据定义与注入 |
| `RelationSeedData` | CardRelation 跨模块关联注入 |
| `SeedModule` | Hilt 依赖注入配置 |

### 2.2 文件路径

所有种子数据相关文件统一放置在 `data/seed` 包下：

```
app/src/main/java/com/corgimemo/app/
├── data/
│   └── seed/
│       ├── DemoDataSeeder.kt          # 总编排器
│       ├── DemoResourceManager.kt     # 资源处理
│       ├── TodoSeedData.kt            # 待办种子数据
│       ├── InspirationSeedData.kt     # 灵感种子数据
│       ├── DateSeedData.kt            # 日期种子数据
│       └── RelationSeedData.kt        # 关联种子数据
├── di/
│   └── SeedModule.kt                  # Hilt 模块
└── CorgiMemoApplication.kt            # 修改：添加注入触发
```

## 三、数据主题与跨模块关联

### 3.1 数据主题

以 **"项目开发旅程"** 为主题，种子数据内容取材于参赛贴 `初赛demo贴初稿.md` 中描述的实际开发过程，使演示数据具有真实性和连贯性。

### 3.2 跨模块关联（5 条 CardRelation）

| 关联编号 | sourceType | sourceId | targetType | targetId | 关联含义 |
|---------|-----------|----------|-----------|----------|---------|
| R1 | todo | T1 | date | D1 | 待办"参赛贴撰写"关联日期"初赛报名截止" |
| R2 | inspiration | I2 | todo | T2 | 灵感"状态机架构"关联待办"数据库架构设计" |
| R3 | inspiration | I3 | inspiration | I1 | 灵感"卡片关联设计"关联灵感"陪伴系统设计" |
| R4 | todo | T3 | inspiration | I4 | 待办"悬浮按钮优化"关联灵感"Snackbar 规范" |
| R5 | date | D2 | todo | T6 | 日期"项目启动日"关联待办"种子数据注入" |

关联覆盖三种类型：
- **待办 → 日期**（R1）：待办的截止时间与日期记录相关联
- **灵感 → 待办**（R2、R4）：灵感为待办提供创意支持或解决方案
- **灵感 → 灵感**（R3）：灵感之间的创意关联
- **日期 → 待办**（R5）：日期记录与相关待办的事项关联

## 七、资源处理设计

### 7.1 DemoResourceManager 职责

`DemoResourceManager` 负责将 drawable 资源复制到内部存储，并生成静音音频文件，供种子数据引用。

### 7.2 图片复制实现

**存储位置**：`filesDir/demo/images/`

**核心方法签名**：

```kotlin
/**
 * 将 drawable 资源复制到内部存储，返回文件绝对路径
 * @param drawableId R.drawable.xxx
 * @param fileName 目标文件名（如 "demo_todo_t1_1.png"）
 * @return 文件绝对路径
 */
fun copyDrawableToInternal(drawableId: Int, fileName: String): String
```

**实现逻辑**：
1. 创建 `filesDir/demo/images/` 目录（若不存在）
2. 打开 drawable 的 InputStream（`Resources.openRawResource`）
3. 写入目标文件
4. 返回 `file.absolutePath`

### 7.3 语音文件生成

**存储位置**：`filesDir/demo/voice/`

**核心方法签名**：

```kotlin
/**
 * 生成指定时长的静音 3gp 音频文件
 * @param durationSec 时长（秒）
 * @param fileName 目标文件名（如 "demo_todo_t1_voice.3gp"）
 * @return 文件绝对路径
 */
fun generateSilentAudio(durationSec: Int, fileName: String): String
```

**实现方案（方案 C：纯代码生成哑 3gp 文件）**：
- 生成一个固定大小的"哑"3gp 文件（约 1KB），duration 元数据通过 MediaMetadataRetriever 可读
- 优点：纯代码生成，无需 assets 模板，无需权限，即时完成
- 缺点：播放时无实际音频内容（种子数据仅用于演示 UI，不播放）

## 八、注入编排与触发机制

### 8.1 触发机制：SharedPreferences 标志位

在 `Application.onCreate` 中检查 SharedPreferences 标志位 `demo_data_seeded`：

| 标志值 | 行为 |
|--------|------|
| `false` 或不存在 | 执行种子数据注入，注入成功后置为 `true` |
| `true` | 跳过注入（已注入过） |

**SharedPreferences 文件**：`corgimemo_demo_prefs`

**优点**：
- 不依赖数据库 onCreate 回调（数据库已存在时 onCreate 不触发）
- 标志位持久化，卸载重装后自动重置
- 可通过清除应用数据重置种子

### 8.2 Application.onCreate 修改

在 `CorgiMemoApplication.onCreate()` 末尾添加种子数据注入逻辑：

```kotlin
/** 演示种子数据注入器 */
@Inject
lateinit var demoDataSeeder: DemoDataSeeder

override fun onCreate() {
    super.onCreate()
    // ... 现有初始化代码 ...
    
    // 演示种子数据注入（首次启动自动执行）
    CoroutineScope(Dispatchers.IO).launch {
        try {
            demoDataSeeder.seedIfNeeded()
        } catch (e: Exception) {
            android.util.Log.w("CorgiMemoApplication", "⚠️ 演示数据注入失败: ${e.message}")
        }
    }
}
```

### 8.3 注入顺序（严格依赖关系）

| 步骤 | 操作 | 依赖 | 产出 |
|------|------|------|------|
| 1 | 资源准备 | 无 | 图片路径、语音路径 |
| 2 | Category 注入 | 无 | categoryId（学习/工作/生活/运动） |
| 3 | Todo 注入 | categoryId、图片路径、语音路径 | todoId（T1-T6） |
| 4 | SubTask 注入 | todoId | subTaskId |
| 5 | Inspiration 注入 | categoryId、图片路径 | inspirationId（I1-I6） |
| 6 | SpecialDate 注入 | 图片路径 | specialDateId（D1-D6） |
| 7 | CardRelation 注入 | todoId、inspirationId、specialDateId | 关联记录（5 条） |

### 8.4 事务安全

整个注入过程包裹在 Room 事务中，确保原子性：

```kotlin
class DemoDataSeeder @Inject constructor(
    private val context: Context,
    private val database: CorgiMemoDatabase,
    private val resourceManager: DemoResourceManager
) {
    suspend fun seedIfNeeded() {
        val prefs = context.getSharedPreferences("corgimemo_demo_prefs", MODE_PRIVATE)
        if (prefs.getBoolean("demo_data_seeded", false)) {
            return
        }
        
        database.withTransaction {
            val imagePaths = resourceManager.prepareAllImages()
            val voicePaths = resourceManager.prepareAllVoice()
            val categoryIds = seedCategories()
            val todoIds = seedTodos(categoryIds, imagePaths, voicePaths)
            seedSubTasks(todoIds)
            val inspirationIds = seedInspirations(categoryIds, imagePaths)
            val dateIds = seedSpecialDates(imagePaths)
            seedRelations(todoIds, inspirationIds, dateIds)
        }
        
        prefs.edit().putBoolean("demo_data_seeded", true).apply()
    }
}
```

**事务边界说明**：
- `withTransaction` 包裹所有 DB 写入，任一步骤失败则全部回滚
- 文件 IO（图片复制、语音生成）在事务内执行，若事务回滚，已复制的文件成为孤儿文件（可接受）
- 标志位在事务成功后才置为 `true`，确保失败可重试

### 8.5 错误处理与日志

| 异常场景 | 处理策略 |
|---------|---------|
| SharedPreferences 读取失败 | 视为未注入，执行注入 |
| 资源复制失败（drawable 不存在） | 抛出异常，事务回滚 |
| DB 插入失败（字段不匹配） | 抛出异常，事务回滚，标志位不更新 |
| 整体注入失败 | Application 捕获异常并记录日志，不影响 APP 启动 |

日志标签：`DemoSeeder`，分步骤输出进度。

### 8.6 幂等性保证

- **首次注入**：标志位 false → 执行注入 → 置 true
- **重复启动**：标志位 true → 跳过注入
- **清除应用数据**：SharedPreferences 清除 → 标志位重置 → 重新注入
- **数据库版本升级**：不影响种子注入（标志位仍为 true，跳过）

### 8.7 Hilt 依赖注入配置

```kotlin

    @Provides
    @Singleton
    fun provideDemoResourceManager(context: Context): DemoResourceManager {

    }
    
    @Provides

        context: Context,
        database: CorgiMemoDatabase,

    }
}
```
