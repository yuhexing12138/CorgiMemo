# 灵感字数统计页面 设计文档

> 日期：2026-07-11
> 适用范围：灵感页字数统计入口与统计页面

## 1. 目标与背景

灵感页右上角三点图标左侧当前显示「标签管理」占位图标（`Icons.AutoMirrored.Filled.Label`，onClick 为 TODO）。现需将其替换为与待办页 topBarActionButtons 完全一致的统计图标（`Icons.Default.BarChart`），并跳转到新建的字数统计页面，展示最近 7/30 天的累计总字数与每日输入字数。

参考图布局：顶部 TopAppBar「字数统计」+ 返回箭头；下方两张卡片：累计总字数（折线图）+ 输入字数（柱状图），每张卡片右上角有展开按钮，可在 7 天与 30 天视图间切换。

## 2. 数据口径与边界

| 项 | 决策 |
|----|------|
| 字数定义 | 沿用 `InspirationTextUtils.countInspirationChars`：title + content + tags 拼接后去除所有空白字符，中英文均按 1 字符 |
| 是否包含已归档 | **包含**（`isArchived = true` 的灵感也参与统计） |
| 按日分组字段 | `createdAt` 的日期部分（年-月-日） |
| 累计 = 截至当日的所有日字数之和 | 是 |
| 0 字数处理 | 折线/柱状图自然显示 0，不弹空提示 |
| 完全空数据库 | 全屏空状态（🐶 + CTA） |
| 7 天 vs 30 天 | 每个图表独立切换，状态互不干扰 |

### 3.3 计算逻辑

```kotlin
private fun computeDailyCounts(
    range: ChartRange,
    allInspirations: List<Inspiration>
): List<DailyWordCount> {
    val today = LocalDate.now()
    val days = (0 until range.days).map { offset ->
        today.minusDays((range.days - 1 - offset).toLong())
    }
    val groupedByDay = allInspirations.groupBy { ins ->
        Instant.ofEpochMilli(ins.createdAt)
            .atZone(ZoneId.systemDefault())
            .toLocalDate()
    }
    var cumulative = 0
    return days.map { date ->
        val daily = groupedByDay[date]?.sumOf { ins ->
            InspirationTextUtils.countInspirationChars(ins)
        } ?: 0
        cumulative += daily
        DailyWordCount(date, daily, cumulative)
    }
}
```

## 7. 错误处理与边界

| 场景 | 处理 |
|------|------|
| 数据库无灵感 | 全屏空状态（🐶 + CTA "去记录灵感"） |
| 当日无输入 | 折线/柱状图 0 正常显示 |
| 数据库读取异常 | catch 后展示"加载失败"占位 + 重试按钮 |
| 30 天内无任何输入 | 折线图为水平线（值为 0），柱状图仅显示坐标轴与 0 刻度，不显示柱形 |
| 编辑/删除灵感返回页面 | `WhileSubscribed(5_000)` + DB Flow 自动重算（无需手动 reload） |
| 配置变更（旋转/暗黑） | ViewModel 持有状态，不丢失 |
| 深色模式 | 使用 `MaterialTheme.colorScheme.*`，不写死颜色 |
