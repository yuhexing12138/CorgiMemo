# 待办卡片滑动接触缩放动画 - 设计文档

> 状态：待实施  
> 创建日期：2026-06-29  
> 项目：CorgiMemo（Android 柯基待办 App）  
> 技术栈：Jetpack Compose（BOM 2026.04.01 + Compose 1.9.2）

## 1. 背景与目标

### 1.1 问题陈述
在待办页面上下滑动过程中，手指接触待办卡片时会触发 Material Design 水波纹效果，干扰用户的视觉专注，降低页面滑动体验。

### 1.2 目标
- **禁用**滑动接触时的水波纹效果
- **新增**滑动接触时的卡片缩放反馈：缩小 8%（scale = 0.92f）后立即恢复（恢复过渡 ≤ 100ms）
- **保持**正常点击/长按操作的原有水波纹反馈不变
- 确保动画流畅自然，不影响整体页面滑动性能

### 1.3 非目标
- 不修改水波纹的实现机制
- 不修改左滑（SwipeableTodoBox）相关的交互
- 不修改拖拽排序的视觉反馈
- 不添加触觉反馈（已与用户确认）

## 2. 关键参数

| 参数 | 值 | 备注 |
|------|-----|------|
| 缩小比例 | 8%（scale = 0.92f） | 用户确认 |
| 缩小动画时长 | 60ms tween | 快速缩小 |
| 恢复动画时长 | 80ms tween | 平滑恢复 |
| Press 延迟 | 16ms | 与现有水波纹延迟一致 |
| 触摸阈值 | `viewConfiguration.touchSlop` | 与现有逻辑一致 |
| 触觉反馈 | 无 | 用户确认 |
| 动画缓动 | `FastOutSlowInEasing` | Material 标准 |

## 3. 架构设计

### 3.1 核心思路
抽取一个独立的 `Modifier.pressFeedback()` 扩展函数，将「接触缩放」+「按压水波纹」+「长按」三套反馈逻辑封装在一起，对调用方完全黑盒。

### 3.2 架构图

```
┌──────────────────────────────────────────────────────────────┐
│  TodoListItem.kt (重构后)                                    │
│  ┌────────────────────────────────────────────────────┐     │
│  │  Card(modifier = Modifier.pressFeedback(...))       │     │
│  │    └─ 内部: 单一手势检测 + 三状态状态机              │     │
│  │       ├─ State A: 静止(无任何反馈)                   │     │
│  │       ├─ State B: 滑动接触(scale=0.92f, 无水波纹)   │     │
│  │       └─ State C: 点击/长按(scale=1f, 显示水波纹)    │     │
│  └────────────────────────────────────────────────────┘     │
└──────────────────────────────────────────────────────────────┘
                              ↓
┌──────────────────────────────────────────────────────────────┐
│  PressFeedback.kt (新建)                                     │
│  - fun Modifier.pressFeedback(                              │
│      onTap, onLongClick, ...                                │
│    )                                                         │
│  - 内部: awaitEachGesture + Animatable<Float>               │
│  - 内部算法: object PressFeedbackLogic (可测试)              │
└──────────────────────────────────────────────────────────────┘
```

### 3.3 模块化收益
- **TodoListItem.kt 减少约 150 行**：手势状态机代码移出
- **子任务卡片可复用同一 Modifier**
- **未来其他卡片可复用**：如日期分组、分类标签等
- **算法可独立测试**：`PressFeedbackLogic` 是纯函数 object

```kotlin
package com.corgimemo.app.ui.components

/**
 * 为 Composable 提供统一的按压反馈修饰符
 *
 * 整合三套反馈逻辑：
 * 1. 滑动接触反馈：滑动中手指接触时，卡片缩放至 0.92f 后恢复
 * 2. 点击/长按水波纹：静止点击时显示水波纹
 * 3. 长按检测：500ms 长按触发 onLongClick

fun Modifier.pressFeedback(
    interactionSource: MutableInteractionSource,
    enabled: Boolean = true,
    isBatchMode: Boolean = false,
    onTap: () -> Unit = {},
    onLongClick: () -> Unit = {},
    scaleDown: Float = 0.92f,
    scaleDownDurationMs: Int = 60,
    scaleUpDurationMs: Int = 80,
    pressDelayMs: Long = 16
): Modifier
```

## 5. 状态机与数据流

### 5.1 三状态状态机

```
        ┌─────────────────────────────────────────┐
        │  down 事件                              │
        │  → 立即进入 State B                      │
        │  → 启动 60ms tween: scale 1f → 0.92f    │
        │  → 启动 16ms Press 延迟定时器            │
        └──────────┬──────────────────────────────┘
                   │
        ┌──────────▼──────────────────────────────┐
        │  等待 16ms                                │
        └──────────┬──────────────────────────────┘
                   │
       ┌───────────┴─────────────┐
       │                         │
   16ms 内静止               16ms 内移动 > touchSlop
   或 移动 ≤ touchSlop         (判定为滑动接触)
       │                         │
       ▼                         ▼
┌──────────────┐         ┌──────────────┐
│  State C     │         │  State B     │
│  (点击/长按)  │         │  (滑动接触)   │
│              │         │              │
│ scale: 1f    │         │ scale: 0.92f │
│ 发射 Press   │         │ 不发射 Press │
│ 显示水波纹    │         │ 无水波纹     │
└──────┬───────┘         └──────┬───────┘
       │                         │
       └────────┬────────────────┘
                │
        ┌───────▼─────────────────┐
        │  手指抬起                │
        │  → 重置状态              │
        │  → 80ms tween: scale    │
        │    回到 1f              │
        │  → 发射 Release/Cancel   │
        └─────────────────────────┘
```

### 5.2 数据流

1. **scale 值**：
   - 由 `Animatable<Float>` 持有
   - 通过 `graphicsLayer { scaleX = scale; scaleY = scale }` 应用
   - 使用 `Modifier.graphicsLayer(...)` 包裹整个 Card

2. **水波纹事件**（Press/Release/Cancel）：
   - 由 Modifier 内部 `Channel<PressInteraction>` 持有
   - 内部 `LaunchedEffect` 异步消费
   - 通过参数传入的 `MutableInteractionSource` 发射

3. **回调触发**：
   - 短按（< 500ms 抬起）→ `onTap()`
   - 长按（≥ 500ms 后抬起）→ `onLongClick()`
   - 滑动接触（移动 > touchSlop 后抬起）→ 不触发任何回调

### 5.3 性能保障

| 性能维度 | 措施 |
|---------|------|
| 动画帧耗时 | 缩放动画 60-80ms，不会拖慢列表滚动 |
| 状态重组 | `Animatable` + `graphicsLayer` 避免重组 |
| 协程开销 | 单一 `pointerInput` 协程 + `Channel` 异步消费 |
| 触摸响应 | 16ms 延迟与现有水波纹机制保持一致 |

## 6. 错误处理与边界情况

### 6.1 边界情况清单

| 场景 | 预期行为 | 处理策略 |
|------|---------|---------|
| 快速点击（< 16ms） | 视为滑动，缩放 0.92f 后恢复 | 16ms Press 延迟定时器未发射，抬起时检测到无 Press 事件 |
| 长按（≥ 500ms） | 触发 `onLongClick()` + 水波纹 | 现有逻辑保留：500ms 后 `longPressTriggered = true` |
| 滑动到顶/底反弹 | scale 保持 0.92f 直到抬起 | Modifier 内部以 `dragDistance > touchSlop` 为准 |
| 多指触摸 | 仅第一根手指触发 | `awaitFirstDown()` + `event.changes.first()` |
| 系统返回键中断 | 缩放状态恢复 | `awaitEachGesture` 的 `finally` 块保证 Cancel 发射 |
| 多选模式点击 | 缩放 + 走 `onSelectClick()` | 现有 `isBatchMode` 分支保留 |
| 拖拽排序中断 | scale 恢复 | `onLongClick` 触发时由外层 `isDragActive` 协调 |
| 左滑中接触 | scale 保持 0.92f，水波纹禁用 | 复用现有 `LocalContentIndication` |
| 屏幕旋转/Activity 重建 | 状态自动重置 | `remember` + 重建时重新初始化 |

### 6.2 错误防御

1. **空指针防护**：
   - `interactionSource` 必须在 Modifier 内部创建
   - 回调函数（`onTap` / `onLongClick`）使用 `rememberUpdatedState` 防止闭包过期

2. **协程泄漏防护**：
   - `scope.launch { delay(16) }` 在 `awaitEachGesture` 的 `try/finally` 中 `cancel()`
   - 每次 gesture 重置时所有协程一起清理

3. **状态一致性**：
   - `scale` Animatable 的 `targetValue` 总是显式设置（1f 或 0.92f），不依赖中间态
   - `interactionSource.emit` 通过 Channel 串行化，避免并发问题
