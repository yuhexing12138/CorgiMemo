## 一、背景与问题

灵感页长按灵感卡片后弹出操作面板（置顶/标签/改日期/删除），其中「标签」和「改日期」两个按钮当前为空实现或未接线状态：

| 编号 | 当前问题 | 期望行为 |
|---|---|---|
| 1 | 点击「标签」无反应 | 进入标签管理弹窗，可修改对应灵感的标签 |
| 2 | 点击「改日期」无反应 | 进入日期时间滚轮选择器，修改后在时间轴上同步重排位置 |

### 决策3：隐藏的高级选项

## 四、详细设计

### 4.1 数据层改造（InspirationViewModel）

#### 新增 `savedTags` 流

```kotlin
/** 历史标签列表（从所有灵感聚合去重，用于 TagPickerSheet 快速选择） */
val savedTags: StateFlow<List<String>> =
    _inspirations.map { list ->
        list.flatMap { decodeTags(it.tags) }
            .distinct()
            .sorted()
    }.stateIn(
        scope = viewModelScope,
        started = SharingStarted.WhileSubscribed(5000),
        initialValue = emptyList()
    )
```

#### 新增 `updateTags(id, newTags)` 方法

```kotlin
/**
 * 更新指定灵感的标签
 * @param id 灵感ID
 * @param newTags 新标签列表
 */
fun updateTags(id: Long, newTags: List<String>) {
    viewModelScope.launch {
        val currentList = _inspirations.value
        val inspiration = currentList.find { it.id == id } ?: return@launch
        inspirationRepository.update(
            inspiration.copy(
                tags = encodeTags(newTags),
                updatedAt = System.currentTimeMillis()
            )
        )
    }
}
```

#### 渲染 TagPickerSheet

```kotlin
if (showTagPicker && longPressedInspiration != null) {
    val tagSheetState = rememberModalBottomSheetState(skipPartiallyExpanded = true)
    val savedTags by viewModel.savedTags.collectAsState()
    TagPickerSheet(
        sheetState = tagSheetState,
        tags = viewModel.decodeTags(longPressedInspiration!!.tags),
        savedTags = savedTags,
        onTagsChange = { newTags ->
            viewModel.updateTags(longPressedInspiration!!.id, newTags)
        },
        onDismiss = {
            showTagPicker = false
            longPressedInspiration = null
        }
    )
}
```

#### 渲染 ReminderPickerBottomSheet

```kotlin
if (showDateTimePicker && longPressedInspiration != null) {
    val inspiration = longPressedInspiration!!
    val calendar = Calendar.getInstance().apply { timeInMillis = inspiration.createdAt }
    ReminderPickerBottomSheet(
        initialDateMillis = inspiration.createdAt,
        initialHour = calendar.get(Calendar.HOUR_OF_DAY),
        initialMinute = calendar.get(Calendar.MINUTE),
        showAdvancedOptions = false,        // 隐藏重复提醒和农历
        title = "修改日期时间",
        rowLabel = "日期时间",
        onDismiss = {
            showDateTimePicker = false
            longPressedInspiration = null
        },
        onConfirm = { dateMillis, _, _, _, _ ->
            if (dateMillis != null) {
                viewModel.updateInspirationDateTime(inspiration.id, dateMillis)
            }
            showDateTimePicker = false
            longPressedInspiration = null
        }
    )
}
```

## 五、数据流验证

### 5.1 标签修改数据流

```
用户点击"确认"
  → TagPickerSheet 回调 onTagsChange(newTags)
  → viewModel.updateTags(id, newTags)
  → inspirationRepository.update(inspiration.copy(tags=...))
  → Room 数据库更新
  → InspirationDao.getAllInspirations() Flow 触发
  → _inspirations 自动更新
  → displayInspirations 自动重组
  → UI 自动刷新（标签显示更新）
```

### 5.2 日期修改数据流（自动重排）

```
用户点击"确定"
  → ReminderPickerBottomSheet 回调 onConfirm(dateMillis, ...)
  → viewModel.updateInspirationDateTime(id, dateMillis)
  → inspirationRepository.update(inspiration.copy(createdAt=...))
  → Room 数据库更新
  → SQL: SELECT * FROM inspirations ORDER BY isPinned DESC, createdAt DESC
  → 数据库层自动按新日期重排
  → _inspirations 自动更新（重排后的列表）
  → displayInspirations 重组（重新计算 showDate 标记）
  → 灵感在时间轴上自动移动到新位置
```

**核心保障**：DAO 层 `ORDER BY isPinned DESC, createdAt DESC` 保证日期变更后列表顺序自动更新，无需 ViewModel 手动排序。

## 七、UI 设计规范遵循

- **标签管理弹窗**：使用现有 `TagPickerSheet` 组件，遵循已定义的野兽派风格（3px粗边框+8px硬阴影+弹性缩放动画）
- **日期时间选择器**：复用 `ReminderPickerBottomSheet`，主题色橙色 #FF9A5C 与 app 默认浅色主题一致
- **背景色**：#F8F6F3（页面）/ #FFFFFF（卡片表面）
- **文字色**：#2D2D2D（主文字）/ #666666（次要文字）/ #999999（提示文字）
- **文案**：「修改日期时间」/「日期时间」已确认

- 不修改 `TagPickerSheet` 组件本身
