# 灵感日期时间修改弹窗优化 — 设计文档

> **创建日期**: 2026-07-09
> **状态**: 已批准
> **关联模块**: 灵感记录（Inspiration）/ 日期时间选择器（ReminderPickerBottomSheet）

---

## 一、背景与需求

灵感长按「改日期」弹窗（ReminderPickerBottomSheet）当前存在以下不足，需优化：

| 编号 | 当前问题 | 期望行为 |
|---|---|---|
| 1 | 弹窗打开时显示灵感原始 createdAt 时间 | 显示系统当前时间（now），用户从当前时间开始调整 |
| 2 | 日历区使用 SpaceEvenly 等间距布局，高度较大 | 灵感场景下间距缩小为 4dp，降低日历区高度（待办编辑页不变） |
| 3 | 日历区下方无灵感内容预览 | 展示当前修改的灵感内容（日期/标题/时间/内容/标签/图片） |
| 4 | 无同步反馈 | 用户修改日期时间时，预览区的日期和时间实时更新 |

## 二、设计决策

### 决策1：实现方案 — 方案1A 参数化 + Composable lambda

| 方案 | 描述 | 优点 | 缺点 | 选择 |
|---|---|---|---|---|
| **方案1A** | 参数化 ReminderPickerBottomSheet + Composable lambda 预览 | 最小改动，职责分离好 | 参数增多 | **采用** |
| 方案2 | 拆分子组件 + 新建独立包装组件 | 职责最清晰 | 工作量大 | 否决 |
| 方案3 | 在 ModalBottomSheet content 中追加预览区 | 不改内部 | 不可行（状态访问问题） | 否决 |

### 决策2：预览区样式

参考 `CalendarInspirationItem`（`InspirationCalendarDialog.kt:708`）样式：
- 无背景，纯展示
- 字号/间距体系与 TimelineInspirationItem 同步
- 新增"日期"字段（CalendarInspirationItem 无日期，因为日历弹窗已选中日期）

### 决策3：日历间距参数化

- `calendarRowSpacing: Dp? = null`
- `null` → `Arrangement.SpaceEvenly`（默认，待办编辑页不变）
- `4.dp` → `Arrangement.spacedBy(4.dp)`（灵感场景）

## 三、详细设计

### 3.1 ReminderPickerBottomSheet 参数化改造

#### 新增 2 个参数

```kotlin
fun ReminderPickerBottomSheet(
    // ...现有参数...
    showAdvancedOptions: Boolean = true,
    title: String = "设置提醒时间",
    rowLabel: String = "提醒时间",
    calendarRowSpacing: Dp? = null,           // 新增：null=SpaceEvenly, 4.dp=spacedBy
    inspirationPreview: @Composable ((date: LocalDate, hour: Int, minute: Int) -> Unit)? = null,  // 新增
    onDismiss: () -> Unit,
    onConfirm: (dateMillis: Long?, hour: Int, minute: Int, repeatType: Int, calendarEnabled: Boolean) -> Unit
)
```

#### 改造点

| 位置 | 改造内容 |
|---|---|
| CalendarGridView 调用处 | 传入 `rowSpacing = calendarRowSpacing` |
| CalendarGridView 函数签名 | 新增 `rowSpacing: Dp? = null` 参数 |
| CalendarGridView L657 | `verticalArrangement` 改为 `if (rowSpacing != null) Arrangement.spacedBy(rowSpacing) else Arrangement.SpaceEvenly` |
| 日历区下方、按钮上方 | 插入 `inspirationPreview?.invoke(selectedDate, selectedHour, selectedMinute)` |

#### 预览区插入位置（布局结构）

```
Column(白底圆角) {
    标题
    日期时间行（日期芯片 + 时间芯片）
    日历区/时间滚轮
    ── ★ inspirationPreview?.invoke(selectedDate, selectedHour, selectedMinute) ★ ──
    重复提醒行（if showAdvancedOptions）
    农历行（if showAdvancedOptions）
    取消/确定按钮
}
```

#### 回归保护

待办编辑页不传新参数 → `calendarRowSpacing=null`（SpaceEvenly）+ `inspirationPreview=null`（无预览区）→ 行为完全不变。

### 3.2 新建 InspirationDateTimePreview 组件

**文件**: `app/src/main/java/com/corgimemo/app/ui/screens/inspiration/components/InspirationDateTimePreview.kt`

**参考样式**: `CalendarInspirationItem`（`InspirationCalendarDialog.kt:708`）

#### 函数签名

```kotlin
@Composable
fun InspirationDateTimePreview(
    inspiration: Inspiration,      // 静态字段：标题/内容/标签/图片
    date: LocalDate,               // 动态：用户选中的日期
    hour: Int,                     // 动态：用户选中的小时
    minute: Int                    // 动态：用户选中的分钟
)
```

#### 展示内容与样式

| 字段 | 来源 | 格式 | 样式 |
|---|---|---|---|
| 日期 | 动态 `date` | "M月d日 周X" | 14sp SemiBold #2D2D2D |
| 标题 | 静态 `inspiration.title` | 原文 | 16sp Medium #2D2D2D |
| 时间 | 动态 `hour:minute` | "HH:mm" | 11sp #999999 |
| 内容 | 静态 `inspiration.content` | 去 HTML 标签 | 14sp 行高21sp #666666 |
| 标签 | 静态 `decodeTags(inspiration.tags)` | "#tag" | 11sp 橙色，最多3个 |
| 图片 | 静态 `decodePaths(inspiration.imagePaths)` | 缩略图 | 圆角，最多3张 |

#### 间距体系（同 CalendarInspirationItem）

```
日期 → 标题 8dp
标题 → 时间 4dp
时间 → 正文 9dp
正文 → 标签 7dp
标签 → 图片 4dp
```

无背景，无点击事件，纯展示。

### 3.3 InspirationScreen 接线改造

#### 需求1：系统当前时间

```kotlin
if (showDateTimePicker && longPressedInspiration != null) {
    val inspiration = longPressedInspiration!!
    val nowCalendar = remember { Calendar.getInstance() }
    val dateTimeSheetState = rememberModalBottomSheetState(skipPartiallyExpanded = true)
    ModalBottomSheet(
        onDismissRequest = { ... },
        sheetState = dateTimeSheetState,
        containerColor = MaterialTheme.colorScheme.surface
    ) {
        ReminderPickerBottomSheet(
            initialDateMillis = System.currentTimeMillis(),  // 系统当前时间
            initialHour = nowCalendar.get(Calendar.HOUR_OF_DAY),
            initialMinute = nowCalendar.get(Calendar.MINUTE),
            showAdvancedOptions = false,
            title = "修改日期时间",
            rowLabel = "日期时间",
            calendarRowSpacing = 4.dp,
            inspirationPreview = { selectedDate, selectedHour, selectedMinute ->
                InspirationDateTimePreview(
                    inspiration = inspiration,
                    date = selectedDate,
                    hour = selectedHour,
                    minute = selectedMinute
                )
            },
            onDismiss = { ... },
            onConfirm = { dateMillis, _, _, _, _ ->
                if (dateMillis != null) {
                    viewModel.updateInspirationDateTime(inspiration.id, dateMillis)
                }
                ...
            }
        )
    }
}
```

#### 需求4：同步更新机制

```
用户修改日期/时间
  → ReminderPickerBottomSheet 内部 selectedDate/selectedHour/selectedMinute 变化
  → inspirationPreview lambda 重新调用（新参数值）
  → InspirationDateTimePreview 重组（新 date/hour/minute）
  → 预览区日期和时间实时更新
```

无需额外状态管理，Compose 响应式机制自动处理。

## 四、数据流

### 4.1 弹窗打开流程

```
用户长按灵感 → 点击"改日期"
  → InspirationScreen 设置 showDateTimePicker=true
  → ReminderPickerBottomSheet 渲染
  → 初始日期 = 系统当前日期
  → 初始时间 = 系统当前时间
  → 日历间距 = 4.dp
  → inspirationPreview lambda 被调用
  → InspirationDateTimePreview 渲染（日期=今天, 时间=当前, 标题/内容/标签/图片=灵感原始数据）
```

### 4.2 修改日期时间流程

```
用户在日历选择新日期 / 在滚轮选择新时间
  → ReminderPickerBottomSheet 内部 selectedDate/selectedHour/selectedMinute 更新
  → inspirationPreview lambda 重新调用（新值）
  → InspirationDateTimePreview 重组
  → 预览区日期/时间实时更新
```

### 4.3 确认流程

```
用户点击"确定"
  → onConfirm(dateMillis, ...) 回调
  → viewModel.updateInspirationDateTime(inspiration.id, dateMillis)
  → Repository 更新
  → DAO ORDER BY isPinned DESC, createdAt DESC 自动重排
  → 时间轴位置同步更新
```

## 六、UI 设计规范遵循

- 预览区样式参考 `CalendarInspirationItem`，与时间轴灵感卡片视觉统一
- 字号/间距体系与 TimelineInspirationItem 同步

- 无背景，纯展示
