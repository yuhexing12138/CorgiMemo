# 灵感编辑页界面调整设计文档

> 日期：2026-07-08
> 范围：仅限灵感编辑页（InspirationEditScreen.kt 及其组件 TagInputField.kt）
> 目标：界面一致性优化与功能精简

## 1. 背景与目标

灵感编辑页存在以下界面一致性问题，需进行精确修改：

1. 标签添加按钮位置不统一，分散在 TagInputField 内部
2. 标题区"标题"占位文字颜色过重，与下方内容占位"请在这里输入内容"视觉层级不一致
3. 右上角铃铛通知按钮承载的时间设置功能在灵感场景下冗余，且其弹窗交互复杂
4. 左上角返回箭头使用硬编码橙色，与待办编辑页（TodoEditScreen）统一后的返回箭头样式不一致
5. 分组按钮右侧的"低/中/高"优先级按钮在灵感场景下不必要

**修改原则**：仅对灵感编辑页进行修改，不改动 ViewModel / Model 层及其他页面。

## 2. 确认的决策

| 决策项 | 用户选择 |
|---|---|
| 铃铛按钮及其时间设置功能去留 | **彻底删除功能**（开始日期/截止时间/提醒时间/重复类型全部移除） |
| "标题"占位淡色调整范围 | **仅改淡色保留字号**（颜色改 `onSurfaceVariant.copy(alpha = 0.5f)`，保留 `headlineMedium`） |
| 新标签添加按钮视觉样式 | **与分类按钮一致**（Surface + 圆角14dp + surfaceVariant 背景） |
| 标签状态管理方案 | **方案 A：状态上提**（isAdding 状态由 InspirationEditScreen 托管） |
| 地理围栏功能 | **保留不动**（由内容中 `#` 触发，与铃铛弹窗无关） |
| priority 状态 | **仅删 UI 按钮，ViewModel 状态保留**（不主动设置默认值） |

## 3. 架构设计

### 3.1 顶部工具栏改造

**文件**：`InspirationEditScreen.kt`（L595-L783）

**改造前布局**：
```
返回箭头(橙色,20dp) | 撤销 | 重做 | ←弹性间距→ | 铃铛+下拉弹窗 | 锁定 | 完成
```

**改造后布局**：
```
返回箭头(onSurface,24dp) | 撤销 | 重做 | ←弹性间距→ | 锁定 | 完成
```

**具体改动**：

1. **返回箭头统一**（L603-L613）：
   - `tint`: `Color(0xFFFF9A5C)` → `MaterialTheme.colorScheme.onSurface`
   - `modifier size`: `20.dp` → `24.dp`
   - 容器 `IconButton size` 保持 `40.dp`
   - 与 TodoEditScreen.kt（L830-L843）完全一致

2. **铃铛按钮及下拉弹窗删除**（L662-L744）：
   - 删除 `showTimeMenu` 状态
   - 删除 `Box` + `IconButton`(Notifications) + `DropdownMenu` 整块
   - 删除菜单项：开始日期、截止时间、提醒时间、重复类型及 `HorizontalDivider`、`RepeatTaskManager.getRepeatTypeOptions()` 循环

### 3.2 标题区与标题下方交互区改造

**文件**：`InspirationEditScreen.kt`（L832-L925）、`TagInputField.kt`

**改造前布局**：
```
[标题输入框（占位"标题" 全色 onSurfaceVariant）]
[分类按钮] [低][中][高]
[TagInputField: "标签"标题 + 标签列表 + 添加按钮 + 输入框(条件)]
```

**改造后布局**：
```
[标题输入框（占位"标题" 半透明 onSurfaceVariant.copy(alpha=0.5f)）]
[分类按钮] [标签添加按钮]
[TagInputField: "标签"标题 + 标签列表 + 输入框(由外部触发)]
```

**具体改动**：

1. **标题占位淡色化**（L836-L843）：
   - 占位 Text 的 `color` 从 `MaterialTheme.colorScheme.onSurfaceVariant`
   - 改为 `MaterialTheme.colorScheme.onSurfaceVariant.copy(alpha = 0.5f)`
   - 保留 `style = MaterialTheme.typography.headlineMedium`

2. **删除优先级按钮**（L890-L902）：
   - 删除 `listOf(Pair(0, "低"), Pair(1, "中"), Pair(2, "高")).forEach { ... }` 整段
   - 删除关联的 `Spacer(modifier = Modifier.width(2.dp))` 逻辑

3. **新增标签添加按钮**（原优先级位置，分类按钮右侧）：
   - 采用与分类按钮完全一致的 `Surface` 样式：
     - `shape = RoundedCornerShape(14.dp)`
     - `color = MaterialTheme.colorScheme.surfaceVariant`
     - `onClick = { showTagInput = true }`
   - 内部 `Row`：`Add` 图标(16dp) + "标签" 文字(`bodySmall`)
   - 图标与文字均不指定显式颜色，沿用 Surface 默认内容色（与分类按钮完全一致）
   - `enabled = !isLocked`（与分类按钮一致，锁定时禁用）

4. **TagInputField 状态上提**（TagInputField.kt）：
   - 移除内部 `var isAdding by remember { mutableStateOf(false) }`（L60）
   - 移除"添加"按钮整块（L116-L140）
   - 新增参数：`isAdding: Boolean`、`onAddingChange: (Boolean) -> Unit`
   - 输入框显示条件由内部 `isAdding` 改为外部传入的 `isAdding`
   - "确定"按钮确认后调用 `onAddingChange(false)` 重置（沿用现有验证逻辑：空或重复时不添加）

5. **InspirationEditScreen 状态托管**：
   - 新增 `var showTagInput by remember { mutableStateOf(false) }`
   - 传入 `TagInputField(isAdding = showTagInput, onAddingChange = { showTagInput = it })`
   - 新标签按钮 `onClick = { showTagInput = true }`

## 4. 数据流

### 4.1 标签添加按钮交互流

```
用户点击[标签添加按钮]
  → showTagInput = true
  → TagInputField 接收 isAdding=true
  → 显示 OutlinedTextField + "确定"按钮
  → 用户输入标签名 + 点击"确定"
  → 验证非空且不重复 → onTagsChange(updatedTags) → viewModel.updateTags()
  → onAddingChange(false) → showTagInput = false → 输入框隐藏
```

### 4.2 返回箭头交互流（不变）

```
用户点击返回箭头 → navController.popBackStack()
```

## 5. 错误处理

- 标签输入为空或重复时：不添加，直接重置状态（沿用 TagInputField 现有逻辑）
- 锁定状态（isLocked=true）：标签添加按钮禁用（`enabled = !isLocked`），与分类按钮一致
- 删除功能后 ViewModel 中残留的方法（setStartDate 等）不被调用，无副作用
