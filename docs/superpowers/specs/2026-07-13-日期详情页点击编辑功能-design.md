# 日期详情页点击编辑功能设计文档

## 一、功能概述

### 1.1 功能目标
实现日期页卡片的点击编辑功能，用户点击日期主页的卡片后，从当前"编辑功能开发中"状态切换为进入日期详情页。详情页提供完整的查看、编辑、分享等操作，并支持跳转到编辑页面修改基本信息。

### 1.2 功能范围
- 日期详情展示页（卡片展示、备注、底部工具栏）
- 三点操作弹窗（置顶、归档、删除）
- 备注编辑（底部弹窗）
- 主题选择（样式+颜色，底部弹窗）
- 编辑基本信息（跳转 QuickCreate 编辑模式）
- 卡片图片分享与保存

---

## 二、架构与导航设计

### 2.1 新增页面与路由

在 `Screen.kt` 中新增两个路由：

| 路由对象 | 路由路径 | 用途 | 参数 |
|----------|--------|------|------|
| `SpecialDateDetailWithId` | `date_detail/{dateId}` | 日期详情页 | `dateId` (Long) |
| `SpecialDateEditWithId` | `date_edit/{dateId}` | 日期编辑页 | `dateId` (Long) |

> 编辑页复用现有的 `SpecialDateQuickCreateScreen` 组件，通过参数区分"新建"和"编辑"模式。

### 2.2 导航流程

```
日期主页 (SpecialDateScreen)
    ↓ 点击卡片
日期详情页 (SpecialDateDetailScreen)
    ├─ ← 返回按钮 → 返回日期主页
    ├─ ⋮ 三点按钮 → 弹窗（置顶/归档/删除）
    ├─ 底部"备注"按钮 → 底部弹窗编辑备注
    ├─ 底部"主题"按钮 → 底部弹窗选样式+颜色
    ├─ 底部"编辑"按钮 → 跳转编辑页
    └─ 底部"分享"按钮 → 触发卡片截图分享
         ↓
日期编辑页 (QuickCreate 改造版)
    ├─ 顶部："编辑"（原"添加"）
    └─ 底部："保存"（原"下一步"）
         ↓ 保存后
返回详情页（刷新数据）
```

### 2.3 ViewModel 设计

| ViewModel | 职责 |
|-----------|------|
| `SpecialDateViewModel`（已存在） | 管理日期列表数据，提供按 ID 查询单个日期的方法 |
| `SpecialDateDetailViewModel`（新增） | 管理详情页状态：日期数据加载、置顶/归档/删除、备注更新、样式/颜色更新、分享准备 |

## 三、详情页 UI 设计

### 3.1 页面整体结构

```
┌─────────────────────────────────┐
│ ←                        ⋮    │  ← TopAppBar
├─────────────────────────────────┤
│                                 │
│         ┌───────────┐           │
│         │           │           │
│         │  日期卡片  │           │  ← 居中展示
│         │           │           │
│         └───────────┘           │
│                                 │
│      这里是备注，超过一行...     │  ← 备注文字行
│                                 │
│                                 │
│   📝    👕    ✏️    ↗️         │  ← 底部工具栏
│  备注   主题   编辑   分享       │
│                                 │
└─────────────────────────────────┘
```

### 3.2 顶部导航栏
- 左上角：返回箭头，点击返回日期主页
- 右上角：三点按钮，点击弹出操作菜单
- 背景色：与卡片主题色联动的浅色背景

### 3.3 日期卡片区域
- 居中展示，左右边距 32dp
- 复用 `DateCardStyleRenderer` 组件渲染
- 支持所有已有的卡片样式和颜色
- 宽度占屏幕宽度的 75% 左右，高度自适应

### 3.4 备注文字行
- 位于卡片下方 16dp 处，居中显示
- 单行显示，超出末尾加省略号
- 点击可触发备注编辑（与底部"备注"按钮功能一致）

### 3.5 底部工具栏
- 四个图标按钮等距排列
- 图标下方显示文字标签
- 图标使用圆形半透明白色背景
- 距离底部导航栏上方 24dp

## 四、三点操作弹窗设计

### 4.1 弹窗样式
- 参考 `TodoMenuDropdown.kt` 的实现模式
- 使用 Material 3 `DropdownMenu`
- 位置：右上角三点按钮下方

### 4.2 菜单项（从上到下）

| 序号 | 文字 | 图标 | 点击行为 |
|------|------|------|----------|
| 1 | 置顶 | `Icons.Outlined.PushPin` | 切换置顶状态，关闭弹窗，实时更新 |
| 2 | 归档 | `Icons.Outlined.Archive` | 执行归档，关闭弹窗，返回主页并显示撤回 Snackbar |
| 3 | 删除 | `Icons.Outlined.Delete` | 弹出确认对话框，确认后删除并返回主页 |

### 4.3 删除确认对话框
- 标题："确认删除？"
- 内容："删除后无法恢复，确定要删除这个日期吗？"
- 按钮：取消 / 删除（红色强调）

## 五、底部工具栏设计

### 5.1 备注按钮
- **图标**：Material Icons `EditNote`
- **文字**：备注
- **点击行为**：弹出底部弹窗编辑备注

#### 备注编辑弹窗
- 顶部：标题"编辑备注" + 关闭按钮
- 中间：多行文本输入框（预填充当前备注）
- 底部：取消 / 保存 按钮
- 保存后实时更新详情页备注显示

### 5.2 主题按钮
- **图标**：Material Icons `Palette`
- **文字**：主题
- **点击行为**：弹出底部弹窗选择样式和颜色

#### 主题选择弹窗
- Tab 切换：样式 / 颜色
- 样式 Tab：展示所有可选卡片样式
- 颜色 Tab：展示所有可选颜色（圆形色卡）
- 选择后实时预览（实时更新详情页卡片）
- 点击弹窗外部确认并关闭

### 5.3 编辑按钮
- **图标**：项目现有编辑图标
- **文字**：编辑
- **点击行为**：跳转到编辑页面（QuickCreate 改造版）
- 跳转时传入 `dateId` 参数
- 编辑保存后返回详情页并刷新数据

### 5.4 分享按钮
- **图标**：项目现有分享图标
- **文字**：分享
- **点击行为**：触发卡片截图分享

## 六、编辑页设计（QuickCreate 改造）

### 6.1 模式区分
通过导航参数 `dateId` 区分模式：
- `dateId = null` → 新建模式（现有逻辑不变）
- `dateId != null` → 编辑模式（加载数据，修改UI文字）

### 6.2 UI 差异

| 元素 | 新建模式 | 编辑模式 |
|------|----------|----------|
| 顶部标题 | "添加" | "编辑" |
| 底部按钮 | "下一步" → 样式选择页 | "保存" → 直接保存并返回 |
| 初始数据 | 空 / 默认值 | 从数据库加载已有数据 |
| 保存逻辑 | 样式页保存 | 编辑页直接保存 |

### 6.3 保存流程（编辑模式）
1. 校验必填项（名称、日期）
2. 更新数据库中的 `SpecialDate` 记录
3. 通过 `savedStateHandle` 设置 `date_updated = true`
4. `popBackStack()` 返回详情页
5. 详情页监听到更新标记后刷新数据

### 6.4 未保存修改处理
- 用户有修改但未保存直接返回时，弹出"是否保存修改？"确认对话框
- 选项：保存 / 不保存 / 取消

## 七、数据流转设计

### 7.1 详情页数据加载
```
进入详情页 → DetailViewModel 接收 dateId
    → 从数据库查询 SpecialDate
    → 收集为 StateFlow
    → UI 层 collectAsState() 渲染
```

### 7.2 状态更新机制

| 操作 | 数据更新方式 | UI 刷新方式 |
|------|-------------|------------|
| 切换置顶 | 更新 DB isPinned 字段 | StateFlow 自动刷新 |
| 归档 | 更新 DB isArchived 字段 + 返回主页 | 返回主页后列表自动刷新 |
| 删除 | 删除 DB 记录 + 返回主页 | 返回主页后列表自动刷新 |
| 编辑备注 | 更新 DB content 字段 | StateFlow 自动刷新 |
| 修改样式 | 更新 DB cardStyle 字段 | StateFlow 自动刷新 |
| 修改颜色 | 更新 DB cardColor 字段 | StateFlow 自动刷新 |
| 编辑基本信息 | 编辑页保存后返回 | 监听返回结果 + 重新加载 |

### 7.3 页面间数据传递
- 详情页 → 编辑页：通过 `dateId` 导航参数
- 编辑页 → 详情页：通过 `savedStateHandle` 设置 `date_updated` 标记
- 详情页 → 主页：归档/删除后 pop 返回，主页列表自动刷新

## 八、分享功能设计

### 8.1 整体流程
1. 对卡片区域进行截图（生成 Bitmap）
2. 保存到本地缓存目录
3. 调用系统分享 Intent 分享图片
4. 同时提供"保存到本地"选项

### 8.2 截图实现
- 使用 Compose View 截图方式
- 截图范围：仅日期卡片区域（不含工具栏、导航栏）
- 截图背景：使用卡片当前的背景色

### 8.3 分享选项
点击分享按钮后，显示底部弹窗，包含两个选项：
1. **分享图片** → 调用系统分享面板
2. **保存到本地** → 保存到相册/图库

### 8.4 新增文件
- `util/SpecialDateScreenshot.kt`：日期卡片截图工具类（参考 `InspirationScreenshot.kt`）

## 九、错误处理与边界情况

### 9.1 数据加载失败
- 日期不存在（已被删除）：显示错误提示 + 返回按钮
- 数据库异常：显示重试按钮

### 9.2 操作反馈
| 操作 | 反馈方式 |
|------|---------|
| 置顶/取消置顶 | 无额外提示（状态直接变化） |
| 归档 | Snackbar："已归档" + 撤回按钮（3秒） |
| 删除 | 确认对话框（防止误操作） |
| 保存成功 | Snackbar："保存成功" |
| 分享失败 | Snackbar："分享失败，请重试" |

### 9.3 编辑页返回
- 用户未修改直接返回：直接返回，不提示
- 用户有修改但未保存：弹出"是否保存修改？"确认对话框

## 十一、开发顺序建议

1. 基础框架：新增路由 + 详情页基础框架 + 数据加载
2. 卡片展示：复用 DateCardStyleRenderer 居中展示卡片
3. 顶部导航：返回按钮 + 三点弹窗
4. 底部工具栏：四个按钮 UI
5. 备注编辑：底部弹窗 + 保存逻辑
6. 主题选择：样式+颜色底部弹窗 + 实时预览 + 保存
7. 编辑跳转：QuickCreate 编辑模式 + 保存返回
8. 分享功能：截图 + 分享弹窗 + 系统分享
