# 待办页导航栏日期与日历弹窗设计

## 概述

为待办页顶部导航栏中间添加日期显示（与灵感页一致），点击后弹出日历弹窗，用户可在日历上选择日期查看当天（基于 startDate）的所有待办事项。

## 需求确认

| 项目 | 决策 |
|------|------|
| 日期过滤字段 | `startDate`（待办开始时间） |
| 导航栏日期 | 始终显示当天日期，不随日历选择变化 |
| 日历圆点 | 有待办的日期显示圆点 |
| 弹窗内容 | 复用共享日历区 + 待办列表（含已完成） |
| 点击待办条目 | 关闭弹窗 + 滚动到主列表对应位置 |
| 待办条目样式 | 勾选框 + 标题 + 优先级色块 + 截止时间 |

## 实现方案：方案C — 抽取共享日历区 + 各自独立弹窗

### 设计段1：共享日历组件抽取

将 `InspirationCalendarDialog.kt` 中的通用日历组件移到共享包 `ui.components.calendar`：

**移动的组件**：
- `CalendarMonthView`：网格日历视图（从 `internal` 改为 `public`）
- `DateWheelPicker`：年/月/日三列滚轮选择器（从 `internal` 改为 `public`）
- `WheelColumn`：保持 `private`，仅被 `DateWheelPicker` 内部使用
- `decodeTagsJson`、`decodePathsJson`、`removeHtmlTags`：保留在灵感组件包中，与灵感数据模型相关

**新包路径**：`com.corgimemo.app.ui.components.calendar`

**影响**：
- InspirationCalendarDialog 需更新 import 路径
- InspirationCalendarDialog 本身逻辑不修改，仅 import 变更

### 设计段2：待办页导航栏日期显示

在 MainScreen.kt 的 `EnhancedTopBar` 的 `centerContent` 参数中，当 `selectedTab == TabItem.TODO && !effectiveBatchMode` 时注入日期组件。

**布局**（与灵感页完全一致）：
```
[大号日期 25sp Bold] [7dp间距] [月份 16sp] [2dp间距] [箭头 8sp]
```

- 点击：切换 `showTodoCalendar` 状态
- 箭头方向：展开时▲，收起时▼
- 日期值：始终使用 `Calendar.getInstance()` 获取当天日期

**MainScreen 新增状态**：
- `var showTodoCalendar by remember { mutableStateOf(false) }`

### 设计段3：TodoCalendarDialog 弹窗

**文件位置**：`ui/screens/home/components/TodoCalendarDialog.kt`

**整体结构**：
```
┌──────────────────────────────┐
│ 2026年7月 ▼             ×    │  标题行
├──────────────────────────────┤
│ 一  二  三  四  五  六  日    │  星期标题行
│           1   2   3   4      │  日历网格（有圆点）
│  5   6   7   8   9  10  11   │
│ ...                          │
├──────────────────────────────┤
│ 7月12日 周六  共 3 条待办     │  选中日期信息栏
├──────────────────────────────┤
│ ✓ 完成项目报告  🔴 截止7/15  │  待办列表
│ □ 购买生日礼物  🟡 截止7/14  │
│ □ 提交周报     🟢 无截止     │
└──────────────────────────────┘
```

**参数签名**：
```kotlin
@Composable
fun TodoCalendarDialog(
    initialDate: LocalDate = LocalDate.now(),
    todoCountByDate: (year: Int, month: Int) -> Map<Int, Int>,
    getTodosByDate: (year: Int, month: Int, day: Int) -> List<TodoItem>,
    onTodoClick: (TodoItem) -> Unit,
    onDismiss: () -> Unit,
    topPadding: Dp = 0.dp
)
```

**CalendarTodoItem 条目样式**：
- 左侧：勾选框图标（已完成 ✓ / 未完成 □），已完成项标题加删除线
- 中间：标题文本（16sp Medium）
- 右侧：优先级色块（红/黄/绿，4dp圆形）+ 截止时间文本（11sp 灰色）
- 间距：8dp 垂直内边距

**空态**：`"今天还没有待办~"`（与灵感页风格一致）

**选中日期信息栏**：`"{月}月{日}日 周{X}  共 {N} 条待办"`

### 设计段4：数据层与交互流程

**HomeViewModel 新增方法**：

1. `getCalendarTodoCount(year: Int, month: Int): Map<Int, Int>`
   - 从内存中的待办列表过滤 `startDate` 落在指定年月的项
   - 按 `startDate` 的日分组计数
   - 实现：遍历所有待办，Calendar 解析 startDate，匹配年月后 groupBy 日

2. `getTodosByDate(year: Int, month: Int, day: Int): List<TodoItem>`
   - 过滤 `startDate` 落在指定日期的所有待办（含已完成）
   - 排序：未完成在前，已完成在后；同状态内按优先级 DESC

**点击待办 → 滚动到主列表**（通过 HomeViewModel 状态解耦）：
1. 用户点击弹窗中的待办 → `onTodoClick(todo)` 触发
2. 关闭日历弹窗（`showTodoCalendar = false`）
3. MainScreen 调用 `homeViewModel.requestScrollToTodo(todo.id)`
4. HomeViewModel 设置 `_scrollToTodoId` 状态为 `todo.id`
5. HomeScreen 中 LaunchedEffect 监听 `scrollToTodoId` 变化：
   - 判断待办在未完成区还是已完成区
   - 如果在已完成区且已完成区折叠，先展开已完成区
   - 通过 `lazyListState` 滚动到待办对应的 displayItem 位置
   - 滚动完成后重置 `_scrollToTodoId` 为 null

**HomeViewModel 新增状态**：
- `scrollToTodoId: StateFlow<Long?>`：请求滚动到指定待办（消费后自动重置为 null）
- `fun requestScrollToTodo(id: Long)`：设置滚动目标

**MainScreen 中 TodoCalendarDialog 的渲染位置**：
- 与灵感页一致，在 Scaffold 外层 Box 内、Scaffold 同级渲染
- 条件：`showTodoCalendar && selectedTab == TabItem.TODO`

## 边界情况

- **startDate 为 null 的待办**：不显示在日历弹窗中（无法归到具体日期）
- **已完成区折叠时点击已完成待办**：先展开已完成区再滚动
- **待办不在当前过滤结果中**：滚动失败时显示 Toast 提示
- **批量模式下**：不显示日期 centerContent，与灵感页一致
