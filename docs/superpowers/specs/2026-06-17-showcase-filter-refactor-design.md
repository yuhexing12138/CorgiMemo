## 1. 范围与约束

### 1.1 改动范围（严格限定）

| 项目 | 说明 |
|---|---|
| **唯一改动文件** | `【刻记+】APP\corgimemo-showcase.html` |
| **不改动** | Kotlin 代码（`FloatingCorgiButton.kt`、`AppDrawer.kt` 等）、所有 `.md` 文档、`res/` 资源、主题文件 |
| **参考来源** | App 端 `AppDrawer.kt`（侧边栏结构）、`FloatingCorgiButton.kt`（悬浮球行为）仅作参考，不修改 |

### 1.2 全局约束

| 编号 | 约束 | 说明 |
|---|---|---|
| C1 | 动画一致性 | 柯基动画继续使用纯 CSS 帧动画（`.corgi-anim` + `.corgi-frame` + `nth-child` + `animation-delay`），不引入 JS 动画库或新实现方式 |
| C2 | 主题色锁定 | 所有新增元素必须复用现有 `:root` 变量（`--color-primary: #FF9A5C` 等），不引入新颜色值 |
| C3 | 组件复用 | 悬浮球封装为可复用 JS 函数组件，每个原型调用一次注入，避免重复实现 |
| C4 | 原型总数 | 由当前 12 个增至 16 个（+3 侧边导航栏 +1 编辑日期） |

### 1.3 术语定义

| 术语 | 定义 |
|---|---|
| **柯基展示区** | 原型内静态的柯基呈现区域，HTML 结构为 `.phone-corgi-area` + `.corgi-anim`。本次需删除（节日关怀除外） |
| **柯基悬浮球** | 本次新增的可拖动动态元素。每个原型内嵌一个，默认位于手机屏幕左下方，始终显示柯基动画，层级最高 |
| **侧边导航栏原型** | 本次新增的 3 个原型，展示 App 端 `AppDrawerContent` 在待办/灵感/日期三个 Tab 下的视图 |
| **四大核心分类** | 核心功能、柯基系统、智能特性、个人中心 |
| **子筛选器层级** | 仅"核心功能"分类下存在的二级筛选：待办、灵感、日期 |

## 2. 任务依赖关系与执行顺序

原始需求存在循环依赖和顺序混乱。优化后按**自底向上**的依赖关系重组为 6 层：

```
Layer 1: 基础设施层 ──→ Layer 2: 原型清理层 ──→ Layer 3: 原型新增层
                                                          │
Layer 6: 全局约束层 ←── Layer 5: 文档更新层 ←── Layer 4: 筛选器重构层
```

| 层级 | 名称 | 原始需求映射 | 依赖说明 |
|---|---|---|---|
| L1 | 基础设施层 | 需求 3（悬浮球）+ 需求 7（动画不变） | 悬浮球组件必须先实现，L2/L3 才能引用 |
| L2 | 原型清理层 | 需求 4（删柯基展示区） | 依赖 L1 的悬浮球替代静态柯基区 |
| L3 | 原型新增层 | 需求 2（侧栏原型）+ 需求 5（编辑日期） | 新增原型需复用 L1 的悬浮球组件 |
| L4 | 筛选器重构层 | 需求 1（四大分类） | 依赖 L3 确定最终原型数量与归属 |
| L5 | 文档更新层 | 需求 6（指引文档） | 依赖 L1-L4 全部完成后的最终状态 |
| L6 | 全局约束层 | 需求 7（动画不变） | 贯穿所有层的约束条件（已在 1.2 定义） |

### 3.1 L1 · 基础设施层：柯基悬浮球组件

**目标**：在 HTML 原型中实现可复用的柯基悬浮球组件

#### 3.1.1 视觉规范

| 属性 | 值 |
|---|---|
| 尺寸 | 48×48px |
| 形状 | 圆形 |
| 背景 | `rgba(255,154,92,0.15)` |
| 边框 | `1px solid rgba(255,154,92,0.4)` |
| 默认位置 | 手机屏幕左下方（距底部 88px，距左侧 12px） |
| 静止阴影 | `0 4px 12px rgba(255,154,92,0.25)` |
| 拖动阴影 | `0 8px 24px rgba(255,154,92,0.35)` |
| 拖动缩放 | 1.1 倍 |

#### 3.1.2 动画规范

| 属性 | 要求 |
|---|---|
| 动画类型 | 复用现有 CSS 帧动画系统（`.corgi-anim` + `.corgi-frame` + `frameShow2` 关键帧） |

| 周期 | 1s 循环 |
| 图片尺寸 | 36×36px（居中于 48×48 容器内） |

#### 3.1.3 交互规范

| 行为 | 实现 |
|---|---|
| 拖动触发 | 鼠标按下悬浮球即可开始拖动 |
| 移动范围 | 限制在所属 `.phone-screen` 边界内 |
| 松开行为 | 保持当前位置，不再自动吸附 |
| 层级 | `z-index: 50`（高于原型内所有其他元素） |

### 3.2 L2 · 原型清理层：删除静态柯基展示区

#### 3.2.1 清理规则

| 操作 | 对象 | 数量 |
|---|---|---|
| **删除** | `.phone-corgi-area` 及其内部所有子元素（`.corgi-anim`、`.corgi-frame`、`<img>` 等） | 10 个原型 |
| **保留** | 节日关怀原型（手机4）的 `.phone-holiday-card` + `.holiday-corgi` 结构完整保留 | 1 个原型 |
| **无操作** | 「容器分组待办」「灵感富文本编辑」本身无柯基展示区 | 2 个原型 |

#### 3.2.2 受影响的原型清单

| 手机编号 | 原型名称 | 操作 |
|---|---|---|
| 手机1 | 待办列表 | 删除 `.phone-corgi-area` |
| 手机6 | 灵感时间线 | 删除 `.phone-corgi-area` |
| 手机8 | 特殊日期记录 | 删除 `.phone-corgi-area` |
| 手机2 | 8种柯基表情 | 删除 `.phone-corgi-area` |
| 手机9 | 柯基成长系统 | 删除 `.phone-corgi-area` |
| 手机10 | 成就系统 | 删除 `.phone-corgi-area` |
| 手机3 | 身份自适应 | 删除 `.phone-corgi-area` |
| 手机11 | 完成统计 | 删除 `.phone-corgi-area` |
| 手机12 | 个性化设置 | 删除 `.phone-corgi-area` |
| — | 节日关怀（手机4） | **保留** `.phone-holiday-card` |
| — | 容器分组待办（手机5） | 无柯基区，无需操作 |
| — | 灵感富文本编辑（手机7） | 无柯基区，无需操作 |

#### 3.2.3 布局调整原则

删除 `.phone-corgi-area` 后，释放的空间由相邻内容区自然填充：
- 若柯基区位于内容区下方 → 内容区扩展填充
- 若柯基区位于内容区右侧 → 内容区宽度扩展
- 保持原型布局紧凑，避免出现空白间隙

### 3.3 L3 · 原型新增层：4 个新原型

#### 3.3a · 侧边导航栏原型（3 个）

##### 整体设计规范

| 属性 | 要求 |

##### 三个原型的差异

| 属性 | 侧边栏·待办 | 侧边栏·灵感 | 侧边栏·日期 |
|---|---|---|---|
| **分类归属** | 核心功能 → 待办 | 核心功能 → 灵感 | 核心功能 → 日期 |
| **标题文字** | 分组管理 | 标签筛选 | 类型筛选 |
| **列表项** | 全部待办、未分类、学习、工作、生活、运动、最近删除(14) | 全部灵感、标签列表（含自定义标签项） | 全部日期、倒计时、正计时、已过期 |

| **底部操作区** | ⚙️ 设置 / ❓ 帮助与反馈 | ⚙️ 设置 / ❓ 帮助与反馈 | ⚙️ 设置 / ❓ 帮助与反馈 |

每个侧边栏原型的右侧显示对应的主页内容：

#### 3.4.4 交互规则

| 规则 | 说明 |
|---|---|

## 4. 验收标准

### 4.1 功能验收

| 编号 | 验收项 | 通过标准 |
|---|---|---|
| F1 | 悬浮球显示 | 每个（含新增的）原型均显示柯基悬浮球，播放坐姿呼吸动画 |
