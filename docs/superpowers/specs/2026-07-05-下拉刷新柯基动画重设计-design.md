## 1. 背景与目标

### 1.1 现状

当前待办页使用 Material3 的 `PullToRefreshBox` + 自定义 `CorgiPullToRefreshIndicator`，柯基**从屏幕右侧跑入**并切换为摇尾巴动画。

参考：[CorgiPullToRefreshIndicator.kt](file:///c:/Users/EDY/Desktop/CorgiMemo/app/src/main/java/com/corgimemo/app/ui/components/CorgiPullToRefreshIndicator.kt)

存在的问题：
- 柯基从右侧跑入，与"下拉刷新"的语义不一致
- 强依赖 Material3 `PullToRefreshBox`，阻尼曲线与最大高度难以精细控制
- 刷新触发由 Material3 内部阈值决定，无法自定义"达阈值后释放触发"

### 1.2 目标

删除现有"柯基从右侧跑出"逻辑，改为：

1. **空白区**：用户下拉时，搜索框与待办卡片区之间出现空白区，随下拉增大
2. **阻尼递增**：用户越往下拉，阻尼越大，直到达到最大空白高度
3. **居中奔跑柯基**：下拉过程中，空白区水平居中处始终动态显示一个**奔跑姿态**的柯基动画
4. **刷新触发**：用户释放时，若下拉达到阈值则触发待办卡片区刷新
5. **视觉伴侣**：执行过程中启用视觉伴侣辅助设计

### 1.3 关键约束

- 柯基动画**所有阶段都使用 `AnimationType.RUN`**（用户明确修正，不切换为 WAG）
- 刷新触发条件：**达到阈值后释放触发**（非"达阈值立即触发"）
- 最大空白高度：参考用户提供的图片（约 100dp）

## 2. 方案选型

### 2.1 候选方案

| 方案 | 核心 | 阻尼控制 | 最大高度控制 | 复杂度 |
|---|---|---|---|---|
| **方案 A** | 自定义 `NestedScrollConnection` + 自定义状态机 | 完全自定义 | 完全自定义 | 中 |
| 方案 B | 复用 Material3 `PullToRefreshBox` + 自定义 indicator | 受限于 Material3 | 受限于 Material3 | 低 |

### 2.2 选型结论

**采用方案 A**：自定义 `NestedScrollConnection`。

理由：
- 阻尼曲线 `1 - (x/max)²` 需要精细控制，Material3 内部阻尼无法覆盖
- 柯基需要"始终居中 + 始终奔跑"，与 Material3 的 indicator 容器模型冲突
- 列表与搜索框需同步下移，需直接控制 `translationY`，Material3 封装后难以注入

## 3. 核心架构

### 3.1 状态机：`PullRefreshState`

4 值枚举，描述下拉刷新全生命周期：

| 状态 | 含义 | 进入条件 | 退出条件 |
|---|---|---|---|
| `IDLE` | 空闲，无下拉 | 初始 / 刷新完成 | 用户开始下拉 |
| `PULLING` | 用户正在下拉 | 用户手指下拉 | 用户释放 / 达阈值 |
| `RELEASING` | 释放后回弹动画中 | 未达阈值释放 | 回弹完成 → IDLE |
| `REFRESHING` | 刷新中 | 达阈值释放 | isRefreshing=false → IDLE |

### 3.2 组件分层

```
PullRefreshStateHolder（remember 持有）
├── state: PullRefreshState              // 当前状态
├── pullOffset: Float                    // 当前下拉偏移（dp，已阻尼）
├── nestedScrollConnection               // 注入到外层 Box
├── onRefreshComplete()                  // 刷新完成回调
└── maxPullHeight / refreshThreshold     // 参数

CorgiPullRefreshIndicator（Composable）
├── pullOffset: Float
├── state: PullRefreshState
└── 渲染：空白区 + 居中奔跑柯基 + 提示文字
```

## 4. 阻尼曲线与关键参数

### 4.1 阻尼公式

```kotlin
/**
 * 阻尼曲线：随下拉量增大，阻尼系数递减
 * dampingFactor = 1 - (rawOffset / maxPullHeight)²
 *
 * @param rawOffset 原始下拉量（px 或 dp，单位与 maxPullHeight 一致）
 * @param maxPullHeight 最大下拉高度
 * @return 阻尼后的实际偏移
 */
fun computeDampedOffset(rawOffset: Float, maxPullHeight: Float): Float {
    if (rawOffset <= 0f) return 0f
    if (rawOffset >= maxPullHeight) return maxPullHeight
    val ratio = rawOffset / maxPullHeight
    val dampingFactor = 1f - ratio * ratio
    return rawOffset * dampingFactor
}
```

**阻尼特性**：
- `rawOffset = 0`：阻尼系数 1.0（无阻尼，跟随手指）
- `rawOffset = maxPullHeight / 2`：阻尼系数 0.75（轻微阻尼）
- `rawOffset = maxPullHeight`：阻尼系数 0.0（完全锁死，无法继续下拉）

### 4.2 关键参数

| 参数 | 值 | 含义 |
|---|---|---|
| `maxPullHeight` | 100.dp | 空白区最大高度 |
| `refreshThreshold` | 60.dp | 触发刷新的阈值（释放时下拉量 ≥ 此值则刷新） |
| `releaseAnimDuration` | 300ms | 释放后回弹动画时长 |
| `corgiSizeMin` | 48.dp | 柯基最小尺寸（下拉量为 0 时） |
| `corgiSizeMax` | 64.dp | 柯基最大尺寸（下拉量达 maxPullHeight 时） |
| `corgiFrameRate` | 8 fps | 帧动画帧率（与现有 CorgiPullToRefreshIndicator 一致） |

### 4.3 柯基大小渐变公式

```kotlin
val pullProgress = (pullOffset / maxPullHeight).coerceIn(0f, 1f)
val corgiSize = lerp(corgiSizeMin, corgiSizeMax, pullProgress)
```

## 5. 动画与提示文字

### 5.1 柯基动画类型

**所有阶段都使用 `AnimationType.RUN`**（用户明确修正）。

| 阶段 | 状态 | 动画类型 | 是否播放 |
|---|---|---|---|
| 未下拉 | IDLE | RUN | 不播放（不可见） |
| 下拉中 | PULLING | RUN | 播放 |
| 释放回弹 | RELEASING | RUN | 播放 |
| 刷新中 | REFRESHING | RUN | 播放 |

### 5.2 柯基显示条件

```kotlin
val showCorgi = pullOffset > 0.01f || state == PullRefreshState.REFRESHING
```

### 5.3 提示文字

| 状态 | 文字 | 颜色 |
|---|---|---|
| PULLING（pullOffset < refreshThreshold） | "下拉刷新" | onSurfaceVariant α=0.6 |
| PULLING（pullOffset ≥ refreshThreshold） | "释放刷新" | onSurfaceVariant α=0.6 |
| REFRESHING | "柯基努力加载中~" | onSurfaceVariant α=0.6 |
| IDLE / RELEASING | 无 | - |

### 5.4 帧资源

复用 `AnimationResourceManager.getAnimationFrames(AnimationType.RUN)`，无需新增资源。

## 8. 风险与注意事项

### 8.1 `NestedScrollConnection` 与 LazyColumn 滚动冲突

- **风险**：LazyColumn 在顶部继续上滑时，`onPreScroll` 必须正确消费未消费的滚动量，否则会出现"列表先滚动再下拉"的卡顿
- **缓解**：`onPreScroll` 中判断 `lazyListState.firstVisibleItemIndex == 0 && lazyListState.firstVisibleItemScrollOffset == 0`，仅在列表已到顶时接管下拉

### 8.2 `translationY` 与 LazyColumn 内部状态

- **风险**：`graphicsLayer { translationY = pullOffset }` 不影响 LazyColumn 的布局，仅视觉位移，拖拽排序时应保持正常
- **缓解**：拖拽进行中（`isDragActive == true`）时禁用 `nestedScrollConnection`，避免手势冲突

### 8.3 `Animatable` 与 `onRefreshComplete` 时序

- **风险**：刷新完成时 `isRefreshing` 从 true → false，若 `onRefreshComplete` 在 `Animatable.animateTo` 执行中调用，可能产生竞态
- **缓解**：`onRefreshComplete` 内部使用 `launch` 启动协程，调用 `pullOffsetAnimatable.animateTo(0f)`，`Animatable` 内部保证原子性

### 8.4 import 检查

- **规则**：每次任务编辑完所有代码后都要检查是否有 import 语句缺失
- **重点**：`HomeScreen.kt` 删除 `PullToRefreshBox` 相关 import 后，需添加 `rememberPullRefreshStateHolder`、`CorgiPullRefreshIndicator`（新）、`nestedScroll`、`graphicsLayer`、`lerp` 等 import

## 9. 后续优化建议

1. **柯基奔跑方向随下拉量变化**：下拉量越大，柯基奔跑速度越快（动态 fps）
2. **柯基表情随状态变化**：PULLING 时专注，REFRESHING 时开心（需新增表情资源）
3. **触觉反馈**：达阈值时触发轻震动（`HapticFeedbackManager`）
4. **音效**：刷新触发时播放短音效（可选）
5. **刷新成功动画**：柯基跳跃后消失（替代直接消失）
6. **自适应最大高度**：根据屏幕高度动态调整 `maxPullHeight`（小屏 80dp，大屏 120dp）
