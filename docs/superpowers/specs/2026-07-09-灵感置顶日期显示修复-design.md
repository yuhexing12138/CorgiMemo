# 灵感置顶日期显示修复 设计文档

> 创建日期：2026-07-09
> 模块：灵感记录页（InspirationScreen）
> 类型：Bug 修复 + 重构

## 一、问题描述

### 1.1 复现场景

灵感数据：
- 灵感1：2026.07.08 23:27
- 灵感2：2026.07.08 21:29
- 灵感3：2026.07.09 21:27

| 操作 | 列表顺序 | 当前日期显示 | 期望日期显示 |
|---|---|---|---|
| 置顶灵感1 | 灵感1-灵感3-灵感2 | 07.08-07.09-07.08 | 07.08-07.09-07.08 |
| 再置顶灵感3 | 灵感3-灵感1-灵感2 | 07.09-无-07.08 | 07.09-07.08-无 |
| 再置顶灵感2 | 灵感3-灵感1-灵感2 | 07.09-无-无 | 07.09-07.08-无 |

### 1.2 根因定位

代码链路：

| 层级 | 文件 | 关键逻辑 |
|---|---|---|
| DAO | `app/src/main/java/com/corgimemo/app/data/local/db/InspirationDao.kt:65` | `ORDER BY isPinned DESC, createdAt DESC` |
| ViewModel | `app/src/main/java/com/corgimemo/app/viewmodel/InspirationViewModel.kt:58-78` | 拆为 `pinnedInspirations` 与 `normalGroupedInspirations` 两个流 |
| UI | `app/src/main/java/com/corgimemo/app/ui/screens/inspiration/InspirationScreen.kt:130` | `val showDate = pinnedInspirations.indexOf(inspiration) == 0` |

**核心 Bug**：置顶区域硬编码只有第一条灵感显示日期，其余全部隐藏，导致：
- 置顶区域非首条灵感即使日期与前一条不同，也不显示日期
- 置顶区域最后一条与普通区域第一条若日期相同，会重复显示

## 二、修复规则

> **整个灵感列表按显示顺序遍历（置顶按 createdAt 倒序在前 → 非置顶按 createdAt 倒序在后），相邻相同日期只显示一次：**
> - 第 1 条灵感：始终显示日期
> - 后续每条：仅当与前一条日期（年月日）不同时才显示日期

| 场景 | 列表顺序 | 当前显示 | 期望显示 | 新规则结果 |
|---|---|---|---|---|
| 置顶灵感1 | 灵感1(07.08)-灵感3(07.09)-灵感2(07.08) | 07.08-07.09-07.08 | 07.08-07.09-07.08 | ✓ |
| 置顶灵感3 | 灵感3(07.09)-灵感1(07.08)-灵感2(07.08) | 07.09-无-07.08 | 07.09-07.08-无 | ✓ |
| 置顶灵感2 | 灵感3(07.09)-灵感1(07.08)-灵感2(07.08) | 07.09-无-无 | 07.09-07.08-无 | ✓ |

| 跨年月边界 | 灵感X(06.30,置顶)-灵感Y(07.01) | 06.30-07.01 | 06.30-07.01 | ✓ |
| 连续3条同日 | A(07.09)-B(07.09)-C(07.09) | A显 B/C隐 | A显 B/C隐 | ✓ |

## 三、实现方案

### 3.1 方案对比

| 方案 | 思路 | 优缺点 |
|---|---|---|
| **方案1：ViewModel 预处理统一显示列表**（选定） | 新增 `displayInspirations: StateFlow<List<InspirationDisplayItem>>`，UI 简化为单一 LazyColumn | 优：逻辑集中可测试、UI 简化、统一处理边界；缺：需小幅重构 UI |
| 方案2：UI 层跨区域协调 | 保留两个流，UI 中扁平化后计算 showDate | 优：ViewModel 改动小；缺：UI 逻辑复杂不易测试 |
| 方案3：仅修补置顶区域 | 改第130行为相邻比较 | 优：改动最小；缺：不修复边界重复，遗留技术债 |

### 3.2 数据结构

新建数据类作为 UI 渲染的统一数据模型：

```kotlin
data class InspirationDisplayItem(
    val inspiration: Inspiration,
    val showDate: Boolean,      // 是否显示左侧日期列
    val isPinned: Boolean       // 是否为置顶项
)
```

### 3.3 ViewModel 改造

在 `InspirationViewModel` 中新增统一显示流，**保留** 现有 `pinnedInspirations`、`normalGroupedInspirations` 流（避免影响其他调用方）：

```kotlin
val displayInspirations: StateFlow<List<InspirationDisplayItem>> =
    _inspirations.map { list -> buildDisplayItems(list) }
        .stateIn(
            scope = viewModelScope,
            started = SharingStarted.WhileSubscribed(5000),
            initialValue = emptyList()
        )

/**
 * 构建统一显示列表：相邻相同日期只显示一次
 */
private fun buildDisplayItems(list: List<Inspiration>): List<InspirationDisplayItem> {
    if (list.isEmpty()) return emptyList()
    return list.mapIndexed { index, inspiration ->
        val showDate = if (index == 0) {
            true
        } else {
            !isSameDay(list[index - 1].createdAt, inspiration.createdAt)
        }
        InspirationDisplayItem(
            inspiration = inspiration,
            showDate = showDate,
            isPinned = inspiration.isPinned
        )
    }
}

/** 判断两个时间戳是否同一天 */
private fun isSameDay(t1: Long, t2: Long): Boolean {
    val c1 = Calendar.getInstance().apply { timeInMillis = t1 }
    val c2 = Calendar.getInstance().apply { timeInMillis = t2 }
    return c1.get(Calendar.YEAR) == c2.get(Calendar.YEAR) &&
           c1.get(Calendar.MONTH) == c2.get(Calendar.MONTH) &&
           c1.get(Calendar.DAY_OF_MONTH) == c2.get(Calendar.DAY_OF_MONTH)
}
```

### 3.4 UI 改造

`InspirationScreen.kt` 的 LazyColumn 部分简化为单一循环：

```kotlin
val displayItems by viewModel.displayInspirations.collectAsState()

// 空状态判断改为 displayItems.isEmpty()
// LazyColumn 内部：
items(
    items = displayItems,
    key = { "inspiration_${it.inspiration.id}" }
) { item ->
    val inspiration = item.inspiration
    val tags = viewModel.decodeTags(inspiration.tags)
    val imagePaths = viewModel.decodePaths(inspiration.imagePaths)
    val formattedTime = viewModel.formatTime(inspiration.createdAt)

    TimelineInspirationItem(
        inspiration = inspiration,
        tags = tags,
        imagePaths = imagePaths,
        formattedTime = formattedTime,
        showDate = item.showDate,
        isPinnedItem = item.isPinned,
        onClick = { navController.navigate("inspiration_edit/${inspiration.id}") },
        onLongClick = {
            longPressedInspiration = inspiration
            showLongPressSheet = true
        }
    )
}
```

## 五、影响范围与回归检查

### 5.1 修改文件

- `app/src/main/java/com/corgimemo/app/viewmodel/InspirationViewModel.kt`：新增 `InspirationDisplayItem` 数据类、`displayInspirations` 流、`buildDisplayItems` 与 `isSameDay` 辅助方法

### 5.2 不修改

- `TimelineInspirationItem.kt`：组件本身正确，只是被错误传参
- DAO / Entity / Repository：排序逻辑正确

### 5.3 回归检查点

- 长按操作面板：仍通过 `longPressedInspiration` 传递 Inspiration 实体，不受影响
- 置顶图标显示：通过 `item.isPinned` 传递，等价于原 `isPinnedItem`
- 节点颜色：置顶橙色 / 普通主题色，通过 `isPinnedItem` 控制，不变
- 搜索状态：搜索时 `_inspirations` 被替换，`displayInspirations` 自动重算，规则一致
- 数据初始化闪烁：`isDataInitialized` 状态保留，逻辑不变
- 时间线竖线连续性：`LazyColumn.verticalArrangement = spacedBy(18.dp)` 不变，竖线延伸 18dp 逻辑不受影响

## 六、后续可优化方向

1. **删除冗余流**：确认 `pinnedInspirations`、`normalGroupedInspirations` 无其他调用方后，可删除以减少状态冗余
2. **扩展显示项**：未来若需添加日期分组 Header（如"今天"、"昨天"），可在 `InspirationDisplayItem` 之上扩展为 sealed class
3. **性能**：当前 `buildDisplayItems` 在每次数据变化时全量重算，列表规模通常较小（<1000），无需优化；若未来扩展到大列表，可考虑增量计算
