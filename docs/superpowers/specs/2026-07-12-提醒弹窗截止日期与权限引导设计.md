# 提醒弹窗截止日期集成与权限引导设计

> 日期：2026-07-12
> 状态：已确认

## 概述

本次设计包含三项改动：

1. 将"截止日期"设置集成到 ReminderPickerBottomSheet 弹窗中（提醒时间下一行）
2. 通知权限未授权时的引导流程（含永久拒绝处理）
3. 精确闹钟权限引导流程（Android 12+）

## 1. 截止日期集成到提醒弹窗

### 1.1 布局变更

**修改前**（ReminderPickerBottomSheet）：

```
标题
提醒时间行 [日期芯片] [时间芯片]
重复提醒行
农历行
──────── 分割线 ────────
日历/时间滚轮内容区
[取消] [确定]
```

**修改后**：

```
标题
提醒时间行 [日期芯片] [时间芯片] [×清除]    ← 右侧增加清除按钮
截止日期行 [日期芯片] [时间芯片] [×清除]    ← 新增行，紧接提醒时间行
重复提醒行
农历行
──────── 分割线 ────────
日历/时间滚轮内容区                        ← 共用，根据 editTarget 切换
[取消] [确定]
```

### 1.2 编辑目标状态

- 新增枚举 `EditTarget`：`REMINDER` | `DUE_DATE`，默认 `REMINDER`
- 点击**提醒时间行**的日期/时间芯片 → `editTarget = REMINDER`，滚轮控制提醒时间
- 点击**截止日期行**的日期/时间芯片 → `editTarget = DUE_DATE`，滚轮控制截止日期
- 当前编辑目标行的标签文字颜色变为主题色（橙色 `#FF9A5C`），非编辑目标行为灰色 `#999999`
- 当前编辑目标行的芯片背景使用主题色（选中态），非编辑目标行使用浅橙 `#FFE4CC`

### 1.3 截止日期行行为

- 默认显示"未设置"占位符，日期芯片文字为"未设置"，时间芯片隐藏
- 首次点击日期/时间芯片时，自动初始化为提醒时间同一天 23:59
- ×清除按钮：点击后清空截止日期，回到"未设置"状态
- 提醒时间行的×清除按钮：点击后清空提醒时间（等同原"删除提醒"操作）

### 1.4 滚轮内容区切换

日历/时间滚轮区域根据 `editTarget` 分别维护两组日期/时间状态：

- `reminderDate` / `reminderHour` / `reminderMinute`
- `dueDateDate` / `dueDateHour` / `dueDateMinute`

切换 `editTarget` 时，滚轮立即反映对应目标的时间值。

### 1.5 参数变更

**ReminderPickerBottomSheet 新增参数**：

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `initialDueDateMillis` | `Long?` | `null` | 初始截止日期时间戳 |
| `onConfirm` 回调变更 | - | - | 增加 `dueDateMillis: Long?` 参数 |

**onConfirm 新签名**：

```kotlin
onConfirm: (
    dateMillis: Long?,      // 提醒日期
    hour: Int,              // 提醒小时
    minute: Int,            // 提醒分钟
    repeatType: Int,        // 重复类型
    calendarEnabled: Boolean, // 农历开关
    dueDateMillis: Long?    // 新增：截止日期时间戳
) -> Unit
```

### 1.6 通知内容变更

`NotificationHelper.buildContentText()` 追加截止日期信息：

- 仅当 `todo.dueDate != null` 时追加
- 格式：`截止：yyyy/MM/dd HH:mm`
- 插入位置：优先级之后、内容摘要之前

修改后示例：`[工作] 中优先级 截止：2026/07/12 18:00 整理项目文档`

### 1.7 旧截止日期UI移除

- 移除 TodoEditScreen 中的 `showDueDatePicker` / `showDueTimePicker` 状态及对应的 `DatePickerDialog` / `TimePicker` 弹窗代码
- 移除编辑页工具栏中独立的"截止日期"按钮入口（如果存在）
- 截止日期统一由 ReminderPickerBottomSheet 的截止日期行管理

### 1.8 TodoEditScreen 回调变更

`onConfirm` 回调中新增 `dueDateMillis` 参数处理：

```kotlin
onConfirm = { dateMillis, hour, minute, repeatTypeNew, calendarEnabled, dueDateMillis ->
    val gid = editingReminderGroupId ?: return@ReminderPickerBottomSheet
    viewModel.setGroupReminder(gid, dateMillis ?: System.currentTimeMillis())
    viewModel.setGroupRepeatType(gid, repeatTypeNew)
    viewModel.setDueDate(dueDateMillis)
    editingReminderGroupId = null
    // ... 权限引导逻辑
}
```

## 2. 通知权限引导（Android 13+）

### 2.1 触发时机

用户在 ReminderPickerBottomSheet 点击"确定"后，检测 `POST_NOTIFICATIONS` 权限。

### 2.2 流程

```
点击确定 → 保存提醒时间 → 检查通知权限
  → 已授权：无操作
  → 未授权且可请求：弹出系统权限请求对话框
  → 未授权且永久拒绝：弹出 AlertDialog 引导去应用设置页
```

### 2.3 永久拒绝检测

使用 `shouldShowRequestPermissionRationale()` 判断：

- 返回 `true`：用户之前拒绝过但仍可请求 → 调用 `launcher.launch(POST_NOTIFICATIONS)`
- 返回 `false` 且权限未授予：永久拒绝 → 弹出引导对话框

### 2.4 永久拒绝引导对话框

```
标题：开启通知权限
内容：需要在系统设置中允许 CorgiMemo 发送通知，才能接收待办提醒。
按钮：[取消] [去设置]
```

点击"去设置"跳转 `Settings.ACTION_APPLICATION_DETAILS_SETTINGS`（data = `package:com.corgimemo.app`）。

### 2.5 本次会话防重复

使用 `remember` 保存 `hasRequestedNotificationPermission: Boolean` 标志，本次会话中仅引导一次，用户拒绝后不再重复弹出。

## 3. 精确闹钟权限引导（Android 12+）

### 3.1 触发时机

通知权限已授予（或无需授权）后，检测 `SCHEDULE_EXACT_ALARM` 权限。

### 3.2 注意事项

- Android 13+ 新安装应用默认拥有 `SCHEDULE_EXACT_ALARM` 权限
- 仅用户手动关闭后才需要引导
- Android 12 的 `SCHEDULE_EXACT_ALARM` 默认授予
- Android 14+ 引入 `USE_EXACT_ALARM`（闹钟类应用自动获得），本应用不在其列

### 3.3 引导对话框

```
标题：开启精确闹钟权限
内容：开启后可以确保提醒在设定时间准时响起，避免延迟。
按钮：[以后再说] [去设置]
```

点击"去设置"跳转 `Settings.ACTION_REQUEST_SCHEDULE_EXACT_ALARM`（Android 12+ API）。

### 3.4 降级逻辑

若用户拒绝引导，保持现有行为：`AlarmScheduler` 使用 `setAndAllowWhileIdle()` 替代 `setExactAndAllowWhileIdle()`，提醒可能延迟但仍会触发。

## 4. 权限引导时序

两个权限引导**串行执行**，避免同时弹出多个对话框：

```
确定 → 保存提醒 → 通知权限检查 → [引导通知权限] → 精确闹钟权限检查 → [引导精确闹钟权限]
```
