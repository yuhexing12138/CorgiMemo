# 三页面风格统一设计文档

> **日期**：2026-05-29
> **范围**：待办页、灵感页、特殊日期页的风格统一
> **方案**：MainScreen 统一容器模式

---

## 1. 背景与目标

### 1.1 当前问题

待办、灵感、特殊日期三个核心页面存在以下不一致：

| # | 差异类别 | 待办页 | 灵感页 | 日期页 |
|---|---------|--------|--------|--------|
| 1 | 顶部导航栏 | `EnhancedTopBar`（菜单+标题+下划线+统计+柯基） | 普通 `TopAppBar`（仅标题） | 普通 `TopAppBar`（标题+搜索切换） |
| 2 | 侧滑导航栏 | ✅ 有 `ModalNavigationDrawer` | ❌ 无 | ❌ 无 |
| 3 | 搜索栏 | 共享 `SearchBar`（暖橙色背景） | 共享 `SearchBar` | 内嵌 `OutlinedTextField` |
| 4 | FAB按钮 | `AnimatedFAB`（铅笔图标） | 标准 `FloatingActionButton`（+图标） | 标准 `FloatingActionButton`（+图标） |
| 5 | 标题字号 | 18sp Bold | 24sp Bold | 24sp Bold |
| 6 | 内容边距 | 搜索栏 16dp 水平 | 搜索栏 20dp 水平 | 列表 20dp 水平 |
| 7 | 空状态 | `EmptyState`（不同风格） | `InspirationEmptyState`（渐变CTA） | `SpecialDateEmptyState`（渐变CTA，动画/尺寸不同） |
| 8 | 悬浮柯基 | ✅ 有 | ❌ 无 | ❌ 无 |

### 1.2 目标

- 保证现有功能不变
- 统一三个页面的视觉风格（文字间距、文字大小、配色）
- 相似功能按钮位置统一
- 三个页面都有顶部导航栏、底部导航栏（含中央⊕按钮）、侧滑导航栏
- 悬浮柯基按钮在所有页面显示

---

## 2. 整体架构

### 2.1 MainScreen 统一容器

采用方案A（MainScreen统一容器模式），将共享组件提升到 MainScreen 容器层：

```
┌─────────────────────────────────────────────────────┐
│  ModalNavigationDrawer                               │
│  ┌───────────────────────────────────────────────┐  │
│  │  AppDrawerContent (根据 selectedTab 动态切换)   │  │
│  └───────────────────────────────────────────────┘  │
│                                                      │
│  ┌───────────────────────────────────────────────┐  │
│  │  EnhancedTopBar                                │  │
│  │  ☰菜单 │ {title} │ {actions} 🐕               │  │
│  │  ════════════════════════ (暖橙色下划线)        │  │
│  ├───────────────────────────────────────────────┤  │
│  │  [子页面内容区域]                               │  │
│  │  搜索栏 + 列表 + 空状态 + FAB                   │  │
│  │                                               │  │
│  │  🐕 (悬浮柯基按钮)                              │  │
│  ├───────────────────────────────────────────────┤  │
│  │  📝待办  💡灵感  ⊕  📅日期  👤我的             │  │
│  └───────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────┘
```

### 2.2 组件位置变更

| 组件 | 当前位置 | 新位置 | 说明 |
|------|---------|--------|------|
| `ModalNavigationDrawer` | HomeScreen 内部 | **MainScreen** | 统一侧滑栏 |
| `EnhancedTopBar` | HomeScreen 内部 | **MainScreen** | 统一顶部栏 |
| `FloatingCorgiButton` | HomeScreen 内部 | **MainScreen** | 全局悬浮 |
| `CorgiBottomNavigationBar` | MainScreen | MainScreen（不变） | 底部导航 |
| `CenterEditButton` | MainScreen | MainScreen（不变） | 中央编辑 |
| `BubbleMenuOverlay` | MainScreen | MainScreen（不变） | 气泡菜单 |
| FAB | 各子页面 | 各子页面（统一风格） | 保留但统一 |
| SearchBar | 各子页面 | 各子页面（统一组件） | 保留但统一 |

### 2.3 子页面职责变化

| 子页面 | 当前职责 | 新职责 |
|--------|---------|--------|
| HomeScreen | Scaffold + TopBar + 侧滑栏 + 搜索 + 列表 + FAB + 悬浮柯基 | **搜索 + 过滤器 + 列表 + 空状态 + FAB** |
| InspirationScreen | Scaffold + TopBar + 搜索 + 列表 + FAB | **搜索 + 列表 + 空状态 + FAB** |
| SpecialDateScreen | Scaffold + TopBar + 内嵌搜索 + 列表 + FAB | **搜索 + 列表 + 空状态 + FAB** |

## 3. 统一顶部导航栏

### 3.1 EnhancedTopBar 改造

升级为可配置的统一组件，三个页面共用：

```
┌─────────────────────────────────────────────────┐
│  ☰菜单 │ {title} │ {action1} {action2} 🐕      │
│         ═══════════════════════════              │  ← 暖橙色下划线
└─────────────────────────────────────────────────┘
```

### 3.2 各页面配置

| 页面 | title | 左侧按钮 | 右侧按钮 | 下划线 |
|------|-------|---------|---------|--------|
| **待办** | "📝 我的待办" | ☰ 打开侧滑栏 | 📊 统计 + 🐕 柯基 | ✅ 暖橙色 |
| **灵感** | "💡 我的灵感" | ☰ 打开侧滑栏 | 🏷️ 标签管理 + 🐕 柯基 | ✅ 暖橙色 |
| **日期** | "📅 特殊日期" | ☰ 打开侧滑栏 | 📊 排序 + 🐕 柯基 | ✅ 暖橙色 |
| **我的** | "👤 我的" | ☰ 打开侧滑栏 | 🐕 柯基 | ✅ 暖橙色 |

### 3.3 统一规范

| 属性 | 规范值 | 说明 |
|------|--------|------|
| 标题字号 | **18sp Bold** | 统一为18sp |
| 标题颜色 | `MaterialTheme.colorScheme.onSurface` | 跟随主题 |
| 下划线高度 | 3dp | 暖橙色渐变 |
| 下划线动画 | 600ms EaseOutQuart | 从左到右展开 |
| 菜单按钮 | `Icons.Default.Menu` | 统一打开侧滑栏 |
| 柯基按钮 | `Icons.Default.Pets` | 统一跳转柯基详情页 |
| 右侧功能按钮 | 可配置 `List<@Composable () -> Unit>` | 各页面自定义 |

### 3.4 新接口

```kotlin
@Composable
fun EnhancedTopBar(
    title: String,
    onMenuClick: () -> Unit,
    onCorgiClick: () -> Unit,
    actionButtons: List<@Composable () -> Unit> = emptyList(),
    modifier: Modifier = Modifier
)
```

## 4. 统一侧滑导航栏

### 4.1 侧滑栏内容动态切换

侧滑栏在 MainScreen 统一管理，内容根据 `selectedTab` 动态切换：

**待办页** — 分组管理：
```
📂 分组管理
══════════════
📋 全部待办 (12)
📁 工作 (5)
📚 学习 (2)
🏠 生活 (4)
＋ 添加分组
```

**灵感页** — 标签筛选：
```
🏷️ 标签筛选
══════════════
📋 全部灵感 (8)
#产品 (3)
#设计 (2)
#生活 (3)
＋ 添加标签
```

**日期页** — 类型筛选：
```
📅 类型筛选
══════════════
📋 全部日期 (6)
⏳ 倒计时 (3)
⏱️ 正计时 (2)
📌 已过期 (1)
```

**我的页** — 快捷导航：
```
🔗 快捷导航
══════════════
📊 统计
🏆 成就
⚙️ 设置
```

**底部操作区（所有页面共享）**：
```
🗑️ 最近删除 (2)
⚙️ 设置  ❓ 帮助与反馈
```

### 4.2 AppDrawerContent 接口改造

```kotlin
@Composable
fun AppDrawerContent(
    currentTab: TabItem,
    corgiData: CorgiData?,
    onCategoryClick: (Long?) -> Unit,

    onDateTypeClick: (GroupType?) -> Unit,
    onRecentlyDeletedClick: () -> Unit,

    onHelpClick: () -> Unit,
)
```

### 4.3 统一规范

| 属性 | 规范值 |
|------|--------|
| 抽屉宽度 | 280dp |
| 用户区域 | 所有页面共享，显示柯基头像+名称+等级 |
| 分组/筛选区域 | 根据当前Tab动态切换内容 |
| 底部操作区 | 所有页面共享：最近删除+设置+帮助 |
| 分组项高度 | 56dp |
| 选中态 | 主色浅色背景 + 主色文字 |
| 分割线 | 暖橙色下划线（标题下方） |

### 5.1 文字规范

| 用途 | 规范字号 | 规范字重 | 变更说明 |
|------|---------|---------|---------|

| 分组标题 | **16sp** | SemiBold | 统一 |
| 卡片标题 | **16sp** | Medium | 不变 |

| 空状态主文案 | **17sp** | Medium | 日期页从18sp Bold改为17sp Medium |
| 空状态副文案 | **14sp** | Regular | 不变 |
| CTA按钮文字 | **16sp** | SemiBold | 不变 |

### 5.2 间距规范

| 场景 | 规范值 | 变更说明 |
|------|--------|---------|
| 页面水平边距 | **20dp** | 待办页搜索栏从16dp改为20dp |
| 搜索栏水平边距 | **20dp** | 待办页从16dp改为20dp |
| 搜索栏垂直边距 | **8dp** | 不变 |
| 卡片间距 | **12dp** | 不变 |
| 卡片内边距 | **16dp** | 不变 |
| 区域间距 | **24dp** | 统一 |
| 列表项间距 | **12dp** | 不变 |

### 5.3 配色规范

| 元素 | 规范值 | 变更说明 |
|------|--------|---------|
| 页面背景 | `MaterialTheme.colorScheme.background` | 统一使用主题色 |
| 卡片背景 | `MaterialTheme.colorScheme.surface` | 统一 |
| 主文字 | `MaterialTheme.colorScheme.onSurface` | 空状态硬编码#2D2D2D改为主题色 |
| 次要文字 | `MaterialTheme.colorScheme.onSurfaceVariant` | 空状态硬编码#999999改为主题色 |
| 搜索栏背景 | `UiColors.SearchBackground` | 日期页统一使用共享SearchBar |
| CTA按钮渐变 | `UiColors.Primary` → 浅一号主色 | 灵感/日期页渐变统一 |
| 空状态图标背景 | 统一为渐变圆形容器 | 灵感页添加背景 |

### 6.2 悬浮柯基按钮全局化

将 `FloatingCorgiButton` 从 HomeScreen 提升到 MainScreen 层级，使其在所有页面可见。

**实现方式**：
- 在 MainScreen 的 Box 容器中添加 `FloatingCorgiButton`（与底部导航栏同级）
- 切换页面时柯基按钮不重建，保持位置和动画状态
- 非底部导航页面（设置页、统计页等）通过 NavHost 外层包裹

| 属性 | 规范值 |
|------|--------|
| 尺寸 | 48×48dp |
| 初始位置 | 右下角，距底部88dp，距右侧16dp |
| 呼吸动画 | 缓慢放大缩小，2s循环 |
| 点击 | 进入柯基详情页 |
| 长按 | 进入拖动模式 |
| 贴边吸附 | 释放后自动贴边 |
| 位置记忆 | DataStore 保存百分比坐标 |
