# 日期页面引导指引设计

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

## 目标

为 CorgiMemo（刻记⁺）首次启动引导流程的 **Step 7（index 6）`DateFeaturePage`** 添加实际可交互的"创建首个日期"功能，替换当前的占位实现。

参考 Step 5（TodoFeaturePage）和 Step 6（InspirationFeaturePage）的实现模式，让用户通过实际操作体验日期功能的创建流程。

## 背景

在 2026-07-13 完成的首启引导重新设计中，`DateFeaturePage` 由于当时日期功能不完整而作为**纯占位页**实现，仅显示 SLEEP 动画 + "敬请期待" 标签。

经过近期迭代，日期功能（卡片样式 / 颜色选择 / 提醒 / 倒计时）已基本完善，**用户希望补齐该引导步骤**，使其与 TodoFeaturePage / InspirationFeaturePage 风格一致。

## 需求与设计决策汇总

通过 brainstorming 流程收集的关键决策：

| 决策点 | 选择 | 备注 |
|--------|------|------|
| 交互范围 | **A** - 完整创建（标题 + 日期 + 类别） | 与 Todo/Inspiration 一致 |
| 类别数量 | **4 个**（BIRTHDAY/ANNIVERSARY/HOLIDAY/OTHER） | DateCategory 实际有 8 个值，引导仅展示最常见的 3 个 + OTHER |
| 卡片样式/颜色 | **A** - 不选择，使用默认 | cardStyle=ORANGE_TEAR_OFF, cardColor=DEFAULT |
| 默认日期 | **今天** | 允许选择过去日期（用于记录已过的生日/纪念日） |
| 成功态 | **C** - PROUD 动画 + 卡片预览组合 | 上方 PROUD 动画，下方迷你倒计时卡片 |
| 跳过选项 | **A** - 可跳过 | 与 TodoFeaturePage 一致 |
| 默认类别 | **BIRTHDAY** | 最常见用例 |
| 架构方案 | **A** - 沿用现有模式 | 最小侵入，参考 TodoFeaturePage |

### 状态管理

**OnboardingViewModel 新增**：
- `createdDateCount: StateFlow<Int>` — 已创建日期计数
- `createFirstDate(title: String, targetDate: Long, category: String)` — 创建首个日期
  - 内部调用 `specialDateRepository.insert(SpecialDate(...))`
  - 成功后 `_createdDateCount.value++`
  - 失败静默处理（与 createFirstTodo 一致）

**DateFeaturePage 内部状态**（与 TodoFeaturePage 风格一致）：
- `dateTitle: String` — 标题（默认空）
- `selectedDateMillis: Long?` — 日期时间戳（默认今天）
- `selectedCategory: DateCategory` — 类别（默认 BIRTHDAY）
- `showDatePicker: Boolean` — 日期选择器显示
- `showSuccessAnimation: Boolean` — 成功态切换

**注意**：参考 TodoFeaturePage 的"不持有日期格式"思路，`selectedDateMillis` 存原始时间戳（毫秒），UI 显示时用 `SimpleDateFormat` 格式化。

### 倒计时天数计算

```kotlin
// 与 SpecialDateCard / SpecialDateScreen 保持一致：
// 倒计时天数 = (目标日期 - 今天) / (24 * 60 * 60 * 1000)
val daysUntil = ((selectedDateMillis - today) / DAY_IN_MILLIS).toInt()
```

未来日期显示正数（如"还有 32 天"），过去日期显示负数（如"已过 7 天"）。

### DatePickerDialog 选型

| 选项 | 优缺点 |
|------|--------|
| **A. Material3 DatePickerDialog** | 官方组件，UI 现代化，与项目风格一致；需要 ExperimentalMaterial3Api |
| **B. 现有 `DatePickerRow` / `DateWheelPicker` 组件** | 项目已有的滚动选择器；样式较老，与 Material3 风格有差异 |
| **C. 自定义简单 DatePicker** | 灵活但工作量大 |

**选择 A** - Material3 DatePickerDialog，与 PermissionPage 等保持一致风格。

### 卡片预览内容

迷你卡片包含：
- **倒计时天数**（大字号 32sp，主题色）
- **目标日期**（"距离 2026/08/15" 格式）
- **标题**（用户输入）
- **类别 emoji + 名称**（如"🎂 生日"）

**不包含**（避免视觉过载）：
- 实际倒计时数（天数已足够示意）
- 卡片样式/颜色（已确认使用默认）
- 操作按钮（仅展示，不交互）

```kotlin
fun DateFeaturePage(
    viewModel: OnboardingViewModel
```

### OnboardingViewModel 新增方法

```kotlin
/** 引导中创建的日期计数 */
private val _createdDateCount = MutableStateFlow(0)
val createdDateCount: StateFlow<Int> = _createdDateCount.asStateFlow()

/**
 * 创建首个日期
 * 在引导流程中创建并保存到 Room 数据库
 *
 * @param title 日期标题
 * @param targetDate 目标日期时间戳（毫秒）
 * @param category 类别名称（BIRTHDAY / ANNIVERSARY / HOLIDAY / OTHER）
 */
fun createFirstDate(title: String, targetDate: Long, category: String) {
    if (title.isBlank()) return

    viewModelScope.launch {
        try {
            val now = System.currentTimeMillis()
            val date = SpecialDate(
                title = title,
                targetDate = targetDate,
                category = category,
                countMode = 0,       // 倒计时
                repeatType = 0,      // 不重复
                reminderDays = 0,    // 不提醒
                isPinned = false,
                isArchived = false,
                cardStyle = "ORANGE_TEAR_OFF",  // 默认样式
                cardColor = "DEFAULT",           // 默认颜色
                createdAt = now,
                updatedAt = now
            )
            specialDateRepository.insert(date)
            _createdDateCount.value = _createdDateCount.value + 1
        } catch (e: Exception) {
            // 创建失败静默处理
        }
    }
}
```

### 辅助函数

以下辅助函数需要在新文件中定义（或使用内联表达式）：

```kotlin
/** 一天的毫秒数 */
private const val DAY_IN_MILLIS = 24 * 60 * 60 * 1000L

/** 获取今天 0 点的时间戳（毫秒） */
private fun todayStartMillis(): Long {
    val calendar = Calendar.getInstance()
    calendar.set(Calendar.HOUR_OF_DAY, 0)
    calendar.set(Calendar.MINUTE, 0)
    calendar.set(Calendar.SECOND, 0)
    calendar.set(Calendar.MILLISECOND, 0)
    return calendar.timeInMillis
}

/** 格式化日期为 "2026/08/15 周六" 格式 */
private fun formatDate(millis: Long): String {
    val dateFormat = SimpleDateFormat("yyyy/MM/dd EEE", Locale.CHINA)
    return dateFormat.format(Date(millis))
}

/** 计算从今天到目标日期的天数差 */
private fun daysUntil(targetMillis: Long): Int {
    val today = todayStartMillis()
    return ((targetMillis - today) / DAY_IN_MILLIS).toInt()
}
```

**注意**：
- `FeatureListItem` 可直接用 `Text(emoji + " 文字", style = bodyMedium)` 内联实现，无需封装
- `formatDate` 使用 `EEE` 格式（Locale.CHINA）会输出"周一/周二/.../周日"
- `todayStartMillis` 使用 0 点对齐，确保天数差为整数

## 数据流

### 用户流程

```
进入 Step 7 DateFeaturePage
    ↓
看到上半部分功能介绍（5 卖点）
    ↓
看到下半部分表单
    ├── 标题：默认空
    ├── 日期：默认今天
    └── 类别：默认 BIRTHDAY
    ↓
用户输入标题（必填）
    ↓ (可选) 点击日期 → DatePickerDialog
    ↓ (可选) 点击类别 Chip
    ↓
点击"创建日期"按钮
    ├── 标题为空：按钮禁用
    └── 标题非空：
        ├── 调用 viewModel.createFirstDate(...)
        ├── Repository 写入 Room special_dates 表
        ├── 切换为成功态（PROUD 动画 + 卡片预览）
        └── _createdDateCount++
    ↓
用户点击"下一步"（OnboardingScreen 控制栏）
    ↓
进入 Step 8 CorgiSystemPage
```

### 跳过流程

```
进入 Step 7 DateFeaturePage
    ↓
直接点击"下一步"（OnboardingScreen 控制栏）
    ↓
进入 Step 8 CorgiSystemPage
    （未创建任何日期）
```

## 错误处理

| 场景 | 处理 |
|------|------|
| 标题为空 | 创建按钮禁用（`enabled = dateTitle.isNotBlank()`） |
| 日期未选 | 默认选中今天，理论上不存在未选情况 |
| Room 写入失败 | `try-catch` 静默处理（与 createFirstTodo 一致） |
| 用户输入超长标题 | OutlinedTextField 默认无 maxLength，参考 TodoFeaturePage 行为（不限制） |
