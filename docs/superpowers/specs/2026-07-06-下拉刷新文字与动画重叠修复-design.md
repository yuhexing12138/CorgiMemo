## 1. 背景与目标

### 1.1 现状

当前待办页使用 [CorgiPullToRefreshIndicator.kt](file:///c:/Users/EDY/Desktop/CorgiMemo/app/src/main/java/com/corgimemo/app/ui/components/CorgiPullToRefreshIndicator.kt) 渲染下拉刷新提示。该组件使用 `Box(contentAlignment=Center, height=pullOffsetDp)` 作为外层容器，柯基居中（默认 `Modifier.align(Center)`），提示文字底部对齐（`Modifier.align(BottomCenter)`）。

参考截图（用户提供）：「下拉刷新/释放刷新」文字与柯基奔跑动画发生重叠；释放后「柯基努力加载中~」文字也与柯基动画重叠。

### 1.2 根因（定量分析）

| 参数 | 值 |
|---|---|
| 柯基最小尺寸 | 48dp |
| 柯基最大尺寸 | 64dp |
| 提示文字字号 | 12sp（≈ 16dp） |
| 文字颜色 | onSurfaceVariant α=0.6 |
| maxPullHeight | 100dp |
| refreshThreshold | 60dp |

**冲突场景**：当 `pullOffset = 60dp`（阈值）时，柯基尺寸 = 48 + (64-48)×0.6 = 57.6dp。Box(contentAlignment=Center) 中柯基垂直居中，柯基中心在 30dp，柯基底部 = 30 + 28.8 = **58.8dp**。文字在 Box 底部，文字顶部 ≈ 44dp。**重叠约 14.8dp**。

### 1.3 目标

- 下拉中、释放时、加载中三个阶段，文字与柯基动画**清晰分层、无重叠**
- 极小下拉量时不出现裁切/溢出
- 保持现有的「下拉刷新 / 释放刷新 / 柯基努力加载中~」三段文字与柯基奔跑动画
- 不改变行为约束（柯基始终 RUN、达阈值释放触发刷新等）
- 不改变 HomeScreen 调用方式（接口兼容）

### 1.4 已确定的设计方向（经 3 轮澄清）

| 决策点 | 选择 |
|---|---|
| 文字处置 | 保留三段文字 + 优化布局 |
| 文字位置 | 文字在柯基上方 |
| 小下拉处理 | 设置最小空白高度后才渲染柯基和文字 |
| 最小空白高度 | 72dp |
| 跨越阈值处理 | 文字 + 柯基用 alpha 淡入淡出 |

## 2. 方案选型

### 2.1 候选方案

| 方案 | 文字位置 | 核心思路 | 优势 | 劣势 |
|---|---|---|---|---|
| **方案 A（采用）** | 文字在上、柯基在下 | Column 居中布局 + 固定 4dp 间距 + 早返 8dp + 最小高度 72dp + alpha 渐变 | 分层清晰，符合阅读顺序「先读文字再看图」 | 需引入 Column 嵌套 |
| 方案 B | 文字在下、柯基在上 | 反转 Column 顺序，强制底部 | 柯基视觉更突出 | 阅读顺序不自然，柯基仍可能被文字「盖住」 |
| 方案 C | 柯基 + 文字整体居中 | 把两者视为一个组合居中 | 实现简单 | 小空白区时整体被裁切 |
| 方案 D | 柯基 + 文字水平并排 | Row 水平布局 | 绝不重叠 | 占用水平空间，窄屏文字易截断 |

### 2.2 选型结论

**采用方案 A**。理由：
- 文字在上符合「先读提示再看动画」的视觉阅读顺序
- Column 强制分层 + 固定间距，从根本上消除重叠
- 早返 + 最小高度解决小下拉量裁切问题
- alpha 渐变避免硬切违和感
- 接口完全兼容，无需修改 HomeScreen

### 3.2 核心纯函数

```kotlin
/**
 * 下拉刷新指示器布局计算结果
 *
 * @property displayHeightDp 空白区展示高度（已应用最小 72dp）
 * @property corgiSizeDp 柯基尺寸（48~64dp）
 * @property contentAlpha 文字+柯基整体透明度（0~1）
 * @property shouldRender 是否进入渲染分支（IDLE 且无下拉时为 false）
 * @property tipText 提示文字（null = 不渲染文字）
 */
data class IndicatorLayout(
    val displayHeightDp: Float,
    val corgiSizeDp: Float,
    val contentAlpha: Float,
    val shouldRender: Boolean,
    val tipText: String?
)

/**
 * 计算下拉刷新指示器布局
 *
 * 单位约定：所有几何参数均以 dp 为单位，调用方负责从 px 转换。
 *
 * @param pullOffsetDp 当前下拉偏移（已阻尼，单位 dp）
 * @param maxPullHeightDp 最大下拉高度（dp）
 * @param refreshThresholdDp 刷新阈值（dp）
 * @param state 当前下拉刷新状态
 * @param minEmptyHeightDp 最小空白高度（dp，默认 72）
 * @param fadeInStartDp 文字+柯基开始淡入的 pullOffset 阈值（dp，默认 8）
 * @param fadeInRangeDp 淡入完成区间（dp，默认 8）
 * @param corgiSizeMinDp 柯基最小尺寸（dp，默认 48）
 * @param corgiSizeMaxDp 柯基最大尺寸（dp，默认 64）
 */
fun computePullRefreshIndicatorLayout(
    pullOffsetDp: Float,
    maxPullHeightDp: Float,
    refreshThresholdDp: Float,
    state: PullRefreshState,
    minEmptyHeightDp: Float = 72f,
    fadeInStartDp: Float = 8f,
    fadeInRangeDp: Float = 8f,
    corgiSizeMinDp: Float = 48f,
    corgiSizeMaxDp: Float = 64f
): IndicatorLayout {
    // 1. IDLE 且无下拉 → 不渲染
    if (state == PullRefreshState.IDLE && pullOffsetDp <= 0f) {
        return IndicatorLayout(0f, corgiSizeMinDp, 0f, false, null)
    }

    // 2. 空白区展示高度：取 pullOffset 与最小高度的较大值
    val displayHeightDp = maxOf(pullOffsetDp, minEmptyHeightDp)

    // 3. 柯基尺寸：48~64dp 渐变
    val pullProgress = (pullOffsetDp / maxPullHeightDp).coerceIn(0f, 1f)
    val corgiSizeDp = corgiSizeMinDp + (corgiSizeMaxDp - corgiSizeMinDp) * pullProgress

    // 4. 文字 + 柯基 alpha
    val contentAlpha = if (state == PullRefreshState.REFRESHING) {
        1f
    } else {
        ((pullOffsetDp - fadeInStartDp) / fadeInRangeDp).coerceIn(0f, 1f)
    }

    // 5. 提示文字（反映该状态下应有的文字，与 alpha 独立）
    val tipText: String? = when {
        state == PullRefreshState.REFRESHING -> "柯基努力加载中~"
        state == PullRefreshState.PULLING && pullOffsetDp >= refreshThresholdDp -> "释放刷新"
        state == PullRefreshState.PULLING -> "下拉刷新"
        state == PullRefreshState.RELEASING -> "下拉刷新"
        state == PullRefreshState.IDLE -> null
        else -> null
    }

    return IndicatorLayout(
        displayHeightDp = displayHeightDp,
        corgiSizeDp = corgiSizeDp,
        contentAlpha = contentAlpha,
        shouldRender = true,
        tipText = tipText
    )
}
```

### 3.3 组件结构（CorgiPullToRefreshIndicator）

```kotlin
@Composable
fun CorgiPullRefreshIndicator(
    pullOffset: Float,
    state: PullRefreshState,
    maxPullHeightPx: Float,
    refreshThresholdPx: Float,
    modifier: Modifier = Modifier
) {
    val density = LocalDensity.current

    // 1. px → dp 转换（在调用纯函数前完成单位转换）
    val pullOffsetDp = with(density) { pullOffset.toDp().value }
    val maxPullHeightDp = with(density) { maxPullHeightPx.toDp().value }
    val refreshThresholdDp = with(density) { refreshThresholdPx.toDp().value }

    // 2. 调用纯函数计算布局
    val layout = remember(pullOffsetDp, state, maxPullHeightDp, refreshThresholdDp) {
        computePullRefreshIndicatorLayout(
            pullOffsetDp = pullOffsetDp,
            maxPullHeightDp = maxPullHeightDp,
            refreshThresholdDp = refreshThresholdDp,
            state = state
        )
    }

    if (!layout.shouldRender) return

    // 3. dp → Dp 转换
    val displayHeight = layout.displayHeightDp.dp
    val corgiSize = layout.corgiSizeDp.dp

    Column(
        modifier = modifier
            .fillMaxWidth()
            .height(displayHeight)
            .graphicsLayer { alpha = layout.contentAlpha },
        verticalArrangement = Arrangement.Center,
        horizontalAlignment = Alignment.CenterHorizontally
    ) {
        // 4. 提示文字（位于柯基上方）
        if (layout.tipText != null) {
            Text(
                text = layout.tipText!!,
                fontSize = 12.sp,
                color = MaterialTheme.colorScheme.onSurfaceVariant.copy(alpha = 0.6f)
            )
            Spacer(Modifier.height(4.dp))
        }

        // 5. 柯基动画
        val frames = remember(AnimationType.RUN) {
            AnimationResourceManager.getAnimationFrames(AnimationType.RUN)
        }
        if (frames.isNotEmpty()) {
            FrameAnimation(
                frames = frames,
                fps = 8,
                isLooping = true,
                isPlaying = true,
                modifier = Modifier.size(corgiSize)
            )
        }
    }
}
```

| 项目 | 公式 | 值（默认值） |

| 文字与柯基间距 | 固定 `4.dp` | 4dp |
| 柯基尺寸 | `48 + (64-48) × (pullOffset / maxPullHeight)` | 48~64dp |
| 淡入起点 | `fadeInStartDp` | 8dp |
| 淡入完成 | `fadeInStartDp + fadeInRangeDp` | 16dp |

## 4. 状态映射与边界情况

### 4.1 各状态渲染表

| PullRefreshState | pullOffset 范围 | 渲染内容 | 空白区高度 | contentAlpha | tipText |
|---|---|---|---|---|---|
| IDLE | 0 | 不渲染 | 0 | - | - |
| PULLING（< 8dp） | 0 < x < 8 | 空白区 + 文字 + 柯基（alpha=0 不可见） | 72dp | 0 | "下拉刷新" |
| PULLING（< 阈值） | 8 ≤ x < 60 | 文字「下拉刷新」+ 柯基 | 72dp | (x-8)/8 | "下拉刷新" |
| PULLING（≥ 阈值） | 60 ≤ x ≤ 100 | 文字「释放刷新」+ 柯基 | max(x, 72)dp | 1 | "释放刷新" |
| RELEASING | 0 → 60（动画） | 文字 + 柯基（alpha 渐变） | max(x, 72)dp | (x-8)/8 | "下拉刷新" |
| REFRESHING | 60（锁） | 文字「柯基努力加载中~」+ 柯基 | 72dp | 1（强制） | "柯基努力加载中~" |

### 4.2 关键边界处理

| 边界 | 行为 | 原因 |
|---|---|---|
| `state=IDLE && pullOffset ≤ 0` | 早返（shouldRender=false） | 避免初始时 72dp 空白闪现 |
| `state=REFRESHING && pullOffset=0` | 不可能发生 | REFRESHING 锁定时 pullOffset 已被动画到 60dp 阈值 |
| `state=PULLING/RELEASING && pullOffset=0` | 渲染空白 72dp，文字/柯基 alpha=0 不可见 | 边界态极少出现，但保持布局稳定 |
| `pullOffset = 0.5dp`（微小下拉） | 空白 72dp 仍展开 | 空白区是父容器，72dp 是视觉缓冲；柯基+文字 alpha=0 视觉不可见，但布局稳定 |
| `pullOffset = 8dp` | 柯基+文字 alpha 仍为 0 | 淡入起点（fadeInStartDp） |
| `pullOffset = 16dp` | 柯基+文字 alpha=1 | 淡入完成 |
| `pullOffset = 100dp` | 柯基 64dp、文字 16dp，间距 4dp | 总占用 84dp < 100dp，完美显示 |
| `pullOffset` 在 RELEASING 阶段从 60 → 0 动画 | 文字 + 柯基 alpha 渐变到 0 | 避免硬切 |
| REFRESHING 锁定时 `pullOffset` 被动画到 60dp | 文字强制「柯基努力加载中~」 | REFRESHING 独立分支 |

### 4.3 不变的部分

- 阻尼曲线（`computeDampedOffset`、`computeRawOffsetForDamped`）不变
- 状态机转换（`computeNextPullRefreshState`）不变
- `maxPullHeight = 100dp`、`refreshThreshold = 60dp` 不变
- 柯基动画类型（始终 RUN）不变
- HomeScreen 调用方式不变

### 6.2 风险与缓解

| 风险 | 缓解 |
|---|---|
| 淡入淡出导致性能问题 | 用 `Modifier.graphicsLayer { alpha = it }` 而非 `Modifier.alpha()`，避免重组 |
| Column 嵌套改变 NestedScroll 行为 | 不动 `pullRefreshState.nestedScrollConnection`，仅改 UI 渲染层 |
| 文字 alpha 在 REFRESHING 锁时异常 | REFRESHING 强制 contentAlpha=1，独立分支 |
| IDLE 早返导致 72dp 空白闪现 | `shouldRender=false` 时整个 `Column` 不返回，连空白都不渲染 |
| dp↔px 转换错误 | 在纯函数中以 dp 为单位，UI 层统一用 `with(density) { xxx.toDp() }` |

## 8. 后续优化建议

1. **柯基方向随下拉量变化**：下拉越大，柯基奔跑越快（动态 fps）
2. **柯基表情随状态变化**：PULLING 时专注，REFRESHING 时开心（需新增表情资源）
3. **触觉反馈**：达阈值时触发轻震动（`HapticFeedbackManager`）
4. **音效**：刷新触发时播放短音效（可选）
5. **文字颜色对比度提升**：alpha 0.6 在某些背景下偏淡，可调至 0.7 或 0.8
6. **响应式最小高度**：根据屏幕宽度动态调整 `minEmptyHeightDp`（小屏 60dp，大屏 84dp）
7. **柯基 + 文字 fade in 异步**：柯基比文字稍晚 100ms 出现，增强层次感
