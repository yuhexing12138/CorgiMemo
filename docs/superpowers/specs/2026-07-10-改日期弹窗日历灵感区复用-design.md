## 一、背景与目标

### 当前问题

灵感长按弹窗中"改日期"按钮弹出的日期时间修改弹窗，其日历区和灵感预览区与灵感页顶部导航栏日期按钮弹出的弹窗（`InspirationCalendarDialog`）存在以下差异：

1. **日历区不同**：改日期弹窗使用 `DateWheelPickerView`（带"2026/07 ◄ ►"导航行），而 `InspirationCalendarDialog` 使用 `CalendarMonthView`（网格日历 + 左右滑动切换月份 + "年月 ▼"标题行）
2. **灵感区不同**：改日期弹窗使用 `InspirationDateTimePreview`（独立组件），而 `InspirationCalendarDialog` 使用 `CalendarInspirationItem`（不同的视觉格式）
3. **缺少分隔线**：改日期弹窗的日历区和灵感区之间没有灰色分隔线

### 目标

1. 删除改日期弹窗中的日历区，直接复用 `InspirationCalendarDialog` 的日历区（`CalendarMonthView`）
2. 删除改日期弹窗中的灵感区，直接复用 `InspirationCalendarDialog` 的灵感区格式（`CalendarInspirationItem`），但年月日仍要在灵感标题上方显示，日期和时间要动态变化
3. 在日历区和灵感区之间加上灰色分隔线
4. 保留时间芯片 + 时间滚轮用于时间选择
5. 删除原有的"2026/07"以及同行的左右箭头

## 四、数据流与状态管理

### 4.1 状态变量

```kotlin
@Composable
fun InspirationDateTimePickerDialog(
    inspiration: Inspiration,
    onDismiss: () -> Unit,
    onConfirm: (dateMillis: Long, hour: Int, minute: Int) -> Unit
) {
    // 初始值：系统当前时间
    var selectedDate by remember { mutableStateOf(LocalDate.now()) }
    val now = remember { Calendar.getInstance() }
    var selectedHour by remember { mutableIntStateOf(now.get(Calendar.HOUR_OF_DAY)) }
    var selectedMinute by remember { mutableIntStateOf(now.get(Calendar.MINUTE)) }

    // 顶层视图模式："calendar"(日期) vs "time"(时间) — 日期/时间芯片切换
    var viewMode by remember { mutableStateOf("calendar") }

    // 日期子模式："grid"(网格) vs "wheel"(滚轮) — 年月▼切换
    var dateViewMode by remember { mutableStateOf("grid") }

    // 月份导航
    var navMonth by remember { mutableStateOf(YearMonth.from(selectedDate)) }
}
```

### 4.2 状态流转

| 用户操作 | 状态变化 | UI联动 |
|---------|---------|--------|
| 点击日期网格中的某天 | `selectedDate` 更新 | 灵感区日期行 + 时间行更新 |
| 滑动切换月份 | `navMonth` 更新 | 日历区月份切换 |
| 点击年月▼ | `dateViewMode` 切换 | 日历区 grid↔wheel 切换 |
| 点击日期芯片 | `viewMode = "calendar"` | 显示年月行+日历区 |
| 点击时间芯片 | `viewMode = "time"` | 显示时间滚轮(隐藏年月行) |
| 滚动时间滚轮 | `selectedHour/Minute` 更新 | 灵感区时间行更新 |
| 点击确定 | 组合 date+time → millis | 调用 `onConfirm` |

### 4.3 确定按钮回调

```kotlin
onConfirm = {
    val cal = Calendar.getInstance().apply {
        set(selectedDate.year, selectedDate.monthValue - 1, selectedDate.dayOfMonth,
            selectedHour, selectedMinute, 0)
        set(Calendar.MILLISECOND, 0)
    }
    onConfirm(cal.timeInMillis, selectedHour, selectedMinute)
}
```

## 六、边界情况与错误处理

### 6.1 内容溢出处理

| 场景 | 处理方式 |
|------|---------|
| 灵感正文过长 | 正文 `maxLines = 6` + `TextOverflow.Ellipsis`；灵感区用 `verticalScroll(rememberScrollState())` 包裹 |
| 标题过长 | `maxLines = 1` + `TextOverflow.Ellipsis`（已有） |
| 标签超过 3 个 | 显示前 3 个 + `+N`（已有） |
| 图片超过 2 张 | 显示前 2 个 + `+N`（已有） |

### 6.2 状态保持

| 场景 | 行为 |
|------|------|
| 日期芯片 ↔ 时间芯片切换 | `selectedDate`/`selectedHour`/`selectedMinute` 保持不变 |
| 网格 ↔ 滚轮（年月▼）切换 | `selectedDate`/`navMonth` 保持不变 |
| 弹窗关闭再打开 | 重置为系统当前时间（与当前行为一致） |

### 6.3 弹窗关闭方式

| 触发 | 行为 |
|------|------|
| 点击遮罩 | `onDismiss` → 关闭弹窗 |
| 按返回键 | `Dialog` 的 `dismissOnBackPress = true` → `onDismissRequest` |
| 点击"取消"按钮 | `onDismiss` |
| 点击"确定"按钮 | `onConfirm(dateMillis, hour, minute)` → 关闭弹窗 |

### 6.4 空字段处理

| 字段 | 空值行为 |
|------|---------|
| `inspiration.content` 为空白 | 不渲染正文区域（已有逻辑） |
| `inspiration.tags` 为空 | 不渲染标签行（已有逻辑） |
| `inspiration.imagePaths` 为空 | 不渲染图片行（已有逻辑） |
| `inspiration.title` 为空 | 正常渲染空标题（不阻断） |

## 八、实现要点

### 8.1 CalendarMonthView 无圆点

`CalendarMonthView` 的 `countMap` 参数传 `emptyMap()` 即可隐藏所有日期的灵感计数圆点。

### 8.2 年月标题行

从 `InspirationCalendarDialog` 提取标题行逻辑：
- 显示"2026年7月"（16sp Bold）
- ▼/▲ 切换图标（12sp 灰色）
- 点击整行切换 `dateViewMode`（grid ↔ wheel）
- 仅在 `viewMode == "calendar"` 时显示

### 8.3 Dialog 窗口层级

新弹窗继续使用 `androidx.compose.ui.window.Dialog` 渲染至窗口层级：
- `DialogProperties(usePlatformDefaultWidth = false, decorFitsSystemWindows = false)`
- 遮罩 `Box(fillMaxSize).navigationBarsPadding().background(Color(0x99000000))`
- 弹窗容器 4:1 比例定位，高度为屏幕 90%
- 淡入 + 缩放动画

### 8.4 删除文件清理

`InspirationDateTimePreview.kt` 删除后，需确保：
- `InspirationScreen.kt` 中删除对该文件的 import
- 无其他文件引用 `InspirationDateTimePreview`
- 该文件中的 `private fun removeHtmlTags` 和 `decodeTagsJson`/`decodePathsJson` 调用改为引用共享文件中的版本
