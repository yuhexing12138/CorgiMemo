# 回收站功能设计

> 日期：2026-07-12
> 状态：已确认，待实施

## 概述

为灵感页新增回收站功能（对标待办页"最近删除"），同时将待办页"最近删除"入口从侧滑栏迁移至三点菜单，并统一命名为"回收站"。待办和灵感的回收站合并为统一页面，使用 Tab 切换，并根据来源页面自动定位到对应 Tab。

## 需求摘要

| 项目 | 决策 |
|------|------|
| 保留策略 | 待办和灵感均为30天自动永久清理 |
| 侧滑栏入口 | 完全移除"最近删除"，仅通过三点菜单进入回收站 |
| 页面结构 | 待办和灵感统一共用一个回收站页面，Tab切换 |
| 灵感删除流程 | 从直接永久删除改为软删除（先移入回收站） |
| 命名 | 全应用统一使用"回收站"，弃用"最近删除" |

## 方案选择

采用**方案C：默认Tab + 自动跳转**。统一回收站页面内使用Tab（"待办"/"灵感"）切换内容，根据来源页面自动选中对应Tab。

- 从待办页三点菜单进入 → 默认选中"待办"Tab
- 从灵感页三点菜单进入 → 默认选中"灵感"Tab

## 第1部分：数据层 — 灵感软删除机制

### 新增 `DeletedInspiration` 实体

- 表名：`deleted_inspirations`
- 字段与 `Inspiration` 一一对应，额外增加 `deletedAt: Long = System.currentTimeMillis()`
- 提供 `fromInspiration(inspiration: Inspiration): DeletedInspiration` 转换方法
- 提供 `toInspiration(deleted: DeletedInspiration): Inspiration` 转换方法

### 新增 `DeletedInspirationDao`

| 方法 | 说明 |
|------|------|
| `insert(deletedInspiration)` | 插入单条 |
| `insertAll(list)` | 批量插入 |
| `getAllDeletedInspirations(): Flow<List<DeletedInspiration>>` | 订阅全部（按deletedAt倒序） |
| `getDeletedCount(): Flow<Int>` | 计数 |
| `getDeletedInspirationById(id): DeletedInspiration?` | 按ID查询 |
| `deleteById(id)` | 永久删除单条 |
| `deleteAll()` | 清空表 |
| `deleteOlderThan(threshold): Int` | 清理超过阈值的记录 |

### 新增 `DeletedInspirationRepository`

与 `DeletedTodoRepository` 同构，注入 `DeletedInspirationDao` + `IoDispatcher`，提供：
- `insertDeletedInspiration(inspiration: Inspiration)` — 转换后插入
- `getAllDeletedInspirations(): Flow<List<DeletedInspiration>>`
- `getByIdBlocking(id): DeletedInspiration?`
- `permanentlyDelete(id)` / `permanentlyDeleteAll()` / `cleanUpOldDeletedInspirations(threshold)`

### 数据库变更

- `CorgiMemoDatabase` 新增 `DeletedInspiration::class` 实体和 `DeletedInspirationDao`
- 新增 Migration（版本号 +1），DDL：`CREATE TABLE deleted_inspirations (...)`
- Hilt Module 中提供 `DeletedInspirationRepository` 的 `@Provides`

## 第2部分：统一回收站页面（UI层）

### 页面结构 — `RecycleBinScreen`

替代现有 `RecentlyDeletedScreen`。

- **顶栏**：返回箭头 + 标题"回收站" + 清空全部文字按钮（仅当两个Tab中任一有数据时显示）
- **Tab栏**：两个Tab按钮 — "待办" / "灵感"，使用项目现有Tab样式（3px粗边框+5px硬阴影）
- **内容区**：根据选中Tab显示对应删除项列表
  - 待办Tab：复用现有分组逻辑（今天/昨天/本周/更早）和 `DeletedTodoCard`
  - 灵感Tab：新写 `DeletedInspirationCard`，同样按时间分组，卡片显示标题、标签、删除时间
- **空态**：柯基表情 + "回收站是空的" + 返回按钮
- **SnackBar**：恢复/永久删除的撤销支持

### 导航参数

- 路由：`"recycle_bin?source={source}"`
- `source` 取值：`"todo"` / `"inspiration"`
- 待办页三点菜单 → 导航 `recycle_bin?source=todo`
- 灵感页三点菜单 → 导航 `recycle_bin?source=inspiration`

### ViewModel — `RecycleBinViewModel`

替代 `RecentlyDeletedViewModel`。

- 注入：`DeletedTodoRepository` + `DeletedInspirationRepository` + `TodoRepository` + `InspirationRepository` + `CategoryRepository`
- `uiState` 包含：`todoGroups`、`inspirationGroups`、`todoTotalCount`、`inspirationTotalCount`、`selectedTab`、`isLoading`、`showClearAllDialog`
- `observeData()`：同时订阅 `deletedTodoRepository.getAllDeletedTodos()` 和 `deletedInspirationRepository.getAllDeletedInspirations()`，combine 后分别映射为按时间分组
- `init` 时调用 `cleanupOldRecords()`（待办30天 + 灵感30天）
- 方法：`restoreTodo` / `restoreInspiration` / `permanentlyDeleteTodo` / `permanentlyDeleteInspiration` / `confirmClearAll`（清空两个表） / `selectTab`
- SnackBar撤销支持：缓存最近一次永久删除的对象（待办或灵感），5秒内可撤销

## 第6部分：灵感删除流程变更

### `InspirationViewModel` 变更

- 注入 `DeletedInspirationRepository`
- `deleteInspiration(id: Long)`：
  1. 通过 `inspirationRepository.getInspirationById(id)` 获取实体
  2. 插入 `deletedInspirationRepository.insertDeletedInspiration(inspiration)`
  3. 调用 `inspirationRepository.deleteById(id)`
- `deleteInspiration(inspiration: Inspiration)`：同理，先插入回收站再删除
- `batchDeleteInspirations()`：遍历选中ID，对每条执行相同的软删除流程

### `InspirationRepository` 新增方法

- `getInspirationById(id: Long): Inspiration?`（若已有则复用）

### 恢复灵感的完整流程（`RecycleBinViewModel.restoreInspiration`）

1. 从 `deleted_inspirations` 读取 `DeletedInspiration`
2. 转换为 `Inspiration` 实体
3. 若原 `categoryId` 对应分类已不存在，重置为 0
4. 写入 `inspirations` 表
5. 删除 `deleted_inspirations` 记录
6. SnackBar 提示"已恢复"

## 第7部分：错误处理与边界

| 场景 | 处理方式 |
|------|----------|
| 恢复时原分类不存在 | `categoryId` 重置为 0（与待办恢复逻辑一致） |
| 恢复时ID冲突 | 使用原有ID插入，Room `@Insert(onConflict = OnConflictStrategy.REPLACE)`；正常情况下原表已无该ID，不会冲突 |
| 清空全部 | 确认弹窗"将永久删除 N 条待办和 M 条灵感记录，且无法恢复"，确认后同时清空两个表 |
| 自动清理失败 | init 时静默清理30天前的记录，失败吞异常，下次启动重试 |
| 灵感删除前获取实体失败 | SnackBar 提示"删除失败，请重试"，不执行任何写入操作 |
