# 灵感页富文本编辑器设计

> **设计日期**：2026-07-10
> **状态**：已批准（待实施）
> **关联规范**：[UI设计规范.md](./UI设计规范.md)、`app/src/main/res/values/colors.xml`、`app/src/main/res/values-night/colors.xml`

---

## 1. 概述

为灵感编辑页设计富文本编辑器，使用 `compose-rich-editor` 库（位于 `compose-rich-editor-main/`）替换现有自定义实现，并将格式化工具栏以折叠展开式集成到灵感编辑页的底部栏区域。

### 1.1 背景

灵感编辑页已有一套自定义富文本编辑器（`RichTextEditor` + `TextFormatToolbar`），但存在两个问题：
1. `TextFormatToolbar` 当前实际无法显示（`InspirationEditScreen.kt` 中 `showFormatToolbar` 初始为 false，且全文件无任何触发点将其设为 true）
2. 自定义实现能力有限，不支持代码块、链接等高级富文本能力

`compose-rich-editor` 库提供了完整的 WYSIWYG 富文本编辑能力，包括粗体/斜体/下划线/删除线、有序/无序列表、段落对齐、链接、代码块、HTML/Markdown 转换、内置 undo/redo。

### 1.2 核心决策

| 决策项 | 选择 | 说明 |
|---|---|---|
| 替换策略 | 用库完全替换 | 用库的 `RichTextState` + `RichTextEditor` 替换现有自定义实现 |
| 底部栏布局 | 折叠展开式 | 收起态 5 按钮，点击 ⋮ 展开上行格式工具栏 |
| 集成方式 | 源码 module | `include(":compose-rich-editor-main:richeditor-compose")` |
| 迁移策略 | 混合迁移 | 库编辑器+样式+Markdown，保留扩展 Undo/Redo 和内容块联动 |

---

## 2. 目标与非目标

### 2.1 目标

1. 用 `compose-rich-editor` 库的 `RichTextEditor` + `RichTextState` 替换现有自定义 `RichTextEditor` + `RichTextEditorState`
2. 新建 `InspirationEditBottomBar` composable，整合 5 按钮（📷/🎤/#/@/⋮）+ 折叠展开式 `RichTextFormatToolbar`
3. 用库的 `toMarkdown()/setMarkdown()` 替换 `MarkdownParser`
4. 保留现有扩展 Undo/Redo 栈（支持文本+内容块混合操作）
5. 保留内容块联动（两步删除、光标感知）和 @关联 / #位置 触发逻辑
6. 支持的富文本功能：B/I/U/S、有序/无序列表、左/中/右对齐、链接、代码块

### 2.2 非目标

- 不迁移到库的 `state.history` undo/redo（保留现有扩展栈）
- 不使用库的 Triggers 系统（保留现有 @/# 触发逻辑）
- 不修改 Room Entity 的 `contentFormat` 字段类型
- 不修改其他页面的编辑器（本次仅灵感编辑页）
- 不实现自定义主题色切换对工具栏的影响（沿用现有主题色方案）

```
│    ├─ RichTextState（库，新增）                           │
│    ├─ 扩展 Undo/Redo 栈（保留）                           │

│  :compose-rich-editor-main:richeditor-compose（源码）     │
└─────────────────────────────────────────────────────────┘
```

### 3.2 数据流

**编辑流程（实时）：**
```
用户输入 → RichTextState 自动更新 → snapshotFlow 监听
→ 防抖 500ms（复用 scheduleFormatExport）→ toMarkdown()
→ contentFormat → Room 持久化
```

**加载流程（进入页面）：**
```
Room 读取 → contentFormat (Markdown) → setMarkdown()
→ RichTextState 恢复 → 编辑器显示
```

**Undo/Redo 桥接（保留扩展栈）：**
```
文本操作：
  RichTextState 变更 → 包装为 RichTextSnapshot → 推入扩展栈
  撤销时 → 取出上一 snapshot → setMarkdown() 恢复

内容块操作（图片/语音）：
  现有扩展栈逻辑不变
```

**格式工具栏交互：**
```
点击格式按钮 → toggleSpanStyle/toggleOrderedList 等
→ RichTextState 更新 → 同编辑流程防抖导出
```

**工具栏展开状态：**
```
⋮ 按钮点击 → isFormatToolbarExpanded = !isFormatToolbarExpanded
→ AnimatedVisibility 展开/收起
（只有 ⋮ 点击切换，其他操作不切换）
```

### 5.2 收起态（默认）

- 单行 5 图标，无文字标签
- 高度约 52dp
- 5 图标均匀分布：📷 相机 / 🎤 语音 / # 位置 / @ 关联 / ⋮ 格式
- ⋮ 按钮背景：主色浅 `#FFE4CC` + 主色深字 `#E88A4D`

### 5.3 展开态（点击 ⋮ 后）

- 上行：`RichTextFormatToolbar`（`LazyRow` 可横滑），4 组按钮用分隔线隔开
- 下行：5 按钮保持不变
- 总高度约 100dp
- ⋮ 按钮背景切换为：主色 `#FF9A5C` + 白字 `#FFFFFF`（激活态）

### 5.4 交互规则

**展开/收起切换：**
- 点击 ⋮ → 展开上行工具栏
- 再次点击 ⋮ → 收起工具栏
- 点击工具栏内格式按钮 → 不收起
- 点击相机/麦克风/#/@ → 不收起
- 编辑器输入/光标移动 → 不收起
- 软键盘弹出/收起 → 不收起

**格式按钮激活态：**
- 从 `RichTextState` 读取当前光标位置的样式状态
  - 粗体：`state.currentSpanStyle.fontWeight == FontWeight.Bold`
  - 斜体：`state.currentSpanStyle.fontStyle == FontStyle.Italic`
  - 下划线：`state.currentSpanStyle.textDecoration == TextDecoration.Underline`
  - 删除线：`state.currentSpanStyle.textDecoration == TextDecoration.LineThrough`
  - 无序列表：`state.isUnorderedList`（库直接提供）
  - 有序列表：`state.isOrderedList`（库直接提供）
  - 代码块：`state.isCodeSpan`（库直接提供）
  - 链接：`state.isLink`（库直接提供）
  - 对齐：`state.currentParagraphStyle.textAlign`（与目标对齐方式比较）
- 激活态：主色浅背景 `#FFE4CC` + 主色深字 `#E88A4D`
- 默认态：次要文字色 `#666666`

### 5.5 # 与 @ 按钮行为

- **# 位置按钮**：点击触发位置提醒弹窗（`showLocationPopup`），与编辑器内输入 # 行为一致
- **@ 关联按钮**：点击触发关联选择弹窗（`showMentionPopup`），与编辑器内输入 @ 行为一致

## 6. 状态管理与数据流

### 6.1 ViewModel 变更

`InspirationEditViewModel` 新增字段：

```kotlin
// 富文本编辑器状态（库）
val richTextState: RichTextState

// 工具栏展开状态
var isFormatToolbarExpanded: Boolean by mutableStateOf(false)
    private set
```

### 6.2 RichTextState 初始化

```kotlin
// 进入页面时从 contentFormat 加载
fun loadContent(contentFormat: String) {
    try {
        richTextState.setMarkdown(contentFormat)
    } catch (e: Exception) {
        // fallback: 纯文本显示
        richTextState.setText(contentFormat)
    }
}
```

### 6.3 编辑监听与导出

用 `snapshotFlow` 监听 `RichTextState` 变更，替代现有 `onValueChanged` 回调：

```kotlin
init {
    viewModelScope.launch {
        snapshotFlow { richTextState.toMarkdown() }
            .debounce(500)  // 复用现有 scheduleFormatExport 防抖逻辑
            .collect { markdown ->
                _contentFormat.value = markdown
                // 触发 Room 持久化
            }
    }
}
```

### 6.4 Undo/Redo 桥接

```kotlin
/**
 * 富文本快照：封装 Markdown + 光标位置，推入扩展 Undo/Redo 栈
 */
data class RichTextSnapshot(
    val markdown: String,
    val selection: TextRange
)

// RichTextState 变更时包装为快照推入扩展栈
fun onRichTextChanged() {
    val snapshot = RichTextSnapshot(
        markdown = richTextState.toMarkdown(),
        selection = richTextState.selection
    )
    extendedUndoStack.push(snapshot)
}

// 撤销时恢复
fun undo() {
    val snapshot = extendedUndoStack.pop()
    richTextState.setMarkdown(snapshot.markdown)
    richTextState.selection = snapshot.selection
}
```

### 6.5 isFormatToolbarExpanded 管理

- 存于 ViewModel，配置改变（屏幕旋转）时自动保留
- 只有 ⋮ 按钮点击调用 `toggleFormatToolbar()` 切换
- 其他操作均不调用此方法

```kotlin
fun toggleFormatToolbar() {
    isFormatToolbarExpanded = !isFormatToolbarExpanded
}
```

### 6.6 内容块联动保留

现有内容块（图片/语音）操作时，用 `MutableState` 标志暂时禁用 `snapshotFlow` 监听，避免 RichTextState 变更与内容块操作互相触发：

```kotlin
private var isContentBlockOperating = false

fun performContentBlockOperation(block: () -> Unit) {
    isContentBlockOperating = true
    block()
    isContentBlockOperating = false
}

// snapshotFlow 监听中判断
snapshotFlow { richTextState.toMarkdown() }
    .filter { !isContentBlockOperating }
    .debounce(500)
    .collect { ... }
```

## 7. 现有功能迁移

### 7.1 迁移清单

| 现有组件/逻辑 | 处理方式 | 说明 |
|---|---|---|
| 自定义 `RichTextEditor` | 替换 | 用库的 `RichTextEditor` |
| 自定义 `RichTextEditorState` | 替换 | 用库的 `RichTextState` |
| `MarkdownParser`（灵感页用法） | 灵感页改用库 | 灵感页的导入/导出改用库的 `toMarkdown()/setMarkdown()`；`MarkdownParser.kt` 文件保留（其他页面仍使用） |
| `TextFormatToolbar` | 废弃 | 新建 `RichTextFormatToolbar` |
| `EditToolbar`（灵感页用法） | 替换 | 新建 `InspirationEditBottomBar` |
| 扩展 Undo/Redo 栈 | 保留 | 桥接 `RichTextSnapshot` |
| 内容块联动（两步删除/光标感知） | 保留 | 现有逻辑不变 |
| @关联 / #位置 触发弹窗 | 保留 | 底部栏按钮直接调用现有弹窗 |
| `scheduleFormatExport()` 防抖 | 保留 | 复用防抖逻辑（500ms） |
| `contentFormat` 字段 | 保留 | Entity 字段不变 |
| 图片/语音内容块 | 保留 | 现有逻辑不变 |
| `showFormatToolbar` 状态 | 废弃 | 由 `isFormatToolbarExpanded` 替代 |

### 7.2 Markdown 转换兼容性

现有 `MarkdownParser` 是自定义实现，库的 `toMarkdown()/setMarkdown()` 基于 `jetbrains-markdown` 库，更标准。

**迁移注意：**
- 现有 contentFormat 数据需验证库的 `setMarkdown()` 能正确解析
- 如有解析差异（如列表缩进、代码块格式），需做数据迁移或自定义解析
- `toMarkdown()` 输出格式需与现有 contentFormat 兼容，避免历史数据损坏

## 8. 错误处理

### 8.1 错误场景与处理

| 场景 | 处理方式 |
|---|---|
| `setMarkdown()` 解析失败 | catch 异常，fallback 到 `setText()` 纯文本显示，记录日志 |
| `toMarkdown()` 导出失败 | catch 异常，保留上一次成功的 `contentFormat`，不覆盖 |
| 库版本不兼容（编译时） | 调整库 `build.gradle.kts` 对齐项目版本（符合工作区规则） |
| RichTextState 与内容块联动冲突 | 内容块操作时用 `isContentBlockOperating` 标志禁用监听 |
| 软键盘与工具栏展开 | 工具栏展开时不主动收起键盘，保持编辑焦点 |
| 配置改变（旋转屏幕） | `isFormatToolbarExpanded` 存于 ViewModel 自动保留；`RichTextState` 由 ViewModel 持有 |
| 历史 Markdown 数据解析差异 | 首次加载时检测，如 `setMarkdown()` 结果与原文差异大，提示用户或做数据迁移 |

### 8.2 边界情况

- **空内容**：`contentFormat` 为空时，`setMarkdown("")` 正常处理，编辑器显示空
- **超长内容**：`toMarkdown()` 可能耗时，已有 500ms 防抖保护
- **光标在内容块边界**：现有内容块光标感知逻辑保留，RichTextState 的 selection 同步
- **多次快速点击 ⋮**：每次点击都切换状态，无防抖（用户可快速收起）
