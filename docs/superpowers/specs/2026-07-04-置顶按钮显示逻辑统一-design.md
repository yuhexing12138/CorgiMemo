# 置顶按钮显示逻辑统一设计

## 1. 背景

### 1.1 用户需求

> 将置顶按钮显示逻辑改成：只要有置顶待办就显示。同步修改待完成按钮的位置逻辑。

### 1.2 当前逻辑（存在 Case A/B 分支）

[HomeScreen.kt:744-779](file:///c:/Users/EDY/Desktop/CorgiMemo/app/src/main/java/com/corgimemo/app/ui/screens/home/HomeScreen.kt#L744-L779)：

| pinnedCount | displayItems 结构 | 说明 |
|---|---|---|
| `>= 4`（Case A） | `[PinnedDivider, P1-Pn, PendingDivider, N1-Nm, ...]` | 置顶区独立，PendingDivider 只代表非置顶 |
| `< 4`（Case B，0-3） | `[PendingDivider, P1-Pn+N1-Nm 混合, ...]` | 无 PinnedDivider，PendingDivider 代表所有待完成（含置顶） |

### 1.3 当前逻辑的问题

1. **阈值 4 不直观**：用户感知不到为什么是 4，而非 3 或 5
2. **Case B 时置顶项混在待完成区**：置顶项没有独立的视觉分组，UX 不一致
3. **pendingCount 逻辑复杂**：需根据 pinnedCount 分支计算，技术债

### 1.4 确认的需求

- `pinnedCount >= 1` 就显示 PinnedDivider
- `pinnedCount == 0` 时仍显示 PendingDivider（代表所有待完成，此时都是非置顶）
- PendingDivider 始终只代表非置顶待完成

## 2. 新逻辑统一后的 displayItems 结构

| pinnedCount | displayItems 结构 |
|---|---|
| `>= 1` | `[PinnedDivider, P1...Pn, PendingDivider, N1...Nm, CompletedDivider, ...]` |
| `== 0` | `[PendingDivider, N1...Nm, CompletedDivider, ...]`（无 PinnedDivider） |

## 3. 修复方案：统一为单一逻辑，消除 Case A/B 分支（方案 A）

### 3.1 修改 1：HomeViewModel.kt 的 pendingCount 简化

#### 修改位置
[HomeViewModel.kt:200-216](file:///c:/Users/EDY/Desktop/CorgiMemo/app/src/main/java/com/corgimemo/app/viewmodel/HomeViewModel.kt#L200-L216)

#### 变更前
```kotlin
/**
 * V2.11: 待完成待办总数(动态计算)
 *
 * 位置语义根据置顶数量动态调整:
 * - 置顶 ≤ 3:待完成按钮在最前,代表所有待完成(含置顶)
 * - 置顶 ≥ 4:待完成按钮在置顶区后,仅代表非置顶
 */
val pendingCount: StateFlow<Int> = combine(_todos, pinnedCount) { todos, pinnedN ->
    val nonPinned = todos.count { !it.isPinned && it.status == 0 }
    if (pinnedN <= 3) {
        // Case B:按钮在最前,代表所有待完成(含置顶)
        todos.count { it.status == 0 }
    } else {
        // Case A:按钮在置顶区后,仅代表非置顶
        nonPinned
    }
}.stateIn(viewModelScope, SharingStarted.WhileSubscribed(5000), 0)
```

#### 变更后
```kotlin
/**
 * 待完成待办总数（仅非置顶）
 *
 * 新逻辑：pinnedCount >= 1 时显示 PinnedDivider，PendingDivider 始终只代表非置顶待完成。
 * - pinnedCount >= 1：PendingDivider 在置顶区后，count = 非置顶待完成数
 * - pinnedCount == 0：无 PinnedDivider，PendingDivider 代表所有待完成（此时都是非置顶）
 *
 * 两种情况 count 都是"非置顶待完成数"，逻辑统一。
 */
val pendingCount: StateFlow<Int> = _todos.map { todos ->
    todos.count { !it.isPinned && it.status == 0 }
}.stateIn(viewModelScope, SharingStarted.WhileSubscribed(5000), 0)
```

#### 简化说明

- 消除 `combine(_todos, pinnedCount)` 和 `if (pinnedN <= 3)` 分支
- 不再依赖 `pinnedCount`，直接从 `_todos` 计算
- 语义统一：PendingDivider 始终代表非置顶待完成

#### 变更后
```kotlin
buildList {
    // 置顶区（仅当有置顶待办时显示）
    if (pinnedCount >= 1) {
        add(DisplayItem.PinnedDivider(count = pinnedCount, isExpanded = showPinned))
        if (showPinned) {
            filteredPending.filter { it.isPinned }.forEach { add(DisplayItem.Todo(it)) }
        }
    }
    // 待完成区（始终显示，代表非置顶待完成）
    add(DisplayItem.PendingDivider(count = pendingCount, isExpanded = showPending))
    if (showPending) {
        filteredPending.filter { !it.isPinned }.forEach { add(DisplayItem.Todo(it)) }

    // 已完成区（原有逻辑不变）

        if (showCompleted) {
            filteredCompleted.forEach { add(DisplayItem.Todo(it)) }
        }
    }
}
```

#### 说明

- `pinnedCount == 0`：无置顶区，PendingDivider 不分隔（虽然存在但无偏移需求），返回 -1

## 4. pendingOffset 逻辑验证（不变）

`pendingOffset` 用于将 displayItems 全局索引转换为 pendingList 索引。pendingList 已按 isPinned 降序排序（置顶在前）。

```kotlin
fun pendingOffset(displayIdx: Int): Int =
    pendingStartIndex + (if (midPendingDividerIndex >= 0 && displayIdx > midPendingDividerIndex) 1 else 0)
```

### 验证场景：pinnedCount = 2

displayItems: `[PinnedDivider(0), P1(1), P2(2), PendingDivider(3), N1(4), N2(5), CompletedDivider(6), ...]`

- `pendingStartIndex` = 1（第一个 Todo 的索引）
- `midPendingDividerIndex` = 3

| 项 | displayIdx | midPendingDividerIndex 比较 | pendingOffset | pendingList 索引 |
|---|---|---|---|---|
| P1 | 1 | 1 > 3? 否 | 1 | 0 ✓ |
| P2 | 2 | 2 > 3? 否 | 1 | 1 ✓ |
| N1 | 4 | 4 > 3? 是 | 1+1=2 | 2 ✓ |
| N2 | 5 | 5 > 3? 是 | 2 | 3 ✓ |

pendingList 顺序：`[P1, P2, N1, N2]`（置顶在前）✓

### 验证场景：pinnedCount = 0

displayItems: `[PendingDivider(0), N1(1), N2(2), CompletedDivider(3), ...]`

- `pendingStartIndex` = 1
- `midPendingDividerIndex` = -1

| 项 | displayIdx | midPendingDividerIndex 比较 | pendingOffset | pendingList 索引 |
|---|---|---|---|---|
| N1 | 1 | midPendingDividerIndex=-1，条件为否 | 1 | 0 ✓ |
| N2 | 2 | 同上 | 1 | 1 ✓ |

pendingList 顺序：`[N1, N2]`（全是非置顶）✓
