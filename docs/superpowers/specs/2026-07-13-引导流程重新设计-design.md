# 引导流程重新设计

> **创建日期**: 2026-07-13
> **状态**: 设计已确认，待编写实现计划
> **方案**: 方案C - 分阶段流程（3阶段10步）

---

## 1. 背景与目标

### 1.1 背景

CorgiMemo（刻记⁺）APP 已迭代多次，现有的首次启动引导流程（5页 HorizontalPager）已过时：

- **身份选择**页声称提供"个性化待办推荐"，但选择后未被实际使用
- **问候页**为纯展示页，无交互，浪费时间
- **权限请求**页仅展示权限列表，未实际触发系统权限对话框
- 缺少对 APP 核心功能（待办/灵感/日期/柯基养成）的介绍
- 运行时操作指引组件（OperationGuide、FirstTimeGuideOverlay、CorgiGuideAnimation）已不适用

### 1.2 目标

1. **删除**过时的运行时操作指引组件
2. **重新设计**首次启动引导流程，采用分阶段方案（3阶段10步）
3. **实际触发**系统权限请求（通知/存储/麦克风/位置）
4. **交互式创建**首个待办和灵感，让用户在引导中实际体验 APP
5. **复用**项目中现有的柯基帧动画资源（13种动画类型）

## 4. 数据流与状态管理

### 4.1 OnboardingViewModel 状态

```kotlin
@HiltViewModel
class OnboardingViewModel @Inject constructor(
    private val corgiPreferences: CorgiPreferences,
    private val corgiRepository: CorgiRepository,
    private val todoRepository: TodoRepository,
    private val inspirationRepository: InspirationRepository
) : ViewModel() {

    // StateFlow
    val currentPage: StateFlow<Int>          // 0-9, 当前页面索引
    val selectedUserType: StateFlow<UserType?>  // 仅 STUDENT 可选
    val corgiName: StateFlow<String>          // 1-8字符
    val createdTodo: StateFlow<Todo?>         // 引导中创建的待办
    val createdInspiration: StateFlow<Inspiration?>  // 引导中创建的灵感
    val permissionStates: StateFlow<Map<PermissionType, PermissionState>>
    val isCompleting: StateFlow<Boolean>
    val skippedFunctions: StateFlow<Set<FunctionType>>  // 已跳过的功能

    // Derived State
    fun canGoNext(): Boolean    // 根据 currentPage 计算
    val isLastPage: Boolean     // currentPage == 9
    val mustComplete: Boolean   // currentPage == 2 || currentPage == 8
}
```

## 5. 权限处理详细设计

### 5.1 权限请求流程

```
用户进入 Step 9 (PermissionPage)
    ↓
显示权限列表（4项）
    ↓
逐个请求权限（通知 → 存储 → 麦克风 → 位置）
    ↓
ActivityResultContracts.RequestPermission() 触发系统对话框
    ↓
├─ 授权 → 标记 ✓ → 请求下一项
├─ 拒绝 → 标记 ✗ → 请求下一项
└─ 永久拒绝 → 标记 ⚠ → 显示"去设置"按钮
    ↓
所有权限请求完毕
    ↓
"完成"按钮可用（必须请求过所有权限）
```

### 5.2 Android 版本兼容

| 权限 | Android < 13 | Android 13+ (API 33+) | Android 14+ (API 34+) |
|------|--------------|----------------------|----------------------|
| 🔔 通知 | 默认开启 | `POST_NOTIFICATIONS` | `POST_NOTIFICATIONS` |
| 📁 存储 | `WRITE_EXTERNAL_STORAGE` | `READ_MEDIA_IMAGES`（Scoped Storage） | 同 Android 13 |
| 🎤 麦克风 | `RECORD_AUDIO` | `RECORD_AUDIO` | `RECORD_AUDIO` |
| 📍 位置 | `ACCESS_COARSE_LOCATION` | `ACCESS_COARSE_LOCATION` | `ACCESS_COARSE_LOCATION` |

### 5.3 权限状态 UI

| 状态 | 图标 | 背景色 | 说明 |
|------|------|--------|------|
| 未请求 | ⏳ | 灰色 | 初始状态 |
| 已授权 | ✓ | 绿色 | 用户同意 |
| 已拒绝 | ✗ | 红色 | 用户拒绝，显示"稍后设置" |
| 永久拒绝 | ⚠ | 橙色 | 勾选"不再询问"，显示"去设置" |

### 5.4 永久拒绝的处理

当用户永久拒绝权限时（勾选"不再询问"）：
1. 权限项显示橙色 ⚠ 图标和"去设置"按钮
2. 点击"去设置"跳转到应用设置页：`Intent(Settings.ACTION_APPLICATION_DETAILS_SETTINGS)`
3. 用户从设置返回后，重新检查权限状态并更新 UI
4. 不阻塞引导流程，用户可以继续完成引导

### 5.5 核心技术实现

```kotlin
@Composable
fun PermissionPage(viewModel: OnboardingViewModel) {
    val context = LocalContext.current

    val permissionLauncher = rememberLauncherForActivityResult(
        ActivityResultContracts.RequestPermission()
    ) { isGranted ->
        viewModel.updatePermissionState(
            currentPermission,
            if (isGranted) PermissionState.GRANTED
            else PermissionState.DENIED
        )
        viewModel.requestNextPermission()
    }

    // 检查是否永久拒绝
    fun checkPermanentlyDenied(permission: String): Boolean {
        return !ActivityCompat.shouldShowRequestPermissionRationale(
            context as Activity, permission
        ) && ContextCompat.checkSelfPermission(
            context, permission
        ) != PackageManager.PERMISSION_GRANTED
    }

    fun requestPermission(permission: String) {
        permissionLauncher.launch(permission)
    }
}
```

```
│                             │
│  [取消]      [跳过]         │
└─────────────────────────────┘
```

### 6.3 错误处理策略

| 错误场景 | 处理方式 | 恢复策略 |
|----------|----------|----------|
| 创建待办/灵感失败 | 显示错误 Toast "创建失败，请稍后重试" | 留在当前页，允许重试或跳过 |
| 柯基数据保存失败 | 使用默认值重试一次 | 重试成功则继续，失败则跳过 |
| DataStore 写入失败 | 静默重试（用户无感知） | 下次启动重新检查 |
| 权限请求异常 | 标记为"未请求"，继续下一项 | 允许用户在设置中手动开启 |

### 6.4 边界情况处理

- **进程被杀**：引导中途进程被杀，下次启动重新开始引导（不保留中间状态）
- **旋转屏幕**：ViewModel 保留状态，HorizontalPager 自动恢复页面位置
- **返回键**：Step 1 按返回键退出APP；其他步骤按返回键回到上一步
- **柯基名字为空**：Step 3 下一步按钮禁用，显示"请输入柯基名字"
- **柯基名字过长**：输入限制8字符，超过不接收；显示 `WORRY` 动画
- **待办标题为空**：Step 5 创建按钮禁用；下一步仍可点击（跳过创建）
