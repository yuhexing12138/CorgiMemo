# 「我的」页面 UI 布局设计

> 日期：2026-07-19
> 关联文件：`app/src/main/java/com/corgimemo/app/ui/screens/profile/ProfileScreen.kt`
> 规范引用：`docs/superpowers/specs/UI设计规范.md`（12.1 节）
> 状态：待用户审核

## 1. 概述

### 1.1 背景

现有「我的」页面（`ProfileScreen.kt`）本质是"柯基宠物中心"，仅承载柯基信息、成就统计、情绪图表、装扮系统四个模块，且未聚合设置、主题、备份、关于等常规个人中心入口。同时页面布局与 `UI设计规范.md` 存在偏差（页面 padding 16dp / 卡片圆角 16dp / 卡片间距 16dp，均小于规范值）。

### 1.2 目标

将「我的」页面从"柯基宠物中心"扩展为**完整的个人中心枢纽**：

- 保留现有柯基体验（信息卡 / 成就 / 情绪图表 / 装扮）
- 新增主题配色快选、设置入口组、数据备份、关于与帮助四个模块
- 对齐 `UI设计规范.md` 的间距、圆角、字号规范
- 将占用大块篇幅的"装扮详情"和"成就横滑列表"折叠为入口卡，外移到独立详情页，保持本页清爽

### 1.3 非目标

- 不改造柯基动画组件（`InteractiveCorgi`）本身
- 不改造 `SettingsScreen` 内部实现，仅复用其作为跳转目标
- 不改造 `AchievementScreen`，仅复用其作为成就详情跳转目标
- 不引入新的数据持久化结构（主题偏好已有 DataStore key `theme_color_scheme`）

## 2. 现状分析

### 2.1 现有模块

| 模块 | 现状 | 问题 |
|------|------|------|
| 柯基信息卡 | 圆形"柯"字头像 + 名字 + 等级/经验/情绪三栏 + 进度条 | 头像无沉浸感；统计与下方成就卡重复 |
| 成就统计卡 | 4 行统计 + "查看详情 ›" | 内容与下方成就横滑卡割裂 |
| 7天情绪图表 | `MoodHistoryChart` 组件 | OK，保留 |
| 成就横滑卡 | `LazyRow` 展示所有成就 | 占用大块篇幅，与本页"概览"定位冲突 |
| 装扮推荐横幅 | `OutfitRecommendationBanner` | 与装扮卡重复，可合并 |
| 装扮卡 | 柯基动画 + 预览模式 + 横滑列表 | 占用最大篇幅，应外移 |
| 改名设置 | 单项 | 可并入设置入口组 |
| 返回首页按钮 | `Button` | 底部导航已提供返回，多余 |

### 2.2 规范偏差

| 项 | 现状 | 规范 | 处理 |
|----|------|------|------|
| 页面左右 padding | 16dp | 20dp | 调整 |
| 卡片圆角 | 16dp | 20dp | 调整 |
| 卡片间距 | 16dp | 12dp | 调整 |
| 卡片内边距 | 16dp | 16dp | 保持 |
| 列表项最小高度 | 未保证 48dp | 48dp | 调整 |

## 3. 设计方案

采用**方案 A · 分层长滚动**：按"身份 → 个性化 → 数据 → 设置 → 关于"层级单列滚动，装扮/成就的重内容外移到独立详情页，本页仅保留轻量入口卡。

### 3.1 方案选型理由

| 维度 | 方案 A 分层长滚动（选定） | 方案 B 头卡+Tab | 方案 C 网格+列表 |
|------|--------------------------|------------------|-------------------|
| 信息层级 | 清晰，自然分层 | Tab 切换有认知成本 | 网格让功能并列缺主次 |
| 滚动深度 | 中等，外移后显著缩短 | 浅但需多次点击 | 浅，首屏全貌 |
| 柯基沉浸感 | 头卡可做大色块，氛围足 | 头卡偏小 | 头卡被网格挤压 |
| 与现有代码 | 改动小，顺序调整即可 | 需引入 Tab 状态管理 | 需重排为网格 |
| 可扩展性 | 新增模块直接追加 | Tab 数量受限 | 网格格数固定 |

## 6. 装扮详情页（新建 OutfitScreen）

将现有 `ProfileScreen` 中的装扮相关内容整体外移到新建的 `OutfitScreen`。

### 6.1 路由

新增 `Screen.Outfit : Screen("outfit")`，在 `NavGraph` 注册。

### 6.2 内容

`OutfitScreen` 承载以下从 `ProfileScreen` 迁移来的内容：

1. **装扮推荐横幅**（`OutfitRecommendationBanner`）
2. **柯基动画预览区**（`InteractiveCorgi`，含预览模式背景提示）
3. **装扮横滑列表**（`LazyRow` + `OutfitCard`）
4. **预览模式操作栏**（取消 / 应用装扮按钮）
5. **预览模式状态管理**（`enterPreviewMode` / `previewOutfit` / `cancelPreview` / `applyPreview`）

### 6.3 顶部导航

`OutfitScreen` 使用独立 `TopAppBar`，标题"🎩 装扮"，左侧返回按钮。

### 6.4 ViewModel

复用 `ProfileViewModel`。通过 Hilt 在 `OutfitScreen` 与 `ProfileScreen` 间共享同一 `ProfileViewModel` 实例（使用 `hiltViewModel()` 在导航图同一 backStackEntry scope 下），避免装扮状态重复。预览模式的 `DisposableEffect` 退出清理逻辑（`onDispose { viewModel.cancelPreview() }`）迁移到 `OutfitScreen`。

## 8. 数据流

### 8.1 ProfileViewModel 新增/调整

| StateFlow | 类型 | 说明 |
|-----------|------|------|
| `themeColorScheme` | `StateFlow<String>` | 新增，读取 DataStore `theme_color_scheme`，供主题快选卡显示当前选中 |
| `outfitCount` | `StateFlow<Int>` | 新增（或直接用 `outfits.size`），装扮入口卡显示总数 |

其余现有 StateFlow（`corgiData` / `achievements` / `outfits` / `levelStage` / `levelProgress` / `progressText` / `moodHistory7Days`）保持不变。

### 8.2 主题切换流

```
用户点击色点
  → ProfileViewModel.setThemeColor(schemeName)
  → 写入 DataStore(key="theme_color_scheme")
  → 顶层 Theme 状态观察者读取 DataStore
  → MaterialTheme.colorScheme 更新
  → 全 App 即时生效
```

### 8.3 装扮预览流

预览模式相关状态（`isPreviewMode` / `previewOutfit` / `recommendedOutfit`）从 `ProfileScreen` 迁移到 `OutfitScreen`，`ProfileScreen` 不再持有预览状态，仅通过 `corgiData.currentOutfit` 显示当前装扮。

## 9. 错误处理

| 场景 | 处理 |
|------|------|
| 柯基数据未加载（`corgiData == null`） | 头卡显示占位（名字"小柯基"、等级 Lv.1、数值 0），不阻塞页面 |
| 成就列表为空 | 成就统计卡显示 0/0，徽章预览行显示 5 个灰锁占位 |
| 主题 DataStore 读取失败 | 降级为默认"暖阳橙"，不抛异常 |
| 装扮列表为空 | 装扮入口卡描述显示"共 0 套"，点击仍可跳 `OutfitScreen` |
| 帮助/反馈/协议页未实现 | 入口点击后显示 Snackbar"功能开发中"（用 `AppSnackbarHost`），不崩溃 |

## 10. 性能考虑

| 点 | 措施 |
|----|------|
| 头卡柯基动画 | 使用 `InteractiveCorgi` 小尺寸版本；若帧动画导致滚动卡顿，降级为静态柯基插画 + 点击触发动画 |
| 主题切换重组合 | 主题色通过 `MaterialTheme.colorScheme` 统一引用，切换时全 App 重组不可避免，但本页仅 8 个卡片，影响可控 |
| LazyColumn | 8 个模块均为 `item{}`，无大列表，`LazyColumn` 性能充足 |
| 装扮外移 | `OutfitScreen` 独立承载柯基动画 + 横滑列表，本页不渲染重动画 |
