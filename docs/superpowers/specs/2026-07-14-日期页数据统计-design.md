## 1. 概述

为日期页添加数据统计功能，统计各类型日期的数量和占比。参考待办页和灵感页的统计模式，采用独立 Screen + ViewModel 架构。

### 统计维度

1. **分类统计**：按 `SpecialDate.category` 字段统计 8 种预设分类（生日/纪念日/节日/生活/学习/工作/娱乐/其他）及自定义分类的数量和占比
2. **分组统计**：按 `DateGroup`（倒计时 COUNTDOWN / 正计时 COUNTUP / 已归档 EXPIRED）统计数量和占比

### 数据范围

包含所有日期（含已归档），与灵感页统计一致。

## 3. 数据层设计

### 3.1 数据模型

```kotlin
/** 分组统计项 */
data class DateGroupStat(
    val group: DateGroup,        // COUNTDOWN / COUNTUP / EXPIRED
    val displayName: String,     // "倒计时" / "正计时" / "已归档"
    val count: Int,
    val percentage: Float        // 0.0-1.0
)

/** 分类统计项 */
data class DateCategoryStat(
    val categoryKey: String,     // DB原始值（如 "BIRTHDAY" 或 "CUSTOM:xxx"）
    val displayName: String,     // "生日" 或自定义名称
    val emoji: String,           // "🎂" 或默认 "📅"
    val color: Color,            // 分类颜色
    val count: Int,
    val percentage: Float         // 0.0-1.0
)

/** 统计数据聚合 */
data class DateStatsData(
    val totalCount: Int = 0,
    val groupStats: List<DateGroupStat> = emptyList(),
    val categoryStats: List<DateCategoryStat> = emptyList()
)
```

### 3.2 ViewModel 设计

```kotlin
@HiltViewModel
class DateStatsViewModel @Inject constructor(
    private val specialDateRepository: SpecialDateRepository
) : ViewModel()
```

**StateFlow 暴露：**

| StateFlow | 类型 | 说明 |
|---|---|---|
| `isEmpty` | `StateFlow<Boolean>` | 无数据时显示空状态 |
| `stats` | `StateFlow<DateStatsData>` | 统计结果聚合 |

**数据加载：**
- `init` 时调用 `specialDateRepository.getAllSpecialDatesBlocking()` 一次性加载全部日期（含已归档）
- 使用 `MutableStateFlow<List<SpecialDate>>` 缓存数据，通过 `map` 计算 `stats`

### 3.3 统计计算逻辑

#### 分组统计

复用 `SpecialDateViewModel.groupByDisplayDates()` 计算各组数量。该方法已处理 `repeatType` 重复规则，确保与主页分组逻辑完全一致。

```kotlin
val grouped = SpecialDateViewModel.groupByDisplayDates(dates)
// grouped: Map<DateGroup, List<DisplayDate>>
```

各组百分比 = 组内数量 / 总数。

#### 分类统计

按 `SpecialDate.category` 原始字符串分组计数，按数量降序排列。仅展示 count > 0 的分类。

### 3.4 自定义分类处理逻辑

| `category` 值 | 处理方式 |
|---|---|
| 匹配 `DateCategory` 枚举名 | 使用枚举的 `displayName`、`emoji`、`color` |
| 以 `"CUSTOM:"` 开头 | 提取前缀后的名称，emoji 用 `"📅"`，color 用 `DateCategory.OTHER.color` |
| 其他未知值 | 直接用原始字符串作为 `displayName`，emoji 用 `"📅"`，color 用 `DateCategory.OTHER.color` |

```kotlin
fun resolveCategoryDisplay(categoryKey: String): Triple<String, String, Color> {
    return try {
        val cat = DateCategory.valueOf(categoryKey)
        Triple(cat.displayName, cat.emoji, cat.color)
    } catch (e: IllegalArgumentException) {
        if (categoryKey.startsWith("CUSTOM:")) {
            val name = categoryKey.removePrefix("CUSTOM:")
            Triple(name, "\uD83D\uDCC5", DateCategory.OTHER.color)
        } else {
            Triple(categoryKey, "\uD83D\uDCC5", DateCategory.OTHER.color)
        }
    }
}
```

## 5. 边界情况与错误处理

| 场景 | 处理方式 |
|---|---|
| 无日期数据 | 显示 EmptyState，CTA 按钮导航到 `date_create` |
| 只有一个分类 | 该分类显示 100%，其他分类不显示（仅展示 count > 0 的分类） |
| 自定义分类 | 按 3.4 节逻辑处理，动态显示在条形图中 |
| 某分组为空 | 该分组的环形进度显示 0%，3 个环始终渲染 |
| 除零保护 | `totalCount == 0` 时 `percentage = 0f` |
| 返回导航 | `savedStateHandle["targetTab"] = TabItem.DATE.name` + `popBackStack()` |
| 数据刷新 | `init` 时加载一次；重新进入页面时重新 `init` 加载最新数据 |

## 6. 不涉及的内容（YAGNI）

- 不添加横屏全屏视图（日期统计是全量快照，非时间序列，不需要 30 天扩展）
- 不添加时间范围切换（与灵感/待办统计不同，日期没有时间序列维度）
- 不添加导出/分享功能
- 不添加交互筛选（如点击饼图扇区筛选对应分类）
- 不修改 `SpecialDateViewModel`（保持现有代码不变）
