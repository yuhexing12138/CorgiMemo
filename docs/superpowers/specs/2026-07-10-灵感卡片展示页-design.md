# 灵感卡片展示页设计文档

> **日期:** 2026-07-10
> **状态:** 已确认
> **方案:** 方案 A — 独立路由 + HorizontalPager + rememberGraphicsLayer

---

## 1. 目标

为灵感页添加一个**过渡展示页**，用户从 `InspirationScreen` 时间线点击某条灵感后，先进入展示页查看完整内容，再选择复制/编辑/分享。**不直接进入编辑页**。

支持在同一展示页内通过**左右滑动**浏览所有灵感。

### 2.5 关键设计决策

- **复用 `InspirationViewModel`**：不创建新 ViewModel，复用 `filteredDisplayInspirations` 状态
- **HorizontalPager 跨所有灵感**：不是只显示当前一张，左/右滑可浏览所有灵感
- **截图 API 选择**：使用 Compose 1.9 提供的 `rememberGraphicsLayer`（官方推荐方式），不依赖 View 层级

## 4. 三个功能按钮实现

### 4.1 复制按钮

**复制格式**（纯文本，四部分用换行分隔）：

```kotlin
fun buildInspirationPlainText(inspiration: Inspiration, formattedDate: String): String {
    val tagsStr = parseTags(inspiration.tags).joinToString(" ") { "#$it" }
    return buildString {
        appendLine(inspiration.title)
        appendLine(formattedDate)   // "2019.07.29 09:00"
        appendLine()
        appendLine(inspiration.content)
        if (tagsStr.isNotBlank()) {
            appendLine()
            append(tagsStr)
        }
    }.trimEnd()
}
```

**示例输出**：
```
就这样，《简记事App》诞生了！
2019.07.29 09:00

今天，持续3天的高密度多方向的努力，直至凌晨1点钟，把对想法的要求都实现了出来，最终我把它叫作"简记事"。

#简记事 #闲笔
```

**实现**：
- 使用 `LocalClipboardManager.current.setText(AnnotatedString(text))`
- 复制成功后弹 `Snackbar`「文本内容已复制到剪切板」，2 秒自动消失

### 4.2 编辑按钮

- 直接 `navController.navigate("inspiration_edit/${inspiration.id}")`
- 不需要任何额外处理

### 4.3 分享按钮

点击后弹出 `ModalBottomSheet`（`ShareInspirationSheet`），三个选项：

| 选项 | 图标 | 行为 |
|------|------|------|
| 保存到相册 | `Icons.Outlined.Image` | 截图 → 保存到 `getExternalFilesDir(DIRECTORY_PICTURES)` → Snackbar「已保存到相册：Inspiration_{timestamp}.png」 |
| 更多分享 | `Icons.Outlined.Share` | 启动系统 `Intent.ACTION_SEND` + PNG 附件（`FileProvider`） |
| 取消 | `Icons.Outlined.Close` | 关闭弹窗 |

## 5. HorizontalPager 左右滑动切换

```kotlin
val pagerState = rememberPagerState(
    initialPage = initialIndex,  // inspirationId 在列表中的位置
    pageCount = { inspirations.size }
)

HorizontalPager(
    state = pagerState,
    beyondViewportPageCount = 1,  // 预加载相邻页
    pageSpacing = 16dp,           // 卡片间距
    contentPadding = PaddingValues(horizontal = 18dp)
) { page ->
    InspirationViewCard(
        inspiration = inspirations[page],
        onImageClick = { index -> showImageGallery(index) }
    )
}
```

**行为**：
- 默认 `HorizontalPager` 在边界有回弹动画（`snapPosition`），不提示
- 切换时通过 `LaunchedEffect(pagerState.currentPage)` 同步当前 `inspiration`，三个功能按钮的回调随 currentPage 更新
- 切换时不重新创建 ViewModel，状态保留

## 6. 字数统计

```kotlin
fun countInspirationChars(inspiration: Inspiration): Int {
    val allText = buildString {
        append(inspiration.title)
        append(inspiration.content)
        append(parseTags(inspiration.tags).joinToString(""))
    }
    return allText.count { !it.isWhitespace() }
}
```

- 范围：标题 + 正文 + 标签，去除所有空白字符
- 中英文均按 1 个字符计数
- 实时计算（不需要持久化）

## 7. 图片预览（InspirationImageGallery）

```kotlin
@Composable
fun InspirationImageGallery(
    imagePaths: List<String>,
    initialIndex: Int,
    onDismiss: () -> Unit
) {
    Box(modifier = Modifier.fillMaxSize().background(Color.Black)) {
        HorizontalPager(state = rememberPagerState(initialIndex, pageCount = { imagePaths.size })) { i ->
            ZoomableImage(
                path = imagePaths[i],
                modifier = Modifier.fillMaxSize()
            )
        }
        IconButton(
            onClick = onDismiss,
            modifier = Modifier.align(Alignment.TopEnd).padding(16dp)
        ) {
            Icon(Icons.Close, "关闭", tint = Color.White)
        }
        // 顶部半透明栏：显示页码 "1/3"
        Text(
            text = "${initialIndex + 1}/${imagePaths.size}",
            color = Color.White,
            modifier = Modifier.align(Alignment.TopCenter).padding(top = 24dp)
        )
    }
}
```

**`ZoomableImage` 实现**：

```kotlin
@Composable
fun ZoomableImage(path: String, modifier: Modifier = Modifier) {
    var scale by remember { mutableStateOf(1f) }
    var offsetX by remember { mutableStateOf(0f) }
    var offsetY by remember { mutableStateOf(0f) }
    
    AsyncImage(
        model = path,
        contentDescription = null,
        modifier = modifier
            .pointerInput(Unit) {
                detectTransformGestures { _, pan, zoom, _ ->
                    scale = (scale * zoom).coerceIn(1f, 4f)
                    offsetX += pan.x
                    offsetY += pan.y
                }
            }
            .pointerInput(Unit) {
                detectTapGestures(
                    onTap = { /* 关闭 */ },
                    onDoubleTap = {
                        if (scale > 1f) {
                            scale = 1f; offsetX = 0f; offsetY = 0f
                        } else {
                            scale = 2f
                        }
                    }
                )
            }
            .graphicsLayer(
                scaleX = scale, scaleY = scale,
                translationX = offsetX, translationY = offsetY
            )
    )
}
```

**交互**：
- 双指捏合：缩放（1x ~ 4x）
- 单指拖动：平移（仅缩放 > 1x 时）
- 双击：放大/还原
- 单击：关闭预览

## 8. 截图实现（rememberGraphicsLayer）

使用 Compose 1.9 提供的官方截图 API：

```kotlin
val graphicsLayer = rememberGraphicsLayer()

Box(
    modifier = Modifier.drawWithContent {
        graphicsLayer.record { this@drawWithContent.drawContent() }
        drawLayer(graphicsLayer)
    }
) {
    InspirationViewCard(...)
}

// 截图
suspend fun captureInspiration(graphicsLayer: GraphicsLayer): Bitmap {
    awaitFrame()  // 等待当前帧绘制完成
    return graphicsLayer.toImageBitmap().asAndroidBitmap()
}
```

**实现要点**：
- `rememberGraphicsLayer()` 替代旧的 `captureBitmap` / `View.draw`
- 卡片内容用 `Modifier.drawWithContent` 包裹，将绘制内容录制到 graphicsLayer
- 截图范围：仅卡片内容（不包括 TopBar、字数徽章）
- 图片加载使用 `coil` 异步加载，需在截图前确保 `onSuccess` 回调已完成

**保存到相册**：

```kotlin
fun saveBitmapToGallery(context: Context, bitmap: Bitmap): Uri? {
    val picturesDir = context.getExternalFilesDir(Environment.DIRECTORY_PICTURES)
    val file = File(picturesDir, "Inspiration_${System.currentTimeMillis()}.png")
    return try {
        FileOutputStream(file).use { out ->
            bitmap.compress(Bitmap.CompressFormat.PNG, 100, out)
        }
        Uri.fromFile(file)
    } catch (e: Exception) {
        null
    }
}
```

**系统分享**：

```kotlin
fun shareBitmap(context: Context, uri: Uri) {
    val intent = Intent(Intent.ACTION_SEND).apply {
        type = "image/png"
        putExtra(Intent.EXTRA_STREAM, uri)
        addFlags(Intent.FLAG_GRANT_READ_URI_PERMISSION)
    }
    context.startActivity(Intent.createChooser(intent, "分享灵感"))
}
```

## 10. 编辑后刷新机制

**方案**：使用 `LifecycleEventObserver` 监听 `ON_RESUME` 事件

```kotlin
val lifecycleOwner = LocalLifecycleOwner.current
DisposableEffect(lifecycleOwner) {
    val observer = LifecycleEventObserver { _, event ->
        if (event == Lifecycle.Event.ON_RESUME) {
            viewModel.refresh()
        }
    }
    lifecycleOwner.lifecycle.addObserver(observer)
    onDispose { lifecycleOwner.lifecycle.removeObserver(observer) }
}
```

**为什么用 ON_RESUME 而不是 SavedStateHandle Result**：
- 用户从编辑页返回时，ViewModel 已存在
- 简单 `refresh()` 重新查 DB 即可，性能可接受（SQL 查询有索引）
- 不需要在编辑页额外写入 SavedStateHandle

## 11. 错误处理

| 场景 | 处理 |
|------|------|
| 灵感 ID 不存在 | 显示空状态 + 返回按钮 + Snackbar「灵感不存在或已删除」 |
| 图片文件丢失 | 图片占位 + 文字提示「图片已失效」 |
| 截图失败 | Snackbar「分享失败，请重试」 |
| 保存到相册失败 | Snackbar「保存失败，请检查存储权限」 |
| 复制失败 | 罕见，ClipboardManager 几乎不会失败，无处理 |
| 加载中 | 整个屏幕显示 `CircularProgressIndicator` |
| 数据库查询失败 | 显示错误状态 + 重试按钮 |

**统一错误展示**：使用 `SnackbarHostState` + `Snackbar` 统一处理
