# 骨架屏适配最新 UI 设计文档

> **日期**：2026-07-13
> **状态**：待审核
> **优先级**：中
> **范围**：待办区（TodoSkeleton）+ 灵感区（InspirationSkeleton）

---

## 1. 背景与问题

### 1.1 问题描述
待办区与灵感区的真实 UI 经过多轮迭代，布局结构已发生较大变化，而两处骨架屏仍停留在旧版布局，导致冷启动时骨架屏到真实内容过渡出现明显布局跳变。

### 1.2 现状差距

| 区域 | 问题 | 现状骨架 | 实际 UI |
|------|------|---------|---------|
| 待办 | 过滤按钮行 | 3 个胶囊按钮（全部/待办/已完成） | 已移除，改为可折叠分区头 |
| 待办 | 分区头 | 缺失 | 3 个 `CollapsibleSectionHeader`（置顶/待完成/已完成） |
| 待办 | 列表水平内边距 | 20.dp | 8.dp（`ZonedReorderableLazyColumn`） |
| 待办 | 卡片间距 | 12.dp | 8.dp（`itemSpacing`） |
| 待办 | 卡片圆角 | 20.dp（`SkeletonDefaults.CardShape`） | 16.dp（`SwipeableTodoBox.cornerRadiusDp`） |
| 灵感 | 布局形态 | 简单卡片堆叠 | 时间线（左侧日期栏 + 节点 + 竖线 + 右侧内容区） |
| 灵感 | 列表水平内边距 | 20.dp | 18.dp |
| 灵感 | 列表项间距 | 12.dp | 18.dp |

### 1.3 真实 UI 结构参考

**待办区（HomeScreen）**：
- `SearchBar`（horizontal=20.dp, bottom=`ui_search_bar_bottom_margin`）
- `ZonedReorderableLazyColumn`（horizontal=8.dp, itemSpacing=8.dp）：
  - `PinnedDivider` → 置顶待办项（`SwipeableTodoBox` + `TodoListItem`，圆角 16.dp）
  - `PendingDivider` → 待完成待办项
  - `CompletedDivider`（绿色 #7EC8A0）→ 已完成待办项
  - footer 80.dp 留白

**灵感区（InspirationScreen）**：
- `SearchBar`（horizontal=20.dp）
- Spacer 8.dp
- `LazyColumn`（horizontal=18.dp, spacedBy=18.dp）：
  - `TimelineInspirationItem`（左侧 50.dp 日期栏 + 7.dp 间距 + 6.dp 节点 + 7.dp 间距 + 右侧内容区）
  - footer 80.dp 留白

## 2. 设计目标

| 目标 | 说明 |
|------|------|
| **消除布局跳变** | 骨架屏布局与真实页面高度一致，冷启动过渡无位移 |
| **结构对齐** | 待办区含 3 分区头；灵感区为时间线布局 |
| **主题适配** | 沿用 `SkeletonDefaults`，亮色/深色自动适配 |
| **最小改动** | 不侵入真实组件，仅更新骨架屏文件 + 扩展通用组件 |

---

## 3. 技术方案

### 3.1 实现方案：独立常量 + 文档锚点（方案 A）

骨架屏文件内定义独立布局常量，每项常量添加注释锚点指向真实组件来源。

**理由**：骨架屏本质是视觉占位，目标是布局相似而非像素级复用。独立常量方式改动最小、不侵入真实组件，漂移风险通过注释提示控制。

### 3.2 文件变更清单

| 文件路径 | 操作 | 说明 |
|---------|------|------|
| `app/src/main/java/com/corgimemo/app/ui/components/Skeleton.kt` | 修改 | 新增 `SkeletonSectionHeader`、`SkeletonTimelineNode` 通用组件 |
| `app/src/main/java/com/corgimemo/app/ui/screens/home/TodoSkeleton.kt` | 重写 | 移除过滤按钮行，新增 3 分区头 + 分区卡片 |
| `app/src/main/java/com/corgimemo/app/ui/screens/inspiration/InspirationSkeleton.kt` | 重写 | 改为时间线布局 |

**不改动**：`SpecialDateSkeleton.kt`、`HomeScreen.kt`、`InspirationScreen.kt`、ViewModel。

### 4.4 单卡片骨架（TodoItemSkeleton）

> **注意**：不复用 `SkeletonCard` 组件（其固定使用 `CardShape` 20.dp 圆角），改为 `Box` + `RoundedCornerShape(16.dp)` + `background(SkeletonColor)` + `padding(16.dp)` 手动构建，以匹配 `SwipeableTodoBox.cornerRadiusDp = 16.dp`。

- 外层 `Box`（圆角 `RoundedCornerShape(16.dp)`，background=`SkeletonColor`，padding=16.dp）
- `Row`（verticalAlignment = Top）：
  - 左侧 `SkeletonCircle(24.dp)` 复选框占位
  - 12.dp 间距
  - 右侧内容列（weight 1，spacedBy 8.dp）：
    - 标题行 `SkeletonText(width=0.7f, height=20.dp)`
    - 内容行 `SkeletonText(width=0.9f, height=14.dp)`
    - 元信息行（spacedBy 12.dp）：分类标签 `SkeletonButton(50.dp, 24.dp)` + 时间 `SkeletonText(0.25f, 12.dp)`

### 4.5 分区头骨架（SkeletonSectionHeader）

模拟 `CollapsibleSectionHeader` 布局：
- `Row`（padding horizontal=16.dp, vertical=8.dp）
- 箭头图标占位（20.dp，固定展开态 ▼，使用 `Icons.Filled.KeyboardArrowDown`，rotate=0f）
- 6.dp 间距
- 标签文字占位（如"置顶"/"待完成"/"已完成"，`bodyMedium` 字号）
- 计数占位（`(▭)` 形式，灰色块）

### 5.4 时间线条目骨架（TimelineItemSkeleton）

- 外层 `Box` + `drawBehind`：绘制竖线（X=60.dp，起点 Y=-18.dp 向上延伸覆盖间距，终点=Item 高度）。第一条向上延伸会被 LazyColumn 顶部裁剪，与真实组件行为一致
- `Row`：
  - 左侧日期栏（50.dp 宽，Column 水平居中）：
    - 年月占位 `SkeletonText(width=0.6f, height=12.dp)`（仅第一条显示）
    - 日期数字占位 `SkeletonText(width=0.5f, height=24.dp)`（仅第一条显示）
  - 7.dp 间距
  - 节点 `SkeletonTimelineNode(6.dp, primary)`（Y 偏移 11.dp 对齐标题中心）
  - 7.dp 间距
  - 右侧内容区（weight 1，Column）：
    - 标题 `SkeletonText(width=0.85f, height=16.dp)`
    - 4.dp 间距 + 时间 `SkeletonText(width=0.3f, height=11.dp)`
    - 9.dp 间距 + 正文行1 `SkeletonText(width=1f, height=14.dp)`
    - 正文行2 `SkeletonText(width=0.7f, height=14.dp)`
    - 7.dp 间距 + 标签行（2 个 `SkeletonButton(40.dp, 20.dp)`，spacedBy 6.dp）

## 6. 通用组件扩展（Skeleton.kt）

### 6.1 新增 SkeletonSectionHeader

```kotlin
/**
 * 可折叠分区头骨架
 * 模拟 CollapsibleSectionHeader 的布局（箭头 + 文字 + 计数），无点击行为
 *
 * @param label 区头标签文字（如"置顶"/"待完成"/"已完成"）
 * @param color 文字与箭头颜色
 * @param modifier 修饰符
 */
@Composable
fun SkeletonSectionHeader(
    label: String,
    color: Color,
    modifier: Modifier = Modifier
)
```

布局：`Row`（padding horizontal=16.dp, vertical=8.dp）→ 箭头图标(20.dp, ▼) + 6.dp 间距 + 文字(label) + 6.dp 间距 + 计数占位块(24.dp x 14.dp)。

### 6.2 新增 SkeletonTimelineNode

```kotlin
/**
 * 时间线节点骨架
 * 模拟 TimelineInspirationItem 的节点圆点
 *
 * @param diameter 节点直径
 * @param color 节点颜色
 * @param modifier 修饰符
 */
@Composable
fun SkeletonTimelineNode(
    diameter: Dp,
    color: Color,
    modifier: Modifier = Modifier
)
```

布局：`Box`（size=diameter, clip=CircleShape, background=color），无 shimmer 动画（节点为纯色实心点）。

### 6.3 现有组件保持不变

`SkeletonText`、`SkeletonCard`、`SkeletonSearchBar`、`SkeletonCircle`、`SkeletonButton`、`skeleton()` 修饰符均不改动。

## 7. 边界情况与错误处理

| 场景 | 处理方式 |
|------|---------|
| 骨架屏不可点击 | 所有占位组件不加 `clickable`，避免拦截用户操作 |
| 亮色/深色主题 | 沿用 `SkeletonDefaults.SkeletonColor`（基于 `surfaceVariant`），自动适配 |
| shimmer 性能 | 保持现有 `shimmerEnabled = true` 默认开启，骨架屏仅在数据未初始化时短暂显示 |
| 时间线竖线裁剪 | 第一条向上延伸 18.dp 会被 LazyColumn 顶部裁剪（与真实组件行为一致），无需特殊处理 |
| 日期栏仅首条显示 | 第二条时间线条目左侧日期栏保留 50.dp 占位宽度，不绘制文字占位 |

## 8. 验收标准

| 标准 | 验证方法 |
|------|---------|
| **无布局跳变** | 冷启动时骨架屏到真实内容过渡，列表项位置无明显位移 |

| **主题适配** | 亮色/深色模式下骨架颜色正确 |
| **无功能影响** | 骨架屏不拦截点击，数据加载后正常切换到真实列表 |
