# 日期侧滑栏类型筛选与帧动画设计

> **日期**: 2026-07-14
> **状态**: 已批准
> **参考**: 待办页面侧滑栏的设计与交互模式

## 1. 背景与目标

### 1.1 当前问题

日期页面侧滑栏（`DateTypeFilterSection`）存在以下问题：
1. **筛选选项不实用**：当前按时间状态（倒计时/正计时/已归档）筛选，但该筛选在 MainScreen 中未接线（`selectedDateType` 和 `onDateTypeClick` 使用默认值，点击无效果）
2. **缺少类型分类**：用户无法按日期类型（生日/纪念日/节日等）筛选，而 `DateCategory` 枚举已存在但未在侧滑栏使用
3. **无自定义类型能力**：不像待办页面的分组管理，日期类型无法自定义创建
4. **柯基头像为静态 emoji**：`UserProfileSection` 使用 `🐕` 文本 emoji，未利用已有的帧动画基础设施

### 1.2 目标

1. 将侧滑栏筛选从 `DateGroup`（时间状态）替换为 `DateCategory`（类型分类）+ 自定义类型
2. 添加"添加类型"按钮，支持自定义类型的完整管理（增/改/删）
3. 将所有页面侧滑栏左上角的柯基 emoji 替换为趴卧姿态帧动画
4. 参考待办页面侧滑栏的交互模式，保持一致性

### 1.3 设计约束

- 主屏幕仍按 `DateGroup`（倒计时/正计时/已归档）分组展示，**不变更主屏分组逻辑**
- 类型筛选在主屏分组之前执行（先筛选类型，再按时间分组）
- 自定义类型持久化采用 Room 实体（与待办页 `Category` 模式一致）
- 帧动画使用已有的 `FrameAnimation` 组件和 `AnimationType.LIE` 资源映射

## 2. 数据层设计

### 2.1 CustomDateType Entity

**新建文件**: `app/src/main/java/com/corgimemo/app/data/model/CustomDateType.kt`

```kotlin
@Entity(tableName = "custom_date_types")
data class CustomDateType(
    @PrimaryKey(autoGenerate = true)
    val id: Long = 0,
    val name: String,
    @ColumnInfo(defaultValue = "📅")
    val emoji: String = "📅",
    val sortOrder: Int = 0,
    val createdAt: Long = System.currentTimeMillis()
)
```

**字段说明**:
- `id`: 自增主键，用作 `SpecialDate.category` 中 `"CUSTOM:<id>"` 的标识
- `name`: 类型名称（如"宠物生日"），用户可修改
- `emoji`: 类型图标 emoji，添加时可选，默认 `📅`
- `sortOrder`: 排序位置，侧滑栏中按此字段 ASC 排列
- `createdAt`: 创建时间戳

### 2.2 CustomDateTypeDao

**新建文件**: `app/src/main/java/com/corgimemo/app/data/local/db/CustomDateTypeDao.kt`

```kotlin
@Dao
interface CustomDateTypeDao {
    @Insert
    suspend fun insert(customType: CustomDateType): Long

    @Update
    suspend fun update(customType: CustomDateType)

    @Query("SELECT * FROM custom_date_types ORDER BY sortOrder ASC, createdAt ASC")
    fun getAllCustomDateTypes(): Flow<List<CustomDateType>>

    @Query("SELECT * FROM custom_date_types WHERE id = :id")
    suspend fun getById(id: Long): CustomDateType?

    @Delete
    suspend fun delete(customType: CustomDateType)
}
```

### 2.4 SpecialDate.category 存储约定

现有 `SpecialDate.category` 为 `String` 类型，**无需修改 Entity**:

| 类型 | 存储格式 | 示例 |
|------|---------|------|
| 内置类型 | `DateCategory.name()` | `"BIRTHDAY"`, `"ANNIVERSARY"` |
| 自定义类型 | `"CUSTOM:<id>"` | `"CUSTOM:42"` |

**重命名优势**: 重命名自定义类型只需更新 `custom_date_types` 表，无需批量更新 `special_dates` 记录（ID 不变）。

**删除处理**: 删除自定义类型时:
1. 将该类型下所有 `special_dates.category` 批量回退为 `"OTHER"`
2. 然后删除 `custom_date_types` 记录

## 3. 侧滑栏 UI 设计

### 3.1 DateTypeFilterSection 改造

**修改文件**: `app/src/main/java/com/corgimemo/app/ui/components/AppDrawer.kt`

**参数变更**:

移除:
- `selectedDateType: DateGroup?`
- `onDateTypeClick: (DateGroup?) -> Unit`

新增:
- `selectedDateCategory: String?` — null=全部, "BIRTHDAY"=内置, "CUSTOM:42"=自定义
- `dateCountByCategory: Map<String, Int>` — 每个类型对应的日期计数
- `customDateTypes: List<CustomDateType>` — 从 DB 加载的自定义类型列表
- `onDateCategoryClick: (String?) -> Unit` — 类型点击回调
- `onAddCustomTypeClick: () -> Unit` — 添加类型按钮回调
- `onCustomTypeAction: (DateTypeAction) -> Unit` — 自定义类型操作回调

**渲染逻辑**:
1. **"全部日期"** 项 → `selectedDateCategory == null` 时选中，显示总计数
2. **8 个内置类型** → 遍历 `DateCategory.values()`，使用 `displayName` / `emoji`，无菜单按钮（显示 `›` 箭头），不可删除
3. **自定义类型** → 遍历 `customDateTypes`，使用 `name` / `emoji`，有菜单按钮（显示 `⋮`），可重命名/删除
4. **"添加类型"按钮** → 底部 `AddCategoryButton(text = "添加类型", onClick = onAddCustomTypeClick)`

### 3.2 管理弹窗（复用现有组件）

| 操作 | 组件 | 复用方式 |
|------|------|---------|
| 添加类型 | `AddCategoryDialog` | 复用，新增可选参数 `showEmojiPicker: Boolean = false`；日期类型传 `true`，待办分组传 `false`（默认）保持向后兼容 |
| 重命名类型 | `RenameCategoryDialog` | 直接复用（仅传名称，无 emoji 修改） |
| 删除确认 | `DeleteCategoryConfirmDialog` | 复用，文案改为"该类型下的日期将变为'其他'类型" |
| 操作菜单 | 新建 `DateTypeOperationSheet` | 不复用 `CategoryOperationSheet`（参数类型不兼容）；新建仅含重命名/删除两项的 BottomSheet |

**AddCategoryDialog 向后兼容扩展**:
- 新增可选参数: `showEmojiPicker: Boolean = false`, `onConfirm: (name: String, emoji: String) -> Unit = { name, _ -> onConfirmName(name) }`
- 保留原有 `onConfirm: (String) -> Unit` 作为 `onConfirmName`，待办页面调用不变
- 当 `showEmojiPicker = true` 时，在名称输入框下方显示 emoji 选择行
- 预设 12 个常用 emoji: 📅 🎂 💕 🎉 🌱 📚 💼 🎮 ✈️ 🐹 🏠 ⭐
- 默认选中 `📅`

**DateTypeOperationSheet**:
```kotlin
@Composable
fun DateTypeOperationSheet(
    sheetState: SheetState = rememberModalBottomSheetState(skipPartiallyExpanded = true),
    customType: CustomDateType,
    onRename: () -> Unit,
    onDelete: () -> Unit,
```

仅含"编辑类型"和"删除类型"两项，不含"置顶"。

### 4.2 新增方法

### 4.3 筛选逻辑

当前流程: `allDates → groupedByDates (按DateGroup分组) → displayDates`

新增类型筛选后: `allDates → filterByCategory(selectedDateCategory) → groupedByDates → displayDates`

```kotlin
val filteredDates = selectedDateCategory?.let { category ->
    allDates.filter { it.category == category }
} ?: allDates  // null = 全部
```

**关键点**: 主屏幕的 `DateGroup` 分组逻辑不变，类型筛选在分组之前执行。

### 4.4 计数逻辑

```kotlin
val dateCountByCategory = allDates.combine(customDateTypes) { dates, customTypes ->
    val counts = mutableMapOf<String, Int>()
    DateCategory.values().forEach { dc ->
        counts[dc.name] = dates.count { it.category == dc.name }
    }
    customTypes.forEach { ct ->
        counts["CUSTOM:${ct.id}"] = dates.count { it.category == "CUSTOM:${ct.id}" }
    }
    counts
}
```

### 4.5 删除自定义类型的处理

```kotlin
fun deleteCustomType(id: Long) {
    viewModelScope.launch {
        // 1. 将该类型下所有 special_dates.category 回退为 "OTHER"
        specialDateDao.updateCategoryByCategory("CUSTOM:$id", "OTHER")
        // 2. 删除 custom_date_types 记录
        customDateTypeDao.delete(customDateTypeDao.getById(id)!!)
    }
}
```

## 5. 帧动画集成

### 5.1 UserProfileSection 改造

**修改文件**: `app/src/main/java/com/corgimemo/app/ui/components/AppDrawer.kt` (L194-L234)

将 `Box + Text("🐕")` 替换为 `FrameAnimation`:

```kotlin
FrameAnimation(
    animationType = AnimationType.LIE,
    fps = 8,
    isLooping = true,
    modifier = Modifier.size(48.dp)
)
```

### 5.2 帧率选择理由

| 姿态 | 帧数 | fps | 理由 |
|------|------|-----|------|
| 趴卧(LIE) | 3帧 | 8fps | 缓慢呼吸感，每帧125ms，3帧循环周期~375ms |
| 奔跑(RUN) | 4帧 | 15fps | 快速动作需要流畅 |
| 摇尾巴(WAG) | 4帧 | 12fps | 中等速度 |

### 5.3 生命周期

- `FrameAnimation` 内部使用 `LaunchedEffect`，当 `UserProfileSection` 进入组合（侧滑栏打开）时自动开始播放
- 侧滑栏关闭时，`UserProfileSection` 离开组合，`LaunchedEffect` 自动取消，动画停止
- 无需额外生命周期管理

### 5.4 所有页面共享

`UserProfileSection` 是 `AppDrawerContent` 的公共部分，所有 4 个 Tab（待办/灵感/日期/我的）的侧滑栏都会显示帧动画，无需重复修改。

### 5.5 资源确认

`AnimationType.LIE` 已在 `AnimationResourceManager.kt` 中映射到:
- `R.drawable.corgi_lie_3frames_01`
- `R.drawable.corgi_lie_3frames_02`
- `R.drawable.corgi_lie_3frames_03`

这三个文件已确认存在于 `app/src/main/res/drawable/` 目录。无需新增动画资源。
