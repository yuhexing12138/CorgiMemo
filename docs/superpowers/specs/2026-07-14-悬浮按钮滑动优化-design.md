## 1. 背景与问题

悬浮柯基按钮当前实现存在三个交互异常,严重影响拖动体验:

| 症状 | 表现 |
|---|---|
| 1. 滑动阻尼感 | 滑动过程明显卡顿、不流畅,有"被拽住"的感觉 |
| 2. 跟不上手指 | 向右滑动一段距离后,按钮位置与手指位置出现显著偏差,且差距随滑动持续增大 |
| 3. 无法到达右侧 | 按钮无法滑动至屏幕右侧区域,即使手指已触达右边缘 |

## 2. 根因分析

### 2.1 阻尼感根因

| 编号 | 位置 | 问题 |
|---|---|---|
| A | `FloatingCorgiButton.kt` L258、L289-290 | `MAX_DELTA_PER_FRAME = 30f` 把每帧位移强制截断到 30px 内,快速滑动时单帧真实位移(80~120px)被截掉 60%+ |
| B | `FloatingCorgiButton.kt` L310-311 | 像素转 dp 在 xxhdpi(3.0x)屏幕上进一步缩小位移,30px ≈ 10dp |
| C | `FloatingCorgiButton.kt` L243 | 使用 `padding` 而非 `offset`,触发 measure + layout,高频更新掉帧 |

### 2.2 跟不上手指根因(核心)

**位置:`FloatingCorgiButton.kt` L287-292**

```kotlin
val rawDeltaX = currentX - prevX
val deltaX = rawDeltaX.coerceIn(-30f, 30f)   // 截断
prevX = currentX                               // prevX 直接跳到 currentX
```

这是**累积误差的根源**。单帧内手指移动 100px 时:
- `rawDeltaX = 100`,`deltaX` 被截成 30
- 但 `prevX` 直接更新为 `currentX`(跳了 100)
- **被截掉的 70px 永久丢失**,后续帧不会补偿

多次丢帧后,手指已到屏幕右边,按钮却只移动了一半,出现"永远追不上"的偏差。向右滑特别明显是因为右滑通常速度更快、单帧位移更大,被截断更多。

### 2.3 无法到达右侧根因

| 编号 | 说明 |
|---|---|
| A | 2.2 节截断累积导致总位移不足,即使手指滑到屏幕右边缘,按钮累计移动量也远小于手指 |
| B | `snapToEdge()`(L331)在松手时强制吸附到左/右边缘,但吸附逻辑本身正确,非主因 |
| C | `configuration.screenWidthDp` 与 `Box(fillMaxSize)` 实际宽度可能不一致(状态栏/导航栏 insets 偏差),边界计算偏差 |

### 2.4 本质问题

当前实现采用"**逐帧增量 delta + 截断限速**"架构,是三个症状的共同根因。必须改为"**绝对位置追踪**"架构。

## 3. 设计决策

经与用户确认,本次优化采用以下决策:

| 决策项 | 选择 | 理由 |
|---|---|---|
| 贴边吸附 | **保留** | 符合现有设计文档约定,改动小 |
| 布局容器 | **改为 BoxWithConstraints** | 获取真实可用尺寸,解决 insets 偏差 |
| 快速滑动手势 | **去掉** `onSwipeLeft`/`onSwipeRight` | 消除手势冲突,长按阈值可恢复正常值 |
| 手势算法 | **方案 B:绝对位置手动追踪** | 彻底解决跟丢问题,可控性最高 |

### 3.1 手势算法方案对比

| 方案 | 核心思路 | 跟手精度 | 复杂度 | 风险 |
|---|---|---|---|---|
| A. `detectDragGestures` | 用 Compose 内置 API,内部绝对位置追踪 | 高 | 低 | 长按判定需手动封装 |
| **B. 绝对位置手动追踪(采用)** | 记录按下时手指-按钮偏移,每帧用 `手指绝对位置 - 偏移量` 定位 | 最高 | 中 | 需手动处理坐标系 |
| C. 修复现有增量逻辑 | 移除截断,改用累积 | 中 | 低 | 极端丢帧仍会偏移 |

**采用方案 B 的理由**:
1. 彻底解决"跟不上手指":绝对坐标直接定位,无截断丢失,丢帧后下一帧立即追上
2. 可控性最高:精确控制按下偏移、边界 clamp、吸附逻辑
3. 与现有 `awaitPointerEventScope` 结构兼容,只改算法不换 API
4. 方案 A 的 `dragAmount` 在某些 Compose 版本仍有累积误差

## 4. 详细设计

### 4.1 架构调整

#### 4.1.1 布局层

外层容器从 `Box(modifier.fillMaxSize())` 改为 `BoxWithConstraints`:

```kotlin
BoxWithConstraints(modifier = modifier.fillMaxSize()) {
    val containerWidthPx = with(density) { maxWidth.toPx() }
    val containerHeightPx = with(density) { maxHeight.toPx() }
    // ... 所有边界计算使用 containerWidthPx / containerHeightPx
}
```

替换原有的 `configuration.screenWidthDp` / `screenHeightDp`,确保与 `fillMaxSize` 实际可用尺寸一致。

#### 4.1.2 位置存储

位置状态从 dp 改为像素 Int:

```kotlin
// 旧
var paddingStartDp by remember { mutableStateOf(defaultPaddingStart) }
var paddingBottomDp by remember { mutableStateOf(defaultPaddingBottom) }

// 新
var offsetX by remember { mutableStateOf(0) }
var offsetY by remember { mutableStateOf(0) }
```

布局修饰符从 `padding` 改为 `offset`:

```kotlin
// 旧
Surface(modifier = Modifier.padding(start = paddingStartDp, bottom = paddingBottomDp) ...)

// 新
Surface(modifier = Modifier.offset { IntOffset(offsetX, offsetY) } ...)
```

`offset` 不触发重测量,性能显著优于 `padding`,且 `offset { lambda }` 版本每帧重组开销最小。

#### 4.1.3 坐标系统一

所有内部计算统一在**像素坐标系**下完成。仅在以下边界转换:
- **持久化**:像素 → 百分比(DataStore)
- **对外回调**:`onPositionChanged(percentX, percentY)` 保持百分比接口不变

### 4.2 手势处理核心算法

#### 4.2.1 手势简化

去掉 `onSwipeLeft`/`onSwipeRight` 后,只剩两种手势:

| 手势 | 判定条件 |
|---|---|
| 点击 | 按下→抬起,未进入拖动模式,且总位移 < 8px 且时长 < 200ms |
| 长按拖动 | 按下后停留 > 200ms,或按下后位移 > 8px(任一满足即进入拖动) |

#### 4.2.2 长按阈值恢复

长按阈值从 100ms 恢复到 **200ms**。原 100ms 是为快速滑动让步(避免右滑被 `onSwipeRight` 截胡),去掉快速滑动后无需此妥协,200ms 降低误触率。

#### 4.2.3 绝对位置追踪算法

**关键改动:`pointerInput` 挂载位置从 48dp 的 `Surface` 改为外层 `BoxWithConstraints` 的 `Box`**,使 `change.position` 成为相对整个可用区域的坐标,直接可用。

**命中区域判断**:由于 `pointerInput` 挂在 `fillMaxSize` 的 Box 上,按下点若不在按钮矩形内必须直接 `continue`,不进入手势循环,避免拦截下层内容点击:

```kotlin
BoxWithConstraints(modifier = modifier.fillMaxSize()) {
    // ... 容器尺寸、状态声明

    Surface(
        modifier = Modifier
            .offset { IntOffset(offsetX, offsetY) }
            .size(buttonSize)
            // ... 视觉修饰
    ) {
        FrameAnimation(...)
    }

    // 手势处理挂在外层 Box 上,position 即容器内绝对坐标
    Box(
        modifier = Modifier
            .fillMaxSize()
            .pointerInput(Unit) {
                awaitPointerEventScope {
                    while (true) {
                        val down = awaitPointerEvent()
                        val downChange = down.changes.firstOrNull() ?: continue

                        val downX = downChange.position.x
                        val downY = downChange.position.y

                        // 0. 命中区域判断:按下点必须在按钮矩形内,否则不消费事件
                        val insideButton = downX in offsetX..(offsetX + buttonSizePx) &&
                                           downY in offsetY..(offsetY + buttonSizePx)
                        if (!insideButton) continue

                        // 1. 按下时记录手指相对按钮左上角的偏移
                        val touchOffsetX = downX - offsetX
                        val touchOffsetY = downY - offsetY

                        var dragStartTime = System.currentTimeMillis()
                        var totalDragX = 0f
                        var totalDragY = 0f
                        var isDragging = false

                        while (true) {
                            val event = awaitPointerEvent()
                            val change = event.changes.firstOrNull() ?: break
                            if (change.isConsumed) break

                            val currentX = change.position.x
                            val currentY = change.position.y

                            if (!isDragging) {
                                totalDragX += currentX - downX
                                totalDragY += currentY - downY

                                val elapsed = System.currentTimeMillis() - dragStartTime
                                val moved = abs(totalDragX) > 8f || abs(totalDragY) > 8f
                                if (elapsed > 200 || moved) {
                                    isDragging = true
                                }
                            }

                            if (isDragging) {
                                // 2. 绝对位置直接定位:无截断、无累积误差
                                val targetX = (currentX - touchOffsetX)
                                    .coerceIn(minX, maxX)
                                val targetY = (currentY - touchOffsetY)
                                    .coerceIn(minY, maxY)
                                offsetX = targetX.toInt()
                                offsetY = targetY.toInt()
                                change.consume()
                            }

                            if (!change.pressed) {
                                if (isDragging) {
                                    isDragging = false
                                    snapToEdge()  // 保留贴边吸附
                                } else if (abs(totalDragX) < 8f && abs(totalDragY) < 8f) {
                                    // 点击
                                    coroutineScope.launch {
                                        playClickAnimation()
                                        onClick()
                                    }
                                }
                                break
                            }
                        }
                    }
                }
            }
    )
}
```

#### 4.2.4 核心改进总结

| 旧实现 | 新实现 | 效果 |
|---|---|---|
| `deltaX = coerceIn(-30, 30)` 截断 | 无截断,直接用 `currentX - touchOffsetX` | 阻尼感消失 |
| `prevX = currentX` 导致丢失累积 | 每帧绝对计算,不依赖上一帧 | 跟手延迟 = 0 |
| `padding` 触发重布局 | `offset { lambda }` 仅位移 | 性能提升 |
| `MAX_DELTA_PER_FRAME` 防瞬移 | 绝对位置天然防瞬移(手指在哪按钮在哪) | 移除该常量 |

### 4.3 边界与吸附

#### 4.3.1 安全边界(像素)

```kotlin
val buttonSizePx = with(density) { buttonSize.toPx() }
val horizontalMarginPx = with(density) { horizontalMargin.toPx() }
val topSafeMarginPx = with(density) { topSafeMargin.toPx() }
val bottomSafeMarginPx = with(density) { bottomSafeMargin.toPx() }

val minX = horizontalMarginPx
val maxX = containerWidthPx - buttonSizePx - horizontalMarginPx
val minY = topSafeMarginPx
val maxY = containerHeightPx - buttonSizePx - bottomSafeMarginPx
```

#### 4.3.2 贴边吸附(保留,改为平滑动画)

```kotlin
fun snapToEdge() {
    val halfWidth = containerWidthPx / 2
    val targetX = if (offsetX < halfWidth) minX else maxX

    coroutineScope.launch {
        playBounceAnimation()
        // 用 Animatable + spring 平滑过渡,而非直接赋值
        val animX = Animatable(offsetX.toFloat())
        animX.animateTo(
            targetValue = targetX,
            animationSpec = spring(
                dampingRatio = Spring.DampingRatioMediumBouncy,
                stiffness = Spring.StiffnessMediumLow
            )
        )
        offsetX = animX.value.toInt()

        // 持久化(百分比)
        val percentX = offsetX / containerWidthPx
        val percentY = 1f - offsetY / containerHeightPx
        onPositionChanged(percentX, percentY)
    }
}
```

#### 4.3.3 持久化兼容

DataStore 中已存的百分比坐标格式不变,无需 migration。首次加载时按新尺寸重新 clamp:

```kotlin
LaunchedEffect(initialPosition) {
    if (!isPositionInitialized && initialPosition != null) {
        val (xPercent, yPercent) = initialPosition
        offsetX = (xPercent * containerWidthPx).toInt().coerceIn(minX.toInt(), maxX.toInt())
        offsetY = ((1f - yPercent) * containerHeightPx).toInt().coerceIn(minY.toInt(), maxY.toInt())
    } else if (!isPositionInitialized) {
        // 默认左下角
        offsetX = minX.toInt()
        offsetY = maxY.toInt()
    }
    isPositionInitialized = true
}
```

### 4.4 动画参数

| 参数 | 旧值 | 新值 | 说明 |
|---|---|---|---|
| 长按阈值 | 100ms | **200ms** | 不再为快速滑动让步,降低误触 |
| 拖动启动位移阈值 | 无(仅时长) | **8px** | 超过即视为拖动,响应更快 |
| 拖动时缩放 | 1.2x | 1.2x | 不变 |
| 吸附动画 | 直接赋值 | `spring(DampingRatioMediumBouncy, StiffnessMediumLow)` 约 250ms | 平滑过渡 |
| 回弹动画 | 1.0→0.9→1.0 | 1.0→0.9→1.0 | 不变 |
| 呼吸动画 | 0.95~1.05,2s | 0.95~1.05,2s | 不变 |
| 点击缩放 | 1.0→1.2→1.0,300ms | 1.0→1.2→1.0,300ms | 不变 |
| `MAX_DELTA_PER_FRAME` | 30px | **移除** | 绝对位置算法无需限速 |

## 5. 验证标准

实现完成后,以下场景必须通过:

| 场景 | 预期结果 |
|---|---|
| 慢速拖动 | 按钮完全贴住手指,无阻尼感 |
| 快速右滑 | 按钮紧跟手指,无偏差累积 |
| 拖到屏幕右边缘 | 按钮到达 `maxX`(距右边缘 16dp),不被截断 |
| 拖到屏幕左边缘 | 按钮到达 `minX`(距左边缘 16dp) |
| 拖到顶部 | 按钮到达 `minY`(距顶部 56dp,避开状态栏) |
| 拖到底部 | 按钮到达 `maxY`(距底部 88dp,避开导航栏/FAB) |
| 松手 | 平滑 spring 动画吸附到最近左/右边缘 |
| 短按(< 200ms,位移 < 8px) | 触发点击缩放 + `onClick` |
| 长按后松手(未移动) | 进入拖动模式,松手吸附边缘 |
| App 重启 | 位置从 DataStore 恢复,百分比格式不变 |
| 旋转屏幕/配置变更 | 位置按新尺寸重新 clamp,不越界 |

## 7. 风险与缓解

| 风险 | 缓解 |
|---|---|
| `BoxWithConstraints` 的 `maxWidth` 在首次组合时可能为 0 | 用 `LaunchedEffect` 监听尺寸变化,尺寸 > 0 后再初始化位置 |
| `offset { lambda }` 在某些 Compose 版本可能与 `shadow` 冲突 | 实现时验证,必要时调整修饰符顺序(shadow 在 offset 之前) |

| ~~`pointerInput` 挂在外层 Box 可能拦截下层点击~~ | **已解决**:4.2.3 节命中区域判断 `insideButton` 保证非按钮区域不消费事件 |
