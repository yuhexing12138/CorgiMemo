# 搜索框显隐动画跳跃感修复 设计文档

## 背景与问题

待办页搜索框当前采用 `Animatable + LaunchedEffect + snapTo` 架构响应滚动显隐。在添加置顶按钮后,用户反馈手指上滑搜索框隐藏时和手指下滑搜索框出现时,过程有跳跃感,不平滑。

## 根因分析

### 当前架构(问题代码)

[HomeScreen.kt#L533-L574](file:///c:/Users/EDY/Desktop/CorgiMemo/app/src/main/java/com/corgimemo/app/ui/screens/home/HomeScreen.kt#L533-L574) 的状态链路:

```
LazyListState.firstVisibleItemScrollOffset (每帧变化)
    ↓ derivedStateOf
scrollOffsetFromTop
    ↓ LaunchedEffect 重启 (异步, 1-3 帧延迟)
searchRevealProgress.snapTo(target) (协程调度)
    ↓
searchRevealProgress.value (滞后)
    ↓ layout/graphicsLayer 读取
UI 渲染
```

### 跳跃感根本原因

| # | 问题 | 影响 |
|---|------|------|
| 1 | **`LaunchedEffect + snapTo` 是异步的** | `scrollOffsetFromTop` 每帧变化时,`LaunchedEffect` 重启(取消旧协程 + 启动新协程 + 执行 body)有 1-3 帧延迟。`searchRevealProgress.value` 滞后于真实滚动,layout/graphicsLayer 用旧值渲染,产生跳跃 |
| 2 | **`Animatable.snapTo` 在 effect 协程中执行** | `snapTo` 本身是同步赋值,但被 `LaunchedEffect` 包裹后变成异步。每帧滚动 → 重启 effect → snapTo 延迟,快速滚动时累积明显 |
| 3 | **`derivedStateOf` + `LaunchedEffect` 的间接性** | 链条过长,无法做到 1:1 跟手 |

**本质**: 搜索框高度应该 1:1 跟随滚动 delta(像 iOS 大标题那样),但当前架构通过 `Animatable + LaunchedEffect + snapTo` 中转,引入了不可避免的协程调度延迟。

## 设计目标

1. **滚动中**: 搜索框高度 1:1 跟随滚动 delta,无协程延迟
2. **滚动停止**: 保留吸附到端点(0f 或 1f)的动画,过渡平滑无跳跃
3. **搜索词输入/清空**: 保留现有行为(立即显示/基于 progress 吸附)
4. **视觉行为不变**: 上边缘固定,底部边缘收缩/展开 + 淡入淡出
5. **级联偏移同步**: [L843](file:///c:/Users/EDY/Desktop/CorgiMemo/app/src/main/java/com/corgimemo/app/ui/screens/home/HomeScreen.kt#L843) 列表的级联偏移也用新 progress

## 架构设计

### 状态管理:两阶段分离

将"滚动跟手"和"停止吸附"两个阶段的状态源彻底分离:

```
┌─────────────────────────────────────────────────┐
│  阶段 1: 滚动中 (isScrollInProgress = true)     │
│  ─────────────────────────────────────────       │
│  progress = derivedStateOf {                    │
│      1f - scrollOffsetFromTop / searchBarHeight │
│  }.coerceIn(0f, 1f)  ← 同步, 1:1 跟手           │
│                                                  │
│  Animatable 被锁定, 不参与计算                    │
└─────────────────────────────────────────────────┘
                       │
                       │ 滚动停止 (isScrollInProgress: true → false)
                       ▼
┌─────────────────────────────────────────────────┐
│  阶段 2: 停止吸附 (isScrollInProgress = false)  │
│  ─────────────────────────────────────────       │
│  1. 读取 derivedStateOf 当前值作为起点            │
│  2. Animatable.snapTo(起点)  ← 同步对齐          │
│  3. Animatable.animateTo(0f 或 1f)  ← 动画吸附   │
│  4. progress 切换为读取 Animatable.value         │
└─────────────────────────────────────────────────┘
                       │
                       │ 用户再次触摸屏幕滚动
                       ▼
                   回到阶段 1
```

### 关键设计点

| 关键点 | 当前实现 | 新设计 |
|--------|---------|--------|
| 滚动中 progress 来源 | `Animatable.snapTo` 异步同步 | `derivedStateOf` 同步计算,无中转 |
| 滚动停止 progress 来源 | 同上 | `Animatable.animateTo` 吸附到端点 |
| 阶段切换时机 | 无明确切换 | `isScrollInProgress` 变化时切换 |
| layout/graphicsLayer 读取 | `searchRevealProgress.value` (滞后) | `currentProgress` (派生状态,无延迟) |
| 搜索词输入时 | `animateTo(1f)` | 不变 |
| 搜索词清空时 | `animateTo(0f or 1f)` | 不变 |

### 核心代码结构

```kotlin
// ━━━ 阶段 1: 滚动中同步跟手 ━━━
val scrollDrivenProgress by remember {
    derivedStateOf {
        if (lazyListState.firstVisibleItemIndex == 0) {
            (1f - lazyListState.firstVisibleItemScrollOffset / searchBarFullHeightPx)
                .coerceIn(0f, 1f)
        } else {
            0f
        }
    }
}

// ━━━ 阶段 2: 停止吸附动画 ━━━
val animatableProgress = remember { Animatable(1f) }

// ━━━ 当前生效的 progress (二选一) ━━━
val currentProgress = if (isScrolling) scrollDrivenProgress else animatableProgress.value

// ━━━ 阶段切换: 滚动停止时同步对齐 + animateTo 吸附 ━━━
LaunchedEffect(isScrolling) {
    if (!isScrolling && searchQuery.isBlank()) {
        val start = scrollDrivenProgress
        if (start > 0.05f && start < 0.95f) {
            val target = if (start < 0.5f) 0f else 1f
            animatableProgress.snapTo(start)  // 同步对齐,无跳跃
            animatableProgress.animateTo(
                targetValue = target,
                animationSpec = tween(if (target == 0f) 200 else 250, easing = FastOutSlowInEasing)
            )
        }
    }
}
```

                placeable.place(0, 0)

### 搜索词输入/清空逻辑保留

```kotlin
LaunchedEffect(searchQuery) {
    if (searchQuery.isNotBlank()) {
        // 输入搜索词: 立即显示 (基于 animatableProgress)
        animatableProgress.snapTo(if (isScrolling) scrollDrivenProgress else animatableProgress.value)
        animatableProgress.animateTo(1f, animationSpec = tween(250, easing = FastOutSlowInEasing))
    } else {
        // 清空搜索词: 基于当前 progress 吸附到端点
        val current = if (isScrolling) scrollDrivenProgress else animatableProgress.value
        if (current > 0.05f && current < 0.95f) {
            val target = if (current < 0.5f) 0f else 1f
            animatableProgress.snapTo(current)
            animatableProgress.animateTo(
                targetValue = target,
                animationSpec = tween(if (target == 0f) 200 else 250, easing = FastOutSlowInEasing)
            )
        }
    }
}
```

## 边界情况处理

| 场景 | 处理方式 |
|------|---------|
| 滚动中输入搜索词 | `snapTo` 当前 `scrollDrivenProgress` → `animateTo(1f)` |
| 滚动中清空搜索词 | 等滚动停止后由 `LaunchedEffect(isScrolling)` 处理吸附 |
| 搜索词非空时滚动停止 | `LaunchedEffect(isScrolling)` 中 `searchQuery.isBlank()` 检查跳过 |
| `firstVisibleItemIndex > 0` | `scrollDrivenProgress` 强制返回 0f |
| `isScrolling` 在 animateTo 中变 true | `currentProgress` 立即切换为 `scrollDrivenProgress`,中断吸附动画 |

1. 慢速上滑: 搜索框平滑收缩,无跳跃
2. 快速上滑 fling: 搜索框跟手收缩,停止后吸附到 0f
3. 慢速下滑: 搜索框平滑展开,无跳跃
4. 快速下滑 fling: 搜索框跟手展开,停止后吸附到 1f
5. 滚动中输入搜索词: 立即展开到 1f
6. 滚动中清空搜索词: 等停止后吸附
7. 列表为空时滚动: 无异常
8. 置顶按钮存在时: 不影响搜索框显隐

## 风险与缓解

| 风险 | 缓解 |
|------|------|
| `Animatable.snapTo` 是 suspend,不能在同步代码中调用 | 仅在 `LaunchedEffect` 协程中调用,符合规范 |
| `isScrolling` 在 animateTo 中变 true 导致动画中断 | 这是预期行为:用户触摸滚动时应立即跟手,中断吸附 |
| `scrollDrivenProgress` 与 `animatableProgress` 切换时数值不一致 | 切换前 `snapTo` 同步对齐,保证连续 |
