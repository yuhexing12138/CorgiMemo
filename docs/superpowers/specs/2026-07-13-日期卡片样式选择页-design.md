## 一、项目背景与目标

### 1.1 背景

- `SpecialDateQuickCreateScreen` 的 "下一步" 按钮当前是占位 Snackbar("功能开发中"),无实际导航逻辑
- `SpecialDate` Entity 没有"卡片样式"字段,所有日期只能用统一的主页 `SpecialDateCard` 渲染
- 用户希望分阶段实现:先支持"选卡片样式 → 保存到数据库",主页渲染按样式区分作为后续任务

### 1.2 目标

1. **跳转功能开发**:`SpecialDateQuickCreateScreen` 的 "下一步" 跳转到新页 `SpecialDateCardStyleScreen`
2. **样式选择页**:在该页实现 2 种参考图样式(橙色撕页 + 米色日历撕页),支持横向切换预览
3. **样式/颜色 Tab**:显示 "样式" / "颜色" 切换条;颜色 tab 占位 Snackbar("颜色功能开发中")
4. **底部缩略图条**:横向滚动 2 个样式缩略图,选中态有橙色边框 + 勾选标记
5. **真实保存**:点 "保存" 创建 SpecialDate 实体,带 `cardStyle` 字段,持久化到 Room
6. **架构可扩展**:用 sealed class 建模样式,加新样式只需新增一个 `object` + 一个 Composable,编译器自动提示补全
7. **平滑过渡**:使用 Navigation Compose 默认水平滑入动画

### 1.3 不在本设计内

- 主页 `SpecialDateCard` 不读 `cardStyle`,继续渲染原样式(仅 Entity 加字段不读)
- "颜色" tab 的真实实现(占位 Snackbar)
- 4 个被隐藏的样式(蓝色气球/拍立得等,后续任务实现)
- 主页展示按 `cardStyle` 渲染(后续任务)
- 备注/标签/图片/关联等详细字段(后续任务)

## 二、设计方案选型

### 2.1 方案对比

| 方案 | 描述 | 优点 | 缺点 | 推荐度 |
|------|------|------|------|--------|
| **方案 A:Sealed Class + when 分发(已选)** | `sealed class DateCardStyle` + 2 个 `object` 子类,DB 存 `serialName` 字符串 | 类型安全,`when` 编译期穷尽匹配;Kotlin 风格与项目一致;扩展只需新增 object | 序列化需 `fromSerialName` 解析(成本极低) | ⭐⭐⭐⭐⭐ |
| 方案 B:Enum + Dispatcher | `enum class DateCardStyle { ORANGE_TEAR_OFF, CALENDAR_TEAR_OFF }` | 最简单,序列化直接 `name` | 加新枚举值时 `when` 不会编译报错(会落到 else) | ⭐⭐⭐ |
| 方案 C:Interface + 工厂注册 | `interface DateCardStyle` + 实现类 + `Map<String, DateCardStyle>` 注册 | 真正运行时可扩展 | Kotlin 风格弱,样板代码多;2 个样式用不上这种灵活性 | ⭐⭐ |

### 2.2 选型结论

采用**方案 A:Sealed Class + when 分发**。理由:

1. **类型安全**:`when (style) { is X -> ... }` 编译期保证所有 case 已处理,未来加样式时编译器会提醒补全分支
2. **Kotlin 风格**:与项目内 `DateGroup`(SpecialDateViewModel.kt)、`TodoZone` 等 sealed class 一致
3. **可扩展但不复杂**:未来加样式只需新增一个 `object` 子类,主 `when` 编译报错指引补全
4. **与"预留可扩展架构"诉求契合**:方案 B 一旦用 enum,加样式时容易漏改 `when`;方案 C 过度设计

### 4.2 路由对象(Screen.kt 新增)

```kotlin
object SpecialDateCardStyle : Screen("date_card_style?title={title}&date={date}&category={category}&pin={pin}") {
    /**
     * 构建路由字符串
     * @param title 名称(自动 URL 编码;空时 fallback 到 "未命名")
     * @param date 目标日期时间戳(0 表示未选)
     * @param category DateCategory.name 或 "CUSTOM:xxx"(自动 URL 编码;空时 fallback 到 "OTHER")
     * @param pin 是否置顶
     */
    fun createRoute(title: String, date: Long, category: String, pin: Boolean): String {
        val encodedTitle = Uri.encode(title.ifBlank { "未命名" })
        val encodedCategory = Uri.encode(category.ifBlank { "OTHER" })
        return "date_card_style?title=$encodedTitle&date=$date&category=$encodedCategory&pin=$pin"
    }
}
```

### 4.6 保存流程

```
用户点击"保存"
  ↓
ViewModel.saveNewDate(title, date, category, isPinned, cardStyle)
  ↓
saveState = Saving
  ↓
构造 SpecialDate(
    title = title,
    targetDate = date,
    category = category,                       // 来自 URL,预设枚举名(如 "BIRTHDAY")或自定义名(如 "旅行")
    countMode = 0,                              // 倒计时(QuickCreate 未设置,默认)
    repeatType = 0,                             // 不重复
    reminderDays = 0,                           // 不提醒
    content = "",
    tags = "",
    imagePaths = "",
    imageUrls = "",
    isPinned = pin,
    isArchived = false,
    cardStyle = cardStyle.serialName,
    createdAt = System.currentTimeMillis(),
    updatedAt = System.currentTimeMillis()
)
  ↓
repository.insert(date)
  ↓
saveState = Success
  ↓
UI 监听 saveState:
  - Success → navController.previousBackStackEntry?.savedStateHandle?.set("date_saved", true)

  - Error   → 显示 Snackbar "保存失败,请重试" + 重试按钮
```

## 六、组件详细设计

### 6.1 `DateCardStyle`(sealed class)

```kotlin
package com.corgimemo.app.data.model

/**
 * 日期卡片样式(主页/QuickCreate 后的样式选择)
 *
 * 采用 sealed class + serialName 模式:
 * - 类型安全:`when` 编译期穷尽匹配
 * - 持久化:数据库存 serialName 字符串
 * - 解析容错:fromSerialName 失败时 fallback 到 DEFAULT
 *
 * 新增样式步骤(预留扩展):
 * 1. 在本 sealed class 中添加 `object NewStyle : DateCardStyle("NEW_STYLE")`
 * 2. 在 components/cardstyle/ 目录下新增 NewStyleCard.kt
 * 3. 在 DateCardStyleRenderer.kt 的 when 分支中补全 case(编译器会报错提示)
 * 4. 在 DateCardStyleSelector.kt 的 styles 列表中追加
 */
sealed class DateCardStyle(val serialName: String) {
    /** 橙色撕页样式:顶部橙色块 + 倒计文字 + 大数字 + 标题(参考图 2) */
    object OrangeTearOff : DateCardStyle("ORANGE_TEAR_OFF")

    /** 米色日历撕页样式:顶部月历 + 大数字 + 标题(参考图 3) */
    object CalendarTearOff : DateCardStyle("CALENDAR_TEAR_OFF")

    companion object {
        val DEFAULT: DateCardStyle = OrangeTearOff

        /**
         * 从 serialName 解析为 DateCardStyle 实例
         * @param name serialName 字符串(null/非法值 fallback 到 DEFAULT)
         */
        fun fromSerialName(name: String?): DateCardStyle {
            return when (name) {
                OrangeTearOff.serialName -> OrangeTearOff
                CalendarTearOff.serialName -> CalendarTearOff
                else -> DEFAULT
            }
        }

        /** 全部样式列表(用于选择器与遍历) */
        val all: List<DateCardStyle> = listOf(OrangeTearOff, CalendarTearOff)
    }
}
```

### 6.2 `DateCardStyleRenderer` 分发器

```kotlin
package com.corgimemo.app.ui.screens.date.components.cardstyle

import androidx.compose.runtime.Composable
import androidx.compose.ui.Modifier
import com.corgimemo.app.data.model.DateCardStyle

/**
 * DateCardStyle 渲染分发器
 *
 * 集中处理 sealed class -> 具体样式组件的映射,避免业务页面直接依赖具体实现。
 * 编译器保证 when 穷尽(新加 DateCardStyle 子类时编译报错提示补全)。
 */
@Composable
fun DateCardStyleRenderer(
    style: DateCardStyle,
    title: String,
    targetDateMillis: Long,
    modifier: Modifier = Modifier
) {
    when (style) {
        is DateCardStyle.OrangeTearOff -> OrangeTearOffCard(
            title = title,
            targetDateMillis = targetDateMillis,
            modifier = modifier
        )
        is DateCardStyle.CalendarTearOff -> CalendarTearOffCard(
            title = title,
            targetDateMillis = targetDateMillis,
            modifier = modifier
        )
    }
}
```

### 6.5 `MiniCalendar` 单月小日历

```kotlin
/**
 * 单月小日历(日历样式专用)
 *
 * 固定显示一个月,无月份导航。目标日用红色圆圈高亮。
 * 上月/下月填充日用浅灰显示。
 *
 * @param year 年份
 * @param month 月份(1-12)
 * @param targetDay 目标日(高亮显示;0 表示不高亮)
 * @param textSize 单元格数字字号(默认 14sp)
 */
@Composable
fun MiniCalendar(
    year: Int,
    month: Int,
    targetDay: Int,
    modifier: Modifier = Modifier,
    textSize: TextUnit = 14.sp
) {
    val firstDayOfWeek = (YearMonth.of(year, month).atDay(1).dayOfWeek.value - 1 + 7) % 7
    val daysInMonth = YearMonth.of(year, month).lengthOfMonth()
    val prevMonth = YearMonth.of(year, month).minusMonths(1)
    val prevMonthDays = prevMonth.lengthOfMonth()

    // 构建 5x7 网格数据(Triple<day, isPrevOrNextMonth, isTarget>)
    val gridItems = mutableListOf<Triple<Int, Boolean, Boolean>>()
    for (i in firstDayOfWeek downTo 1) {
        gridItems.add(Triple(prevMonthDays - i + 1, true, false))
    }
    for (day in 1..daysInMonth) {
        gridItems.add(Triple(day, false, day == targetDay))
    }
    var nextDay = 1
    while (gridItems.size < 35) {
        gridItems.add(Triple(nextDay++, true, false))
    }

    Column(modifier = modifier) {
        // 星期表头
        Row(modifier = Modifier.fillMaxWidth(), horizontalArrangement = Arrangement.SpaceAround) {
            listOf("一", "二", "三", "四", "五", "六", "日").forEach { dow ->
                Text(dow, fontSize = 12.sp, color = Color(0xFF999999), modifier = Modifier.weight(1f), textAlign = TextAlign.Center)
            }
        }
        Spacer(Modifier.height(4.dp))
        // 5 行日期
        repeat(5) { weekIndex ->
            Row(modifier = Modifier.fillMaxWidth(), horizontalArrangement = Arrangement.SpaceAround) {
                repeat(7) { dayIndex ->
                    val (day, isPrevOrNext, isTarget) = gridItems[weekIndex * 7 + dayIndex]
                    val textColor = when {
                        isTarget -> MaterialTheme.colorScheme.primary
                        isPrevOrNext -> Color(0xFFCCCCCC)
                        else -> Color(0xFF2D2D2D)
                    }
                    Box(
                        modifier = Modifier
                            .weight(1f)
                            .height(if (textSize == 14.sp) 24.dp else 16.dp)
                            .then(
                                if (isTarget) Modifier
                                    .border(1.5.dp, Color(0xFFFF8A80), CircleShape)
                                else Modifier
                            ),
                        contentAlignment = Alignment.Center
                    ) {
                        Text(day.toString(), fontSize = textSize, color = textColor, textAlign = TextAlign.Center)
                    }
                }
            }
        }
    }
}
```

### 6.6 `WavyBottomShape` 撕页波浪 Shape

```kotlin
package com.corgimemo.app.ui.screens.date.components.cardstyle

import androidx.compose.ui.geometry.Size
import androidx.compose.ui.graphics.Outline
import androidx.compose.ui.graphics.Path
import androidx.compose.ui.graphics.Shape
import androidx.compose.ui.unit.Density
import androidx.compose.ui.unit.LayoutDirection

/**
 * 撕页效果底部波浪 Shape
 *
 * 仅修改底部边缘为波浪形,顶部/左侧/右侧为直线。
 * 配合 Card 的 clip 使用,实现"撕日历"纸张效果。
 *
 * @param waveHeightPx 波浪高度(像素)
 * @param waveCount 波浪数量
 */
class WavyBottomShape(
    private val waveHeightPx: Float = 8f,
    private val waveCount: Int = 12
) : Shape {
    override fun createOutline(
        size: Size,
        layoutDirection: LayoutDirection,
        density: Density
    ): Outline {
        val path = Path().apply {
            moveTo(0f, 0f)
            lineTo(size.width, 0f)
            lineTo(size.width, size.height)
            val waveWidth = size.width / waveCount
            // 从右下到左下画波浪
            for (i in 0 until waveCount) {
                val endX = size.width - (i + 1) * waveWidth
                val startX = size.width - i * waveWidth
                val midX = (startX + endX) / 2
                quadraticBezierTo(
                    midX, size.height + waveHeightPx,
                    endX, size.height
                )
            }
            lineTo(0f, size.height)
            close()
        }
        return Outline.Generic(path)
    }
}
```

### 6.10 时间计算工具

```kotlin
// 文件位置: components/cardstyle/DateCardTimeUtils.kt

/**
 * 距离天数(取整,abs)
 *
 * @param target 目标日期时间戳(毫秒)
 * @param now 当前时间戳(毫秒,默认 System.currentTimeMillis())
 * @return 距离天数的绝对值(无论过去还是未来)
 */
fun daysUntil(target: Long, now: Long = System.currentTimeMillis()): Long {
    return kotlin.math.abs(target - now) / 86_400_000L
}

/**
 * 判断目标日期相对当前是过去还是未来
 *
 * @param target 目标日期时间戳(毫秒)
 * @param now 当前时间戳(毫秒,默认 System.currentTimeMillis())
 * @return true=目标日期在未来(还未到),false=目标日期在过去(已过)
 */
fun isFuture(target: Long, now: Long = System.currentTimeMillis()): Boolean {
    return target - now >= 0
}

/**
 * 距离文字(用于橙色撕页样式顶部)
 *
 * 格式:
 * - 未来:"距离 yyyy/MM/dd 周X 还有"
 * - 过去:"已过 yyyy/MM/dd 周X"
 *
 * 例: "距离 2026/07/17 周五 还有" / "已过 2026/07/17 周五"
 */
fun formatDistanceTextWithWeekday(target: Long, now: Long = System.currentTimeMillis()): String {
    val future = isFuture(target, now)
    val dateText = formatDateWithWeekday(target)
    return if (future) "距离 $dateText 还有" else "已过 $dateText"
}

/**
 * 距离文字(用于米色日历撕页样式底部)
 *
 * 格式:
 * - 未来:"距离 yyyy/MM/dd 还有"
 * - 过去:"已过 yyyy/MM/dd"
 *
 * 例: "距离 2026/07/17 还有" / "已过 2026/07/17"
 */
fun formatDistanceTextShort(target: Long, now: Long = System.currentTimeMillis()): String {
    val future = isFuture(target, now)
    val dateText = formatShortDate(target)
    return if (future) "距离 $dateText 还有" else "已过 $dateText"
}

/**
 * 完整日期+星期(用于橙色样式顶部)
 * 例: "2026/07/17 周五"
 */
fun formatDateWithWeekday(timestamp: Long): String {
    val date = Instant.ofEpochMilli(timestamp).atZone(ZoneId.systemDefault()).toLocalDate()
    val weekdays = arrayOf("周一", "周二", "周三", "周四", "周五", "周六", "周日")
    val weekday = weekdays[date.dayOfWeek.value - 1]
    return "${date.year}/${String.format("%02d", date.monthValue)}/${String.format("%02d", date.dayOfMonth)} $weekday"
}

/**
 * 短日期(用于日历样式底部)
 * 例: "2026/07/17"
 */
fun formatShortDate(timestamp: Long): String {
    val date = Instant.ofEpochMilli(timestamp).atZone(ZoneId.systemDefault()).toLocalDate()
    return "${date.year}/${String.format("%02d", date.monthValue)}/${String.format("%02d", date.dayOfMonth)}"
}
```

## 七、错误处理

| 场景 | 处理 |
|------|------|
| QuickCreate 未选日期点"下一步" | Snackbar "请先选择日期",不跳转 |
| CardStyle 页未选日期(0L)点"保存" | Snackbar "请先返回选择日期",不调用 ViewModel |
| ViewModel 保存失败(DB 抛异常) | `SaveState.Error` → Snackbar "保存失败,请重试" + 重试按钮 |
| 路由参数 URL 解析失败/为空 | title 默认 "未命名"、category 默认 "OTHER"、dateMillis 0L、isPinned false |
| 路由参数包含特殊字符(中文/emoji) | `Uri.encode()` 处理(标题 URL 编码) |
| 重复点击"保存"按钮 | ViewModel 内部判断 `saveState == Saving` 时直接忽略 |
| 主页 `SpecialDateScreen` 已销毁 | `savedStateHandle.get` 返回 null,LaunchedEffect 不触发,无副作用 |
| 撕页波浪 Shape 在低密度屏异常 | `waveHeightPx` 用物理像素,不随密度变化;`waveCount` 固定 12,小屏仍有可见波浪 |
| 周次计算偏差 | 使用 `java.time.YearMonth` + `dayOfWeek.value`,与 `ReminderPickerBottomSheet` 中算法一致 |
