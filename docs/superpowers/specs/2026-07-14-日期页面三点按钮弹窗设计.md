# 日期页面三点按钮弹窗设计

> 创建日期：2026-07-14
> 状态：设计待实施
> 关联模块：特殊日期页面、回收站页面、MainScreen

## 一、背景与目标

日期页面（SpecialDateScreen）右上角三点按钮目前仅显示 Toast"功能开发中..."。需开发功能完整的弹窗组件，严格参考待办页面（TodoMenuDropdown）和灵感页面（InspirationMenuDropdown）的现有实现方案，确保 UI 风格、交互逻辑和用户体验的一致性。

弹窗包含 6 个功能选项（从上到下）：

1. **隐藏详情/显示详情**：隐藏日期卡片倒计时时间文本，标题垂直居中
2. **隐藏已归档/显示已归档**：过滤已归档日期，逻辑与待办页"隐藏已完成"一致
3. **批量选择**：进入批量选择模式，工具栏含删除/归档/创建副本等操作
4. **分享**（占位）：显示"开发中"状态
5. **创建日期副本**（占位）：显示"开发中"状态
6. **回收站**：跳转回收站页面，新增日期 Tab 展示已删除日期卡片

## 四、隐藏详情与隐藏已归档

### 4.1 隐藏详情（卡片布局变更）

**当前 `SpecialDateCard` 布局**：

```
[圆形图片 48dp] [12dp] [Column: 标题行 + 时间信息行] [8dp] [Column: 天数+单位]
```

**隐藏详情后**：

```
[圆形图片 48dp] [12dp] [Column: 仅标题行(垂直居中)] [8dp] [Column: 天数+单位]
```

**实现方式**：

- `SpecialDateCard` 新增 `isSimpleMode: Boolean` 参数（与 `TodoListItem` 的 `isSimpleMode` 命名一致）
- `isSimpleMode=true` 时：不渲染时间信息行（`formatDuration(abs(diff))` 的 `Text`）
- `Column(weight=1f)` 仅剩标题行，因 `Row.verticalAlignment = CenterVertically`，标题自然垂直居中
- 右侧天数+单位保持不变

**数据流**：

```
SpecialDateViewModel.hideDetails → SpecialDateScreen → SpecialDateCard(isSimpleMode=hideDetails)
```

### 4.2 隐藏已归档（列表过滤）

**实现方式**：

- `SpecialDateViewModel.groupedDates` Flow 增加过滤逻辑：
  ```kotlin
  combine(specialDates, _searchQuery, _hideArchivedItems) { dates, query, hideArchived ->
      val filtered = if (hideArchived) dates.filter { !it.isArchived } else dates
      // ... 搜索过滤 + 分组
  }
  ```
- 隐藏后"已归档"分组（`DateGroup.EXPIRED`）整体不显示
- 与待办页 `hideCompletedItems` 逻辑完全一致

### 4.3 持久化

- `CorgiPreferences` 新增：
  - `hideSpecialDateDetails: Flow<Boolean>` + `setHideSpecialDateDetails(hide: Boolean)`
  - `hideArchivedDateItems: Flow<Boolean>` + `setHideArchivedDateItems(hide: Boolean)`
- `SpecialDateViewModel.init` 订阅这两个 Flow 同步到对应 StateFlow
- `toggleHideDetails()` / `toggleHideArchivedItems()` 同时更新 StateFlow 和持久化

## 六、回收站集成

### 6.1 数据实体 `DeletedSpecialDate`

```kotlin
@Entity(tableName = "deleted_special_dates")
data class DeletedSpecialDate(
    @PrimaryKey val id: Long,
    val title: String,
    val targetDate: Long,
    val category: String,
    val countMode: Int,
    val repeatType: Int,
    val reminderDays: Int,
    val content: String,
    val tags: String,
    val imagePaths: String,
    val imageUrls: String,
    val isPinned: Boolean,
    val isArchived: Boolean,
    val cardStyle: String,
    val cardColor: String,
    val createdAt: Long,
    val updatedAt: Long,
    val deletedAt: Long = System.currentTimeMillis()
) {
    companion object {
        fun fromSpecialDate(date: SpecialDate): DeletedSpecialDate
        fun toSpecialDate(deleted: DeletedSpecialDate): SpecialDate

}
```

**字段说明**：与 `SpecialDate` 完全一致 + `deletedAt` 字段记录删除时间。`fromSpecialDate()` 拷贝所有字段并设置 `deletedAt`；`toSpecialDate()` 恢复时将 `isPinned=false`、`isArchived=false`。

## 八、删除行为变更

### 8.1 当前行为

`SpecialDateViewModel.deleteDate(id)` 直接调用 `repository.delete(id)`，从 `special_dates` 表硬删除，无撤回。

### 8.2 新行为（软删除）

```kotlin
fun deleteDate(id: Long) {
    viewModelScope.launch {
        val date = repository.getByIdBlocking(id) ?: return@launch
        deletedSpecialDateRepository.insert(DeletedSpecialDate.fromSpecialDate(date))
        repository.delete(id)
        _pendingDeletedDate.value = date  // 缓存供撤回
        // 触发 Snackbar
    }
}

fun undoDelete() {
    _pendingDeletedDate.value?.let { date ->
        viewModelScope.launch {
            deletedSpecialDateRepository.permanentlyDelete(date.id)
            repository.insert(date)  // 恢复原始数据（保留 isPinned/isArchived）
            _pendingDeletedDate.value = null
        }
    }
}
```

### 8.3 左滑删除交互

- 左滑删除按钮点击后调用新的 `deleteDate()`（软删除）
- 显示 Snackbar "已删除『title』" + "撤回"按钮
- 3 秒超时后清空 `pendingDeletedDate`，永久保留在回收站
