## 一、问题背景

### 现象
1. **待办编辑页底部工具栏的"分享"按钮按下没反应**（`onShareClick = {}` 空实现，见 [TodoEditScreen.kt:877](file:///c:/Users/EDY/Desktop/CorgiMemo/app/src/main/java/com/corgimemo/app/ui/screens/todo/TodoEditScreen.kt#L877)）。
2. **多选模式下分享多个待办会连续弹 N 次系统分享面板**（[MainScreen.kt:380-385](file:///c:/Users/EDY/Desktop/CorgiMemo/app/src/main/java/com/corgimemo/app/ui/screens/main/MainScreen.kt#L380) 遍历 `selectedTodoIds.forEach` 调用 `shareTodoAsImage`），用户体验极差。

### 设计目标
- 修复待办编辑页分享按钮的空实现
- 当存在多个待办（多分组/多卡片或多选）时，**让用户在"合并分享"和"一条条分享"之间选择**
- 单个待办直接分享，不弹选择弹窗
- 拼图策略：智能分档（≤6 网格，7-10 垂直，>10 禁用合并）
- 未保存的分组不允许分享（提示"请先保存所有分组"）

## 四、模块详细设计

### 4.1 ShareCoordinator

#### 公开 API

```kotlin
object ShareCoordinator {
    /**
     * 从待办编辑页调用
     *
     * @param savedSubTodos 已保存的子 todo 列表（由 ViewModel 过滤 isSaved==true 的分组）
     * @param hasUnsavedGroups 是否有未保存的分组（用于 SnackBar 提示）
     */
    suspend fun shareTodosFromEdit(
        context: Context,
        mainTodo: TodoItem,
        savedSubTodos: List<TodoItem>,
        categories: List<Category>,
        hasUnsavedGroups: Boolean,
        onShowSnackBar: (String) -> Unit
    )

    /**
     * 从多选模式调用
     */
    suspend fun shareTodos(
        context: Context,
        todos: List<TodoItem>,
        categories: List<Category>
    )
}
```

#### 内部决策流程

```
1. shareTodosFromEdit 被调用
2. if (hasUnsavedGroups) { onShowSnackBar("请先保存所有分组"); return }
3. val all = listOf(mainTodo) + savedSubTodos
4. when (all.size) {
       0 -> return  // 边界保护
       1 -> shareTodoAsImage(context, all[0], categories)
       else -> 显示 ShareModeDialog → 用户选择 → 调对应分支
   }
```

### 4.2 ShareModeDialog

fun ShareModeDialog(

#### 视觉规范

### 4.3 ImageExporter.mergeBitmaps

#### 函数签名
```kotlin
suspend fun mergeBitmaps(
    bitmaps: List<Bitmap>,
    spacingPx: Int = 24          // 拼图间隔（12dp × density × 2）
): Bitmap

enum class MergeStrategy {
    GRID_2_COL,   // 2-6 张：2 列网格
    VERTICAL,    // 7-10 张：垂直
}
```

#### 拼接策略表

| 数量 | 策略 | 布局 |
|---|---|---|
| 2 张 | GRID_2_COL | 1 行 × 2 列 |
| 3-4 张 | GRID_2_COL | 2 行 × 2 列 |
| 5-6 张 | GRID_2_COL | 3 行 × 2 列 |
| 7-10 张 | VERTICAL | N 行 × 1 列 |
| >10 张 | （不调用此函数） | — |

#### 实现要点
- 输出 Bitmap 宽度固定为 720dp（与单张卡片相同），保证分享面板识别
- 每张子图按比例缩放，等宽排版
- 网格布局：行高按子图比例分配
- 垂直布局：宽度统一为 720dp，高度按子图原始比例累加
- 子图间距：白色 `spacingPx` 像素
- 背景：纯白 (#FFFFFF)
- 所有 Canvas 绘制在 `Dispatchers.Default` 协程中完成

### 4.4 共享/单独分享执行

#### 一条条分享（循环）
```kotlin
suspend fun shareOneByOne(
    context: Context,
    todos: List<TodoItem>,
    categories: List<Category>
) {
    todos.forEach { todo ->
        shareTodoAsImage(context, todo, categories)
        delay(800)  // 避免系统分享面板叠加混乱
    }
}
```

#### 合并分享
```kotlin
suspend fun shareMerged(
    context: Context,
    todos: List<TodoItem>,
    categories: List<Category>
) {
    val bitmaps = todos.map { todo ->
        val category = categories.find { it.id == todo.categoryId }
        ImageExporter.createTodoShareCard(context, todo, category)
    }
    val merged = ImageExporter.mergeBitmaps(bitmaps)
    val file = ImageExporter.saveBitmapToCache(context, merged)
    val intent = ShareIntentHelper.createShareImageIntent(
        context = context,
        imageFile = file,
        text = "我在 CorgiMemo 创建了 ${todos.size} 条待办"
    )
    intent.addFlags(Intent.FLAG_ACTIVITY_NEW_TASK)
    withContext(Dispatchers.Main) { context.startActivity(intent) }
}
```

## 五、数据流

### 5.1 待办编辑页分享流程

```
1. 用户点击分享按钮
   ↓
2. onShareClick lambda 触发
   ↓
3. 构造 savedSubTodos = viewModel.getSavedSubTodos()
   ↓
4. val hasUnsavedGroups = (groupSaveStates.values.any { !it.isSaved })
   ↓
5. ShareCoordinator.shareTodosFromEdit(
       context, mainTodo, savedSubTodos, categories, hasUnsavedGroups, onShowSnackBar
   )
   ↓
6. Coordinator 内部决策：
   ├─ hasUnsavedGroups → onShowSnackBar("请先保存所有分组")
   ├─ size == 0 → return
   ├─ size == 1 → shareTodoAsImage(...) 直接系统分享
   └─ size > 1 → 显示 ShareModeDialog
       ├─ onMerge → shareMerged(...) 单次系统分享
       └─ onOneByOne → shareOneByOne(...) 循环系统分享
```

### 5.2 多选模式分享流程

```
1. 用户在批量操作栏点击分享
   ↓
2. selectedTodoIds 非空校验
   ↓
3. todos = filteredTodos.filter { it.id in selectedTodoIds }
   ↓
4. ShareCoordinator.shareTodos(context, todos, categories)
   ↓
5. Coordinator 内部决策（与编辑页相同）
```

## 六、错误处理

| 场景 | 处理方式 |
|---|---|
| 拼图数量 > 10 | ShareModeDialog 接收 `enableMerge=false`，按钮置灰 |
| Bitmap 生成失败（极少见） | Toast "图片生成失败: ${e.message}"，不调分享 Intent |
| 文件保存失败 | Toast "图片保存失败"，不调分享 Intent |
| 系统分享 Intent 无应用可处理 | 静默忽略，符合 Android 惯例（不弹错） |
| 多选模式 selectedTodoIds 为空 | 入口处直接 return（不调 Coordinator） |
| 编辑页 0 个分组（数据异常） | Coordinator 内部 size==0 → return |
| 用户在拼图生成中按返回键 | Coroutine 取消，UI 状态由 `LaunchedEffect` 自动清理 |
| 协程作用域取消 | 所有 IO 任务通过 `withContext(Dispatchers.IO)` 隔离，作用域取消时自动终止 |
| 内存压力（拼 10 张图） | Bitmaps 不复用，循环结束由 GC 回收；不缓存中间结果 |

## 八、本次设计未涉及（Out of Scope）

为保持 YAGNI，以下功能**不在本次实现范围**：

- ❌ 分享为 iCal 格式（保持现有 export 在 Settings 中）
- ❌ 分享为纯文本（YAGNI）
- ❌ 自定义拼图样式（间距/边框/标题）
- ❌ 灵感编辑页同步（其分享按钮被复用为"添加子任务"功能，保持原样）
- ❌ 分享进度条（Coroutine 在 IO 线程，用户在 UI 线程看到的是"分享中"状态由系统分享面板覆盖）
- ❌ 分享历史记录
- ❌ 跨页面状态保存（点分享时切走，丢弃分享操作）
