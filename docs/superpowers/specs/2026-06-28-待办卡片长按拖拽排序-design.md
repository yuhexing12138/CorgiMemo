# 待办卡片长按拖拽排序设计

> 创建日期：2026-06-28
> 状态：已确认，待实施
> 方案：A（新建 ReorderableLazyColumn 容器组件）

## 一、问题概述与需求

### 1.1 需求描述

为 HomeScreen 待办列表实现长按触发的拖拽排序功能：

| # | 交互要求 |
|---|---------|
| 1 | 长按卡片启动触摸监听；手指未离开屏幕时检测到 5-10px 位移触发拖拽模式 |
| 2 | 拖拽中卡片保持半透明（60-80%），跟随手指实时移动 |
| 3 | 与其他卡片重叠面积超过 50% 时触发位置互换动画 |
| 4 | 被拖卡片悬浮效果（Z 轴提升 + 阴影）+ 目标位置虚线占位框 |
| 5 | 手指抬起时平滑过渡到最终位置，更新排序数据 |
| 6 | 不同屏幕尺寸与触摸设备均流畅无卡顿 |

### 1.2 与现有功能的关键冲突与决策

| 冲突 | 决策 |
|------|------|
| 长按手势已被"进入批量选择模式"占用 | 长按 + 上下移动 → 拖拽；长按 + 静止抬起 → 批量模式（同一手势按位移区分） |
| 长按后位移方向歧义（纵向拖拽 vs 横向左滑） | 长按激活后（500ms 后）+ 拖拽期间，禁用左滑手势；500ms 内仍可左滑 |
| TodoItem 无 sortOrder 字段，DAO 按 createdAt DESC 排序 | 新增 `sortOrder: Int` 字段 + Migration；任何排序模式下均可拖拽；排序菜单新增"恢复默认排序" |
| 拖拽范围与置顶区关系 | 所有可见待办可拖拽；置顶项拖到非置顶区取消置顶属性（对称规则：反向亦自动置顶） |
| `isPinned` 字段当前未实际参与排序（注释与实现不符） | 修复：`isPinned DESC` 作为首要排序键 |

## 二、架构与组件结构

### 2.1 整体调用结构


### 2.2 手势分层（三层职责严格分离）

| 层级 | 组件 | 检测器 | 触发条件 | 职责 |
|------|------|--------|---------|------|
| L1 容器 | `ReorderableLazyColumn` | `pointerInput(Unit) { awaitEachGesture }` | 长按 500ms + 纵向移动 > 8dp | 拖拽排序 |
| L2 包装 | `SwipeableTodoBox` | `detectHorizontalDragGestures` | 横向移动 > touchSlop | 左滑操作菜单 |
| L3 子项 | `TodoListItem` 内 Card | `pointerInput(...) { awaitEachGesture }` | 短按 / 长按静止抬起 | tap / 批量模式 |

**冲突解决规则**：

```
手指按下
  ├─ L3 启动 500ms 定时器 + emit Press（水波纹）
  ├─ 500ms 内移动 > touchSlop（任意方向）
  │    └─ L3 取消定时器 + emit Cancel + break → 让位 L2 左滑
  ├─ 500ms 到期（长按触发）
  │    ├─ L3 标记 longPressTriggered = true + 震动（仅非批量模式）
  │    ├─ 后续纵向移动 > 8dp
  │    │    └─ L1 接管 → 进入拖拽模式
  │    │       （L3 检测 isDragActive=true → emit Cancel + break，L2 因 isDragActive=true 禁用）
  │    ├─ 后续横向移动 → 忽略（L2 已禁用，L3 不再消费）
  │    └─ 抬手（无移动）→ L3 触发 onLongClick → 进入批量模式
  └─ 500ms 内抬手 → L3 触发 onTap → onClick / onSelectClick
```

### 3.2 数据库 Migration（v30 → v31）

```kotlin
private val MIGRATION_30_31 = object : Migration(30, 31) {
    override fun migrate(db: SupportSQLiteDatabase) {
        // 1. 新增 sortOrder 列，默认值 0
        db.execSQL("ALTER TABLE todo_items ADD COLUMN sortOrder INTEGER NOT NULL DEFAULT 0")

        // 2. 按当前默认排序（createdAt DESC）回填 sortOrder，
        //    使升级后列表顺序与升级前完全一致
        db.execSQL("""
            UPDATE todo_items
            SET sortOrder = (
                SELECT COUNT(*) FROM todo_items AS t2
                WHERE t2.isPinned = todo_items.isPinned
                  AND (t2.createdAt > todo_items.createdAt
                       OR (t2.createdAt = todo_items.createdAt
                           AND t2.id > todo_items.id))
            )
        """.trimIndent())
    }
}
```

`CorgiMemoDatabase.kt` `version = 31`，`addMigrations(..., MIGRATION_29_30, MIGRATION_30_31)`。

### 3.4 HomeViewModel 排序逻辑重构

**修复**：现有排序逻辑（`when (sortType) { ... }`）未处理 `isPinned`，与注释不一致。新增 `isPinned DESC` 作为首要排序键。

**关键模型**：
- `sortOrder` 是列表顺序的**唯一来源**（在 isPinned 分区内）
- `sortType` 仅决定"默认顺序算法"，在切换 sortType 或点击"恢复默认排序"时按 sortType **重算 sortOrder**
- 拖拽仅修改 sortOrder，不改 sortType
- 显示顺序：`isPinned DESC, sortOrder ASC, createdAt DESC`（createdAt 仅作并列兜底，sortOrder 唯一时不触发）

```kotlin
/**
 * 显示顺序：sortOrder 是唯一来源，sortType 不直接参与显示排序
 * sortType 的作用在 onSortTypeChanged 和 restoreDefaultOrder 中重算 sortOrder
 */
result = result
    .sortedWith(compareByDescending<TodoItem> { it.isPinned }
        .thenBy { it.sortOrder }
        .thenByDescending { it.createdAt })  // sortOrder 并列时兜底（理论上不会发生）
```

**sortType 切换时重算 sortOrder**：

```kotlin
/**
 * 切换排序模式时调用：按新 sortType 重新生成 sortOrder
 * 之后列表顺序由新 sortOrder 决定
 */
fun onSortTypeChanged(newSortType: String) {
    viewModelScope.launch {
        _sortType.value = newSortType
        val currentList = _todos.value
        val naturalOrder = when (newSortType) {
            "updated_desc" -> currentList.sortedByDescending { it.updatedAt }
            "updated_asc"  -> currentList.sortedBy { it.updatedAt }
            "created_desc" -> currentList.sortedByDescending { it.createdAt }
            "created_asc"  -> currentList.sortedBy { it.createdAt }
            else -> currentList.sortedByDescending { it.updatedAt }
        }
        val updates = naturalOrder.mapIndexed { index, item -> item.id to index }
        todoDao.updateSortOrders(updates)
    }
}
```

> 注：`restoreDefaultOrder()`（第 3.5 节）与 `onSortTypeChanged()` 逻辑相同，可合并为一个方法 `applyDefaultOrder()`。

### 3.5 新增 ViewModel 方法

```kotlin
/**
 * 拖拽完成后调用：更新被拖项及受影响项的 sortOrder
 * 同时处理置顶区跨越
 *
 * @param fromIndex 被拖项原始位置
 * @param toIndex 被拖项最终位置
 * @param crossedPinnedZone 是否跨越置顶区分界线
 */
fun reorderTodos(fromIndex: Int, toIndex: Int, crossedPinnedZone: Boolean) {
    viewModelScope.launch {
        val currentList = _filteredTodos.value
        val updatedList = currentList.toMutableList()

        // 1. 移除被拖项，插入到目标位置
        val draggedItem = updatedList.removeAt(fromIndex)
        updatedList.add(toIndex, draggedItem)

        // 2. 处理置顶区跨越（对称规则：自动切换 isPinned）
        val finalDraggedItem = if (crossedPinnedZone) {
            draggedItem.copy(isPinned = !draggedItem.isPinned)
        } else draggedItem

        // 3. 重新分配 sortOrder（基于列表位置）
        val updates = updatedList.mapIndexed { index, item ->
            item.id to index
        }

        // 4. 持久化
        todoDao.updateSortOrders(updates)
        if (crossedPinnedZone) {
            todoDao.update(finalDraggedItem)
        }
    }
}

/**
 * 恢复默认排序：按当前 sortType 重新生成 sortOrder
 */
fun restoreDefaultOrder() {
    viewModelScope.launch {
        val currentList = _todos.value
        val naturalOrder = when (_sortType.value) {
            "updated_desc" -> currentList.sortedByDescending { it.updatedAt }
            "updated_asc"  -> currentList.sortedBy { it.updatedAt }
            "created_desc" -> currentList.sortedByDescending { it.createdAt }
            "created_asc"  -> currentList.sortedBy { it.createdAt }
            else -> currentList.sortedByDescending { it.updatedAt }
        }
        val updates = naturalOrder.mapIndexed { index, item -> item.id to index }
        todoDao.updateSortOrders(updates)
    }
}
```

## 四、手势状态机

### 4.1 核心协调机制

容器与子项**各自维护 `awaitEachGesture`**，通过共享 `isDragActive` 状态协调：

```
┌─────────────────────────────────────────────────────────┐
│ ReorderableLazyColumn (容器)                             │
│  pointerInput(Unit) { awaitEachGesture { ... } }        │
│  职责：长按 500ms + 纵向移动 → 拖拽                      │
│  共享：var isDragActive by remember { mutableStateOf() } │
└─────────────────────────────────────────────────────────┘
                          ↓ isDragActive
┌─────────────────────────────────────────────────────────┐
│ TodoListItem.Card (子项)                                 │
│  pointerInput(...) { awaitEachGesture { ... } }         │
│  职责：短按 → tap；长按静止 → 批量模式                    │
│  协调：检测到 isDragActive=true → emit Cancel + break    │
└─────────────────────────────────────────────────────────┘
```

### 4.2 容器状态机（ReorderableLazyColumn）

```
State: Idle
  ├─ onDown(position)
  │    ├─ record downPosition
  │    ├─ start longPressJob (500ms)
  │    └─ 不 emit Press（子项会 emit，避免双重水波纹）
  │
  ├─ onEvent (longPressJob 未触发)
  │    ├─ changedToUp → cancel longPressJob, break
  │    │              （子项处理 tap）
  │    └─ move > touchSlop → cancel longPressJob, break
  │                          （让位 SwipeableTodoBox 左滑）
  │
  ├─ longPressJob 触发 (500ms 到期)
  │    └─ State: LongPressed
  │         ├─ 不震动（子项负责 LONG_CLICK 震动）
  │         │
  │         ├─ onEvent
  │         │    ├─ changedToUp → break
  │         │    │              （子项处理 longClick → 批量模式）
  │         │    ├─ 纵向移动 > 8dp (dragThreshold)
  │         │    │    └─ State: Dragging
  │         │    │         ├─ isDragActive = true
  │         │    │         ├─ draggedKey = findKeyFromY(downPosition.y)
  │         │    │         ├─ draggedOriginalIndex = currentIndex
  │         │    │         ├─ draggedCurrentIndex = currentIndex
  │         │    │         ├─ draggedBaseCenterY = layoutInfo[key].centerY
  │         │    │         ├─ fingerY = currentY
  │         │    │         ├─ 震动 TEXT_MOVE（标记进入拖拽）
  │         │    │         └─ 进入拖拽主循环
  │         │    └─ 横向移动 → 忽略（左滑已禁用）
  │         │
  │         └─ (子项此时检测到 isDragActive 或自行 break)
  │
  └─ State: Dragging
       ├─ onEvent
       │    ├─ move → fingerY = currentY
       │    │         change.consume()
       │    │         findSwapTarget() → 若有交换 → 同步更新基线 + displayItems
       │    │         边缘检测 → 触发自动滚动
       │    └─ changedToUp → commit reorder
       │                     震动 CONFIRM
       │                     isDragActive = false
       │                     draggedKey = null
       │                     State: Idle
       │
       └─ 异常/协程取消 → finally 块清理 isDragActive、draggedKey 等
```

### 4.3 子项状态机（TodoListItem）调整

**当前行为（需修改）**：移动 > touchSlop 无条件 break，让位左滑。这导致长按后纵向移动也会 break，无法触发拖拽。

**新行为**：长按触发后，**不再因移动 break**；仅当 `isDragActive=true` 时 break 让位容器。

```kotlin
// 修改后的 awaitEachGesture 主循环
while (true) {
    val event = awaitPointerEvent()
    val change = event.changes.first()

    if (change.changedToUp()) {
        longPressJob.cancel()
        interactionChannel.trySend(PressInteraction.Release(pressInteraction))
        if (longPressTriggered) onLongClick()    // 批量模式
        else if (isBatchMode) onSelectClick()
        else onClick()
        break
    }

    val dragDistance = (change.position - downPosition).getDistance()

    if (!longPressTriggered) {
        // 500ms 内：保持当前行为，移动 > touchSlop → 让位左滑
        if (dragDistance > viewConfiguration.touchSlop) {
            longPressJob.cancel()
            interactionChannel.trySend(PressInteraction.Cancel(pressInteraction))
            break
        }
    } else {
        // 500ms 后：仅当容器接管拖拽时 break
        if (isDragActive) {
            interactionChannel.trySend(PressInteraction.Cancel(pressInteraction))
            break
        }
        // 否则继续等待（手指静止 → 抬起时触发 longClick）
    }
}
```

### 4.4 关键阈值参数

| 参数 | 值 | 说明 |
|------|-----|------|
| `touchSlop` | `viewConfiguration.touchSlop`（约 8dp） | 500ms 内移动判左滑 |
| `longPressTimeout` | 500ms | 长按触发延时（与子项一致） |
| `dragThreshold` | 8dp（约 24px @ xxhdpi） | 长按后纵向移动触发拖拽 |
| `hapticThrottle` | 200ms | 跨项交换震动节流 |
| `overlapRatio` | 0.5 | 重叠面积超 50% 触发交换 |

### 4.5 pointerInput Key 策略

| 组件 | key | 理由 |
|------|-----|------|
| 容器 `ReorderableLazyColumn` | `Unit` | 避免重组导致手势中断（项目记忆教训） |
| 子项 `TodoListItem` | `(isBatchMode, onClick, onLongClick, onSelectClick, isDragActive)` | 新增 `isDragActive`，确保拖拽激活时子项手势能及时退出 |

### 4.6 与 SwipeableTodoBox 的协调

`SwipeableTodoBox` 已有 `isEnabled` 参数。HomeScreen 调用处：

```kotlin
SwipeableTodoBox(
    isEnabled = !isBatchMode && !isDragActive,  // 拖拽激活时禁用左滑
    ...
```

由于 `isDragActive` 仅在长按 500ms 后才可能变 true，500ms 内的横向移动仍由 SwipeableTodoBox 正常处理，符合"长按期间和拖拽期间都禁止左滑"的需求。

## 五、拖拽核心算法与视觉反馈

### 5.1 消除 1 帧跳变：手动追踪基线

**问题根源**：`derivedStateOf` 依赖 `layoutInfo`，而 `layoutInfo` 在 `displayItems` 重排后需等下一帧才更新，导致 1 帧跳变。

**解决方案**：手动追踪 `draggedBaseCenterY`（拖拽项中心在视口中的逻辑基线），`dragOffsetY` 直接同步计算，**不依赖 `layoutInfo`**。

```kotlin
// 拖拽状态
var draggedKey by remember { mutableStateOf<Any?>(null) }
var draggedOriginalIndex by remember { mutableIntStateOf(-1) }
var draggedCurrentIndex by remember { mutableIntStateOf(-1) }
var fingerY by remember { mutableFloatStateOf(0f) }

/**
 * 拖拽项中心的逻辑基线 Y（视口坐标）
 *
 * - 拖拽开始时：从 layoutInfo 读取初始值
 * - 交换时：同步更新为被交换项的中心 Y
 * - 自动滚动时：同步按 scrollDelta 调整
 *
 * dragOffsetY = fingerY - draggedBaseCenterY（纯同步计算，无 layoutInfo 依赖）
 */
var draggedBaseCenterY by remember { mutableFloatStateOf(0f) }

val dragOffsetY: Float
    get() = if (isDragActive) fingerY - draggedBaseCenterY else 0f
```

### 5.2 关键操作：同步更新基线

```kotlin
// ━━━ 拖拽开始 ━━━
val info = listState.layoutInfo.visibleItemsInfo.find { it.key == draggedKey }!!
draggedBaseCenterY = (info.offset + info.size / 2f)  // 视口坐标
fingerY = change.position.y

// ━━━ 拖拽移动 ━━━
fingerY = change.position.y
// dragOffsetY 自动 = fingerY - draggedBaseCenterY（同步正确）

// ━━━ 交换（同步，无延迟）━━━
val otherInfo = layoutInfo.visibleItemsInfo.find { it.key == otherKey }!!
val otherCenterY = otherInfo.offset + otherInfo.size / 2f

// 1. 更新基线 = 被交换项的中心（交换即位置互换）
draggedBaseCenterY = otherCenterY

// 2. 重排 displayItems
val newDisplay = displayItems.toMutableList()
val draggedItem = newDisplay.removeAt(draggedCurrentIndex)
newDisplay.add(targetIndex, draggedItem)
displayItems = newDisplay
draggedCurrentIndex = targetIndex

// dragOffsetY 立即 = fingerY - draggedBaseCenterY（正确，无需等 layout 更新）

// ━━━ 自动滚动（同步调整基线）━━━
// scrollBy(delta): delta > 0 列表向起始（上）滚动，视口内容下移，item.offset 增加
if (scrollDelta != 0f) {
    listState.scrollBy(scrollDelta)
    draggedBaseCenterY += scrollDelta  // 基线随视口同步调整
}
```

### 5.3 50% 重叠检测算法

```kotlin
/**
 * 检测被拖项与哪个可见项重叠超过 50%
 * @return 目标项的 key，无交换返回 null
 */
fun findSwapTarget(
    draggedKey: Any,
    fingerY: Float,
    draggedSize: Int,
    layoutInfo: LazyListLayoutInfo
): Any? {
    val draggedCenter = fingerY  // 拖拽项中心 = 手指 Y
    val draggedTop = draggedCenter - draggedSize / 2f
    val draggedBottom = draggedCenter + draggedSize / 2f

    for (other in layoutInfo.visibleItemsInfo) {
        if (other.key == draggedKey) continue

        val otherTop = other.offset.toFloat()
        val otherBottom = (other.offset + other.size).toFloat()

        // 计算重叠高度
        val overlapTop = maxOf(draggedTop, otherTop)
        val overlapBottom = minOf(draggedBottom, otherBottom)
        val overlapHeight = maxOf(0f, overlapBottom - overlapTop)

        // 重叠比例 = 重叠高度 / 较小项高度
        val minSize = minOf(draggedSize, other.size).toFloat()
        if (minSize > 0 && overlapHeight / minSize > 0.5f) {
            return other.key
        }
    }
    return null
}
```

### 5.4 置顶区跨越检测

```kotlin
/**
 * 检测拖拽是否跨越置顶区分界线
 * 置顶区 = 列表顶部连续的 isPinned=true 项
 */
fun checkPinnedZoneCrossed(
    displayItems: List<TodoItem>,
    draggedKey: Any,
    draggedOriginalIsPinned: Boolean,
    draggedCurrentIndex: Int
): Boolean {
    if (draggedCurrentIndex < 0 || draggedCurrentIndex >= displayItems.size) return false
    val currentItem = displayItems[draggedCurrentIndex]
    // 简化：以当前位置上下邻居的 isPinned 判断
    val neighborIsPinned = when {
        draggedCurrentIndex > 0 -> displayItems[draggedCurrentIndex - 1].isPinned
        draggedCurrentIndex < displayItems.size - 1 -> displayItems[draggedCurrentIndex + 1].isPinned
        else -> draggedOriginalIsPinned
    }
    return draggedOriginalIsPinned != neighborIsPinned
}
```

### 5.5 边缘自动滚动（变速）

```kotlin
LaunchedEffect(isDragActive, fingerY) {
    while (isDragActive) {
        val viewportHeight = listState.layoutInfo.viewportSize.height.toFloat()
        val edgeThreshold = with(density) { 80.dp.toPx() }
        val maxSpeed = with(density) { 20.dp.toPx() }  // 最大速度 20dp/帧

        val scrollDelta = when {
            fingerY < edgeThreshold -> {
                // 上边缘：越靠近边缘速度越快
                val ratio = (edgeThreshold - fingerY) / edgeThreshold
                ratio * maxSpeed  // 正值 → 向上滚动
            }
            fingerY > viewportHeight - edgeThreshold -> {
                // 下边缘：越靠近边缘速度越快
                val ratio = (fingerY - (viewportHeight - edgeThreshold)) / edgeThreshold
                -ratio * maxSpeed  // 负值 → 向下滚动
            }
            else -> 0f
        }

        if (scrollDelta != 0f) {
            listState.scrollBy(scrollDelta)
            draggedBaseCenterY += scrollDelta  // 同步调整基线
        }
        delay(16)  // ~60fps
    }
}
```

### 5.8 拖拽结束提交流程

```kotlin
fun commitReorder() {
    if (draggedOriginalIndex != draggedCurrentIndex) {
        val crossedPinnedZone = checkPinnedZoneCrossed(
            displayItems, draggedKey, originalIsPinned, draggedCurrentIndex
        )
        onReorder(
            fromIndex = draggedOriginalIndex,
            toIndex = draggedCurrentIndex,
            crossedPinnedZone = crossedPinnedZone
        )
    }
    // 重置状态
    isDragActive = false
    draggedKey = null
    draggedOriginalIndex = -1
    draggedCurrentIndex = -1
    fingerY = 0f
    draggedBaseCenterY = 0f
}
```

### 5.9 关键保证：无 1 帧跳变

| 操作 | 同步性 | 机制 |
|------|--------|------|
| 拖拽开始 | ✅ 同步 | `draggedBaseCenterY` 从 `layoutInfo` 读取一次，缓存 |
| 拖拽移动 | ✅ 同步 | `dragOffsetY = fingerY - draggedBaseCenterY`（纯计算） |
| 交换 | ✅ 同步 | `draggedBaseCenterY = otherCenterY`（同步赋值） |
| 自动滚动 | ✅ 同步 | `draggedBaseCenterY += scrollDelta`（同步调整） |
| 其他项让位 | ⚠️ 依赖 `animateItem()` | 由 Compose 动画驱动，不涉及被拖项跳变 |

被拖项的视觉位置**永远** = `draggedBaseCenterY + dragOffsetY` = `fingerY`，完全同步跟随手指，**不依赖任何异步 layout 更新**。

## 六、边界情况与错误处理

### 6.1 边界情况处理

| # | 场景 | 处理方式 |
|---|------|---------|
| 1 | 列表为空或仅 1 项 | `pointerInput` 内部 `if (items.size < 2) return@awaitEachGesture` |
| 2 | 拖拽中 `items` 被 ViewModel 更新（如同步） | `displayItems` 由 `remember(items) { mutableStateOf(items) }` 派生，items 变化会重置 displayItems；拖拽中通过 `LaunchedEffect(items)` 监测，若变化则**取消拖拽**（重置状态） |
| 3 | 拖拽中重组导致 `pointerInput` 重启 | `pointerInput(Unit)` 避免；协程取消时 `finally` 块清理 `isDragActive`、`draggedKey` 等 |
| 4 | 批量模式下尝试长按拖拽 | `isDragEnabled = !isBatchMode`；手势内部检查 `if (!isDragEnabled) return@awaitEachGesture`，子项保留原行为（进入批量模式） |
| 5 | 左滑菜单展开时尝试长按 | `isDragEnabled = !isBatchMode && swipeExpandedTodoId == null` |
| 6 | 拖拽中切换排序模式 | 排序菜单 BottomSheet 在拖拽中不可点击（`isDragActive` 时屏蔽 UI 交互） |
| 7 | 拖拽到列表最顶部 | `findSwapTarget` 自然返回 index 0；`draggedCurrentIndex` 不会 < 0 |
| 8 | 拖拽到列表最底部 | 同上，`findSwapTarget` 自然返回最后一个 index |
| 9 | 多指触摸 | `awaitFirstDown` 默认只取第一根手指；`awaitPointerEvent` 主循环中 `event.changes.first()` 只处理第一根 |
| 10 | 手指超出视口上下边界 | `fingerY` 不 clamp（允许超出视口）；`findSwapTarget` 仅检测可见项，超出视口时无交换目标；自动滚动会拉回可见区 |
| 11 | 拖拽中 Activity 销毁/配置变更 | Compose 状态 `remember` 自动保存；`isDragActive` 等用 `rememberSaveable` 持久化（除 `draggedKey` 外） |
| 12 | `draggedKey` 对应的 item 被删除 | `layoutInfo.visibleItemsInfo.find { it.key == draggedKey }` 返回 null → 取消拖拽，重置状态 |
| 13 | 快速长按 + 立即抬起（无移动） | 子项 awaitEachGesture 处理 → `onLongClick` 进入批量模式；容器未进入 Dragging 状态，无副作用 |
| 14 | 长按后小幅抖动（< 8dp） | 不进入拖拽；子项保持 LongPressed 状态等待抬起 → 触发批量模式 |

## 八、已知限制

| 限制 | 影响 | 缓解措施 |
|------|------|---------|
| `displayItems.toMutableList()` 每次交换都复制 | 大列表（>100 项）可能产生 GC 压力 | 待办列表通常 <50 项，可接受 |
| 边缘自动滚动速度上限 20dp/帧 | 极长列表拖到中间位置体验一般 | 后续可改为按距离边缘比例变速（已实现） |
| `draggedKey` 不用 `rememberSaveable` | Activity 销毁时拖拽状态丢失 | 拖拽中销毁 Activity 极罕见，可接受 |
| `findSwapTarget` 依赖 `layoutInfo` 检测其他项 | 极端情况下连续快速交换可能漏检 | 16ms 间隔通常足够 layout 更新 |
