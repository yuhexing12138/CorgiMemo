# 灵感侧边栏标签筛选设计

> **日期**: 2026-07-10
> **范围**: 灵感页侧边栏（AppDrawer.kt 中的 InspirationFilterSection）
> **目标**: 支持多选标签筛选（AND/OR/NOT）、标签计数、搜索过滤、添加标签按钮固定底部

## 背景

灵感页侧边栏当前的 `InspirationFilterSection` 存在以下局限：
- **单选模式**：`selectedTag: String?` 仅支持单标签筛选
- **无计数**：所有标签的 `count` 硬编码为 0
- **添加按钮位置不当**："添加标签"按钮在 `InspirationFilterSection` 内部，不在固定底部
- **无搜索**：无法在侧边栏中搜索标签

需参考待办页侧边栏模式（`CategoryGroupSection` + 外部 `AddCategoryButton`），对灵感页侧边栏进行全面升级。

## 改动前状态

| 属性 | 当前值 |
|------|--------|
| 选择模式 | 单选 (`selectedTag: String?`) |
| 标签计数 | 硬编码 `count = 0` |
| 搜索功能 | 无 |
| "添加标签"位置 | `InspirationFilterSection` 内部 (line 454) |
| 筛选模式 | 无 (仅单一选择) |

## 设计决策

### 1. 多选标签筛选 + 三种模式

**状态变更**：
- `selectedTag: String?` → `selectedTags: Set<String>` (多选集合)
- 新增 `TagFilterMode` 枚举：`OR` / `AND` / `NOT`

**筛选逻辑**：

| 模式 | 行为 | 示例（选中"产品"和"设计"）|
|------|------|------|
| OR (默认) | 显示包含任意选中标签的灵感 | 有"产品"或有"设计"或两者皆有的灵感 |
| AND | 显示同时包含所有选中标签的灵感 | 同时有"产品"和"设计"标签的灵感 |
| NOT | 显示不包含任何选中标签的灵感 | 既没有"产品"也没有"设计"的灵感 |

**模式切换**：3 个芯片按钮（OR / AND / NOT），当前选中项高亮，默认 OR。

### 2. 标签计数

**ViewModel 新增**：
- `tagCounts: StateFlow<Map<String, Int>>` — 每个标签对应的灵感数量
- 计算方式：遍历所有灵感，解码 tags JSON，统计每个标签出现次数

**UI 显示**：每个标签右侧显示计数徽章（与待办页 `CategoryItem` 一致）。

### 3. 搜索框

**位置**：标签筛选标题下方、筛选模式芯片上方。

**行为**：
- 实时过滤标签列表（输入即过滤）
- 支持精确匹配和模糊匹配（`contains` 忽略大小写）
- 搜索为可选操作：用户可以不搜索直接选择标签
- 搜索框为本地 Compose 状态（`remember` + `mutableStateOf`），不需要 ViewModel 持久化

### 4. "全部灵感"选项

- 保留在标签列表顶部
- 点击时清空 `selectedTags`（清空所有选中标签）
- 显示总灵感数作为计数
- 与多选标签互斥：选中"全部灵感"时不能同时选中其他标签

### 5. "添加标签"按钮固定底部

**改动**：
- 从 `InspirationFilterSection` 内部移除
- 移到 `AppDrawerContent` 中 `TabItem.INSPIRE` 分支的外部（与待办页 `AddCategoryButton` 同级）

**改动前**（AppDrawer.kt line 146-154）：
```kotlin
TabItem.INSPIRE -> {
    InspirationFilterSection(
        tags = inspirationTags,
        selectedTag = selectedTag,
        onTagClick = onTagClick,
        onAddTagClick = onAddTagClick,
        modifier = Modifier.weight(1f)

```

**改动后**：
```kotlin
TabItem.INSPIRE -> {
    InspirationFilterSection(
        tags = inspirationTags,
        selectedTags = selectedTags,
        filterMode = tagFilterMode,
        tagCounts = tagCounts,
        onTagClick = onTagClick,
        onFilterModeChange = onFilterModeChange,
        modifier = Modifier.weight(1f)
    )
    AddCategoryButton(text = "添加标签", onClick = onAddTagClick)
}
```

## 数据模型

### 新增枚举

```kotlin
enum class TagFilterMode {
    OR,   // 并集：包含任意选中标签
    AND,  // 交集：包含所有选中标签
    NOT   // 非：不包含任何选中标签
}
```

### ViewModel 新增状态

```kotlin
/** 选中的标签集合（多选） */
val selectedTags: StateFlow<Set<String>>

/** 当前筛选模式 */
val tagFilterMode: StateFlow<TagFilterMode>

/** 每个标签对应的灵感数量 */
val tagCounts: StateFlow<Map<String, Int>>
```

### 筛选逻辑

```kotlin
/** 根据选中标签和筛选模式过滤灵感列表 */
fun filterInspirationsByTags(
    inspirations: List<Inspiration>,
    selectedTags: Set<String>,
    mode: TagFilterMode
): List<Inspiration> {
    if (selectedTags.isEmpty()) return inspirations
    return when (mode) {
        TagFilterMode.OR -> inspirations.filter { insp ->
            val tags = decodeTags(insp.tags)
            tags.any { it in selectedTags }
        }
        TagFilterMode.AND -> inspirations.filter { insp ->
            val tags = decodeTags(insp.tags)
            selectedTags.all { it in tags }
        }
        TagFilterMode.NOT -> inspirations.filter { insp ->
            val tags = decodeTags(insp.tags)
            selectedTags.none { it in tags }
        }
    }
}
```

## 验证标准

1. 侧边栏显示所有标签，每个标签右侧显示灵感计数
2. 点击标签可切换选中/取消（多选）
3. OR/AND/NOT 三种模式可通过芯片按钮切换
4. 搜索框实时过滤标签列表
5. "全部灵感"点击清空所有选中标签
6. "添加标签"按钮固定在底部（不随列表滚动）
7. 灵感列表根据选中标签和筛选模式正确过滤
8. 编译通过，无 import 缺失
