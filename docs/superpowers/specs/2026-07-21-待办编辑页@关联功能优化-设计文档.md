## 1. 概述

### 1.1 目标

优化待办编辑页的「@关联」功能，从单选 Popup 升级为多选 BottomSheet，支持跨三种类型（待办/灵感/日期）卡片关联，并将关联显示位置从编辑页底部上移到每个分组容器内部的 Chips 行和按钮行之间。

### 1.2 范围

**包含**：
- 待办编辑页（`TodoEditScreen`）的 @关联功能升级
- `CardRelation` 表新增 `groupId` 字段（分组级别关联）
- 新增 3 个 UI 组件：`RelationPickerBottomSheet`、`LinkedCardsRow`、`LinkedCardPreviewDialog`
- 修改 `TodoGroupContainer` 在 Chips 行和按钮行之间插入已关联区域
- ViewModel/Repository/DAO 全链路适配 `groupId`

**不包含**：
- 灵感编辑页、特殊日期编辑页的关联功能改造（保留 todo 级别关联）
- `MentionTriggerPopup` 组件的删除（保留供其他模块继续使用）
- 已关联卡片在列表页（首页）的提示显示（已有 `RelationSummary` 机制）

### 1.3 核心设计决策（已通过视觉原型确认）

| # | 决策点 | 选定方案 | 备选 |
|---|--------|---------|------|
| 1 | 弹窗形态 | 底部 BottomSheet + 完成按钮 | Popup / Dialog |
| 2 | 已关联卡片交互 | 显示 + ×删除 + 点击弹出预览 | 仅删除 / 跳转详情 |
| 3 | 已关联卡片位置 | 分组容器内部，Chips 行和按钮行之间 | 编辑页底部 / 预计时长下方 |
| 4 | 已关联卡片样式 | 横向 Chip 流（与 Chips 行风格统一） | 纵向列表 / 网格 |
| 5 | 多分组关联策略 | 关联改为分组级别（CardRelation 加 groupId） | 仅主分组显示 / 所有分组共享 |
| 6 | BottomSheet 内分组 | 按类型分组（待办/灵感/日期），可点击折叠/展开 | 不分组 / 双列布局 |
| 7 | 触发方式 | @触发 + ＋按钮双入口 | 仅 @ / 仅 ＋ |
| 8 | 预览弹窗 | 居中 Dialog，三种类型差异化展示 | BottomSheet 预览 / 直接跳转 |

### 3.1 CardRelation Entity 修改

**文件**: `app/src/main/java/com/corgimemo/app/data/model/CardRelation.kt`

```kotlin
@Entity(
    tableName = "card_relations",
    indices = [
        // 修改：source 索引加上 groupId，支持按分组查询
        Index(value = ["sourceType", "sourceId", "groupId"]),
        Index(value = ["targetType", "targetId"]),
        // 修改：唯一约束加上 groupId，同一分组内不能重复关联

data class CardRelation(
    @PrimaryKey(autoGenerate = true)
    val id: Long = 0,

    /** 关联发起方类型: "todo" | "inspiration" | "date" */
    val sourceType: String,

    /** 关联发起方ID */
    val sourceId: Long,

    /** 关联发起方分组ID（仅 todo 类型有意义；inspiration/date 默认 0） */
    @ColumnInfo(defaultValue = "0")
    val groupId: Int = 0,

    /** 关联目标方类型: "todo" | "inspiration" | "date" */
    val targetType: String,

    /** 关联目标方ID */
    val targetId: Long,

    /** 创建时间戳 */
    @ColumnInfo(defaultValue = "0")
    val createdAt: Long = System.currentTimeMillis()
)
```

### 3.2 Migration

**新增文件**: `app/src/main/java/com/corgimemo/app/data/local/Migration42To43AddGroupIdToCardRelations.kt`

当前数据库版本为 42（见 `CorgiMemoDatabase.kt:34`），本次升级到 43。需同步修改 `CorgiMemoDatabase.kt` 的 `version = 43`，并在 `migrations` 数组中注册新 Migration。

```kotlin
val MIGRATION_42_TO_43 = object : Migration(42, 43) {
    override fun migrate(database: SupportSQLiteDatabase) {
        // 新增 groupId 字段，默认值 0（兼容旧数据）
        database.execSQL("ALTER TABLE card_relations ADD COLUMN groupId INTEGER NOT NULL DEFAULT 0")

        // 重建索引（删除旧索引 + 创建新索引）
        database.execSQL("DROP INDEX IF EXISTS index_card_relations_sourceType_sourceId")
        database.execSQL("DROP INDEX IF EXISTS index_card_relations_sourceType_sourceId_targetType_targetId")

        database.execSQL("CREATE INDEX index_card_relations_sourceType_sourceId_groupId ON card_relations(sourceType, sourceId, groupId)")
        database.execSQL("CREATE UNIQUE INDEX index_card_relations_sourceType_sourceId_groupId_targetType_targetId ON card_relations(sourceType, sourceId, groupId, targetType, targetId)")
    }
}
```

**规则遵循**: 按 [entity与migration同步检查规则](../../.trae/rules/entity与migration同步检查.md)，`@ColumnInfo(defaultValue = "0")` 与 SQL `DEFAULT 0` 一致。

### 4.2 方法变更

```kotlin
/**
 * 添加关联（分组级别）
 * @param targetType 目标类型
 * @param targetId 目标ID
 * @param groupId 当前分组ID
 */
fun addRelation(targetType: String, targetId: Long, groupId: Int) {
    viewModelScope.launch {
        val todoId = existingTodo?.id ?: 0L
        if (todoId == 0L) {
            _errorEvent.emit("请先保存待办再添加关联")
            return@launch
        }
        val relation = CardRelation(
            sourceType = "todo",
            sourceId = todoId,
            groupId = groupId,
            targetType = targetType,
            targetId = targetId
        )
        val result = cardRelationRepository.addRelation(relation)
        when (result) {
            -1L -> _errorEvent.emit("该卡片已关联")  // 静默或提示
            -2L -> _errorEvent.emit("每组最多关联 10 张卡片")
            else -> {
                val updatedList = (_groupRelations.value[groupId] ?: emptyList()) + relation.copy(id = result)
                _groupRelations.value = _groupRelations.value + (groupId to updatedList)
            }
        }
    }
}

/**
 * 删除关联
 * @param relationId 关联ID
 * @param groupId 当前分组ID
 */
fun deleteRelation(relationId: Long, groupId: Int) {
    viewModelScope.launch {
        cardRelationRepository.removeRelationById(relationId)
        val currentList = _groupRelations.value[groupId] ?: emptyList()
        _groupRelations.value = _groupRelations.value + (groupId to currentList.filter { it.id != relationId })
    }
}

/**
 * 加载某 todo 的所有分组的关联
 * @param todoId 待办ID
 * @param groupIds 所有分组ID列表
 */
fun loadGroupRelations(todoId: Long, groupIds: List<Int>) {
    viewModelScope.launch {
        val relationsMap = mutableMapOf<Int, List<CardRelation>>()
        groupIds.forEach { gid ->
            relationsMap[gid] = cardRelationRepository.getRelationsBlocking("todo", todoId, gid)
        }
        _groupRelations.value = relationsMap
    }
}

// searchCards 不变
fun searchCards(query: String, callback: (List<CardSearchResult>) -> Unit) { ... }
```

**文件**: `app/src/main/java/com/corgimemo/app/ui/components/LinkedCardPreviewDialog.kt`

**三种类型的差异化展示**:

| 类型 | 显示字段 |
|------|---------|
| 📝 待办 | 截止时间(橙高亮) + 分类 + 优先级 + 子任务进度条 |
| 💡 灵感 | 创建时间 + 内容预览(3行渐变遮罩) + 图片缩略图(最多3张+N) + 标签 |
| 📅 日期 | 目标日期(粉红) + 倒计时大数字(橙色渐变背景) + 备注 |

**底部三个按钮**:
- **取消关联**（红边框透明底）→ `onUnlink(relation.id)` 后 `onDismiss()`
- **跳转详情**（橙底白字）→ `onJumpToDetail(relation.targetType, relation.targetId)` 后 `onDismiss()`
- **关闭**（灰底）→ `onDismiss()`

**数据加载**: Dialog 内部用 `LaunchedEffect(relation)` 调用 ViewModel 获取完整卡片数据（如 `getTodoDetail`、`getInspirationDetail`、`getSpecialDateDetail`），加载中显示进度条。

```kotlin
            pickerTargetGroupId = null
        },
        onConfirm = { selectedCards ->
            selectedCards.forEach { card ->
                viewModel.addRelation(card.cardType, card.cardId, targetGid)
            }
            showRelationPicker = false
            pickerTargetGroupId = null
        },
        searchCards = { query, callback -> viewModel.searchCards(query, callback) }
    )
}

// 🆕 预览 Dialog
previewingRelation?.let { relation ->
    LinkedCardPreviewDialog(
        relation = relation,
        onDismiss = { previewingRelation = null },
        onUnlink = { relationId ->
            viewModel.deleteRelation(relationId, relation.groupId)
            previewingRelation = null
        },
        onJumpToDetail = { cardType, cardId ->
            // 跳转逻辑：根据 cardType 路由到不同页面
            when (cardType) {
                "todo" -> navController.navigate("todo_edit?todoId=$cardId")
                "inspiration" -> navController.navigate("inspiration_edit?inspirationId=$cardId")
                "date" -> navController.navigate("special_date_detail?dateId=$cardId")
            }
            previewingRelation = null
        }
    )
}
```

## 6. 数据流

### 6.1 添加关联流程

```
[用户操作]
  ├─ A. 在编辑框输入 @
  │     ↓
  │  CheckboxEditText 检测到 @ → onSpecialCharDetected("@", query)
  │     ↓
  │  TodoEditScreen: pickerTargetGroupId = 当前行 groupId; showRelationPicker = true
  │
  └─ B. 点击 LinkedCardsRow 的 ＋ 按钮
        ↓
       onAddRelationClick(groupId)
        ↓
       TodoEditScreen: pickerTargetGroupId = groupId; showRelationPicker = true

[BottomSheet 弹窗]
  RelationPickerBottomSheet 显示
     ↓
  LaunchedEffect: viewModel.searchCards("", callback) → 默认列出全部
     ↓
  用户输入关键词 → 实时 searchCards(query) → 过滤
     ↓
  用户点击卡片项 → selectedIds 切换
     ↓
  点击"完成" → onConfirm(selectedCards)

[ViewModel 处理]
  TodoEditScreen.onConfirm:
    for each card in selectedCards:
        viewModel.addRelation(card.cardType, card.cardId, groupId)
     ↓
  ViewModel.addRelation:
    - 校验 todoId > 0（已保存）
    - cardRelationRepository.addRelation(relation)
    - 成功: _groupRelations[groupId] += relation
    - 失败: _errorEvent 发送错误消息

[UI 更新]
  groupRelations StateFlow 变化
     ↓
  TodoGroupContainer 重组
     ↓
  LinkedCardsRow 渲染新的 Chip
```

### 6.2 删除关联流程

```
用户点击 Chip 的 ×
  ↓
LinkedCardsRow.onChipDelete(relationId, groupId)
  ↓
TodoEditScreen: viewModel.deleteRelation(relationId, groupId)
  ↓
ViewModel.deleteRelation:
  - cardRelationRepository.removeRelationById(relationId)
  - _groupRelations[groupId] = filter { it.id != relationId }
  ↓
UI 自动重组，Chip 消失
```

### 6.3 预览流程

```
用户点击 Chip（非 × 区域）
  ↓
LinkedCardsRow.onChipClick(relation)
  ↓
TodoEditScreen: previewingRelation = relation
  ↓
LinkedCardPreviewDialog 显示
  ↓
LaunchedEffect: 根据 relation.targetType 加载完整卡片数据
  ↓
用户操作:
  ├─ 取消关联 → onUnlink(relation.id) → viewModel.deleteRelation → 关闭 Dialog
  ├─ 跳转详情 → onJumpToDetail(type, id) → navController.navigate(...) → 关闭 Dialog
  └─ 关闭/遮罩/返回键 → onDismiss()
```

## 7. 错误处理

| 场景 | 检测点 | 处理方式 |
|------|--------|---------|
| 关联数量超过 10 个/分组 | `Repository.addRelation` 返回 -2L | ViewModel `_errorEvent.emit("每组最多关联 10 张卡片")`；UI 通过 Snackbar 显示 |
| 重复关联 | `Repository.addRelation` 返回 -1L | 静默忽略（已存在），不提示 |
| 被关联卡片被删除 | `removeAllForCard` 已有逻辑 | 自动清理所有指向它的关联，UI 下一帧自动消失 |
| 搜索结果为空 | BottomSheet 内 `searchResults.isEmpty()` | 显示"无匹配结果"提示文字 |
| 关联的卡片标题变更 | ViewModel 持有 `relationTitles` 缓存 | 当 `_groupRelations` 变化时异步刷新标题；考虑跨页返回时刷新 |
| 未保存就尝试关联 | `todoId == 0` | ViewModel `_errorEvent.emit("请先保存待办再添加关联")`；Snackbar 显示 |
| BottomSheet 加载卡片数据失败 | `LaunchedEffect` 内 try-catch | 显示"加载失败，请重试" + 重试按钮 |
| 预览 Dialog 加载卡片数据失败 | 同上 | 同上 |
| DAO Migration 失败 | `Migration` 抛异常 | 应用 fallback 到 `fallbackToDestructiveMigration`（已在 Database 配置） |

## 9. 兼容性处理

### 9.1 灵感/特殊日期编辑页

**保持不变**: 灵感编辑页和特殊日期编辑页的关联功能暂不改造，仍使用 `MentionTriggerPopup` 单选模式。

**原因**:
- 灵感和特殊日期没有"多分组"概念，todo 级别关联已够用
- 减少本次改造范围，避免过度工程化

**后续扩展**: 如未来灵感/日期也需要多选，可复用 `RelationPickerBottomSheet`，传入 `groupId = 0`。

### 9.2 旧数据迁移

- Migration 后所有旧关联记录 `groupId = 0`（默认归到主分组）
- 旧关联在待办编辑页的主分组（groupId=0）容器内显示
- 用户视觉上无感知差异

### 9.3 MentionTriggerPopup 组件

**保留**: 不删除 `MentionTriggerPopup.kt` 文件，因为：
- 灵感编辑页（`InspirationEditScreen.kt:1164`）确实仍在使用 `MentionTriggerPopup`，删除会破坏灵感编辑页的 @关联功能
- 作为单选模式的备选实现，未来特殊日期编辑页也可复用

**待办编辑页不再使用**: `TodoEditScreen.kt:1400` 移除 `MentionTriggerPopup` 调用，替换为 `RelationPickerBottomSheet`。

> ⚠️ **重要约束**: 修改 `TodoEditScreen.kt` 时只能移除待办编辑页的调用，不能删除 `MentionTriggerPopup.kt` 文件本身，否则会导致灵感编辑页编译失败。
