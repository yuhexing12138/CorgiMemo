# 提醒弹窗芯片布局与默认值修复

## 背景

提醒设置底部弹窗（`ReminderPickerBottomSheet`）在集成截止日期功能后出现以下视觉与交互问题：

1. **提醒时间行与截止日期行的芯片竖向重叠**：两行 `DateTimeRow` 之间没有间距，且当芯片文本较长时行高被撑开，导致相邻行的芯片在垂直方向重叠。
2. **"未设置"占位文字多余**：用户希望芯片始终显示具体日期/时间，而非"未设置"占位。
3. **标签颜色不符合预期**：未激活状态下"提醒时间"/"截止日期"标签为灰色 `#999999`，应为黑色 `#2D2D2D`。
4. **行间距不均等**：提醒时间行 → 截止日期行 → 重复提醒行 → 农历行 之间的 Spacer 不统一。
5. **待办卡片不应显示截止时间**：`TodoListItem` 中截止时间显示需要移除（通知弹窗仍保留）。

## 设计目标

- 消除芯片垂直重叠，确保两行 `DateTimeRow` 视觉清晰、对齐
- 移除"未设置"占位，打开弹窗即显示默认日期/时间
- 截止时间默认与提醒时间一致（日期 + 时分完全相同）
- 统一四行（提醒时间、截止日期、重复提醒、农历）的间距为 16dp
- 待办卡片不再显示截止时间

## 详细设计

### 1. DateTimeRow 标签颜色调整

**文件**：`ReminderPickerBottomSheet.kt` → `DateTimeRow` 组件

**当前**：
```kotlin
color = if (isActive) MaterialTheme.colorScheme.primary else Color(0xFF999999),
```

**修改后**：
```kotlin
color = if (isActive) MaterialTheme.colorScheme.primary else Color(0xFF2D2D2D),
```

**影响范围**：仅 `DateTimeRow` 内的标签 Text，不影响其他模块。

### 2. 移除"未设置"显示，芯片始终显示日期/时间

**文件**：`ReminderPickerBottomSheet.kt`

#### 2.1 状态简化

**当前**：通过 `isReminderSet` / `isDueDateSet` 两个布尔标志控制是否显示"未设置"。

**修改后**：
- 移除 `isReminderSet` 和 `isDueDateSet` 状态变量
- 弹窗打开时，`selectedDate` 默认为今天，`selectedHour/Minute` 默认为当前系统时间
- `dueDate` 默认等于 `selectedDate`，`dueHour/Minute` 默认等于 `selectedHour/Minute`
- 编辑旧待办时：由调用方传入 `initialDateMillis` / `initialDueDateMillis`，若非 null 则用旧值初始化

#### 2.2 DateTimeRow 调用修改

**提醒时间行**：
```kotlin
DateTimeRow(
    label = rowLabel,
    isActive = editTarget == EditTarget.REMINDER,
    dateText = "${selectedDate.year}/${String.format("%02d", selectedDate.monthValue)}/${String.format("%02d", selectedDate.dayOfMonth)}",
    timeText = "${String.format("%02d", selectedHour)}:${String.format("%02d", selectedMinute)}",
    isCalendarActive = viewMode == "calendar",
    isTimeActive = viewMode == "time",
    onDateClick = {
        editTarget = EditTarget.REMINDER
        viewMode = "calendar"
    },
    onTimeClick = {
        editTarget = EditTarget.REMINDER
        viewMode = "time"
    },
    onClear = null  // 提醒时间行不显示清除按钮（移除 isReminderSet 后清除语义不明确）
)
```

**截止日期行**：
```kotlin
DateTimeRow(
    label = "截止日期",
    isActive = editTarget == EditTarget.DUE_DATE,
    dateText = "${dueDate.year}/${String.format("%02d", dueDate.monthValue)}/${String.format("%02d", dueDate.dayOfMonth)}",
    timeText = "${String.format("%02d", dueHour)}:${String.format("%02d", dueMinute)}",
    isCalendarActive = viewMode == "calendar",
    isTimeActive = viewMode == "time",
    onDateClick = {
        editTarget = EditTarget.DUE_DATE
        viewMode = "calendar"
    },
    onTimeClick = {
        editTarget = EditTarget.DUE_DATE
        viewMode = "time"
    },
    onClear = null,  // 截止日期行不显示清除按钮
    highlightEnabled = isDueDateAutoFixed
)
```

#### 2.3 确定按钮逻辑调整

由于移除了 `isReminderSet` / `isDueDateSet`，确定按钮的提交逻辑简化：

```kotlin
.clickable {
    // 计算提醒时间戳（始终有值，因为默认为今天当前时间）
    val reminderMillis = selectedDate.atTime(selectedHour, selectedMinute)
        .atZone(java.time.ZoneId.systemDefault())
        .toInstant().toEpochMilli()

    // 计算截止日期时间戳（始终有值，默认等于提醒时间）
    val dueDateMillis = dueDate.atTime(dueHour, dueMinute)
        .atZone(java.time.ZoneId.systemDefault())
        .toInstant().toEpochMilli()

    // 截止时间自动修正：早于提醒时间时自动调整为提醒时间
    if (dueDateMillis < reminderMillis) {
        val adjustedDateTime = java.time.Instant.ofEpochMilli(reminderMillis)
            .atZone(java.time.ZoneId.systemDefault())
        dueDate = adjustedDateTime.toLocalDate()
        dueHour = adjustedDateTime.hour
        dueMinute = adjustedDateTime.minute
        editTarget = EditTarget.DUE_DATE
        isDueDateAutoFixed = true
        Toast.makeText(context, "截止时间已自动调整为提醒时间，请确认", Toast.LENGTH_SHORT).show()
        return@clickable
    }

    onConfirm(
        reminderMillis,
        selectedHour,
        selectedMinute,
        repeatType,
        calendarEnabled && showAdvancedOptions,
        dueDateMillis
    )
}
```

**注意**：移除"至少设置一个"校验，因为默认状态下两者都有值。

#### 2.4 标题动态化逻辑调整

由于 `isReminderSet` / `isDueDateSet` 已移除，标题判断需要改为基于 `initialDateMillis`：

```kotlin
val displayTitle = if (initialDateMillis == null && initialDueDateMillis != null) "设置截止日期" else title
```

### 3. 截止时间默认与提醒时间一致

**文件**：`ReminderPickerBottomSheet.kt`

#### 3.1 初始化逻辑

**当前**：
```kotlin
var dueDate by remember { mutableStateOf(initDueLocalDate) }
var dueHour by remember { mutableIntStateOf(23) }  // 默认 23:59
var dueMinute by remember { mutableIntStateOf(59) }
```

**修改后**：
```kotlin
// 截止时间默认与提醒时间一致
val initDueZoned = if (initialDueDateMillis != null) {
    java.time.Instant.ofEpochMilli(initialDueDateMillis)
        .atZone(java.time.ZoneId.systemDefault())
} else {
    // 未传入截止日期时，使用提醒时间的时区日期（与 selectedDate/selectedHour/Minute 一致）
    initLocalDate.atTime(initialHour, initialMinute)
        .atZone(java.time.ZoneId.systemDefault())
}
var dueDate by remember { mutableStateOf(initDueZoned.toLocalDate()) }
var dueHour by remember { mutableIntStateOf(initDueZoned.hour) }
var dueMinute by remember { mutableIntStateOf(initDueZoned.minute) }
```

当 `initialDueDateMillis != null` 时，从中解析出日期/时分；否则使用 `initLocalDate` + `initialHour` + `initialMinute`（与提醒时间完全一致）。

#### 3.2 联动同步（可选，YAGNI）

**决策**：**不实现联动同步**。原因：
- 截止时间默认值在弹窗打开时已设为与提醒时间一致
- 用户修改提醒时间后，截止时间保持不变是合理行为（用户可能想设不同的截止时间）
- 联动同步会增加状态复杂度（需跟踪 `isDueDateIndependent`），违反 YAGNI

### 4. 芯片垂直重叠修复

**文件**：`ReminderPickerBottomSheet.kt` → `DateTimeRow` 组件

#### 4.1 根因分析

当前 `DateTimeRow` 使用 `Row` 容器，布局结构：
```
[标签] [Spacer(weight=1f)] [日期芯片] [Spacer(8dp)] [时间芯片] [Spacer(8dp)] [清除图标]
```

问题：
- 标签无固定宽度，`Spacer(weight=1f)` 撑开弹性空间时，若标签文本较长，右侧芯片区域被压缩
- 芯片 Text 未设置 `maxLines = 1`，文本换行会撑高芯片高度
- 两行 `DateTimeRow` 之间无 Spacer，行高不稳定时相邻芯片在垂直方向重叠

#### 4.2 修复方案

**布局调整**：
```kotlin
@Composable
private fun DateTimeRow(
    label: String,
    isActive: Boolean,
    dateText: String,
    timeText: String?,
    isCalendarActive: Boolean,
    isTimeActive: Boolean,
    onDateClick: () -> Unit,
    onTimeClick: () -> Unit,
    onClear: (() -> Unit)?,
    highlightEnabled: Boolean = false
) {
    val rowBackground = if (highlightEnabled) Color(0xFFFFF3E8) else Color.Transparent
    Row(
        modifier = Modifier
            .fillMaxWidth()
            .clip(RoundedCornerShape(8.dp))
            .background(rowBackground)
            .padding(vertical = if (highlightEnabled) 4.dp else 0.dp),
        verticalAlignment = Alignment.CenterVertically
    ) {
        // 标签固定宽度，避免压缩右侧芯片区域
        Text(
            text = label,
            fontSize = 15.sp,
            color = if (isActive) MaterialTheme.colorScheme.primary else Color(0xFF2D2D2D),
            fontWeight = FontWeight.Medium,
            modifier = Modifier.width(64.dp)
        )

        // 右侧芯片区域整体 weight(1f) 右对齐
        Row(
            modifier = Modifier.weight(1f),
            horizontalArrangement = Arrangement.spacedBy(8.dp, Alignment.End),
            verticalAlignment = Alignment.CenterVertically
        ) {
            // 日期芯片
            Row(
                modifier = Modifier
                    .clip(RoundedCornerShape(8.dp))
                    .background(if (isActive && isCalendarActive) MaterialTheme.colorScheme.primary else Color(0xFFFFE4CC))
                    .clickable(onClick = onDateClick)
                    .padding(horizontal = 12.dp, vertical = 6.dp),
                verticalAlignment = Alignment.CenterVertically
            ) {
                Text(
                    text = dateText,
                    fontSize = 13.sp,
                    color = if (isActive && isCalendarActive) Color.White else Color(0xFF2D2D2D),
                    fontWeight = if (isActive && isCalendarActive) FontWeight.Medium else FontWeight.Normal,
                    maxLines = 1,
                    softWrap = false
                )
            }

            // 时间芯片
            Row(
                modifier = Modifier
                    .clip(RoundedCornerShape(8.dp))
                    .background(if (isActive && isTimeActive) MaterialTheme.colorScheme.primary else Color(0xFFFFE4CC))
                    .clickable(onClick = onTimeClick)
                    .padding(horizontal = 12.dp, vertical = 6.dp),
                verticalAlignment = Alignment.CenterVertically
            ) {
                Text(
                    text = timeText ?: "",
                    fontSize = 13.sp,
                    color = if (isActive && isTimeActive) Color.White else Color(0xFF2D2D2D),
                    fontWeight = if (isActive && isTimeActive) FontWeight.Medium else FontWeight.Normal,
                    maxLines = 1,
                    softWrap = false
                )
            }

            // 清除按钮（仅在 onClear 非 null 时显示）
            if (onClear != null) {
                Icon(
                    imageVector = Icons.Default.Close,
                    contentDescription = "清除",
                    modifier = Modifier
                        .size(18.dp)
                        .clip(RoundedCornerShape(9.dp))
                        .clickable(onClick = onClear)
                        .padding(2.dp),
                    tint = Color(0xFF999999)
                )
            }
        }
    }
}
```

**关键改动**：
- 标签 `Modifier.width(64.dp)` 固定宽度
- 右侧芯片区域 `Modifier.weight(1f)` + `Arrangement.spacedBy(8.dp, Alignment.End)` 右对齐
- 芯片 Text 添加 `maxLines = 1` + `softWrap = false` 防止换行
- 移除 `isDateSelected` 参数（已无用，因为始终显示具体日期/时间）
- 芯片文字颜色从灰色 `#666666` 改为黑色 `#2D2D2D`（与标签保持一致）

## 风险与验证

### 风险

1. **移除 `isReminderSet` / `isDueDateSet` 后确定按钮始终提交非 null 值**
   - **影响**：调用方（`TodoEditScreen`、`HomeScreen`）原本处理 null 提醒/截止日期的逻辑不再触发
   - **缓解**：调用方已有非 null 分支处理，移除 null 分支不影响功能
   - **验证**：测试新建待办、编辑旧待办、批量设置场景

2. **截止时间默认等于提醒时间可能导致用户困惑**
   - **影响**：用户修改提醒时间后，截止时间不变（仍为打开弹窗时的默认值）
   - **缓解**：这是合理行为，用户可单独修改截止时间
   - **验证**：手动测试修改提醒时间后截止时间是否保持不变

3. **`DateTimeRow` 移除 `isDateSelected` 参数**
   - **影响**：调用方需同步移除该参数
   - **缓解**：仅 `ReminderPickerBottomSheet` 内部调用，影响范围可控

### 验证清单

- [ ] 打开弹窗时，提醒时间和截止时间都显示今天日期 + 当前时间
- [ ] 编辑旧待办时，提醒时间和截止时间显示待办原有值
- [ ] 标签颜色为黑色 `#2D2D2D`，激活时为橙色主题色
- [ ] 两行芯片在垂直方向不重叠，对齐良好
- [ ] 提醒时间行、截止日期行、重复提醒行、农历行间距均为 16dp
- [ ] 待办卡片不显示截止时间
- [ ] 通知弹窗仍显示截止日期
- [ ] 截止时间早于提醒时间时自动修正并高亮
- [ ] 确定按钮提交正确的提醒时间和截止时间

## 不做的事

- **不实现截止时间与提醒时间联动同步**：打开弹窗时默认一致即可，修改一个不影响另一个（YAGNI）
- **不保留清除按钮**：移除"未设置"状态后，清除按钮语义不明确；若用户想清除提醒，可在待办编辑页其他入口操作
- **不修改通知弹窗截止日期显示**：需求明确仅移除待办卡片上的显示
- **不修改数据库字段**：`dueDate` 字段保留，仅 UI 层调整
