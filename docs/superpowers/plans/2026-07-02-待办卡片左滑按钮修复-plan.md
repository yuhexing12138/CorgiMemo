# 待办卡片左滑按钮修复实现计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 修复待办卡片左滑操作面板中分享/置顶/删除三个按钮无响应的问题，并实现置顶按钮根据卡片置顶状态动态切换文字与图标。

**Architecture:** 通过调整 SwipeableTodoBox 内容层 Box 的 modifier 链顺序（`.offset` 移到 `.pointerInput` 之前）使 pointerInput 命中区域跟随视觉偏移，与按钮区域自然分离；外层 Layout pointerInput 增加按钮区域放行逻辑；新增 `isPinned` 参数，`buttons` 列表用 `remember(isPinned)` 动态生成，置顶按钮根据状态切换文字（"置顶"/"取消置顶"）和图标（`Icons.Outlined.PushPin`/`Icons.Filled.PushPin`）。

**Tech Stack:** Jetpack Compose (BOM 2026.04.01 + Compose 1.9.2)、Kotlin、Material Icons

**约束:** 不改动 `detectHorizontalDragGestures` 的任何回调（onDragStart / onDragEnd / onDragCancel / onDrag）；不改动 `ReorderableLazyColumn` 拖拽排序逻辑；不改动 `HomeScreen.kt` 中三个按钮的 onClick 业务接入。

## Task 2: 外层 Layout pointerInput 智能分流

**Files:**
- Modify: `app/src/main/java/com/corgimemo/app/ui/components/SwipeableTodoBox.kt` 第 243-250 行

**目的:** 外层 Layout 的 pointerInput 当前在 `isExpanded=true` 时无条件 `down.consume()`，会拦截按钮点击。修改为：仅在非按钮区域消费 down，按钮区域放行让事件透传给按钮 clickable。

- [ ] **Step 1: 定位外层 Layout 的 pointerInput**

打开 `app/src/main/java/com/corgimemo/app/ui/components/SwipeableTodoBox.kt`，找到第 236-250 行的 Layout modifier：

```kotlin
Layout(
    modifier = modifier
        .clip(RoundedCornerShape(cornerRadiusDp))
        .pointerInput(isEnabled, isExpanded) {
            if (!isEnabled || !isExpanded) return@pointerInput
            awaitEachGesture {
                val down = awaitFirstDown(requireUnconsumed = false)
                down.consume()
            }
        },
    content = { ... }
)
```

- [ ] **Step 2: 修改 pointerInput 增加按钮区域放行逻辑**

将 `down.consume()` 替换为条件消费：down 落在按钮区域（`x >= size.width - actionsWidthPx`）则不消费，让事件透传给按钮。

修改后：

```kotlin
Layout(
    modifier = modifier
        .clip(RoundedCornerShape(cornerRadiusDp))
        .pointerInput(isEnabled, isExpanded) {
            if (!isEnabled || !isExpanded) return@pointerInput
            awaitEachGesture {
                val down = awaitFirstDown(requireUnconsumed = false)
                // 智能分流：down 落在按钮区域则放行，让事件透传给按钮 clickable
                val buttonAreaStartX = size.width - actionsWidthPx
                val isButtonArea = down.position.x >= buttonAreaStartX
                if (!isButtonArea) {
                    down.consume()
                }
            }
        },
    content = { ... }
)
```

**说明:**
- `size.width` 是 PointerInputScope 的属性，表示手势检测区域宽度（即 cardWidth）
- `actionsWidthPx` 是 SwipeableTodoBox 函数体内第 162 行定义的局部变量（= buttonWidthDp * 3 转换为 px），在此作用域可访问
- `down.position.x` 是 down 事件相对于手势检测区域的 x 坐标

确认以下 import 已存在（均在文件顶部，无需新增）：

- `import androidx.compose.foundation.gestures.awaitFirstDown`（第 9 行）

Run: `./gradlew assembleDebug --quiet`
Expected: 编译成功，无错误
