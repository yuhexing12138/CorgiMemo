# 待办卡片长按交互优化实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将待办卡片长按交互从"长按立即触发"改为"手指完全离开屏幕后触发"，并支持长按后移动取消（交给左滑）。

**Architecture:** 在 TodoListItem.kt 中用 `awaitEachGesture` 替换 `detectTapGestures`，自定义手势状态机：down → 启动500ms定时器 → up执行/tap或longPress，move>touchSlop取消。手动管理 interactionSource 水波纹事件。

**Tech Stack:** Kotlin, Jetpack Compose (BOM 2026.04.01 + Compose 1.9.2), `awaitEachGesture`/`awaitFirstDown`/`awaitPointerEvent`

**设计文档:** `docs/superpowers/specs/2026-06-27-待办卡片长按交互优化-design.md`

- [ ] **Step 3: 替换 detectTapGestures 为 awaitEachGesture**

将第 193-265 行的整个 `.pointerInput(...) { detectTapGestures(...) }` 代码块替换为以下代码：

```kotlin
            .pointerInput(isBatchMode, onClick, onLongClick, onSelectClick) {
                awaitEachGesture {
                    val down = awaitFirstDown(requireUnconsumed = false)
                    val downPosition = down.position

                    /**
                     * 长按触发标志：500ms 定时器到期后置为 true
                     *
                     * 仅在主线程读写（scope.launch 默认 Main 调度器，
                     * awaitEachGesture 也在主线程），无需同步原语。
                     */
                    var longPressTriggered = false

                    // 发射 Press 事件，触发水波纹效果
                    val pressInteraction = PressInteraction.Press(downPosition)
                    interactionSource.emit(pressInteraction)

                    /**
                     * 启动长按定时器（500ms）
                     *
                     * 到期后：标记 longPressTriggered + 触发触觉反馈
                     * 被取消时（手指提前抬起或移动）：不执行任何操作
                     */
                    val longPressJob = scope.launch {
                        delay(500)
                        longPressTriggered = true
                        HapticFeedbackManager.performHapticFeedback(
                            context = context,
                            type = InteractionType.LONG_CLICK,
                            enabled = hapticEnabled
                        )
                    }

                    /**
                     * 标记是否已发射终结事件（Release 或 Cancel）
                     *
                     * 沿用原代码的兜底策略：try/finally 确保 Cancel 总是被发射，
                     * 避免水波纹残留（用户感知为"卡片变暗"）。
                     */
                    var terminalEmitted = false

                    try {
                        /**
                         * 主循环：等待手指抬起或移动
                         *
                         * - changedToUp() → 手指抬起，根据 longPressTriggered 执行 tap/longClick
                         * - 移动 > touchSlop → 取消，不消费事件，交给左滑手势
                         */
                        while (true) {
                            val event = awaitPointerEvent()
                            val change = event.changes.first()

                            // 检测手指抬起
                            if (change.changedToUp()) {
                                longPressJob.cancel()

                                // 结束水波纹动画
                                interactionSource.emit(PressInteraction.Release(pressInteraction))
                                terminalEmitted = true

                                if (longPressTriggered) {
                                    // 长按已触发（≥500ms），手指抬起 → 执行 onLongClick
                                    // 普通模式：进入批量模式并选中该条
                                    // 批量模式：切换该条选中状态（toggleSelection 语义）
                                    onLongClick()
                                } else {
                                    // 短按（<500ms），手指抬起 → 执行 onTap
                                    if (isBatchMode) {
                                        onSelectClick()
                                    } else {
                                        onClick()
                                    }
                                }
                                break
                            }

                            // 检测移动距离：超过 touchSlop 则取消长按
                            val dragDistance = (change.position - downPosition).getDistance()
                            if (dragDistance > viewConfiguration.touchSlop) {
                                longPressJob.cancel()
                                // 发射 Cancel，结束水波纹
                                interactionSource.emit(PressInteraction.Cancel(pressInteraction))
                                terminalEmitted = true
                                // 不消费事件，break 退出，让 SwipeableTodoBox 的左滑手势接管
                                break
                            }
                        }
                    } finally {
                        /**
                         * 异常场景兜底：保证 Press 状态被正确清理
                         *
                         * 覆盖：协程因重组/CancellationException 被中断、
                         * pointerInput key 变化导致手势重启等情况。
                         * MutableInteractionSource 对重复 Cancel 是幂等的。
                         */
                        if (!terminalEmitted) {
                            interactionSource.emit(PressInteraction.Cancel(pressInteraction))
                        }
                    }
                }
            }
```

## Task 2: 手动测试验证

**Files:**
- 无文件修改，仅手动测试

由于 Compose 手势检测的自动化测试需要模拟 PointerEvent（复杂度高），采用手动测试验证。

- [ ] **Step 1: 短按测试（普通模式）**

操作：普通模式下，短按（<500ms）一张待办卡片，快速抬起手指。

预期结果：
- 卡片显示水波纹效果
- 跳转到待办编辑页（onClick 触发）
- 不进入批量模式

- [ ] **Step 2: 短按测试（批量模式）**

操作：先长按进入批量模式，再短按另一张卡片，快速抬起手指。

预期结果：
- 该卡片选中状态切换（onSelectClick 触发）
- 水波纹正常显示

- [ ] **Step 3: 长按测试（核心需求 — 抬起触发）**

操作：普通模式下，长按一张待办卡片 **≥500ms 不动**，然后抬起手指。

预期结果：
- 长按 500ms 时：触发触觉反馈（震动）
- **手指抬起后**：进入批量模式 + 该卡片被勾选（√标记显示）
- 水波纹在抬起时正常结束

- [ ] **Step 4: 长按后移动取消测试**

操作：普通模式下，长按一张待办卡片 ≥500ms，然后**水平移动手指**（超过 touchSlop）。

预期结果：
- 长按 500ms 时：触发触觉反馈（震动）
- 手指移动后：**不进入批量模式**，不勾选
- 左滑手势接管，显示左滑操作按钮（分享/置顶/删除）
- 水波纹在移动时取消

- [ ] **Step 5: 长按后移动取消（未到500ms）测试**

操作：普通模式下，手指按下后立即水平移动（<500ms）。

预期结果：
- 不触发长按
- 左滑手势接管，显示左滑操作按钮
- 水波纹在移动时取消

- [ ] **Step 6: 批量模式下长按测试**

操作：已进入批量模式后，长按另一张卡片 ≥500ms 不动，抬起手指。

预期结果：
- 长按 500ms 时：触发触觉反馈
- 手指抬起后：该卡片选中状态切换（toggleSelection）
- 水波纹正常显示和结束

- [ ] **Step 7: 水波纹残留测试**

操作：快速短按多张卡片，或长按后快速移动取消，反复操作。

预期结果：
- 所有卡片水波纹正常消失，无"卡片变暗"残留现象
- 确认 try/finally 兜底 Cancel 机制生效
