v1.0 分组标签角标化：设计要点与样式规范

核心变更

变更点 说明
位置 卡片左上角 从元数据行（卡内第一行）迁移到卡片左上角贴边，与待办分享图分类角标严格一致。
position: absolute; top: 0; left: 4px（从竖条内侧 4dp 开始）
圆角 仅右下 16dp 原：4dp 全圆角胶囊 + 阴影偏移。
新：仅右下 16dp（与首页卡片 16dp 圆角严格一致），其他三边 0 圆角贴合边框。
视觉效果：卡片左上角 = 角标 + 卡片共同构成"双重圆角"，与边框完美贴合。
样式 仿分享图 背景：rgba(224, 224, 224, 0.6)（#E0E0E0 60% 透明，与分享图一致）。
文字色：#666666（次要文字色）。
字号：11sp（从 12sp 缩小）。
padding：4px 10px（与分享图一致）。
letter-spacing：0.3px。
移除 阴影偏移 原 offset(2dp, 2dp) + blur(4dp) 阴影偏移设计废弃。
新设计为贴边角标，无需阴影（视觉上更轻盈、整齐）。
元数据行 简化为单行 原拥挤模式 = 两行（分组/附件/关联 一行 + 提醒 一行）。
新设计：移除"拥挤模式"分支，统一单行布局（附件/关联/提醒），因为分组标签已迁出元数据行。
Z 顺序 z-index=3 角标 z-index=3，被优先级竖条（z=5）和优先级边框（z=4）压住。
与分享图分类角标一致。

对首页布局的影响

  • 卡片顶部 padding 调整：原 card-content padding-top=14dp，新版建议保留 14dp 但额外增加 topCardPadding = 28dp 给角标让出空间（参考分享图 36dp，按比例缩小到首页）
  • 挤压模式判定简化：原 isMetaCrowded = hasCategory && hasReminder && hasAttachment 改写为 isMetaCrowded = hasReminder && hasAttachment（因为 hasCategory 不再是元数据行元素）
  • 元数据行结构调整：第一行不再渲染分组标签，所有元数据元素按 8dp 间距顺序排列（附件/关联/提醒）
  • Category 数据模型不变：无需修改，仅修改 CategoryTagWithShadow 组件为 CategoryBadge（位置+样式重写）
