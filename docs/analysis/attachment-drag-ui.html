行内排序核心特性

  • ✓
    水平方向拖拽
    仅在 LazyRow 内部进行水平方向的重新排列，不涉及垂直移动。
  • ✓
    实时预览排序结果
    拖拽过程中即时显示目标位置的半透明提示，用户可直观看到最终效果。
  • ✓
    平滑动画过渡
    使用 spring 动画（damping=0.6）实现自然的弹跳效果。
  • ✓
    触觉反馈集成
    完成排序时触发轻量级振动（HapticFeedbackType.LIGHT_MOVE）。

跨行移动核心特性

  • ✓
    智能模式切换
    当Y轴偏移超过30dp时，自动从"行内排序"切换为"跨行移动"模式，无需手动选择。
  • ✓
    全局浮层渲染
    跨行模式下，被拖项脱离原容器，作为全局Overlay渲染，确保不被其他元素遮挡。
  • ✓
    动态目标高亮
    实时检测手指位置下方的TodoItem，为该行添加脉冲边框动画和浅色背景。
  • ✓
    数据原子性更新
    释放时执行事务：先从源行的imagePaths删除，再插入到目标行，保证数据一致性。
  • ✓
    层级关系保持
    移入子任务后，图片自动应用28dp左侧缩进，符合UI层级规范。
边界情况处理：
• 如果拖回原行且Y偏移小于阈值，自动降级为行内排序模式
• 如果释放在非目标区域（空白处），图片返回原位置（取消操作）
• 同一行内多次快速拖拽时，通过防抖机制避免状态冲突

性能优化建议

优化领域 具体措施 预期收益
渲染性能 拖拽期间冻结非可见区域的LazyRow重组 减少60%的不必要 recomposition
内存管理 使用remember保存DragState，避免拖拽时重建 降低GC压力
动画流畅度 浮层使用graphicsLayer实现硬件加速 确保120fps顺滑体验
触摸响应 使用PointerInputScope.detectTapGestures分离点击和长按 消除误触，提升准确率
大数据量 单行超过10张图片时启用虚拟化滚动 支持无限附件而不卡顿

可访问性（Accessibility）考虑

  • ♿
    屏幕阅读器支持
    为拖拽项添加 ContentDescription："照片B，第2张，共3张。长按可调整顺序或移动到其他任务。"
  • ♿
    触觉反馈替代
    为听力障碍用户提供明显的视觉提示（颜色变化+图标动画）代替振动反馈。
  • ♿
    大触摸目标
    最小触摸区域44x44dp，长按容差范围扩大至±15dp，方便运动障碍用户操作。
  • ♿
    键盘导航支持
    允许通过Tab键聚焦图片，使用方向键模拟拖拽操作（辅助技术兼容）。
  • ♿
    色彩对比度
    所有文本和边框满足WCAG 2.1 AA标准（对比度>=4.5:1），高亮色在浅色背景上清晰可辨。
