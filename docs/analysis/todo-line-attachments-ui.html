交互流程详解

📸 添加图片到指定行

  1. 步骤1：点击目标行的文本输入框，使该行获得焦点
  2. 步骤2：点击底部工具栏的"📷 相机"图标按钮
  3. 步骤3：选择拍照或从相册选取图片
  4. 步骤4：图片自动添加到当前聚焦行的 imagePaths 列表
  5. 步骤5：附件以动画形式出现在该行文本框正下方
焦点追踪机制：系统会记录最后操作的输入框位置， 工具栏的所有附件操作都会作用于该行。

🎤 添加语音到指定行

  1. 步骤1：点击目标行的文本输入框（主任务或子任务均可）
  2. 步骤2：点击底部工具栏的"🎤 麦克风"图标按钮
  3. 步骤3：开始录音，界面显示录音时长计时
  4. 步骤4：停止录音后，语音文件保存至 voiceAttachments
  5. 步骤5：语音播放器组件渲染在该行下方
语音播放器特性：实时波形动画、时长显示、 点击播放/暂停、支持删除单个语音条目。

🗑️ 删除附件操作

  1. 图片删除：鼠标悬停在图片预览上 → 显示右上角 ✕ 按钮 → 点击删除
  2. 语音删除：鼠标悬停在语音播放器上 → 显示右侧 ✕ 按钮 → 点击删除
  3. 确认提示：删除前弹出轻量级 Toast 提示"已删除"
  4. 数据同步：从对应的 imagePaths 或 voiceAttachments 列表中移除
移动端适配：在触摸设备上，长按附件即可触发删除选项， 避免误触操作。

📏 缩进规则可视化

主任务行 → 附件无缩进（左对齐）
[🖼️ 主任务图片] ← 与复选框对齐
子任务行 → 左侧缩进28dp
[🖼️ 子任务图片] ← 跟随缩进

通过视觉缩进，用户可以快速识别附件的层级归属关系。

数据模型定义

// 修改后的 TodoLine 数据类 data class TodoLine( val text: String = "", val isChecked: Boolean = false, val isSubTask: Boolean = false, // 是否为子任务 val groupId: Int = 0, val order: Int = 0, // ===== 新增字段 ===== val imagePaths: List<String> = emptyList(), // 该行的图片路径列表 val voiceAttachments: List<VoiceInfo> = emptyList() // 该行的语音附件列表 ) // 语音信息数据类 data class VoiceInfo( val path: String, // 语音文件路径 val duration: Int? // 录音时长（秒） )

使用示例

// 创建一个带有附件的主任务 val mainTask = TodoLine( text = "完成项目周报", isSubTask = false, imagePaths = listOf( "/storage/emulated/0/Pictures/report_chart.png", "/storage/emulated/0/Pictures/meeting_photo.jpg" ), voiceAttachments = listOf( VoiceInfo( path = "/storage/emulated/0/Voice/meeting_summary.m4a", duration = 125 // 2分5秒 ) ) ) // 创建一个带有图片的子任务 val subTask = TodoLine( text = "整理数据图表", isSubTask = true, imagePaths = listOf( "/storage/emulated/0/Pictures/data_visualization.png" ), voiceAttachments = emptyList() // 无语音附件 )

💡 设计优势

  • 数据内聚：附件数据与行数据绑定，无需外部映射表
  • 类型安全：Kotlin data class 保证编译时类型检查
  • 可空友好：duration 使用 Int? 支持未知时长场景
  • 不可变性：val 声明确保线程安全

🔄 迁移策略

  • 向后兼容：新字段提供默认值，旧数据自动迁移
  • 增量更新：Room Migration 添加新列即可
  • 零停机：支持数据库版本平滑升级
  • 回滚支持：保留降级路径以应对异常情况
