🔍 根本原因分析

  • 问题根源： 在 ReminderPickerBottomSheet.kt 第 115 行， 最外层容器使用了 Modifier.fillMaxSize()，这告诉 Compose 布局系统： "这个组件想要占据所有可用的空间"！
  • 连锁反应： 当父容器（TodoEditScreen）测量这个弹窗时， 返回的 dialogHeight 不是内容的实际高度，而是父容器的最大约束高度（约等于屏幕高度）。
  • 数学结果： 如果屏幕高 2400px，弹窗也被测量为 2400px：
    bottomSpace = (2400 - 2400) / 5 = 0px
    topSpace = 0 × 4 = 0px
    → 弹窗直接贴着屏幕边缘，4:1 比例完全失效！

📊 技术细节对比表

技术指标 ❌ 当前实现（错误） ✅ 正确实现
外层容器修饰符 fillMaxSize() wrapContentSize()
测量返回的高度 父容器最大约束（≈2400px） 内容固有高度（≈1600px）
dialogHeight 值 2400px（虚高） 1600px（真实）
bottomSpace 计算 (2400-2400)/5 = 0px (2400-1600)/5 = 160px
topSpace 计算 0 × 4 = 0px 160 × 4 = 640px
最终比例 (top:bottom) 0:0 （失效） 640:160 = 4:1 ✓

💡 解决方案

修改 ReminderPickerBottomSheet.kt 第 115 行：

// ❌ 错误：强制填满父容器 Box(modifier = Modifier.fillMaxSize()) { ✅ 正确：使用内容的固有尺寸 Box(modifier = Modifier.wrapContentSize()) {

或者更简单的方法：
直接移除 fillMaxSize()，让 Column 自适应内容高度。

原理：wrapContentSize() 会告诉布局系统 "只占用内容需要的空间"，这样父容器测量时就能获取到真实的 dialogHeight， 从而正确计算出 4:1 的上下间距比例。
