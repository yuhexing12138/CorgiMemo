1. Demo 简介

1.1 产品概述

「刻记+」是一款集待办管理、灵感记录、日期规划于一体的治愈系记录应用，以一只有情绪、会互动、能成长的虚拟柯基犬作为陪伴形象，提供治愈系的生活记录体验。面向用户覆盖学生群体、上班族、老年群体三大核心人群，APP 根据用户身份自动切换界面风格——学生模式活泼可爱，上班族模式简洁高效，老年模式大字体 + 语音交互。
技术架构上，本作是一款原生 Android 应用，UI 层采用 Kotlin + Jetpack Compose (Material3) 构建声明式界面；数据层基于 Room 数据库实现本地持久化，配合 DataStore 与加密 SharedPreferences 保障数据安全。三大功能通过统一的 CardRelation 关联表实现跨模块 @提及，柯基系统以 FloatingCorgiButton 全局悬浮贯穿三大功能提供陪伴反馈。

1.2 主要功能

- **待办管理**：三区域模型（置顶区/待办区/已完成区）+ 长按拖拽跨区排序 + 多卡片子任务架构 + 分享导出
- **灵感记录**：时间线分组 + 富文本编辑 + 多标签 + 图片画廊 + 跨卡片关联 + 删除撤销
- **日期规划**：倒计时/正计时 + 自定义日期类型 + 多种卡片样式（撕历卡/波浪卡/日历卡）+ 农历支持 + 数据统计
- **柯基系统**：全局悬浮陪伴形象 + 情绪反馈 + 任务完成动画 + 贯穿三大功能的情感连接层

下面逐项说明三大核心功能在手机上的简易操作流程，助你快速新建待办、灵感、日期卡片。
#### 1.2.1 待办管理
新用户进入「待办」Tab 后，面对的是一个存有示例的待办区。完整操作流程如下：
1. **添加待办**：点击右下角"铅笔形状"浮动按钮 → 进入编辑页 → 输入标题（必填）→ 选填分类（学习/工作/生活…）、优先级（高/中/低）、提醒时间（精确到分钟）→ 点击右上角"保存"按钮 → 回到主页，列表中自动出现该待办
2. **标记完成**：在主页找到待办 → 点击待办右侧的圆形复选框 → 待办划入"已完成"区域
3. **长按拖拽跨区排序**：长按待办卡片（约 0.5 秒）→ 卡片浮起并显示拖拽状态 → 拖到目标位置（支持在"置顶区"与"待办区"之间跨区拖拽）→ 松手完成排序
4. **删除与撤销**：在主页向左滑动待办卡片 → 出现"删除"按钮 → 点击删除 → 底部出现 Snackbar 提示"已删除"，3 秒内可点击"撤销"恢复

#### 1.2.2 灵感记录

新用户进入「灵感」Tab 后，呈现的是一个存有示例的灵感区。完整操作流程如下：
1. **快速记录灵感**：点击右下角"铅笔形状"浮动按钮 → 进入编辑页 → 输入标题（必填）和正文（富文本编辑器，支持加粗/斜体/列表/引用）→ 点击右上角"完成" → 灵感卡片回到时间线顶部
2. **添加图片**：在编辑页点击工具栏"图片"图标 → 选择图片来源（拍照/从相册选择/多选）→ 图片以画廊形式插入正文中
3. **添加标签**：在编辑页底部"标签"输入框中输入文字 → 点击"+" → 标签以 Chip 形式显示在标题下方，便于后续筛选
4. **查看详情**：在主页点击灵感卡片 → 进入详情页（富文本渲染 + 图片画廊）→ 左右滑动切换上下条灵感
5. **编辑/删除**：长按灵感卡片 → 弹出操作菜单（编辑/删除/分享）→ 选择"删除" → 底部 Snackbar"已删除"，3 秒内可撤销


#### 1.2.3 日期规划

新用户进入「日期」Tab 后，呈现的是一个存有示例的日期区。完整操作流程如下：

1. **快速创建日期**：点击右下角"铅笔形状"浮动按钮 → 进入快速创建页 → 输入名称（必填）→ 点击日期字段选择目标日期（默认今天）→ 选择类型（生日/纪念日/节日/其他等 8 种）→ 点击"完成"按钮 → 主页出现新日期卡片，显示倒计时天数
2. **切换日期类型**：在完整编辑页点击"类型"字段 → 底部弹出类型选择器（8 种类型，每种带专属图标和颜色）→ 点击新类型 → 自动应用对应配色
3. **更换卡片样式**：点击卡片进入完整编辑页 → 点击“主题”下的"样式" → 进入样式选择页 → 查看 2 种样式（橙色撕历卡/波浪日历卡）→ 选中样式 → 新样式立即生效
4. **删除日期**：在主页向左滑动日期卡片 → 出现"删除"按钮 → 点击删除 → 底部 Snackbar 提示"已删除"，3 秒内可撤销



2. Demo 创作思路

2.1 灵感来源
我观察到身边很多人对记录类APP又爱又恨——离不开它，但每次打开都感到压力。同时，独居老人和异地求学、打拼的年轻人普遍存在孤独感。我想，如果有一个工具既能帮你管理生活，又能给你温暖陪伴，会不会让"记录"这件事本身变得令人期待？
2.2 解决的问题
市面上的大部分记录类APP冷冰冰的，只关注效率不关注人的感受。学生被deadline追着跑缺乏动力，上班族被任务淹没感到焦虑，老年人记不住吃药买菜又无人陪伴。待办管理、灵感捕捉、重要日期记录散落在不同工具中，缺乏统一管理，更缺乏情感温度。
2.3 为什么做这个方向
**社会价值：对抗孤独。** 在老龄化与城市化背景下，独居老人和异乡青年的孤独感是真实存在的社会问题。一只随身的柯基陪伴形象，让记录工具承载情感连接——对老人是日常叮咛，对青年是无声陪伴。
**差异化竞争：效率 vs 温度。** 市面记录类工具普遍陷入"功能堆砌"的同质化竞争，刻意追求效率却忽略了使用者的情绪。「刻记+」以柯基陪伴系统为差异化壁垒，证明"有温度的记录"能同时兼顾效率与情感，让用户从"不得不打开"变为"期待打开"。
**AI 赋能独立开发。** AI 编程工具的成熟让"一个人 + AI"的产品模式成为可能。「刻记+」从需求文档、架构设计到功能迭代全程由 TRAE IDE 驱动，验证了独立开发者借助 AI 能交付具备工程深度的完整产品。
**技术实践综合场。** 待办拖拽排序、富文本编辑、跨模块卡片关联、加密备份、桌面小组件等复杂交互需要一个综合载体。「刻记+」以三大记录功能为骨架，把这些技术难点串联成一个有意义的真实产品，而非孤立的 Demo。
四个角度的共同指向是——「刻记+」不只是一个记录工具，更是一次"AI 时代如何用技术回应人的情感需求"的实践探索。

5. 经验总结及开发心得
5.1 在需求文档与技术文档生产阶段，我利用TRAE WORK(TRAE SOLO)中自带的doc-coauthoring和brainstorming技能，前者引导用户通过结构化协作流程撰写文档、提案、技术规格和决策文档，能够快速撰写初稿；后者在任何创作、功能开发或行为修改前，先探索用户意图、需求和设计方向，可以快速迭代。在两者的结合下我高质量的完成了需求文档与技术文档，为后续开发阶段提供了基石。
5.2 在开发阶段，我绝大部分时间都使用trae ide的solo模式，核心经验是组合运用五大能力：
1. Solo模式的思考可见性：可清楚看到模型思考过程与执行细节，更容易掌握开发节奏，及时纠偏
2. MCP工具链：用得最多的是Playwright，原型开发阶段自动进行页面检查、截图；提交报名帖时多次用于审查附件HTML
3. 规则设置（全局+项目）：全局规则贴合个人习惯，如"每完成一个任务询问是否git提交，提交信息用中文"；项目规则规避重复错误，如"每次编辑完代码检查import语句缺失"。规则让AI开发更符合开发者习惯，也避免陷入同一种错误
4. Skills技能：用得最多的是brainstorming技能，实现复杂功能前先创建计划文档与任务步骤，主动与用户确认开发细节，极大减少返工、提升效率
5. 视觉伴侣模式：实际开发前启动服务器提供HTML原型，让开发者直观感受UI界面，在写Compose代码前先对齐视觉方向
