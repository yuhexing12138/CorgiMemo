ViewModel 重构、状态流转、数据持久化策略

OnboardingViewModel 状态流转图

┌─────────────────────────────────────────────────────────────────┐
│                    OnboardingViewModel                          │
│                                                                 │
│  StateFlow                                                      │
│  ├─ currentPage: Int        (0-9, 当前页面索引)                │
│  ├─ selectedUserType: UserType?  (仅 STUDENT 可选)             │
│  ├─ corgiName: String       (1-8字符)                         │
│  ├─ createdTodo: Todo?      (引导中创建的待办)                 │

│  ├─ permissionStates: Map     │
│  ├─ isCompleting: Boolean                                    │
│  └─ skippedFunctions: Set<FunctionType>  (已跳过的功能)  │
│                                                                 │
│  Derived State                                                  │
│  ├─ canGoNext: Boolean     (根据 currentPage 计算)             │
│  ├─ isLastPage: Boolean    (currentPage == 9)                  │
│  └─ mustComplete: Boolean  (currentPage == 2 || currentPage == 8)│
└─────────────────────────────────────────────────────────────────┘

数据持久化策略

数据 存储方式 时机 键/表
引导完成状态 DataStore Step 10 完成时 isOnboardingCompleted
首次启动标记 DataStore Step 10 完成时 isFirstLaunchDone
用户身份 DataStore Step 2 选择时 userType
柯基名字 DataStore + Room Step 3 完成时 corgiName + CorgiData 表
创建的待办 Room Step 5 创建时 Todo 表
创建的灵感 Room Step 6 创建时 Inspiration 表
权限状态 内存（不持久化） Step 9 请求时 ViewModel StateFlow

核心方法设计

// 页面导航
fun nextPage()                    // 下一页（检查 canGoNext）
fun prevPage()                    // 上一页
fun goToPage(page: Int)          // 跳转到指定页
fun skipToPermission()           // 跳过功能介绍 → Step 9

// 个性化设置
fun setUserType(type: UserType)  // 设置身份（仅 STUDENT）
fun setCorgiName(name: String)   // 设置柯基名字
fun isValidName(): Boolean       // 验证名字有效性

// 功能创建
fun createFirstTodo(title: String, priority: Priority)  // 创建首个待办
fun createFirstInspiration(title: String, content: String)  // 创建首个灵感

// 权限处理
fun requestPermission(type: PermissionType)  // 请求权限
fun updatePermissionState(type: PermissionType, state: PermissionState)

// 完成引导
fun completeOnboarding(onComplete: () -> Unit)  // 保存所有数据并完成

权限请求顺序： 通知 → 存储 → 麦克风 → 位置（逐个请求）
拒绝处理： 标记为"已拒绝"，不阻塞流程，显示"可在设置中稍后开启"

与现有代码的集成

  • OnboardingRouter（MainActivity.kt）：保持现有逻辑，读取 isOnboardingCompleted 决定起始页
  • CorgiPreferences：复用现有方法（saveUserType, saveCorgiName, setOnboardingCompleted 等）
  • CorgiRepository：复用现有方法插入 CorgiData
  • TodoRepository：复用现有方法插入 Todo
  • InspirationRepository：复用现有方法插入 Inspiration
  • AppNavHost：保持 Screen.Onboarding.route 路由不变
