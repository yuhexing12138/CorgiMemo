实现方案对比

三种技术实现方案的对比分析

方案A：双列表嵌套

思路：使用两个独立的ReorderableLazyColumn分别渲染未完成和已完成待办，外层用普通Column包裹，"已完成"按钮放在两列表之间。

✅ 优点

  • 不需要修改ReorderableLazyColumn组件
  • 两个区域独立，逻辑简单清晰
  • 拖拽天然隔离，不会跨区域

❌ 缺点

  • 两个LazyColumn嵌套在ScrollableColumn中会有滚动冲突
  • 整体滚动体验不连贯，分隔按钮无法自然"粘"在列表中
  • 无法实现平滑的展开/折叠动画（animateItem不跨列表）
  • FAB、下拉刷新等需要额外处理嵌套滚动

⚠️ 不推荐：嵌套滚动问题难以解决，用户体验差

方案B：Sealed Class + 扩展ReorderableLazyColumn（推荐）

思路：定义DisplayItem密封类（Todo项/CompletedDivider分隔项），扩展ReorderableLazyColumn支持"不可拖拽、不可跨越"的固定项，单列表实现。

✅ 优点

  • 单列表，滚动流畅自然
  • 分隔按钮作为列表项，随列表滚动、参与动画
  • 展开/折叠使用animateItem()实现平滑过渡
  • 拖拽逻辑中分隔项像"墙"一样不可跨越，天然分区
  • 与现有架构一致，下拉刷新、FAB等无需改动

❌ 缺点

  • 需要修改ReorderableLazyColumn（添加isDraggable参数）
  • 索引转换需要小心处理（divider占一个位置）
  • ViewModel排序逻辑需适配区域内排序

✅ 推荐：用户体验最佳，改动可控

方案C：直接在LazyColumn中手写拖拽

思路：放弃ReorderableLazyColumn，直接使用LazyColumn的DSL（item+items），在待办项上各自实现拖拽手势。

✅ 优点

  • 完全控制布局，可任意放置分隔item
  • 不需要修改现有组件

❌ 缺点

  • 重复造轮子：重写长按检测、50%重叠交换、边缘滚动、置顶区跨越、反向交换锁定等所有拖拽逻辑
  • 工作量最大，风险最高
  • 容易引入新的拖拽bug

❌ 不推荐：重复工作量大，风险高

🎯 推荐方案 B 详细设计

1. 数据层（ViewModel）

  • 移除FilterStatus.ALL/PENDING/COMPLETED三态枚举
  • 新增 _showCompleted = MutableStateFlow(false)，持久化到CorgiPreferences
  • 分离待办为两个列表：pendingTodos（status=0）和 completedTodos（status=1），各自按 isPinned→sortOrder→createdAt 排序
  • 新增 completedCount 统计流
  • 修改排序逻辑：拖拽只在各自区域内重排sortOrder，不跨区域

2. UI层（HomeScreen）

  • 移除三个FilterButton及对应Row
  • 定义sealed interface DisplayItem { data class Todo(val item: TodoItem); data object CompletedDivider }
  • 组合displayList：pendingTodos.map(Todo) + (if(showCompleted) listOf(CompletedDivider) + completedTodos.map(Todo) else emptyList())
  • CompletedDivider渲染为"⌄/⌃ 已完成 N"按钮，点击切换showCompleted
  • 使用Modifier.animateContentSize()或依赖LazyColumn.animateItem()实现展开/折叠动画

3. 组件层（ReorderableLazyColumn）

  • 新增参数 isDraggable: (T) -> Boolean = { true }
  • 拖拽检测时跳过isDraggable=false的项（CompletedDivider）
  • findSwapTarget中过滤掉isDraggable=false的项（它们像墙一样不可跨越）
  • content lambda新增isDraggableItem参数，让调用方决定项样式

4. 视觉设计细节

元素 样式
折叠状态按钮 ⌄ 已完成 N，颜色#999，字号15sp，左对齐，水平padding=20dp
展开状态按钮 ⌃ 已完成 N，颜色#999，字号15sp
按钮点击区域 48dp高度，水平padding=16dp，增加触控面积
展开/折叠动画 300ms ease-in-out，箭头旋转180度，列表项使用animateItem
