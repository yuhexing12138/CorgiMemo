🔧 实现方案对比

两种实现方式:完全镜像 showCompleted 模式 vs. 抽象为通用 SectionHeader 组件

方案 1:完全镜像 showCompleted 推荐
最简单、风险最低、与现有模式完全一致

📝 核心思路

完全照搬 `_showCompleted` / `showCompleted` 的实现方式,新增对应的 `_showPinned` / `showPinned`。

📁 涉及文件

+ 新增 按钮组件:PinnedSectionHeader.kt
~ 修改 HomeViewModel.kt:新增 _showPinned 状态、toggleShowPinned() 方法、pinnedCount 计算
~ 修改 HomeScreen.kt:DisplayItem 新增 PinnedDivider 分支,displayItems 构建逻辑新增 pinned
~ 修改 CorgiPreferences.kt:新增 SHOW_PINNED 键 + get/set 方法

💻 关键代码片段

// HomeViewModel.kt private val _showPinned = MutableStateFlow(true) val showPinned: StateFlow<Boolean> = _showPinned.asStateFlow() val pinnedCount: StateFlow<Int> = _todos.map { todos -> todos.count { it.isPinned && it.status == 0 } } fun toggleShowPinned() { val newVal = !_showPinned.value _showPinned.value = newVal viewModelScope.launch { corgiPreferences.setShowPinned(newVal) } }

✅ 优点

  • 代码模式高度一致
  • 改动范围最小
  • 不触碰现有"已完成"逻辑
  • 易于 review 和测试

❌ 缺点

  • PinnedSectionHeader 和 CompletedSectionHeader 有重复代码
  • 未来若要改通用样式需改两处
方案 2:抽象通用 SectionHeader 重构
提取通用的 CollapsibleSectionHeader,两个按钮共享

📝 核心思路

提取一个通用的 `CollapsibleSectionHeader` 组件,接受颜色、图标、文本等参数。PinnedSectionHeader 和 CompletedSectionHeader 都基于它实现。

📁 涉及文件

+ 新增 通用组件:CollapsibleSectionHeader.kt
+ 新增 按钮组件:PinnedSectionHeader.kt(基于通用组件)
~ 修改 HomeViewModel.kt:同方案 1
~ 修改 HomeScreen.kt:同方案 1 + 将 CompletedSectionHeader 改为基于通用组件
~ 修改 CorgiPreferences.kt:同方案 1

💻 关键代码片段

// CollapsibleSectionHeader.kt(通用) @Composable fun CollapsibleSectionHeader( label: String, count: Int, color: Color, isExpanded: Boolean, onClick: () -> Unit ) { val arrowRotation by animateFloatAsState( targetValue = if (isExpanded) 0f else -90f ) Row( modifier = Modifier .fillMaxWidth() .clickable(onClick = onClick), ... ) { Icon(Icons.Filled.PlayArrow, modifier = Modifier.rotate(arrowRotation), tint = color ) Text("$label ($count)", color = color) } } // PinnedSectionHeader.kt(精简) @Composable fun PinnedSectionHeader(count: Int, isExpanded: Boolean, onClick: () -> Unit) = CollapsibleSectionHeader( label = "置顶", count = count, color = MaterialTheme.colorScheme.primary, isExpanded = isExpanded, onClick = onClick )

✅ 优点

  • 代码完全复用
  • 未来调整只改一处
  • 设计语言统一更彻底

❌ 缺点

  • 触碰现有"已完成"代码
  • 需要额外测试
  • 改动面更大、风险略高
💡 建议: 当前阶段采用方案 1(完全镜像),实现简单且风险低;未来若需要扩展更多类似折叠区(如"高优先级"、"今日到期"),再统一重构为方案 2。
