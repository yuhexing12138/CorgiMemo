设计第 4 节：状态管理与数据流

RichTextState 与现有 contentFormat / Undo-Redo 栈的桥接

数据流图
编辑流程（实时）
用户输入 → RichTextState 自动更新 → snapshotFlow 监听 → 防抖 500ms → toMarkdown() → contentFormat → Room 持久化
加载流程（进入页面时）
Room 读取 → contentFormat (Markdown) → setMarkdown() → RichTextState 恢复 → 编辑器显示
Undo/Redo 桥接（保留扩展栈）
文本操作
RichTextState 变更 → 包装为 RichTextSnapshot → 推入扩展栈
撤销时
取出上一 snapshot → setMarkdown() 恢复
内容块操作（图片/语音）
现有扩展栈逻辑 → 不变
格式工具栏交互
点击格式按钮 (如 B) → toggleSpanStyle(Bold) → RichTextState 更新 → 同编辑流程（防抖导出）
工具栏展开状态
⋮ 按钮点击 → isFormatToolbarExpanded = !isFormatToolbarExpanded → AnimatedVisibility 展开/收起
状态存于 ViewModel，配置改变时保留；只有 ⋮ 点击才切换
