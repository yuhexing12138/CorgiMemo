为本APP项目的待办卡片实现一个独立可复用的移动端左滑操作组件，1:1还原飞书左滑按钮的堆叠挤出动效与交互逻辑，组件为独立单元，不绑定列表业务场景，具体需求如下：

一、组件基础结构
采用双层叠加布局，所有元素处于同一行容器内：
1.  **内容层**：位于最上层，默认完全覆盖下层区域，支持传入任意自定义内容，是默认可见的主体区域。
2.  **操作层**：位于内容层下方，固定在组件右侧，内部横向排列多个操作按钮；按钮从左到右 z-index 逐级升高，最右侧按钮层级最高，为堆叠动效提供层级基础。

二、核心交互逻辑
1.  **跟手触发**：在内容区域按住并横向左滑时触发交互，竖向滑动不触发左滑逻辑，不干扰页面垂直滚动。
2.  **阈值吸附规则**
    - 滑动距离小于操作层总宽度的 1/3时，松手后内容层自动向右回弹复位，完全隐藏操作层。
    - 滑动距离大于等于该阈值时，松手后内容层自动向左吸附至终点，完整露出全部操作按钮。
3.  **边界限制**：内容层向左滑动的最大位移等于操作层的总宽度，禁止超出边界滑动。
4.  **收起规则**：点击内容层区域、点击任意操作按钮后，内容层自动向右回弹复位，操作按钮同步收起。

三、操作按钮视觉规范
操作层从左到右排列 3 个等宽操作按钮，按钮高度与组件整体高度保持动态一致：
- 第1个（最左侧）：背景为黄色，内部垂直居中排布分享图标 + 项目主题色文字「分享」，图标在上，文字在下。
- 第2个（中间）：背景为橙色，内部垂直居中排布向上箭头的置顶图标 + 项目主题色文字「置顶」，图标在上，文字在下。
- 第3个（最右侧）：背景为正红色，内部垂直居中排布垃圾桶删除图标 + 项目主题色文字「删除」，图标在上，文字在下。
所有按钮文字使用无衬线字体，图标与文字视觉居中对齐。

四、飞书风格堆叠动效细节
操作按钮并非随内容层整体平移，而是采用**堆叠式挤出动效**，严格还原飞书原生交互质感：
1.  **露出逻辑**：随内容层左滑，最右侧（最高层级）的按钮最先完整露出，中间按钮次之，最左侧按钮最后露出；呈现“从右向左逐个挤出”的堆叠视觉效果，而非所有按钮同步平移。
2.  **过渡细节**：未完全露出的按钮伴随轻微的横向位移偏移与透明度渐变，从堆叠状态平滑过渡到完全展开状态，无生硬跳变。
3.  **收起反向动效**：内容层回弹收起时，按钮按从左向右的顺序依次收回复位，最左侧按钮最先消失，最右侧按钮最后隐藏，与展开逻辑完全对应。
4.  **动画曲线**：所有展开、收起动画使用 ease-out 缓动曲线，整体动画时长 250ms 左右，跟手过程无延迟，松手后吸附/回弹顺滑自然。

五、适配与技术要求
1.  优先支持移动端触摸事件。
2.  组件封装为独立可复用单元，内容区域支持自定义替换，可嵌入任意容器中使用。
3.  使用原生 HTML+CSS+JavaScript 实现，不依赖第三方库；若需框架版本，可基于 Vue3 / React 编写。代码结构清晰，可直接运行调试。
